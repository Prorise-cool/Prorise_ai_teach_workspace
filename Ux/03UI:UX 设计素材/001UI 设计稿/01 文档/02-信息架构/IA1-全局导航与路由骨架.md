# IA1-全局导航与路由骨架

- 文档编号：IA1
- 版本：v1.0
- 日期：2026-03-27
- 范围：全局导航骨架、正式路由清单、受保护路由与未登录拦截语义、非路由交互边界、P0/P1/P2 路由优先级、后端归属（FastAPI / RuoYi）
- 唯一业务事实源：`_bmad-output/planning-artifacts/archive/{prd,ux-design-specification,architecture,epics}.md`

## 1. 文档目的

本文件用于把“小麦”在 MVP 周期内可对外表达的“页面级信息架构（IA）”定为一套最小但可扩展的全局导航与路由骨架，满足：

- 所有 P0 必须能力都有对应可挂载的正式页面或可验收的路由承载点。
- 不把共享等待壳层、风格面板等非路由交互误写为正式主路由；登录则改为独立路由页承载。
- 与现有线框图的“正式路由结构与路径”不冲突，仅做摘要化承接与优先级标注。
- 明确每个路由在双后端体系下的主要归属：哪些主要走 FastAPI（功能执行与 AI 编排），哪些主要走 RuoYi（用户域、权限与长期业务数据）。

## 2. 路由分层原则

### 2.1 路由层级（从浅到深）

- L0 公共入口层：`/`，以及独立登录页（建议：`/login`）。
- L1 场景输入层：`/classroom/input`、`/video/input`，承载“创建任务”与会话级配置（老师风格等）。
- L2 任务过程层：`/classroom/:id/generating`、`/video/:id/generating`，承载长任务进度、阶段、SSE 恢复语义与失败恢复动作。
- L3 结果回看层：`/classroom/:id`、`/video/:id`，承载结果展示、继续学习入口、沉淀动作（收藏、回写历史等）。
- L4 横切功能层：知识问答、测验、学习路径、学习中心、个人域等，作为独立模块路由或在 L3 内以局部交互承载。

### 2.2 “正式页面”与“非路由交互”边界

- 正式页面：必须是可对外验收的 Route 级页面，URL 可直达，可在浏览器刷新后恢复到同一业务语义（至少能通过任务快照或业务查询恢复）。
- 非路由交互：使用弹窗、面板、浮层、Toast 等，不占用主路由，不应被写成 `/auth`、`/style`、`/progress-shell` 这类独立主路由。

### 2.3 命名与路径稳定性

- 以线框图已冻结的路径为准，优先保持现有模块路径不变。
- 如需新增“学习中心”等缺口路由，优先新增不冲突的新路径，不复用已被其他语义占用的路径。

## 3. 全局导航骨架（Global Navigation Skeleton）

### 3.1 导航的最浅原则

首页仅呈现“我想学 / 我有问题”双入口与最浅导航，不强行暴露所有模块入口，避免用户进入时产生“我要去哪”焦虑。

### 3.2 导航形态（建议）

- 顶部右上角悬浮工具栏（Pill / 工具条语义，不是侧边栏信息架构）。
- 永远可见：`首页`、`个人入口`（头像入口）。
- 条件可见：`知识问答`，以及 LearningCenter / Profile 相关入口。若后续冻结为独立顶层路由，可直接展示；若保留为聚合页或 Tab，则收纳到“更多”或用户入口中。

### 3.3 导航项与页面组件映射（骨架级）

| 导航项 | 路由 | 登录要求 | 说明 |
|---|---|---|---|
| 首页 | `/` | 否 | 双入口选择、智能推荐入口（如有）。 |
| 登录 | `建议：/login` | 否 | 主动登录、未登录拦截回流、401 失效恢复统一进入该页。 |
| 我想学 | `/classroom/input` | 是 | 课堂主题输入入口，也可在首页入口卡片直接进入。 |
| 我有问题 | `/video/input` | 是 | 视频题目输入入口，也可在首页入口卡片直接进入。 |
| 学习中心 | `候选：/learning` | 是 | 负责学习结果聚合（历史、收藏、学习沉淀入口等）；是否作为独立顶层路由仍待与全站 IA 一起冻结。 |
| 知识问答 | `/knowledge` | 是 | 知识问答主页面。 |
| 个人 | `候选：/profile` | 是 | 只承接个人资料与平台设置；不再承接学习结果聚合。 |

## 4. 已冻结正式路由与候选承载页

说明：下表以“路由页面”为单位，标注优先级（P0/P1/P2）、是否受保护、以及主要后端归属。部分页面会同时调用 FastAPI 与 RuoYi，但此处标注“主要依赖”用于对齐路由职责与接口契约的主驱动方。

### 4.1 P0（必须，可演示闭环）

| 路由 | 页面名（建议） | 受保护 | 主要后端 | 说明（骨架级） |
|---|---|---|---|---|
| `/` | 首页与双入口 | 否 | RuoYi（可选） | 首页可不依赖后端也可渲染，但“登录态、推荐入口、最近一次任务”若展示则来自 RuoYi。 |
| `建议：/login` | 登录页 | 否 | RuoYi | 主动登录、未登录拦截、401 失效恢复统一进入该页；注册可作为同页状态切换承载。 |
| `/classroom/input` | 课堂主题输入 | 是 | FastAPI | 创建课堂任务，老师风格等会话配置在此完成并透传到 FastAPI。 |
| `/classroom/:id/generating` | 课堂生成进度 | 是 | FastAPI | SSE 进度与断线恢复快照，统一任务状态五态（pending/processing/completed/failed/cancelled）。 |
| `/classroom/:id` | 课堂结果页 | 是 | FastAPI + RuoYi | 结果展示主要来自 FastAPI，沉淀与回看索引写入 RuoYi（历史/学习中心）。 |
| `/video/input` | 视频题目输入 | 是 | FastAPI | 文本/图片输入与 OCR 触发，创建视频任务。 |
| `/video/:id/generating` | 视频生成进度 | 是 | FastAPI | SSE 进度、Provider 切换提示、失败恢复动作。 |
| `/video/:id` | 视频结果页（含播放） | 是 | FastAPI + RuoYi | 播放资源与结果来自 FastAPI/COS，收藏与历史索引在 RuoYi。 |
| `/knowledge` | 知识问答 | 是 | FastAPI | 知识问答与引用来源展示的主承载页。 |

## 5. 非路由交互边界（Non-Route Interactions）

以下交互必须以“弹窗/面板/浮层/局部展开”的形式存在，不进入正式主路由表：

- 登录页内部的登录 / 注册模式切换、错误提示与辅助说明：属于登录页内部交互，不再单独扩展第二条认证主路由。
- 老师风格选择器：输入页内部的会话配置面板或下拉展开，不是独立页面。
- 共享等待壳层：视频/课堂 generating 页内部复用的“进度壳层组件”，不是独立路由。
- 资料初始化弹层：首次进入个人域或需要补全资料时触发，不是独立页面。
- 通用状态资产：`loading`、`empty`、`toast`、`confirm`、`error-404/500/network` 等，不代表业务主流程路由。

## 6. 受保护路由与未登录拦截

### 6.1 受保护路由范围

除 `/` 与独立登录页外，默认所有业务路由均为受保护路由，至少包含：

- `/classroom/*`
- `/video/*`
- `/knowledge`
- LearningCenter / Profile 页面组件（若最终冻结为独立路由，则其路径也属于受保护路由）
- 测验 / 学习路径等横切能力（若最终冻结为独立路由，则同样属于受保护路由）

### 6.2 未登录访问语义（统一）

- 当用户访问受保护路由但未登录时：进入独立登录页；对外产品语义至少保证“登录成功后可回到原目标路由（含参数）”。
- 当 Token 失效或接口返回 401 时：前端按 `architecture.md` §10.3.1 / §10.3.3 的固定规则清除 Token，并跳转独立登录页。
- 当接口返回 403 时：进入“无权限 / 登录失效”提示通道，不把 401 / 403 默认写成 Auth Dialog 语义。

## 8. 后端归属与接口主驱动（FastAPI / RuoYi）

### 8.1 归属原则（摘要）

- FastAPI：AI 功能执行与编排、异步任务协调、视频与课堂生成、Provider 调度、SSE 事件与运行时状态管理。
- RuoYi：用户/角色/权限，以及长期业务数据持久化与查询（学习记录、收藏、任务元数据、问答日志等），并承接审计、导出与管理宿主能力。

### 8.2 路由到后端的“主驱动”映射（摘要）

- “创建任务、展示进度、展示生成结果、SSE 连接与恢复”相关页面：主驱动为 FastAPI。
- “用户域、列表聚合、历史/收藏/设置等长期数据查询与管理”相关页面：主驱动为 RuoYi。
- 结果页通常需要双归属：FastAPI 提供结果与运行时详情，RuoYi 提供长期沉淀索引与个人域动作（收藏、删除、列表等）。

## 9. 冲突 / 待确认点

以下仅记录差异与待确认点，不在本文件中改写事实源：

| 主题 | 现象 | 影响 | 待确认 |
|---|---|---|---|
| 独立登录页路径与注册承载方式 | 当前轮已冻结“登录为独立路由页”，但事实源未显式写出最终路径与注册是否独立页 | 影响导航、拦截与线框补画 | 当前按“建议：`/login`，注册作为同页状态切换”执行；若要拆出 `/register`，需继续冻结。 |
| LearningCenter / Profile 的路径与内部视图边界 | `architecture.md` §14.2 只冻结 `LearningCenter.tsx`、`Profile.tsx` 页面组件 | 若提前写死 `/learning`、`/profile` 以及其下子路由，会超出架构已承诺边界 | 学习中心承担历史 / 收藏聚合，个人中心承担资料 / 设置；具体是内部视图、Tab 还是二级路由，待全站 IA 统一冻结。 |
| `/knowledge`、`/quiz/:sessionId` 的 P0/P1 优先级 | 事实源强调 P0 需“知识问答与学习辅助可演示闭环”，线框图优先级中 `/knowledge` 与 `/quiz/:sessionId` 被标为 P1 | 可能导致“P0 能力”在页面层承载形态不一致 | P0 是否必须提供独立 `/knowledge`、`/quiz` 正式路由页面，或允许先在结果页内以局部交互承载，再在 P1 成品化独立页面。 |
