# 05线框可用性校验｜看板图场景模板

> 节点：校验线框可用性
> 
> 用途：适用于按待办/进行中/已完成管理任务推进，展示责任人与状态。

## 1. 场景目标
围绕“基于任务脚本走查线框可用性，识别断点、冗余与迷失点。”生成可执行的看板图场景方案，用于当前节点的评审、对齐与产出。

## 2. 输入变量（含参考图）
- `{项目名称}`：
- `{业务目标}`：
- `{节点目标}`：默认=`基于任务脚本走查线框可用性，识别断点、冗余与迷失点。`
- `{上游输入}`：默认=`低保真线框、任务脚本、目标用户特征`
- `{预期输出}`：默认=`可用性问题清单+优先级+修订建议`
- `{参考图_1}`：链接或附件说明（必填）
- `{参考图_2}`：链接或附件说明（可选）
- `{品牌关键词}`：如“专业、可信、简洁”
- `{禁用项}`：如“禁止高饱和霓虹色、禁止装饰性过强图标”
- `{平台约束}`：Web/iOS/Android/大屏
- `{补充约束}`：时间、人力、技术限制

## 3. 约束
- 不改动 00 与 01 在《01_总流程图.md》中的核心结论与顺序。
- 必须生成 5 个候选方案，且候选之间差异明确。
- 先执行硬门槛（可读性、完整性、一致性、可执行性、比例适配）；任一不通过即淘汰。
- 仅对通过硬门槛的候选进行五维打分（业务贴合/信息清晰/视觉层级/实现成本/扩展性）。
- 必须输出 Top1 推荐与 Top2 备选，并给出取舍理由。
- 必须提供 16:9、4:3、9:16 三比例参数（画布尺寸、栅格、字号层级、间距策略）。
- 输出应可直接复制到AIGC工具执行，不得出现“待补充/略”。

## 4. 执行提示词（内置 core_prompt_v2 机制，可直接复制）
你是资深UI/UX设计系统专家和信息可视化导演。

【任务】
请为“校验线框可用性”输出看板图场景，并严格执行 core_prompt_v2：
1) 先发散生成5个候选；
2) 再硬门槛筛选；
3) 对通过项打分排序；
4) 输出Top1/Top2与三比例参数。

【输入】
- 项目：{项目名称}
- 业务目标：{业务目标}
- 节点目标：{节点目标}
- 上游输入：{上游输入}
- 预期输出：{预期输出}
- 场景类型：看板图场景
- 参考图：{参考图_1}；{参考图_2}
- 品牌关键词：{品牌关键词}
- 禁用项：{禁用项}
- 平台约束：{平台约束}
- 补充约束：{补充约束}

【硬门槛（全部满足才可进入打分）】
- H1 可读性：标题、主信息、辅助信息层级清晰；
- H2 完整性：覆盖该节点必需输入、处理动作、输出结果；
- H3 一致性：术语与《01_总流程图》节点命名一致；
- H4 可执行性：团队可按图直接落地，不依赖隐含前提；
- H5 比例适配：在16:9、4:3、9:16下不丢关键内容。

【打分规则（仅对过门槛方案，满分100）】
- S1 业务贴合度（25）
- S2 信息结构清晰度（25）
- S3 视觉层级与美观度（20）
- S4 实施成本与效率（15）
- S5 扩展与复用性（15）

【输出结构】
A. 候选方案1~5（逐个给出）
- 方案名
- 核心思路（80字内）
- 关键结构（节点/泳道/卡片分组）
- 风险点
- 硬门槛结论（H1~H5逐项：通过/不通过）
- 得分（S1~S5 + 总分）

B. 排名与推荐
- 排名列表（仅列过门槛方案）
- Top1 推荐：适用条件、为什么最佳
- Top2 备选：何时替代Top1

C. 三比例参数
- 16:9：画布尺寸、栅格、字号、间距、图例位置
- 4:3：画布尺寸、栅格、字号、间距、图例位置
- 9:16：画布尺寸、栅格、字号、间距、图例位置

D. 下一步执行清单（最多5条）

## 5. 失败回退策略
- 若5个候选全部未通过硬门槛：
  1) 优先回退修正输入变量中的`{上游输入}`与`{预期输出}`，确保边界清晰；
  2) 参考图不足时，至少补充2张同类场景参考图再重跑；
  3) 缩减视觉表达自由度（固定配色与版式）后再次生成5候选。
- 若Top1与Top2分差<5分：
  1) 追加“团队实现复杂度”作为决策因子；
  2) 先以Top2做小范围评审，24小时内回收反馈再定版。
- 若三比例中任一比例不达标：
  1) 保留信息结构，单独重排该比例版式；
  2) 禁止删减关键节点或关键说明文字。
