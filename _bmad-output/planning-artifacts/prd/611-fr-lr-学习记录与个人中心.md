# 6.11 FR-LR 学习记录与个人中心

## FR-LR-001 历史记录
**描述**  
用户可以查看已完成的课堂、视频、问答或学习辅助生成记录。

**优先级**  
P0

**业务规则**
- 历史记录属于长期业务数据
- 必须可持久化与重新打开

**验收标准**
- AC1：视频、课堂、问答或学习辅助结果完成后自动写入记录
- AC2：历史记录按时间倒序展示
- AC3：点击记录可重新进入对应结果页

## FR-LR-002 收藏管理
**描述**  
用户可收藏课堂、视频、问答结果或学习辅助内容。

**优先级**  
P1

**验收标准**
- AC1：用户可执行收藏与取消收藏
- AC2：收藏状态在结果页可见
- AC3：收藏结果在个人中心或学习中心可查看

## FR-LR-003 删除记录
**描述**  
用户可删除历史记录。

**优先级**  
P2

**验收标准**
- AC1：用户可删除单条历史记录
- AC2：删除前有确认提示
- AC3：删除后列表即时更新

## FR-LR-004 学习中心聚合回看
**描述**  
学习中心应作为学习数据与结果回看的统一聚合入口。

**优先级**  
P0

**验收标准**
- AC1：学习中心可查看视频、课堂、问答、测验、推荐等结果入口
- AC2：各类结果可重新打开
- AC3：长期数据均有明确归属与查询来源

## FR-LR-005 ToB 数据承接边界
**描述**  
后台侧只承接小麦长期业务数据的管理、查询与审计边界，不在当前版本扩展为独立 ToB 产品。

**优先级**  
P1

**验收标准**
- AC1：关键长期业务数据进入 RuoYi 业务表
- AC2：管理能力聚焦查询、记录、审计和导出准备，不扩展无关复杂后台能力
- AC3：FastAPI 不重复建设完整后台 CRUD
