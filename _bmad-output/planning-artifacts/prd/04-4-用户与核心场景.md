# 4. 用户与核心场景

## 4.1 目标用户

| 用户类型 | 核心需求 | 典型场景 |
|---|---|---|
| 高职学生 | 快速理解知识点与难题，获得练习与推荐 | 课后复习、作业难题、考试冲刺 |
| 高职教师 | 快速生成教学视频与课堂内容 | 备课、课堂资源制作、翻转课堂 |
| 自学者 | 定向突破知识盲点 | 补基础、专项练习、技能提升 |
| 营销访客 / 试点线索 | 快速理解产品价值、判断是否试用或合作 | 活动投放、赛事展示、院校试点咨询 |
| 管理角色（预留） | 查看记录、审计与运营数据 | 后台查询、导出、统计 |

## 4.2 核心场景

### 场景 A：主题课堂学习

用户生成课堂内容，在学习过程中可随时追问当前步骤，并在课后进入 checkpoint / quiz。

### 场景 B：单题视频学习

用户生成讲解视频，在播放过程中可针对当前秒点发问，并获得文本 + 白板解释。

### 场景 C：证据回看与来源补充

用户在会话解释不足时，通过来源抽屉 / 证据面板查看教材、讲义、术语或上传文档中的来源依据，并回看引用来源。

### 场景 D：学习教练

系统基于会话结果与学习行为，提供课后练习、错题解析、推荐与路径规划。

### 场景 E：教师备课

教师输入知识点或题目，快速获得可播放视频、基础课堂结果及后续学习建议。

### 场景 F：营销获客与试点转化

访客通过独立营销落地页了解小麦的双入口价值、老师风格、试点方案与合作方式，再进入默认产品首页或提交试点咨询。

### 场景 G：公开视频发现与复用

用户在视频输入页浏览公开讲解视频卡片，快速判断是否直接观看、复用已有讲解，或基于相似题目继续发起新任务。

### 场景 H：课堂生成前联网搜索增强

用户在课堂输入页按需开启联网搜索，并配置证据范围，让课堂生成在必要时参考公开资料或指定资料源。

### 场景 I：课堂结果导出与教学资料外发

用户在课堂结果页将课堂内容导出为可分享、可备课复用的文件产物，用于后续教学沉淀与外发。

## 4.3 当前版本聚焦场景

MVP / 当前版本优先聚焦：

1. 学生单题讲解
2. 学生主题学习
3. 会话解释后的证据补充与来源回看
4. 学习中心中的基础记录聚合
5. 教师快速备课中的单题视频生成与课堂生成
6. 营销场景下的 landing 获客与试点转化
7. 视频输入页中的公开视频发现与复用
8. 课堂生成前的联网搜索增强
9. 课堂结果导出与教学资料外发

## 4.4 当前版本不重点解决

* 完整 ToB 后台产品体系
* 深度院校运营能力
* 自定义 Agent 市场
* 社交关系与分享生态
* 多语言完整商业化体系
* 高级推荐算法
