# 6.9 FR-SE 实时进度与 SSE

## FR-SE-001 SSE 实时进度推送
**描述**  
长任务执行过程中，系统通过 SSE 向前端推送阶段进度与状态变化。

**优先级**  
P0

**验收标准**
- AC1：任务创建后前端可建立 SSE 连接
- AC2：系统推送 connected、progress、completed、failed、heartbeat 等关键事件
- AC3：前端可基于事件更新任务进度与状态

## FR-SE-002 SSE 断线恢复
**描述**  
当连接短暂中断时，系统应支持基于 Redis 状态与事件缓存恢复任务状态。

**优先级**  
P0

**验收标准**
- AC1：客户端断线后可自动重连
- AC2：重连后可获取当前状态快照或缺失事件
- AC3：恢复机制不依赖数据库完整回放历史事件

## FR-SE-003 状态查询降级
**描述**  
当 SSE 恢复失败或连接受限时，系统可退化为状态查询能力。

**优先级**  
P1

**验收标准**
- AC1：客户端可通过状态接口查询任务当前状态
- AC2：状态接口返回任务状态、进度和当前阶段
- AC3：该降级路径不影响任务最终完成与结果回看
