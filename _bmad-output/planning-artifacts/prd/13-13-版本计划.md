# 13. 版本计划

## 13.1 阶段 A：契约冻结、mock 建立与并行启动

* 冻结统一 API 响应、错误码、状态枚举、示例 payload 与 mock 样例
* 建立前端 `adapter`、`mock handler`、演示数据集与页面状态矩阵
* 启动登录与鉴权、视频 / 课堂任务主流程、首页双入口、输入页、等待页与结果页的并行开发
* 明确高保真视觉稿、关键状态与交互说明的评审基线

## 13.2 阶段 B：双端并行实现与主链路闭环

* 后端完成视频 / 课堂任务主流程、SSE 恢复语义、Provider 基础能力验证与 RuoYi 业务表回写
* 前端基于稳定契约与 mock 完成首页、视频、课堂、学习中心与个人域正式页面
* 完成空态、错态、加载态、禁用态与权限失败态的页面闭环
* 完成接口测试、契约测试、前端状态测试与演示场景数据集维护

## 13.3 阶段 C：真实接口联调与稳定性收口

* 前端从 mock 切换到真实 API 端点，验证 `adapter` 不需要页面级重写
* 补齐视频 / 课堂 / 历史 / 资料相关接口测试、联调验证与稳定性问题修复
* 验证 Provider 主备切换、任务恢复链路与长期数据写入闭环
* 仅在高保真视觉稿、关键状态、交互说明、稳定接口契约四项门禁齐备后执行主分支合并与发布收口

## 13.4 阶段 D：测试与上线

* P0 E2E 回归
* 稳定性优化
* 性能优化
* 演示环境部署
* 演示脚本固化
