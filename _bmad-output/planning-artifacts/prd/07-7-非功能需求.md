# 7. 非功能需求

## 7.1 性能与可用性

| NFR-ID | 需求 | 目标 | 优先级 |
|---|---|---|---|
| NFR-PF-001 | 非生成类 API 响应时间 | P95 < 200ms | P0 |
| NFR-PF-002 | 视频生成端到端延迟 | P95 < 5 分钟 | P0 |
| NFR-PF-003 | 基础课堂生成端到端延迟 | P95 < 5 分钟 | P0 |
| NFR-PF-004 | 证据检索响应时间 | 主观可接受，优先快速返回结果 | P1 |
| NFR-PF-005 | 视频生成成功率 | > 80% | P0 |
| NFR-PF-006 | 基础课堂生成成功率 | > 90% | P0 |
| NFR-PF-007 | 系统可用性 | MVP ≥ 99% | P1 |
| NFR-PF-008 | 首屏加载时间 FCP | < 1.5s | P1 |

## 7.2 安全与合规

| NFR-ID | 需求 | 说明 | 优先级 |
|---|---|---|---|
| NFR-SE-001 | HTTPS 传输 | 所有外部访问必须通过 HTTPS | P0 |
| NFR-SE-002 | 鉴权一致性 | 鉴权体系与 RuoYi 保持一致 | P0 |
| NFR-SE-003 | 输入校验 | 前后端均进行输入校验 | P0 |
| NFR-SE-004 | XSS 防护 | HTML / 富文本场景需明确消毒策略 | P0 |
| NFR-SE-005 | 安全优先 | Manim 沙箱安全边界不得为成功率让步 | P0 |
| NFR-CO-001 | 隐私合规 | 符合学生数据隐私保护要求 | P1 |
| NFR-CO-002 | AI 内容标识 | 对 AI 生成内容提供必要标识 | P1 |

## 7.3 架构与可维护性约束

| NFR-ID | 需求 | 说明 | 优先级 |
|---|---|---|---|
| NFR-AR-001 | 双后端分层 | FastAPI 做功能服务，RuoYi 做长期业务宿主 | P0 |
| NFR-AR-002 | 长期数据归属明确 | 长期业务数据进入 RuoYi/MySQL 或 COS | P0 |
| NFR-AR-003 | Redis 运行态边界 | Redis 仅用于运行态与事件缓存，且必须设置 TTL | P0 |
| NFR-AR-004 | 模块松耦合 | 视频、课堂、Companion、Evidence / Retrieval、Learning Coach 允许共享基础设施但业务独立 | P0 |
| NFR-AR-005 | 外部能力可替换 | Provider 接入需具备可替换性与 Failover | P1 |
| NFR-AR-006 | 日志可追踪 | 关键链路具备 request\_id / task\_id 与统一日志追踪能力 | P1 |
| NFR-AR-007 | 容器化部署 | 生产环境支持 Linux 容器化部署 | P0 |

## 7.4 兼容性与体验约束

| NFR-ID | 需求 | 说明 | 优先级 |
|---|---|---|---|
| NFR-UX-001 | MVP 以桌面端为主 | 最小支持宽度 1024px | P0 |
| NFR-UX-002 | 浏览器支持 | Chrome / Edge / Firefox 主支持 | P1 |
| NFR-UX-003 | 基础无障碍支持 | 关键交互可键盘操作，颜色对比不明显违规 | P1 |
| NFR-UX-004 | 完整 WCAG AA | 后置到 Post-MVP | P2 |
| NFR-UX-005 | i18n 架构预留 | MVP 默认中文，完整语言切换后置 | P2 |

## 7.5 外部依赖降级策略

| 依赖 | 故障时行为 | 优先级 |
|---|---|---|
| LLM | 自动切换备用 Provider；全部不可用则返回明确失败 | P0 |
| TTS | 自动切换备用 Provider；必要时降级或失败返回 | P0 |
| OCR | 提示手动输入题目 | P0 |
| COS | 执行重试或临时存储降级策略 | P1 |
| Redis | 尽量退化到可接受的状态查询能力 | P1 |
| 腾讯云 ADP | 回退到更基础的回答或显式失败提示 | P1 |
