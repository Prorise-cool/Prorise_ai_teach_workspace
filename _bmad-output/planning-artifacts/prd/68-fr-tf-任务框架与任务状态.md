# 6.8 FR-TF 任务框架与任务状态

## FR-TF-001 统一任务模型
**描述**  
视频任务与课堂任务应采用统一任务模型，保证状态、事件、结果格式一致。

**优先级**  
P0

**验收标准**
- AC1：VideoTask 与 ClassroomTask 使用统一状态枚举
- AC2：任务结果结构可被前端统一消费
- AC3：任务状态、失败与完成路径具备一致性

## FR-TF-002 统一任务状态机
**描述**  
长任务应遵循统一状态流转规则。

**优先级**  
P0

**验收标准**
- AC1：任务至少支持 pending、processing、completed、failed、cancelled 五类状态
- AC2：前后端对状态含义保持一致
- AC3：异常路径不会出现悬挂状态

## FR-TF-003 统一任务错误码
**描述**  
任务框架应支持统一错误码，以支持排障与前端可解释提示。

**优先级**  
P1

**验收标准**
- AC1：关键失败类型有对应错误码
- AC2：前端可基于错误码展示明确提示
- AC3：日志中可追踪错误码与 request_id / task_id
