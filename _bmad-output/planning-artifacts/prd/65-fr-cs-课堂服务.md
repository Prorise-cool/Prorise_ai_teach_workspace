# 6.5 FR-CS 课堂服务

## FR-CS-001 输入主题生成基础课堂
**描述**  
用户输入一个主题后，系统生成一套基础课堂内容。

**优先级**  
P0

**业务规则**
- MVP 阶段聚焦课堂基础闭环
- 不以完整复刻所有复杂课堂机制为目标
- 课堂生成属于长任务

**验收标准**
- AC1：用户输入主题后，可在目标时间内生成可展示课堂内容
- AC2：生成失败时返回可理解错误并支持重试
- AC3：生成过程中前端实时看到当前步骤和进度

## FR-CS-002 Agent 风格选择
**描述**  
用户可在预设老师风格中选择一种用于课堂或视频生成。

**优先级**  
P1

**验收标准**
- AC1：系统提供严肃、幽默、耐心、高效四种风格选项
- AC2：不同风格生成结果在语气、讲解方式和举例倾向上存在可辨识差异
- AC3：若用户未选择，系统使用默认风格

## FR-CS-003 幻灯片展示
**描述**  
课堂内容应包含基础幻灯片结构，支持前端展示。

**优先级**  
P0

**验收标准**
- AC1：生成结果包含多张结构化幻灯片
- AC2：每张幻灯片包含标题和正文要点
- AC3：前端可按顺序切换浏览

## FR-CS-004 基础测验
**描述**  
课堂结束后生成基础测验，并可展示结果。

**优先级**  
P0

**验收标准**
- AC1：系统可生成 3-5 道基础测验题
- AC2：用户提交答案后可看到对错和解析
- AC3：测验结果进入学习记录与学习中心闭环

## FR-CS-005 多 Agent 讨论
**描述**  
课堂中支持多角色 Agent 参与讨论。

**优先级**  
P2

**验收标准**
- AC1：课堂中至少可展示多角色参与的讨论片段
- AC2：讨论过程顺序可控且可呈现
- AC3：用户作为学生输入后可获得回应

## FR-CS-006 课堂进度反馈
**描述**  
课堂生成过程中应向用户反馈当前阶段和进度。

**优先级**  
P0

**验收标准**
- AC1：课堂生成时可建立实时进度连接
- AC2：前端可显示当前阶段描述和进度条
- AC3：连接断开后可恢复当前任务状态

## FR-CS-007 白板布局管理
**描述**  
课堂中的白板展示需避免内容重叠，并具备基础可读性。

**优先级**  
P2

**验收标准**
- AC1：同一时刻白板主要内容不发生严重重叠
- AC2：主要元素可读可见
- AC3：复杂布局问题不阻断 MVP 主链路
