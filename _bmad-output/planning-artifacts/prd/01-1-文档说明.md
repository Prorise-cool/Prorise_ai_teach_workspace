# 1. 文档说明

## 1.1 文档目的

本文档用于定义小麦当前版本的：

* 产品目标
* 版本范围
* 功能需求（FR）
* 非功能需求（NFR）
* 验收标准（AC）
* 测试与上线门禁
* 变更控制机制

本文档面向研发执行，不是市场宣传材料。

## 1.2 文档边界

本文档聚焦：

* 做什么
* 为什么做
* 当前版本范围是什么
* 各模块如何验收
* 哪些指标必须达到
* 哪些测试必须通过

本文档不展开以下实现细节：

* 系统详细代码设计
* 完整接口字段清单
* Redis Key 全量清单
* Provider 协议实现细节
* 沙箱参数与资源限制具体值
* 目录级代码样例

上述内容以架构主文档、开发手册、测试计划为准。

## 1.3 与架构文档的关系

* **PRD 负责**：产品范围、用户场景、优先级、功能需求、非功能目标、验收标准、测试门禁、版本计划
* **架构文档负责**：系统边界、职责划分、模块组织、数据边界、集成模式、技术选型、实现约束

冲突处理规则：

1. 产品范围、优先级、验收标准以 PRD 为准
2. 技术边界、职责边界、数据边界以架构文档为准
3. 重大冲突必须走变更控制流程统一

## 1.4 重构原则

本版 PRD 不是旧文档修补版，而是基于当前架构边界的重构版，遵循以下原则：

1. 以当前架构主文档作为系统边界基准。
2. 继承上一版开发级对齐 PRD 的研发表达方式。
3. 吸收原始 PRD 中仍有效的用户旅程、成功指标、范围哲学与交付约束。
4. 将架构中已明确的新模块正式纳入产品主干，而非仅作为技术附录。
