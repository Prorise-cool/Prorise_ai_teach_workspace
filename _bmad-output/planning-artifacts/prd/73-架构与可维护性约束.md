# 7.3 架构与可维护性约束
| NFR-ID | 需求 | 说明 | 优先级 |
|---|---|---|---|
| NFR-AR-001 | 双后端分层 | FastAPI 做功能服务，RuoYi 做长期业务宿主 | P0 |
| NFR-AR-002 | 长期数据归属明确 | 长期业务数据进入 RuoYi/MySQL 或 COS | P0 |
| NFR-AR-003 | Redis 运行态边界 | Redis 仅用于运行态与事件缓存，且必须设置 TTL | P0 |
| NFR-AR-004 | 模块松耦合 | 视频、课堂、知识问答、学习辅助允许共享基础设施但业务独立 | P0 |
| NFR-AR-005 | 外部能力可替换 | Provider 接入需具备可替换性与 Failover | P1 |
| NFR-AR-006 | 日志可追踪 | 关键链路具备 request_id / task_id 与统一日志追踪能力 | P1 |
| NFR-AR-007 | 容器化部署 | 生产环境支持 Linux 容器化部署 | P0 |
