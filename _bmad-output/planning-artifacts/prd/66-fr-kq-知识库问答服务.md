# 6.6 FR-KQ 知识库问答服务

## FR-KQ-001 知识问答入口
**描述**  
用户可进入知识问答页面，就课程知识点、教材内容或术语发起问题。

**优先级**  
P0

**验收标准**
- AC1：用户可在知识问答页输入问题并提交
- AC2：系统返回可阅读的答案结果
- AC3：失败时提供重试或改写问题提示

## FR-KQ-002 文档上传与解析
**描述**  
系统支持将教材、讲义等文档解析后接入知识问答能力。

**优先级**  
P1

**依赖**
- 文档解析能力
- 外部知识库能力

**验收标准**
- AC1：支持上传规定格式文档进行解析
- AC2：解析完成后文档内容可用于问答检索
- AC3：解析失败时返回明确错误原因

## FR-KQ-003 知识点问答
**描述**  
系统基于知识库与外部能力返回知识点解释、课程相关问答结果。

**优先级**  
P0

**验收标准**
- AC1：系统可围绕课程知识返回针对性回答
- AC2：回答结果与问题语义保持相关
- AC3：无结果时系统返回明确说明而不是编造答案

## FR-KQ-004 引用来源展示
**描述**  
问答结果应尽可能展示引用来源，帮助用户理解答案依据。

**优先级**  
P1

**验收标准**
- AC1：有来源时可展示文档片段、章节或来源标记
- AC2：来源展示不影响主答案阅读
- AC3：来源缺失时系统明确说明来源不足

## FR-KQ-005 术语解释
**描述**  
系统支持对知识问答中的专业术语进行简明解释。

**优先级**  
P1

**验收标准**
- AC1：用户可对术语发起解释请求
- AC2：返回内容适合当前目标用户理解水平
- AC3：术语解释结果可与主问答结果联动展示

## FR-KQ-006 问答记录回写
**描述**  
知识问答结果应回写为长期业务数据，供学习中心或后台查询。

**优先级**  
P0

**验收标准**
- AC1：每次有效知识问答可生成记录摘要
- AC2：问答记录可在学习中心重新查看
- AC3：长期记录进入 RuoYi 业务表而非仅保留在 Redis
