# 2. 产品概述

## 2.1 产品愿景
小麦是一款面向中国高职教育场景的 AIGC 原生教学平台，通过双入口与学习辅助能力，帮助学生更快理解知识点、题目与学习路径，帮助教师更高效地产出可播放、可复用的教学内容。

## 2.2 产品定位
小麦当前提供三类核心能力层：

### 核心能力层 A：双入口学习体验
- **入口 1：初学场景**
  - 用户输入主题
  - 系统生成基础课堂内容、幻灯片、课堂测验与互动讲解
- **入口 2：问答场景**
  - 用户输入题目文本或上传题目图片
  - 系统生成单题讲解视频并支持播放复习

### 核心能力层 B：知识与学习辅助
- 课程知识库问答
- 术语解释
- 文档解析后知识问答
- 错题解析
- 课后小测
- 学习路径规划
- 知识点推荐
- 错题本与学习中心聚合

### 核心能力层 C：结果回看与数据闭环
- 历史记录
- 收藏管理
- 测验结果记录
- 问答记录回写
- 学习中心聚合展示

## 2.3 当前版本目标
当前版本目标是在赛事周期内交付一个可稳定演示、可持续扩展的 MVP / 公测版，满足：
- 用户可登录并进入双入口
- 视频生成主链路稳定打通
- 基础课堂链路稳定打通
- 知识问答与学习辅助具备可演示闭环
- 长任务过程可见、状态可恢复、结果可回看
- 长期业务数据有明确持久化归属
- 外部能力具备基础 Failover 与降级策略

## 2.4 核心价值
### 对学生
- 降低抽象知识理解难度
- 缩短单题学习时间
- 提供更具解释性的学习体验
- 提供从“提问—理解—练习—记录”的连续学习闭环

### 对教师
- 降低教学视频制作成本
- 提高备课与资源生成效率
- 快速产出可播放、可管理的教学内容
- 逐步形成可复用的知识资源与学习记录体系

## 2.5 当前版本的产品边界变化
相较于旧版 PRD，本版本已不再只聚焦“课堂 + 视频 + 历史记录”，而是正式纳入：
- 知识库问答服务
- 学习辅助服务
- 学习中心
- 统一任务框架
- SSE 恢复机制
- Provider 抽象与外部能力编排
- Async Worker 执行边界
- RuoYi 业务表数据承接边界

# 3. 架构对齐摘要

## 3.1 系统分层
系统采用双后端分层：

### FastAPI
负责：
- AI 功能执行
- 异步任务协调
- 视频与课堂生成
- Provider 调度
- SSE 进度推送
- 外部能力编排
- 运行时状态管理

### RuoYi
负责：
- 用户、角色、权限
- 小麦长期业务数据持久化
- 学习记录、收藏、任务元数据、问答日志等业务表承接
- 查询、审计、导出与管理宿主能力

## 3.2 数据边界
- Redis 只用于运行时状态、Token 在线态、SSE 事件缓存、短期上下文、Provider 健康状态
- Redis 中的数据必须设置 TTL
- 长期业务数据必须进入 RuoYi/MySQL 或 COS
- FastAPI 不承担长期业务主库存储职责
- 文件产物统一进入 COS

## 3.3 模块边界
- 课堂服务与视频服务是两个独立业务模块
- 知识问答与学习辅助是独立产品能力域
- 各模块可共享任务框架、Provider 抽象层、SSE 基础设施、RuoYi 防腐层、COS 客户端
- 禁止跨业务模块直接耦合具体实现逻辑

## 3.4 外部能力边界
- 腾讯云 ADP 作为外部 AI 能力扩展层接入
- 主链路稳定性不得依赖单一外部平台
- 用户前端体验全部自研，不直接暴露外部平台原生前端组件
- 外部依赖均需具备降级策略

## 3.5 Agent 风格边界
- Agent 风格属于数据层人格预设，不是 UI 全局主题
- 前端整体品牌风格统一
- 不同风格仅体现在内容语气、头像、局部点缀、语音参数等层面

# 4. 用户与核心场景

## 4.1 目标用户
| 用户类型 | 核心需求 | 典型场景 |
|---|---|---|
| 高职学生 | 快速理解知识点与难题，获得练习与推荐 | 课后复习、作业难题、考试冲刺 |
| 高职教师 | 快速生成教学视频与课堂内容 | 备课、课堂资源制作、翻转课堂 |
| 自学者 | 定向突破知识盲点 | 补基础、专项练习、技能提升 |
| 管理角色（预留） | 查看记录、审计与运营数据 | 后台查询、导出、统计 |

## 4.2 核心场景
### 场景 A：主题学习
用户输入“我想学 XXX”，系统生成基础课堂内容，并在课堂结束后提供测验与学习记录。

### 场景 B：单题讲解
用户输入题目文本或上传题目图片，系统生成单题讲解视频，用户可播放、倍速、收藏与回看。

### 场景 C：课程知识问答
用户围绕某课程或已上传教材进行提问，系统返回知识点解释、引用来源和术语说明。

### 场景 D：学习辅助
系统基于课堂、视频、问答或错题结果，提供课后小测、错题解析、学习路径规划、知识点推荐。

### 场景 E：教师备课
教师输入知识点或题目，快速获得可播放视频、基础课堂结果及后续练习建议。

## 4.3 当前版本聚焦场景
MVP / 当前版本优先聚焦：
1. 学生单题讲解
2. 学生主题学习
3. 课程知识问答
4. 学习中心中的基础记录聚合
5. 教师快速备课中的单题视频生成与课堂生成

## 4.4 当前版本不重点解决
- 完整 ToB 后台产品体系
- 深度院校运营能力
- 自定义 Agent 市场
- 社交关系与分享生态
- 多语言完整商业化体系
- 高级推荐算法

- 主链路稳定可演示
- 失败路径可解释
- 数据有归属
- 任务过程可见且可恢复
- 结果可回看

# 6. 功能需求

## 6.1 FR-UM 用户与认证

### FR-UM-001 用户注册/登录
**描述**  
用户可以完成注册、登录、登出，并进入小麦前端功能页面。

**优先级**  
P0

**依赖**
- RuoYi 用户体系
- 前端登录页与鉴权拦截

**业务规则**
- 未登录用户访问受保护页面必须被拦截
- 登录态需在前端、FastAPI 与 RuoYi 层面保持一致
- FastAPI 不自建独立用户体系

**验收标准**
- AC1：用户输入正确凭证后，可在 2 秒内完成登录并进入首页
- AC2：未登录用户访问受保护页面时，自动跳转登录页
- AC3：登录失败时返回明确错误提示，不泄露敏感账户信息
- AC4：用户登出后原登录态立即失效

### FR-UM-002 Token 鉴权一致性
**描述**  
前端、FastAPI 与 RuoYi 对用户登录态保持一致判断。

**优先级**  
P0

**依赖**
- Redis 在线态
- 统一鉴权头

**验收标准**
- AC1：合法登录态可访问受保护的 FastAPI 接口
- AC2：失效或非法 Token 调用接口返回 401
- AC3：用户登出后再次调用受保护接口应失败

### FR-UM-003 个人资料查看与修改
**描述**  
用户可在个人中心查看并修改基础个人信息。

**优先级**  
P1

**验收标准**
- AC1：用户可查看昵称、头像、学校等基础信息
- AC2：用户可修改昵称和头像，并即时看到更新结果
- AC3：非法格式或超限文件上传被拒绝并提示原因

### FR-UM-004 权限分层
**描述**  
系统需基于 RuoYi 角色体系区分学生、教师和管理角色的能力边界。

**优先级**  
P1

**验收标准**
- AC1：学生角色仅可访问 ToC 学习功能
- AC2：教师角色可访问教师范围内的结果管理能力
- AC3：管理角色能力不直接暴露在 ToC 主导航中

P0

P0

**描述**  
播放器页承载视频播放、倍速、全屏、信息查看及结果操作。

**优先级**  
P0

**验收标准**
- AC1：播放器位于页面主体区域
- AC2：播放器下方显示题目信息或知识点标签
- AC3：提供收藏、重新生成、返回等基础操作入口

### FR-UI-005 个人中心
**描述**  
个人中心展示用户基础资料、历史记录、收藏、设置等信息。

**优先级**  
P1

**验收标准**
- AC1：页面展示用户头像、昵称、学校信息
- AC2：页面提供历史记录与设置入口
- AC3：页面支持个人信息修改入口

### FR-UI-006 知识问答页
**描述**  
知识问答页承载知识库问答、术语解释、引用来源与文档解析后的问答体验。

**优先级**  
P0

**验收标准**
- AC1：页面支持用户输入知识问题并发起提问
- AC2：系统返回回答时可展示引用来源或文档片段
- AC3：回答失败时用户获得明确提示并可重试

### FR-UI-007 学习中心页
**描述**  
学习中心用于聚合学习记录、收藏、问答记录、测验结果与推荐内容。

**优先级**  
P0

**验收标准**
- AC1：页面聚合展示至少历史记录、收藏、测验结果三个区域
- AC2：用户可从学习中心进入对应明细结果页
- AC3：页面展示结构清晰，不因聚合导致信息混乱

### FR-UI-008 国际化架构预留
**描述**  
前端需为中英双语切换保留架构能力。

**优先级**  
P2

**验收标准**
- AC1：静态文本具备国际化资源管理方式
- AC2：未来接入语言切换时不需要重写页面结构
- AC3：当前版本默认中文，不阻断 MVP 交付

## 6.3 FR-VS 视频服务

### FR-VS-001 题目输入
**描述**  
用户可通过文本输入或图片上传提交题目。

**优先级**  
P0

**依赖**
- 文本输入组件
- 图片上传组件
- OCR 能力

**业务规则**
- 文本输入为默认方式
- 图片上传是补充输入方式
- OCR 失败时必须提供手动输入替代路径

**验收标准**
- AC1：用户可提交文本题目并创建视频任务
- AC2：用户可上传 JPG/PNG 图片并提交视频任务
- AC3：图片识别失败时系统提示用户改为手动输入

### FR-VS-002 题目理解
**描述**  
系统需将题目解析为结构化理解结果，用于后续分镜与视频生成。

**优先级**  
P0

**验收标准**
- AC1：系统可提取题型、知识点和解题结构
- AC2：无法理解题目时返回明确提示而非直接报错
- AC3：理解结果可供后续阶段继续使用

### FR-VS-003 分镜生成
**描述**  
系统基于题目理解生成讲解视频分镜方案。

**优先级**  
P0

**验收标准**
- AC1：系统生成不少于 3 个有效分镜
- AC2：每个分镜包含场景描述、讲解重点和旁白内容
- AC3：分镜总时长控制在目标范围内，不明显偏离题目复杂度

### FR-VS-004 Manim 代码生成
**描述**  
系统基于分镜生成可渲染的 Manim 动画代码。

**优先级**  
P0

**验收标准**
- AC1：生成代码通过基础语法检查
- AC2：代码包含完整可执行场景定义
- AC3：生成结果可进入渲染阶段继续处理

### FR-VS-005 代码自动修复
**描述**  
渲染失败后，系统自动尝试修复 Manim 代码。

**优先级**  
P0

**业务规则**
- 系统执行预设次数的自动修复
- 修复失败后任务必须正确结束，不得悬挂
- 高质量方案失败时可降级返回较低质量结果

- AC3：修复失败后返回清晰失败结果并记录任务状态
- AC4：存在降级结果时优先返回可用结果

# 8. 数据与集成约束

## 8.1 长期业务数据
以下数据必须可持久化、可查询、可追溯：
- 视频任务元数据
- 课堂会话摘要
- 学习记录
- 收藏记录
- 测验结果
- 问答记录
- 错题本数据
- 学习路径结果
- 推荐结果摘要（如需长期回看）

## 8.2 运行时数据
以下数据属于运行时状态，不要求作为长期业务表实时承接：
- 任务阶段进度
- SSE 过程事件
- 短期会话上下文
- Provider 健康状态
- 临时恢复快照

## 8.3 集成边界
- 前端通过 HTTP / SSE 调用 FastAPI 获取功能能力
- 用户与基础业务能力与 RuoYi 保持一致
- FastAPI 与 RuoYi 之间通过防腐层交互
- 外部平台能力通过抽象层接入，不直接暴露原生前端组件

## 8.4 数据归属原则
- 用户在个人中心、学习中心可见的数据属于长期业务数据，应进入 RuoYi/MySQL
- 文件产物进入 COS
- Redis 不作为学习记录、收藏、问答记录、测验结果的长期存储宿主

# 14. 变更控制

## 14.1 冻结规则
- 第 1 周开始：P0 功能冻结
- 第 2 周开始：MVP 范围冻结
- 第 5 周开始：代码冻结，仅允许高优先级缺陷修复

## 14.2 变更原则
若新增需求满足以下任一条件，应默认不进入当前 MVP：
- 跨模块引入新依赖
- 影响主链路稳定性
- 需要新增复杂后台能力
- 无法在现有测试门禁下验证
- 对演示价值提升有限但开发成本高
