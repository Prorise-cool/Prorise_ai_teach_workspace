# Sprint Change Proposal：Epic 1 首页入口口径改道

日期：2026-04-02  
模式：Batch  
触发 Story：`1.4`

## 1. 问题摘要

- 当前 `Story 1.4`、Epic 1、PRD `FR-UI-001` 与 UX 页面规范，把默认首页 `/` 定义成“首页双入口 + 推荐提示”。
- 该定义与现有成品图 `docs/03UI:UX 设计素材/001UI 设计稿/04-成品图/01-正式路由页面/01-首页与入口/02-home.html` 不一致。
- 最新产品纠偏口径明确：首页不是双入口比较页；首页主入口应直接进入课堂输入页，其它入口通过顶边栏导航进入。

## 2. 影响分析

### 2.1 Epic 影响

- `Epic 1` 需要从“理解双入口差异”改为“首页课堂直达入口 + 顶栏导航分发”。
- `Story 1.4` 需要整体重写，不再承接推荐提示与双入口卡片。
- `Story 1.6` 需把入口级权限可见性从“首页入口壳层”扩展到“首页 + 顶栏导航壳层”。
- `Story 1.7` 需同步默认首页职责描述，避免继续把 `/` 写成双入口页。

### 2.2 PRD / UX / Architecture 影响

- PRD `FR-UI-001` 需从“首页双入口展示”调整为“首页主入口与顶栏导航分发”。
- 路由清单中 `/` 的职责边界需改成“课堂直达入口 + 顶栏导航”，不能再写推荐分流。
- UX `7.2 首页与认证页` 需把首页模块从“双入口卡片 + 推荐提示区”改成“单一主 CTA + 顶栏导航”。
- Architecture 中对 `Home.tsx` 与首页模块的说明需同步改名，避免实现层继续照着双入口结构拆组件。

### 2.3 次级文档影响

- `sprint-status.yaml` 的 Story 标题键值需同步更新。
- 开发手册中的成品图对齐说明、模块总览、M2 里程碑描述需同步修正。

## 3. 推荐路径

- 选择 `Option 1: Direct Adjustment`。
- 理由：
  - 这是需求理解修正，不是产品目标重做。
  - 影响集中在文档与 Story 执行口径，不需要回滚已完成代码。
  - 直接修正事实源后，可避免后续前端继续按错误双入口结构开发。
- 评估：
  - 工作量：Low
  - 风险：Medium
  - 时间影响：低于 1 个工作日，主要是文档与 Story 口径重排

## 4. 详细变更提案

### 4.1 Story 1.4

OLD:
- 首页双入口理解与非阻塞推荐提示
- 双入口卡片
- 推荐提示区
- 点击任一入口后进入目标输入页

NEW:
- 首页课堂直达入口与顶栏导航分发
- 单一主 CTA，默认进入 `/classroom/input`
- 其它学习入口通过顶栏导航进入
- 未登录点击主 CTA 或受保护导航入口时保留 `returnTo`

Rationale：与成品图和最新产品口径保持一致，避免首页继续承担双入口比较页职责。

### 4.2 PRD `FR-UI-001`

OLD:
- 首页双入口展示
- 首页展示两个核心入口

NEW:
- 首页主入口与顶栏导航分发
- 首页首屏展示单一主 CTA，顶栏导航承接其它有效入口

Rationale：把需求语言改成实现可执行的真实页面职责。

### 4.3 UX 页面规范

OLD:
- 首页模块：双入口卡片、推荐提示区

NEW:
- 首页模块：顶栏导航、单一主 CTA、最近学习入口、登录入口

Rationale：恢复为成品图实际结构，避免 UI 继续围绕双卡片推进。

### 4.4 Epic / Sprint 状态

OLD:
- Epic 1 目标强调“双入口差异”
- sprint-status 中 Story 1.4 标题沿用旧口径

NEW:
- Epic 1 目标强调“课堂直达 + 顶栏导航”
- sprint-status 与索引统一改为新 Story 标题

Rationale：保证 backlog、执行文档与状态追踪用同一套命名。

## 5. 实施交接

- 变更级别：Moderate
- 交接对象：
  - PO / SM：确认 Epic 1 与 Story 1.4 的新口径作为后续唯一事实源
  - 前端开发：按新 Story 1.4 实现首页，不再做双入口卡片页
  - QA / Review：把“首页主 CTA 直达 `/classroom/input`”与“顶栏导航进入其他入口”加入回归重点

## 6. 成功标准

- `_bmad-output` 下 PRD、Epic、Story、UX、sprint-status 不再出现首页双入口作为当前事实源。
- 默认首页 `/` 的职责稳定为“课堂直达入口 + 顶栏导航分发”。
- 后续实现与测试不再以“双入口首页”为验收前提。
