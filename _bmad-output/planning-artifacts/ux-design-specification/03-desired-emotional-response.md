## Desired Emotional Response

### Primary Emotional Goals

**核心情感目标：被理解 + 惊喜感 + 分享欲**

小麦的情感设计围绕三个核心情感展开：

1. **被理解感** —— "这个产品懂我"
   * 首次使用时提供清晰入口和合理默认值
   * 用户感觉"这是为我准备的产品"

2. **惊喜感** —— "这也太厉害了"
   * 速度 + 质量的双重超预期
   * Manim 动画流畅呈现，视觉冲击力强

3. **分享欲** —— "要分享给室友"
   * 结果好到忍不住想展示
   * 分享功能便捷，成为传播引擎

### Emotional Journey Mapping

| 旅程阶段 | 用户情绪 | 设计响应 |
|----------|----------|----------|
| **首次打开** | "这是为我准备的" | 入口清晰、默认值可直接使用 |
| **配置用户信息** | "系统懂我了" | 首次引导清晰、后续自动应用 |
| **提交需求** | "开始了，有点期待" | 即时确认，阶段透明 |
| **等待生成** | "系统在认真工作" | 真实进度，不搞花哨（技术加速优先） |
| **看到结果** | "太厉害了，要分享" | 视觉冲击，一键分享 |
| **遇到失败** | "没关系，再试一次" | 失败可控，原因明确 |
| **再次使用** | "还是好用" | 记住最近使用配置，更顺滑 |

### Micro-Emotions

| 微情感 | 重要性 | 设计应对 |
|--------|--------|----------|
| **被理解感** | ⭐⭐⭐⭐⭐ | 清晰入口，智能推荐 |
| **期待感** | ⭐⭐⭐⭐ | 进度透明，阶段可见 |
| **惊叹感** | ⭐⭐⭐⭐⭐ | 视觉质量超预期 |
| **分享欲** | ⭐⭐⭐⭐⭐ | 结果优质 + 分享便捷 |
| **宽容感** | ⭐⭐⭐⭐ | 失败可控，不焦虑 |

**要避免的负面情绪：**

* ❌ 困惑感 —— "我该选哪个？"
* ❌ 焦虑感 —— "是不是卡住了？"
* ❌ 挫败感 —— "又失败了，算了"
* ❌ 失望感 —— "就这？"

### Design Implications

**情感 → 设计连接：**

| 情感目标 | UX 设计决策 |
|----------|-------------|
| **被理解感** | 首次使用引导用户输入个人简介和学习偏好，系统智能适配生成内容 |
| **等待信任感** | 真实进度反馈，阶段描述具体，预计时间准确；**重点：技术加速 > UX 技巧** |
| **结果惊叹感** | Manim 动画质量第一眼就惊艳，TTS 配音清晰专业 |
| **分享欲** | 生成完成时突出分享按钮，支持一键复制链接/生成海报 |
| **失败宽容感** | 失败时明确告知原因（如"Manim 渲染失败，正在重试"），提供"再试一次"的明确路径 |

**关键设计决策：**

* 等待体验优化以技术加速为主，UX 不搞花哨的等待动画
* 分享功能是增长引擎，必须在结果页突出
* 用户配置在首次使用时完成，后续任务自动应用，无需重复操作

### Emotional Design Principles

**原则 1：被理解优先**

* 首次使用先提供清晰入口和合理默认值
* 智能推荐优于增加额外配置步骤

**原则 2：惊喜驱动分享**

* 结果必须超出预期，才能激发分享欲
* 分享功能必须便捷，一键完成

**原则 3：透明建立信任**

* 等待时真实反馈进度，不模糊处理
* 失败时明确告知原因，不隐藏问题

**原则 4：技术解决根本**

* 等待焦虑的最好解药是更快的生成速度
* UX 配合技术，而非掩盖技术瓶颈
