## 10. Learning Coach Layer（学习教练层）

### 10.1 设计原则

* 所有学习教练能力都发生在会话后，而不是会话中强插。
* 重点不是“再解释一遍”，而是“检验理解、指出薄弱点、给下一步动作”。
* 结果必须回流学习中心，可回看、可继续。

### 10.2 Checkpoint

**角色**

* 轻量理解检查。
* 在课堂或视频结束后给用户一个低打扰入口。

**交互要求**

* 题量少、反馈快。
* 可以直接进入 quiz，也可以先结束本次会话。
* 不要求独立大页面视觉复杂度，但要有独立状态闭环。

### 10.3 Quiz

**角色**

* 正式作答、判题、解析、结果回写。

**MVP 题型边界**

* 单选题优先。
* 多选、填空、主观题后置。

**关键状态**

* 作答前。
* 已选择未提交。
* 判题中。
* 单题解析。
* 全部完成后的总结页。

### 10.4 Learning Path

**角色**

* 结合目标、周期与近期学习结果，生成阶段性学习路径。

**关键模块**

* 目标选择。
* 周期选择。
* 周计划 / 阶段任务。
* 保存、调整、开始学习。

### 10.5 与学习中心关系

* Learning Coach 的结果默认进入 `/learning`。
* `/learning` 是查看 quiz 结果、错题、推荐、路径的聚合入口。
* 个人中心不承担 Learning Coach 结果聚合。
