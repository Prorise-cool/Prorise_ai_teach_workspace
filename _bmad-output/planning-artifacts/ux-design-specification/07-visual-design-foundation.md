## Visual Design Foundation

### 1. Color System (色彩系统)

**主色系（小麦品牌色）：**

| 颜色角色 | HEX | 用途 | 暗色模式 |
|----------|-----|------|----------|
| **主色** | #f5c547 | 品牌、强调、按钮、氛围光晕 | #e6b841 |
| **副色** | #f5ede1 | 背景、卡片 | #2a2420 |
| **装饰色** | #3b1701 | 文字、边框、图标 | #f5ede1 |

**语义化颜色映射：**

| 语义 | 亮色模式 | 暗色模式 | 用途 |
|------|----------|----------|------|
| **Primary** | #f5c547 | #e6b841 | 主要按钮、链接、高亮焦点 |
| **Background** | #f5ede1 | #1a1614 | 页面底层背景 |
| **Surface** | #ffffff | #2a2420 | 卡片、弹窗、对话框本体 |
| **Text Primary** | #3b1701 | #f5ede1 | 主要文字 |
| **Text Secondary** | #6b4421 | #c4b8a8 | 次要文字、辅助信息 |
| **Border** | #e6dcc8 | #3d3630 | 边框、分割线 |
| **Success** | #52c41a | #73d13d | 成功状态 |
| **Warning** | #faad14 | #ffc53d | 警告状态 |
| **Error** | #ff4d4f | #ff7875 | 错误状态 |

**颜色使用原则：**

* 亮色模式以暖黄色调为主，营造温暖友好氛围；暗色模式保持温暖感，避免过冷的蓝灰色
* 所有文字颜色对比度严格符合 WCAG AA 标准（4.5:1）

### 3. Layout & Structure (布局与结构)

引入 OpenMAIC 架构理念，页面从"传统的图文左右布局"进化为\*\*"以对话/输入为核心的沉浸式布局"\*\*。

**核心布局原则：**

* **悬浮导航 (Floating Nav)**：摒弃笨重的全宽顶部导航栏，采用右上角悬浮药丸形 (Pill-shape) 工具栏，释放最大垂直空间给内容
* **视觉聚焦 (Centered Hero)**：核心交互区（输入框/对话框）水平居中，最大宽度限制为 800px，确保视线聚焦，减少眼球左右移动
* **留白与呼吸感**：增加模块外边距（以 vh 为单位计算垂直边距），营造轻松、不压抑的对话环境

**间距系统（4px 基础单位）：**

* `--spacing-2` (8px): 紧凑元素间距（如图标与文字）
* `--spacing-4` (16px): 默认间距（如按钮内边距）
* `--spacing-6` (24px): 快捷指令间的气泡间距
* `--spacing-10` (40px): 核心输入框的内部上下留白

### 4. Material & Depth (材质与层级系统)

结合现代 Web 设计趋势，引入\*\*毛玻璃（Glassmorphism）\*\*与多层级阴影，让界面产生通透感与层次感。

**圆角系统 (Radius)：**

* `--radius-sm` (4px): 内部小图标、复选框
* `--radius-md` (8px): 标准卡片、弹窗内元素
* `--radius-lg` (16px): 核心对话框主体 (Main Chat Box)
* `--radius-full` (9999px): 悬浮导航条、快捷指令药丸标签、头像

**材质与阴影 (Shadow & Blur)：**

| Token | 值 (实现细节) | 用途 |
|-------|---------------|------|
| **Glass Nav** | bg-white/60 backdrop-blur-md | 右上角悬浮导航，透出底层光晕 |
| **Glass Surface** | bg-white/80 backdrop-blur-xl | 核心对话框，高级通透感 |
| **Shadow SM** | 0 1px 2px rgba(0,0,0,0.05) | 快捷指令、导航栏 |
| **Shadow MD** | 0 4px 12px rgba(0,0,0,0.08) | 按钮悬浮、活跃态卡片 |
| **Shadow LG** | 0 8px 24px rgba(0,0,0,0.12) | 核心对话框默认状态 |
| **Shadow XL (Glow)** | 0 20px 25px -5px rgba(114,46,209,0.1) | 核心对话框 Focus 状态发光阴影 |

### 5. Animation System (动画与动效系统)

动效是赋予"小麦"生命力的关键，设计需遵循\*\*"克制、流畅、有呼吸感"\*\*的原则。

**1. 氛围光晕 (Background Glows) - 呼吸感**

* **实现方式**：在底层背景上放置超大圆角的绝对定位元素，使用 blur-3xl（极度模糊）和低透明度（10%-20%）
* **动画定义**：animate-pulse (周期 3~4 秒)：模拟呼吸的频率
* **色彩搭配**：左上角为主色 (var(--color-primary))，右下角为风格色或品牌点缀色（如 blue-500），形成对角线色彩呼应

**2. 交互反馈 (Interaction Transitions) - 流畅性**

* **全局过渡**：所有的颜色切换、边框变化、阴影变化默认使用 `transition-all duration-300 ease-in-out`
* **按钮与卡片悬停**：
  * 浮动：`hover:-translate-y-0.5`（轻微上浮 2px）配合阴影加深（SM -> MD）
  * 按压：`active:scale-[0.97]`（轻微下陷，提供真实的点击物理反馈）
* **焦点状态**：当用户在核心聊天框输入时，外层容器阴影由 shadow-lg 扩散至 shadow-2xl，边框颜色平滑过渡为主色（Primary），视觉完全聚焦

**3. 图标微动效 (Icon Micro-interactions)**

* 悬停在特殊操作（如 OpenMAIC Star 按钮或笔形编辑图标）时，图标触发微缩放 `group-hover:scale-110`，增加趣味性

### 6. Accessibility Considerations (无障碍设计规范)

| 要求 | 标准 | 实现 |
|------|------|------|
| **色彩对比度** | WCAG AA (4.5:1) | 无论亮暗色模式，文本与背景对比度强制达标。按钮文字使用深色（#3b1701）确保在黄底上清晰可见 |
| **键盘导航** | 全功能可键盘操作 | 输入框、按钮均支持 Tab 切换，Focus 状态除了阴影变化，强制保留系统 Outline |
| **屏幕阅读器** | ARIA 标签完整 | 图标按钮（如日月切换、录音键）均包含 aria-label 属性 |
| **动画控制** | 尊重系统设置 | 使用 `@media (prefers-reduced-motion)` 媒体查询，在需要时关闭脉冲光晕，防止光敏不适 |
