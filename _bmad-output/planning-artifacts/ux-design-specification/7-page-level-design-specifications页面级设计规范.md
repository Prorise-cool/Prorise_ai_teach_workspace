# 7. Page-Level Design Specifications（页面级设计规范）

> **架构对齐**：根据架构文档，小麦采用双后端分层架构（FastAPI 功能服务 + RuoYi 管理后台），前端统一与双后端交互。本节定义各页面的 UX 设计规范。

## 7.2 首页/双入口页面

**路由**：`/`

**页面目标**：让用户在 3 秒内理解产品，并直觉化选择入口。

**布局结构**：

```
┌─────────────────────────────────────────────────────────────────┐
│  [Logo]                                    [登录] [注册] [🌙]    │  ← 悬浮导航
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                     "你想怎么学？"                               │  ← 主标题
│                                                                 │
│     ┌─────────────────────┐    ┌─────────────────────┐         │
│     │   📚 初学场景        │    │   ❓ 问答 Agent     │         │  ← 双入口卡片
│     │                     │    │                     │         │
│     │   "我想学 XXX"      │    │   "帮我讲这道题"    │         │
│     │   完整虚拟教室体验    │    │   单题视频讲解      │         │
│     │                     │    │                     │         │
│     │   [开始学习] →      │    │   [开始提问] →      │         │
│     └─────────────────────┘    └─────────────────────┘         │
│                                                                 │
│                     💡 智能推荐提示区                            │  ← 可选
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**交互规范**：

| 元素 | 交互 | 反馈 |
|------|------|------|
| **入口卡片** | 鼠标悬停 | 卡片上浮 2px + 阴影加深 + 对应风格色边框 |
| **入口卡片** | 点击 | 页面平滑过渡到对应输入页 |
| **智能推荐** | 检测到历史 | "检测到你最近在做数学题，推荐使用问答入口" |
| **登录/注册** | 点击 | 弹出对话框，不跳转页面 |

**文案规范**：

| 场景 | 文案 |
|------|------|
| **初学场景标题** | "我想系统地学" |
| **初学场景描述** | "输入主题，获得完整的虚拟教室体验：AI 老师讲解、幻灯片、测验" |
| **问答 Agent 标题** | "我有道题不会" |
| **问答 Agent 描述** | "输入题目，5 分钟获得 Manim 高质量动画讲解视频" |

## 7.3 问答 Agent 输入页面

**路由**：`/video/input`

**页面目标**：极简输入，3 秒内开始。

**布局结构**：

```
┌─────────────────────────────────────────────────────────────────┐
│  [← 返回]                                              [登录]   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│              "输入你的题目，AI 老师来帮你"                        │  ← 副标题
│                                                                 │
│     ┌─────────────────────────────────────────────────────┐    │
│     │                                                     │    │
│     │   [📷 拍照]  [📋 粘贴]  [⌨️ 打字]   [🎓 严谨教授 ▾]  │    │  ← 输入方式 + 会话配置触发器
│     │                                                     │    │
│     │   ┌─────────────────────────────────────────────┐   │    │
│     │   │                                             │   │    │  ← 输入区域
│     │   │  在这里输入或粘贴题目...                     │   │    │
│     │   │                                             │   │    │
│     │   └─────────────────────────────────────────────┘   │    │
│     │                                                     │    │
│     │   ┌─────────────────────────────────────────────┐   │    │  ← 选择器展开态（示意）
│     │   │  选择当前会话老师风格                        │   │    │
│     │   │  ✓ 🎓 严谨教授                               │   │    │
│     │   │    😄 风趣老师                               │   │    │
│     │   │    🌸 温和导师                               │   │    │
│     │   │    ⚡ 干练讲师                               │   │    │
│     │   └─────────────────────────────────────────────┘   │    │
│     │                                                     │    │
│     └─────────────────────────────────────────────────────┘    │
│                                                                 │
│                       [开始生成讲解视频]                         │  ← 主按钮
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**输入方式规范**：

| 方式 | 交互 | 识别流程 |
|------|------|----------|
| **打字** | 直接输入文本 | 无需识别，直接提交 |
| **拍照** | 点击 → 调用摄像头/相册 → OCR 识别 | 腾讯云 OCR → 填充输入框 |
| **粘贴** | Ctrl/Cmd+V | 自动检测图片/文本 → 填充输入框 |

**风格选择器规范**：

| 元素 | 交互 | 反馈 |
|------|------|------|
| **触发器（头像 / 胶囊）** | 点击 | 在输入框右上角或近输入框位置展开当前会话风格面板 |
| **风格项** | 点击选中 | 更新当前会话风格、触发器文案/头像与选中态 |
| **展开面板** | 展示列表 | 默认展示 4 种老师风格的头像、名称和选中态，结构支持未来扩展更多老师风格 |
| **选中态** | 当前风格高亮 | 使用勾选、高亮或边框表达当前选择，不要求固定卡片布局 |

**实现约束**：
- 老师风格选择器不是页面中心的 4 张固定卡片，而是附着在核心输入框附近的会话配置入口。
- MVP 阶段默认提供 4 种老师风格，但组件结构必须支持未来扩展更多老师风格，不应写死为仅 4 张固定卡片。

## 7.4 统一生成进度页面

**路由**：`/video/:id/generating` 或 `/classroom/:id/generating`

> **架构对齐**：视频任务和课堂任务遵循**统一任务模型**（见架构文档 5.5），进度页面采用统一的 UX 模式。

**页面目标**：让等待透明、不焦虑，建立信任感。

**布局结构**：

```
┌─────────────────────────────────────────────────────────────────┐
│  [← 返回]                              [任务ID: video_xxx]       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│                     正在为你生成讲解视频                          │  ← 动态标题
│                                                                 │
│     ┌─────────────────────────────────────────────────────┐    │
│     │                                                     │    │
│     │   ✅ 理解题目                         已完成          │    │
│     │   ───────────────────────────                       │    │
│     │                                                     │    │
│     │   ✅ 生成分镜                         已完成          │    │
│     │   ───────────────────────────                       │    │
│     │                                                     │    │
│     │   🔄 生成 Manim 代码                  进行中          │    │  ← 当前阶段
│     │   ═══════════════════░░░░░░░░░░░░░░░  50%            │    │
│     │                                                     │    │
│     │   ⏳ 渲染动画                          等待中          │    │
│     │   ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0%             │    │
│     │                                                     │    │
│     │   ⏳ TTS 配音                          等待中          │    │
│     │   ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0%             │    │
│     │                                                     │    │
│     │   ⏳ 合成视频                          等待中          │    │
│     │   ░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  0%             │    │
│     │                                                     │    │
│     └─────────────────────────────────────────────────────┘    │
│                                                                 │
│                     预计剩余时间：2 分 30 秒                      │  ← 预计时间
│                                                                 │
│                     💡 生成完成后可分享给朋友                     │  ← 提示
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**SSE 事件 → UI 映射**：

| SSE 事件 | UI 响应 |
|----------|---------|
| `connected` | 显示"已连接到服务器" |
| `progress` | 更新对应阶段进度条 + 阶段描述 |
| `provider_switch` | 显示 toast "已切换到备用服务" |
| `completed` | 自动跳转到结果页面 |
| `failed` | 显示错误原因 + "再试一次"按钮 |
| `heartbeat` | 无 UI 变化（后台保活） |
| `snapshot` | 断线重连后恢复进度状态 |

**阶段描述文案**：

| 阶段 | 文案 | 预计耗时 |
|------|------|----------|
| **理解题目** | "正在理解你的题目..." | 10-20s |
| **生成分镜** | "正在设计讲解流程..." | 30-60s |
| **生成 Manim** | "正在生成动画代码..." | 60-120s |
| **渲染动画** | "正在渲染高质量动画..." | 60-120s |
| **TTS 配音** | "正在合成专业配音..." | 20-40s |
| **合成视频** | "正在合成最终视频..." | 20-40s |

**失败处理**：

| 错误码 | 错误描述 | UI 响应 |
|--------|----------|---------|
| `MANIM_RENDER_FAILED` | Manim 渲染失败 | "动画渲染遇到问题，正在自动修复..." |
| `LLM_UNAVAILABLE` | LLM 服务不可用 | "AI 服务暂时繁忙，请稍后重试" + [再试一次] |
| `TTS_ALL_PROVIDERS_FAILED` | 所有 TTS 服务失败 | "语音合成失败，正在重试..." |
| `TIMEOUT` | 任务超时 | "生成超时，请检查网络后重试" |

**播放器交互规范**：

| 功能 | 交互 | 反馈 |
|------|------|------|
| **播放/暂停** | 点击/空格 | 图标切换 + 状态变化 |
| **进度拖动** | 拖拽进度条 | 实时预览时间 |
| **倍速切换** | 点击倍速按钮 | 弹出选项：0.5x / 1x / 1.5x / 2x |
| **全屏** | 点击全屏按钮 | 进入/退出全屏 |
| **截图笔记** | 点击截图按钮 | 保存当前帧 + 时间戳 |

**分享功能规范**：

| 方式 | 交互 | 生成内容 |
|------|------|----------|
| **复制链接** | 点击 → 复制 | 视频播放链接 |
| **生成海报** | 点击 → 下载 | 包含视频封面 + 标题 + 二维码 |
| **分享到微信** | 点击 → 二维码 | 小程序码/公众号链接 |
