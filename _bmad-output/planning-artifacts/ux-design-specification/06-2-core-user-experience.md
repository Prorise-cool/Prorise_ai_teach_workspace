## 2. Core User Experience

### 2.1 Defining Experience

**小麦的定义性体验："输入即惊喜"**

用户输入题目 → 系统智能适配 → 获得超预期的高质量讲解视频

这是一个端到端的体验魔法，不是单一交互。三个支柱缺一不可：

1. **极简输入**：输入题目就行，打字/拍照/粘贴，不要思考
2. **智能适配**：根据用户配置生成适配的 AI 老师，用户感觉"这个老师懂我"
3. **超预期结果**：速度快到惊人（5 分钟内），质量高到恐怖（Manim 级别）

**用户会这样向朋友描述：**
"你输入题目，5 分钟就有讲解视频，这个 AI 老师很懂我！"

### 2.2 User Mental Model

**用户现有的心智模型：**

| 来源 | 心智模型 | 小麦的适配 |
|------|----------|------------|
| **B站** | 搜索视频 → 选择UP主 → 播放 | 输入题目 → 智能适配 → 生成播放 |
| **作业帮** | 拍题 → 等待 → 查看答案 | 输入题目 → 快速生成 → 完整讲解 |
| **线下补课** | 找老师 → 约时间 → 上课 | 首次配置 → 即时生成 → 随时观看 |

**用户期望：**

* 输入方式要简单（打字/拍照）
* 系统能根据我的特点生成适配的内容
* 生成速度要快（比等老师回复快）
* 质量要稳定（不像作业帮那样随机）

**可能困惑的点：**

* 初次使用不知道选哪个入口 → 智能推荐
* 不知道要配置什么 → 清晰的引导说明和示例
* 担心生成失败 → 透明进度 + 失败可控

### 2.3 Success Criteria

**核心体验成功指标：**

| 指标 | 成功标准 | 测量方式 |
|------|----------|----------|
| **输入成功率** | > 95% | 用户成功提交需求的比例 |
| **用户配置完成率** | > 90% | 用户在首次引导时完成配置的比例 |
| **生成速度** | < 5 分钟 | 端到端生成时间 |
| **质量满意度** | > 4.5/5.0 | 用户评分 |
| **分享率** | > 30% | 生成后分享的比例 |

**"哇"的瞬间触发条件：**

* 用户 3 秒内理解配置引导流程用途
* 生成进度比预期快
* 视频第一眼看到 Manim 动画流畅

### 2.4 Novel UX Patterns

**小麦采用的交互模式：**

| 模式类型 | 模式描述 | 小麦的实现 |
|----------|----------|------------|
| **成熟模式** | 视频播放器交互 | 对标 B站，倍速/进度/全屏 |
| **成熟模式** | 图片上传识别 | 腾讯云 OCR，拍题即识别 |
| **创新模式** | 用户配置系统 | 首次使用时引导配置个人简介和学习偏好，系统自动适配 |
| **创新模式** | 透明生成进度 | 分阶段+预计时间+小成就 |
| **创新模式** | 与 AI 老师对话 | 生成后可继续提问 |

**需要教育用户的模式：**

* 双入口的区别 → 首次使用时智能推荐+说明
* AI 智能适配的差异 → 通过首次引导配置让系统理解用户特点 |
* 生成进度的含义 → 分阶段描述让用户理解

### 2.5 Experience Mechanics

**核心交互流程：**

**阶段 1：输入触发**

* 用户看到清晰的输入框："输入题目或知识点"
* 支持三种输入方式：打字 / 拍照 / 粘贴
* 输入框下方有智能推荐："检测到数学题，推荐使用问答入口"

**阶段 2：任务创建与配置应用**

* 输入框附近显示"使用当前配置"提示或图标
* 用户点击可查看当前配置（个人简介等）
* 配置在首次使用时已完成，后续任务自动应用
* 用户可从个人中心重新进入配置页面修改

**阶段 3：生成等待**

* 提交后立即跳转到进度页面
* 分阶段展示：理解题目 → 生成脚本 → 渲染动画 → 合成视频
* 每个阶段有进度条和预计剩余时间
* 阶段完成时有"小成就"提示："脚本已生成！"

**阶段 4：结果呈现**

* 视频生成完成后自动跳转到播放页面
* 第一眼看到的是 Manim 动画的高质量画面
* 播放器下方有突出的分享按钮
* 右侧有"继续提问"入口，支持与 AI 老师对话

**错误处理：**

* 生成失败时明确告知原因（如"Manim 渲染失败"）
* 提供"再试一次"按钮，一键重试
* 失败不超过 3 次时保持友好，超过后转人工
