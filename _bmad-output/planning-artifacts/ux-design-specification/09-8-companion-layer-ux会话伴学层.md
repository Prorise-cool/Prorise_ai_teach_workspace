## 8. Companion Layer UX（会话伴学层）

### 8.1 设计原则

* 所有提问必须绑定上下文锚点：视频时间点、课堂步骤、slide 或白板段落。
* 回答优先解释当前上下文，再决定是否补充外部资料。
* 白板解释是 Companion 的核心差异化，而不是附加彩蛋。
* 展开 Companion 后，用户仍能继续看视频或浏览课堂。

### 8.2 视频页 Companion

**触发方式**

* 点击“问这一秒”。
* 暂停时针对当前帧提问。
* 从推荐追问中快速发起。

**最小结构**

* 当前时间锚点。
* 提问框。
* 问答流。
* 白板标签页 / 折叠区。

**体验要求**

* 提问后默认保持播放器上下文，不自动跳离视频。
* 白板解释优先围绕当前镜头、当前公式或当前推导。

### 8.3 课堂页 Companion

**触发方式**

* 针对当前 slide。
* 针对当前白板步骤。
* 针对当前讨论片段中的某一轮发言。

**最小结构**

* 当前步骤锚点。
* 问答流。
* 白板解释。
* 返回课堂主叙事的显式收起动作。

**体验要求**

* 用户追问后，课堂内容浏览仍可继续。
* Companion 回答不能重写课堂主叙事，只能补充解释。

### 8.4 失败与降级

| 场景 | 处理方式 |
|------|----------|
| Companion 暂时不可用 | 保留主内容区，侧栏显示明确失败与重试入口 |
| 白板解释生成失败 | 回退为结构化文本解释，不阻断问答 |
| 当前锚点缺失 | 提示用户基于当前播放位置或当前课堂步骤重新发问 |
| 资料不足 | 提示打开证据来源抽屉，或在当前页的来源抽屉 / 证据面板内继续深挖依据 |
