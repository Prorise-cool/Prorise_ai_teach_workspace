# Core User Experience

## Defining Experience

小麦的核心用户体验围绕"选择-等待-惊喜"的情感弧线展开：

**核心用户操作循环：**
1. 用户选择 AI 老师风格 → **感受明显差异**
2. 用户输入学习需求 → **极简操作，零思考**
3. 系统生成内容 → **等待透明，进度可见**
4. 用户获得结果 → **速度+质量的双重惊喜**

**关键 UX 诉求：**
- 风格不只是功能选项，而是"有性格的老师"
- 等待不是"空白时间"，而是"期待过程"
- 简单不是"少功能"，而是"懂用户"

## Platform Strategy

| 阶段 | 目标平台 | 交互模式 | 关键约束 |
|------|----------|----------|----------|
| **MVP** | 桌面端（≥1024px） | 鼠标/键盘 | 单一断点，功能优先 |
| **Growth** | 平板适配（768-1024px） | 触控+鼠标 | 响应式布局优化 |
| **Vision** | 移动端（<768px） | 触控为主 | 独立移动端体验 |

**浏览器支持矩阵：**
- Chrome/Edge 90+：✅ 完全支持（主要目标）
- Firefox 88+：✅ 完全支持
- Safari 14+：⚠️ 基础支持（需测试 WebAudio 兼容性）

## Effortless Interactions

**必须做到零负担的交互：**

| 交互场景 | 零负担设计 |
|----------|------------|
| **入口选择** | 智能推荐 + 一键确认，而非让用户自己判断 |
| **风格选择** | 通过附着在输入框附近的头像或胶囊触发器展开会话配置面板，使用头像、名称和选中态让选择直觉化 |
| **题目输入** | 支持"打字/拍照/粘贴"三种方式，无门槛 |
| **等待过程** | 分阶段进度 + 预计剩余时间 + 有意义的阶段描述 |
| **视频播放** | 一键播放，自动推荐相关内容 |

**消除的摩擦点：**
- ❌ 不让用户理解"LLM/Manim/TTS"是什么
- ❌ 不让用户担心"选错入口会怎样"
- ❌ 不让用户在等待时焦虑"是不是卡住了"
- ❌ 不让用户思考"这个功能在哪里"

## Critical Success Moments

**关键时刻 1：风格选择的"哇"瞬间**
- 用户点击输入框右上角的头像或胶囊触发器，立即看到：
  - 当前会话老师风格的头像与名称
  - 其他可选老师风格及其选中态
  - 一个轻量但明确的会话配置入口，不打断核心输入流程
- **成功标准**：用户 3 秒内理解“这里可以切换老师风格”，且不会被迫离开输入任务

**关键时刻 2：输入后的"我在工作"确认**
- 用户提交需求后，立即看到：
  - 清晰的阶段划分："理解题目 → 生成脚本 → 渲染动画 → 合成视频"
  - 当前进度 + 预计剩余时间
  - 阶段性的"小成就"提示（如"脚本已生成，正在画动画..."）
- **成功标准**：用户不焦虑，知道系统在认真工作

**关键时刻 3：结果呈现的"超预期"瞬间**
- 视频生成完成，第一眼看到：
  - 流畅的 Manim 动画（数学公式"活"了）
  - 清晰的 TTS 配音
  - 整体视觉质量远超预期
- **成功标准**：用户产生"这居然是 5 分钟生成的"惊叹

## Experience Principles

**原则 1：风格可见性**
- 4 种 AI 老师必须在视觉、听觉、内容风格上全方位差异
- 风格差异必须"肉眼可见"，不能只停留在文字描述

**原则 2：等待透明性**
- 进度反馈必须真实、有意义
- 避免模糊的"加载中"，要告诉用户具体在做什么
- 预计剩余时间要尽量准确

**原则 3：极简至上**
- 任何需要用户"想一想"的交互都是设计失败
- 首次使用应该"一看就会"，无需教程
- 智能推荐优于让用户选择

**原则 4：质量惊喜**
- 速度和质量的反差是核心差异化
- 在视觉上强化"高质量"的感知
- 让用户忍不住想分享给朋友
