# Epic 8: 首页、认证入口与输入壳层成品化交付

在首页、落地页、认证入口与输入壳层成品图冻结后，交付正式的首页、认证对话框、视频 / 课堂输入壳层和老师风格选择体验。

## Story 8.1: 首页双入口与推荐区成品化实现

As a 首次或回访用户，
I want 看到符合成品图的正式首页与双入口布局，
So that 我能快速理解两个主入口并进入正确路径。

**Acceptance Criteria:**

**Given** 首页成品图已冻结且相关接口契约稳定
**When** 用户打开首页
**Then** 页面按成品图展示双入口、推荐区、文案层级和视觉样式
**And** 不再依赖原型页或临时联调壳层

**Given** 推荐数据缺失或推荐逻辑失败
**When** 首页继续渲染
**Then** 页面保持完整可用
**And** 双入口主路径不被阻塞

## Story 8.2: 认证入口、登录注册对话框与受保护路由成品化

As a 需要进入平台的用户，
I want 使用正式认证入口完成登录注册并回到正确上下文，
So that 我不会在首页与鉴权流程之间迷失。

**Acceptance Criteria:**

**Given** 用户位于正式首页或受保护页面入口
**When** 用户触发登录或注册
**Then** 正式认证对话框按成品图打开
**And** 成功后回到原上下文而不是跳到无关页面

**Given** Token 失效或未授权
**When** 用户访问受保护页面
**Then** 系统展示符合成品图的重新登录引导
**And** 不出现临时调试态或无样式回退页

## Story 8.3: 视频 / 课堂输入壳层与老师风格面板成品化

As a 准备发起学习任务的用户，
I want 在正式输入壳层中完成题目 / 主题输入和老师风格选择，
So that 我可以在正式页面上开始学习而不是依赖原型页。

**Acceptance Criteria:**

**Given** 输入壳层成品图已冻结
**When** 用户进入视频或课堂输入页
**Then** 页面按成品图展示输入区域、主 CTA、上传入口和风格触发器
**And** 老师风格面板与契约字段保持一致

**Given** 接口返回校验失败或输入为空
**When** 用户尝试提交
**Then** 页面展示成品图定义的错误态和禁用态
**And** 不再使用占位提示替代正式交互
