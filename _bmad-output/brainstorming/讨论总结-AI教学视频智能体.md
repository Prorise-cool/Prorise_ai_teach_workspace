## 一、项目背景

### 1.1 赛事约束
- **赛事名称**：教育智能体应用创新赛（高职组）
- **平台要求**：腾讯云智能体开发平台（仅用基础设施）
- **提交截止**：2026/4/25（约 5 周）
- **高职组定位**：基础教学智能体
- **必交材料**：方案说明书、演示视频（≤3分钟）、答辩 PPT、可访问链接

### 1.2 现有基础
- 已有 **ManimToVideoClaw** 项目，完整的视频生成流水线
- 支持：分镜生成 → Manim 代码 → 渲染 → TTS → 上传
- 多 TTS 支持：豆包、百度、Spark、Kokoro

### 1.3 开发策略
- **不是在示例项目上开发**，而是全新开发
- 吸收 ManimToVideoClaw 的精华和能力

## 二、产品定义

### 2.1 核心问题

**用户痛点**：
1. 理解抽象概念（数学公式/算法逻辑太抽象）
2. 学习效率低（看书/看课太慢）
3. 缺乏个性化（现有视频不够针对性）
4. 教师备课负担（制作高质量讲解视频太耗时）

**差异化价值**：
1. 动画质量（3Blue1Brown 级别）
2. 生成速度（输入 → 分钟级出视频）
3. 教学效果（更懂教学法）
4. 多风格 Agent（选择老师 = 选择 Agent）

### 2.2 目标用户

| 用户类型 | 核心场景 |
|----------|----------|
| 高职学生 | 课后复习、考前冲刺 |
| 高职教师 | 备课、制作教学视频 |
| 自学者 | 按需学习、快速理解 |

### 2.3 输入与输出

**输入方式**：
- 文字输入（题目/知识点）
- 图片上传（拍照识别）
- 语音输入（语音提问）

**输出**：
- 带配音的讲解视频
- 多种老师风格可选

### 2.4 MVP 功能范围

**必须包含**：
- ✅ 用户系统（登录/注册/个人中心）
- ✅ 单题生成（核心流程跑通）
- ✅ 学习记录（历史/收藏/重看）
- ✅ 播放体验（播放器/进度/倍速）

**后期迭代**：
- 批量生成
- 风格模板
- 学习分析
- 作业系统

## 三、技术架构

### 3.1 技术选型

| 层级 | 技术选择 | 说明 |
|------|----------|------|
| **前端** | Vue 3 | 精美设计，品牌化，组件库后期选 |
| **后端** | FastAPI + LangChain | Python 生态，支持多 Agent |

## 四、多 Agent 设计

### 4.1 老师风格类型

| Agent 类型 | 语音特点 | 视觉风格 | 内容深度 | 互动频率 |
|------------|----------|----------|----------|----------|
| **严肃老师** | 稳重、清晰 | 简洁、专业 | 深入、严谨 | 少 |
| **幽默老师** | 活泼、有趣 | 多彩、动感 | 轻松、案例多 | 中 |
| **耐心老师** | 温和、慢速 | 温暖、柔和 | 细致、步骤多 | 高 |
| **高效老师** | 快速、干练 | 极简、高效 | 精炼、直击要点 | 少 |

### 4.2 风格维度

每种 Agent 在以下四个维度有差异化配置：
1. **语音风格**：语速、语调、节奏
2. **视觉风格**：动画节奏、配色、字体
3. **内容风格**：讲解深度、案例选择、步骤拆解
4. **互动风格**：习题数量、提问频率、鼓励语

### 4.3 选择机制

- **生成前选择**：用户主动选择老师类型
- **个性化匹配**：根据学习历史智能推荐
