# Story 3.6: 视频输入页公开视频广场与复用入口

Status: backlog

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 想快速判断是否已有相近讲解的用户，
I want 在视频输入页浏览公开视频卡片并把合适内容复用到当前输入流程，
so that 我可以先看已有结果或少走重复创建路径。

## Acceptance Criteria

1. `/video/input` 可展示公开视频卡片列表或推荐集合，且主输入区与主 CTA 仍然保持页面主叙事。
2. 用户可从公开视频卡片进入对应结果页，或把题目 / 摘要复用到当前输入流程。
3. 公开内容为空、加载失败或处于 mock 模式时，页面提供空态 / 降级提示，但不阻断继续创建新视频任务。

## Dev Notes

### Business Context

- 公开视频能力已被正式纳入 BMAD，不再按“伪业务”处理。
- 公开内容区属于视频输入页的辅助发现层，不替代用户自主输入与任务创建主链。
- 当前版本只承认“发现、观看、复用”，不承认完整视频社区生态。

### Technical Guardrails

- 主输入区与主 CTA 仍然是 `/video/input` 的第一优先级。
- 公开卡片只消费最小必要元数据，不直接依赖复杂社区详情结构。
- 复用动作应回填到当前输入流程，而不是新开一套独立创建页。

### Suggested File Targets

- `packages/student-web/src/services/api/adapters/video-adapter.ts`
- `packages/student-web/src/app/routes/index.tsx`

### Project Structure Notes

- 本 Story 消费 `Story 4.10` 产出的公开卡片 payload。
- 当前 `student-web` 仍只有首页骨架；`/video/input` 页面、`video` feature 目录和对应路由注册都需要在后续实现中新增。
- 在真实接口未完成前，允许基于 `adapter + mock handler` 完成正式页面状态。
- 公开卡片进入结果页时仍走既有 `/video/:id` 结果消费链路。
- 在 `Story 4.10` 和视频结果最小卡片 payload 未冻结前，本 Story 维持 `backlog`，不直接进入正式开发。

### Testing Requirements

- 覆盖公开卡片列表成功加载与空列表场景。
- 覆盖“查看讲解”跳转与“复用题目”回填动作。
- 覆盖失败降级时仍可正常提交新视频任务。
