# Story 6.6: 白板动作协议与结构化降级

Status: ready-for-dev

## Story

As a 正在理解难点的用户，
I want 在回答旁看到白板解释，并在失败时得到结构化降级内容，
So that 即使遇到异常也能继续理解当前知识点。

## Acceptance Criteria

1. 当前问题适合白板辅助解释（几何、函数图像、公式推导）时，Ask API 响应包含 `whiteboard_actions` 数组，每个 action 包含 `action_type`、`payload`、可选 `render_uri`。
2. 白板能力失败或问题不适合图形化时，返回结构化文本解释（分步骤说明列表），`persistence_status` 标记为 `whiteboard_degraded`，主回答仍可用。
3. 前端收到白板 action 且 schema 合法时，侧栏内展示白板渲染结果，内容与回答锚点一致。
4. 白板 action schema 不合法或前端无法渲染时，前端安全退回文本型降级展示，整条问答流不崩溃。

## Dev Notes

### 白板动作协议

复用已有的 `WhiteboardActionRecord`（`shared/long_term/models.py`）：
```python
class WhiteboardActionRecord(BaseModel):
    action_type: str           # draw_function, highlight_region, animate_step, show_formula, plot_graph
    payload: dict[str, object] # 动作参数
    object_ref: str | None     # 关联对象引用
    render_uri: str | None     # 渲染产物 URI（如果服务端预渲染）
    render_state: str | None   # 渲染状态
```

### Payload Schema 约定

```typescript
// draw_function
{ expression: string, xRange: [number, number], label?: string }

// plot_graph
{ points: [{x, y}], type: "scatter" | "line", labels?: {xAxis, yAxis} }

// show_formula
{ latex: string, description?: string }

// animate_step
{ steps: [{ description: string, formula?: string }], currentStep: number }

// highlight_region
{ text: string, highlightRanges: [{start, end}] }
```

### LLM Prompt 中的白板指令

```
如果学生的提问适合图形化解释（如函数图像、几何图形、公式推导），请同时生成白板动作。
白板动作以 JSON 数组格式返回，每个动作包含 action_type 和 payload 字段。
如果不确定是否适合图形化，请不要生成白板动作，仅提供文本解释。
```

### 已有基础设施

- `whiteboard/action_schema.py` 已有 `WhiteboardAction`（仅 action + payload），需扩展对齐 `WhiteboardActionRecord`
- `whiteboard/renderer.py` 已有 `render_whiteboard_action()`（直接 dump dict），需实现实际渲染逻辑
- `PersistenceStatus.WHITEBOARD_DEGRADED` 已在枚举中定义
- `CompanionTurnCreateRequest.whiteboard_degraded` 字段已存在，model_validator 自动推导状态
- 前端已集成 KaTeX/Temml（`packages/student-web` 中已有数学公式渲染能力）

### 前端渲染策略

MVP 阶段优先实现：
1. `show_formula` → 使用 Temml/KaTeX 渲染（已有依赖）
2. `animate_step` → 步骤卡片列表（纯 React 组件）
3. `highlight_region` → 高亮文本（简单 CSS）

后续阶段（Post-MVP）：
- `draw_function` / `plot_graph` → Canvas 渲染或 Manim 预渲染

### 技术约束

- 白板动作生成是可选的（不是每次回答都必须有白板）
- 白板失败不阻塞主回答返回
- 前端渲染必须是渐进增强：有白板显示白板，没有或失败不影响文本回答
- 白板动作持久化走 Story 6.7 的 `xm_whiteboard_action_log`
