# Story 4.8: 视频结果页、播放器与结果操作

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 获得结果的用户，
I want 在结果页直接播放讲解视频并执行基础控制与结果操作，
so that 我可以立刻完成一次完整的单题复习体验。

## Acceptance Criteria

1. 视频任务完成后，用户进入 `/video/:id` 时，页面查询并展示视频播放器（主区域）、题目摘要、知识点摘要、AI 内容标识，用户不需要跳到其他工具或临时链接才能观看结果。
2. 播放器基于 Video.js 封装，支持播放、暂停、倍速切换（0.5x / 1x / 1.5x / 2x）、拖动进度条与全屏控制，这些操作立即生效并反馈到 UI；从历史记录再次打开结果页也能稳定消费同一套播放器行为。
3. 页面在 mock 模式下可展示完成态、视频缺失态（URL 失效）、权限失败态（非本人任务）与加载失败态（API 异常），不需要真实视频流水线跑通后才开始实现页面结构和状态机。
4. `Companion`、`Evidence / Retrieval` 与 `Learning Coach` 仅作为后续动作入口（侧边栏按钮或底部 CTA），不抢占播放器的主叙事中心位置；入口按钮在对应 Epic 未实现前展示 `disabled` 或 `coming soon` 状态。
5. 结果操作区包含公开发布/取消公开按钮、结果复用入口（"使用此题目重新生成"）与分享入口（复制链接），即使当前成品图未完整展开操作区，仍必须按 Story 4.10 约定预留完整操作区壳层。
6. 结果页数据通过 `@tanstack/react-query` 的 `useQuery` 获取，支持缓存与重新验证；结果加载中展示 skeleton 而非空白。

### Business Context

- 结果页是视频生成主链路的"最终消费端"，用户在这里完成视频观看和后续动作。
- 播放器是页面的绝对主角，Companion/Evidence/Learning Coach 只是辅助入口，不能喧宾夺主。
- 公开发布操作是 Story 4.10 的核心交互入口，本 Story 必须预留完整操作区壳层而非事后补丁。
- AI 内容标识是合规要求，不可省略或隐藏。
- Video.js 已在 `package.json` 中引入，是架构文档指定的视频播放解决方案。

### Technical Guardrails

- Video.js 封装必须处理组件卸载时的实例销毁（`player.dispose()`），避免内存泄漏。
- Video.js 初始化必须在 DOM 挂载后（`useEffect` + ref），不在 render 阶段调用。
- 结果数据使用 `useQuery` + `staleTime: 5min`，避免每次路由切换重新请求。
- 公开发布/取消操作使用 `useMutation` + `onSuccess` 刷新 `useQuery` 缓存。
- 复制链接使用 `navigator.clipboard.writeText()`，降级到 `document.execCommand('copy')`。
- 页面级样式使用 feature SCSS，播放器皮肤覆盖集中在 `_result.scss` 中。
- 结果页路由 `/video/:id` 为受保护路由，未登录 → 跳登录页，非本人任务 → 展示 403 提示。
- 后续动作入口按钮的 disabled 状态应由 feature flag 或 Epic 完成状态控制，而非硬编码。
