# Story 4.11 FastAPI 渐进预览后端实现说明

> 本文记录 `Story 4.11` 在 `packages/fastapi-backend` 的后端重构收口情况。事实状态仍以 [`../index.md`](./index.md) 与 [`sprint-status.yaml`](./sprint-status.yaml) 为准。

## 范围边界

- 本次只覆盖 FastAPI 后端，不包含 `student-web` 等待页 UI、store、hook 或点击验收步骤。
- 改造原则按本次任务要求执行：已有实现中“有用则留、无用删除”，不为了保留旧 batch 路径而额外做向下兼容层。
- 当前整体 Story 仍是 `in-progress`，原因是前端等待页尚未接入新的 preview endpoint 与 section 级 SSE 事件。

## 已落地能力

- 编排模型重构：
  - `orchestrator.run()` 已从 `generate_codes(ALL) -> render_all_sections(ALL)` 的 batch 模式改为 per-section streaming loop。
  - 每个 section 独立经历 `generate_section_code -> render_section -> compose clip -> publish clip`，成功后立即对外暴露可访问片段。
- TTS 前移：
  - `_run_tts()` 已前移到 storyboard 之后。
  - 每个 section 发布时即可携带对应音频，等待页消费的是“可听可看”的片段，而不是静音预览。
- 预览运行态：
  - 运行态新增 `VideoTaskPreview / VideoPreviewSection / VideoPreviewSectionStatus`。
  - Redis 运行态可保存 summary、knowledge points、section 状态、audioUrl、clipUrl、previewVersion，并在刷新后恢复。
- 对外契约：
  - 新增 `GET /api/v1/video/tasks/{task_id}/preview`。
  - 现有 SSE 事件新增 `section_progress`、`section_ready`。
  - 既有 `progress` 事件 context 增加 `previewAvailable`、`previewVersion`，供前端决定是否拉取 preview。
- 失败隔离：
  - 单 section 渲染失败会标记为 `failed` 并跳过，不会阻断已完成片段的展示。
  - 最终整片只拼接成功 section，并继续沿用成功率门禁。

## 关键接口与事件

### Preview Endpoint

- 路径：`GET /api/v1/video/tasks/{task_id}/preview`
- 返回：`VideoTaskPreviewResponseEnvelope`
- 关键字段：
  - `previewAvailable`
  - `previewVersion`
  - `summary`
  - `knowledgePoints`
  - `sections[].status`
  - `sections[].audioUrl`
  - `sections[].clipUrl`

### SSE 事件

- `section_progress`
  - 用于 section 进入 `generating | rendering | fixing | failed`
- `section_ready`
  - 用于某个 section 片段已可播放，事件内包含 `clipUrl`
