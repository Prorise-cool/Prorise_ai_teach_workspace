# Story 4.11 补丁：视频任务删除 500 与首页状态 404 风暴热修复

## 背景

2026-04-19 排查到视频任务中心存在一组串联问题：

1. 用户在导航栏 bell / 首页 bell 中删除已完成视频任务时，后端返回 `500 COMMON_INTERNAL_ERROR`。
2. 用户取消或删除任务后，首页与输入页的任务中心没有立即把任务从本地缓存中剔除。
3. 任务元数据列表仍残留旧任务时，前端会继续对这些 taskId 轮询 `/api/v1/tasks/{id}/status`，从而形成重复 `404` 请求。

## 根因

### 后端

`delete_video_task()` 原先先删除 Redis 运行态，再调用 `VideoService.persist_task()` 将 RuoYi 元数据标记为 `cancelled`。  
一旦元数据持久化失败，接口会返回 `500`，但 Redis 运行态已经被删掉，导致：

- workspace 列表仍能查到旧任务元数据；
- `/status` 因运行态缺失返回 `404`；
- 前端任务中心持续命中同一批脏 taskId。

### 前端

- `video-workspace-task-adapter` 虽然会在单次请求里跳过 `status 404` 的任务，但不会记住这些 stale taskId，下一轮轮询还会再请求一次。
- 首页、输入页、等待页取消任务成功后，只做 query invalidation，没有先把当前任务从任务中心缓存中移除。
- 删除任务成功后，也没有主动清理 `task-status / task-preview / result` 相关 detail query。

## 本次修复

### FastAPI / Redis

- `packages/fastapi-backend/app/features/video/service/delete_task.py`
  - 调整为“先持久化元数据，再清理运行态”。
  - 持久化时显式写入 `updated_at`，避免删除路径与取消路径的元数据更新时间口径不一致。
  - 删除成功后统一清理：
    - `xm_task_runtime:{task_id}`
    - `xm_task_events:{task_id}`
    - `xm_video_task:{task_id}:preview`
    - `xm_video_task:{task_id}:cancel_request`
    - `xm_video_task:{task_id}:fix_log`
    - `xm_video_task:{task_id}:result_detail`
    - `xm_video_task:{task_id}:understanding`
    - `xm_video_task:{task_id}:storyboard`
    - `xm_video_task:{task_id}:manim_code`
    - `xm_video_task:{task_id}:tts_result`
    - `xm_video_task:{task_id}:upload_result`
    - `xm_video_runtime_auth:{task_id}`
- `packages/fastapi-backend/app/features/video/routes.py`
  - 删除接口成功响应改为显式返回 `taskId/status`，不再把普通 dict 误传给 `build_success_envelope()`。

### student-web

- `packages/student-web/src/services/api/adapters/video-workspace-task-adapter.ts`
  - 增加 dismissed stale-task 内存集合。
  - 单条 `status 404` 后会记住该 taskId，后续轮询不再重复请求它。
  - 聚合结果现在先按 `task_id` 去重，再做快照 enrich。
- `packages/student-web/src/features/video/utils/task-center-cache.ts`
  - 新增任务中心缓存工具：
    - `pruneVideoWorkspaceTaskFromCache()`
    - `cacheCancelledVideoTask()`
    - `clearVideoTaskDetailQueries()`
- `packages/student-web/src/features/home/pages/home-page.tsx`
- `packages/student-web/src/features/video/pages/video-input-page.tsx`
- `packages/student-web/src/features/video/hooks/use-cancel-video-task.ts`
  - 取消成功后立即缓存 cancelled snapshot，并把任务从 bell 列表缓存移除。
  - 删除成功后立即移除 bell 列表项，并清空任务 detail query，避免旧页面/轮询继续追打已删除 taskId。

## 前端点击验收清单

1. 准备一个已完成视频任务，打开首页或输入页右上 bell，点击“删除”。
   期望：出现成功提示；任务立即从 bell 列表消失；删除请求不再返回 `500`。
2. 准备一个 `pending/processing` 视频任务，分别在首页 bell、输入页 bell、等待页中触发“取消任务”。
   期望：任务立即从首页/输入页 bell 列表消失；等待页跳回输入页后，不会继续显示旧任务中心项。
3. 删除或取消后保持首页/输入页停留至少 20 秒，观察浏览器网络面板。
   期望：不会持续对同一个已删除/已取消 taskId 反复请求 `/api/v1/tasks/{id}/status` 并产生 404 风暴。
