# Story 4.1: 视频流水线阶段、进度区间与结果契约冻结

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 前后端协作团队，
I want 先冻结视频流水线的阶段枚举、进度区间和结果 payload，
so that 等待页、后端 stage 实现和结果页可以围绕一致语义并行开发。

## Acceptance Criteria

1. 视频流水线对外公开的 stage 枚举至少覆盖 `understanding`、`storyboard`、`manim_gen`、`manim_fix`、`render`、`tts`、`compose`、`upload`，每个 stage 具备稳定的 `displayLabel`（中文显示名）、`progress` 起止区间与预估时长区间，前端无需通过猜测 message 文本判断任务当前所处阶段。
2. 每个 stage 的 `progress` 区间连续不重叠，总区间覆盖 `0–100`；`upload` 完成时 `progress === 100`；前端只需对比 `stage + progress` 即可渲染等待态，不需要自行拼装进度数学。
3. SSE `task:progress` 事件的 payload 扩展 `stage` 字段后仍与 Story 2.5 统一事件契约兼容，不破坏已有事件消费逻辑；`/api/v1/video/tasks/:id/status` 查询也返回等效 stage 信息。
4. 成功结果 schema 至少稳定包含 `videoUrl`、`coverUrl`、`duration`、`summary`、`knowledgePoints`、`resultId`、`completedAt`、`aiContentFlag`，前端结果页不需要等待后续隐式字段补齐后才能展示主体内容。
5. 失败结果 schema 至少稳定包含 `errorCode`、`errorMessage`、`failedStage`、`failedAt`、`retryable`，前端可基于 `failedStage` 与 `retryable` 渲染可解释失败信息与重试按钮。
6. `contracts/video/v1/` 下至少交付 `pipeline-stages.md`、`video-result.schema.json`、`video-failure.schema.json`，并在 `mocks/video/` 下交付覆盖全 stage 流的 mock SSE 事件序列。
7. 前端 mock handler 注册后，`/video/:id/generating` 等待页可在 mock 模式下完整消费 stage 流并最终跳转到结果页或展示失败态。

### Business Context

- `Story 4.1` 是 `Epic 4` 所有 Story 的共同前置。后端 `4.2–4.6` 围绕 stage 枚举实现各阶段服务，前端 `4.7` 围绕 stage 枚举构建等待页状态机，`4.8` 围绕结果 schema 构建结果页。
- 本 Story 与 `Story 3.1`（创建契约）共同构成视频域完整契约体系：`3.1` 管"怎么创建任务"，`4.1` 管"任务创建后怎么推进、怎么汇报进度、怎么输出结果"。
- 视频目标时长 `120s`（范围 `90–180s`），该参数应在 stage 契约中体现为分镜阶段的约束输入。
- stage 进度分布建议参考架构文档 §5 序列图中的时序比例，但具体 progress 区间需本 Story 显式冻结。

### Technical Guardrails

- stage 枚举一旦冻结，后续 Story 只能扩展（如增加新 stage），不能重命名或重新排序已冻结 stage。
- 2026-04-09 并行阶段补充冻结：`manim_gen=26–42`、`tts=43–58`、`manim_fix=59–65`、`render=66–78`、`compose=79–94`、`upload=95–100`。
- `manim_gen` 与 `tts` 在实现上允许并行，但对外暴露的全局 `progress` 必须单调递增；后端不能再把较晚完成的早期阶段事件回推给前端。
- `manim_fix` 是条件 stage，不一定每次执行都出现；前端状态机必须处理 stage 跳过的情况，也必须允许在不降低全局 `progress` 的前提下进入 `manim_fix` 修复态。
- progress 区间必须在 SSE 事件和 `/status` 查询中保持一致语义，不能出现 SSE 报 45% 但 `/status` 查询返回 30% 的不一致。
- 结果 schema 中的 `videoUrl` 应为 COS 签名 URL 或 CDN 地址，有效期由后端在上传时确定；前端不应假设 URL 永不过期。
- `aiContentFlag` 是合规要求字段，所有 AI 生成视频必须标记为 `true`，不可省略。
- 视频域新增错误码必须注册到统一 `TaskErrorCode` 字典（`contracts/tasks/task-error-codes.md`），不能在视频模块内部私建命名空间。
- 对外字段口径与 Story 2.1、3.1 保持一致：对外 camelCase，Python 内部 snake_case。
