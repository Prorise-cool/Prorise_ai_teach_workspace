# Story 1.5: 输入壳层中的老师风格最小选择配置

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 准备发起学习会话的用户，
I want 在视频或课堂输入壳层中通过轻量下拉选择老师风格，
so that 当前会话可以带着我期望的讲解风格启动。

## Acceptance Criteria

1. 用户进入 `/video/input` 或 `/classroom/input` 时，老师风格以下拉方式出现在核心输入区域附近，存在默认风格，不需要跳转独立配置页。
2. 用户选择风格并发起任务时，所选风格作为 `AgentConfig` 或等效字段稳定透传，只影响会话内容行为与局部 teacher indicator，不触发页面级主题切换。
3. 页面在 mock 模式下切换不同风格时，能通过局部头像、标签或点缀色体现选择结果，不会变成全站 CSS 主题切换。

## Tasks / Subtasks

- [ ] 固化 `AgentConfig` 前端数据模型（AC: 1, 2, 3）
  - [ ] 将 4 种老师风格配置整理为前端可消费数据源。
  - [ ] 明确默认风格、显示名称、局部点缀色、avatar、description 字段。
- [ ] 实现轻量选择器组件（AC: 1, 3）
  - [ ] 组件放在输入区附近，不做大卡片角色选择。
  - [ ] 暴露当前选择值和切换事件，供视频 / 课堂输入页复用。
- [ ] 接入任务创建字段透传（AC: 2）
  - [ ] 以 `agentConfig` 或等效请求字段透传到 mock 请求 / 真实请求。
  - [ ] 保证 mock task、表单 state、后续请求体使用同一字段语义。
- [ ] 增加局部视觉反馈与测试（AC: 2, 3）
  - [ ] 局部头像、标签或指示器随风格变化。
  - [ ] 测试默认值、切换值、透传值和局部视觉反馈。

## Dev Notes

### Business Context

- 老师风格是会话启动配置，不是产品主流程，也不是页面级主题切换能力。
- Epic 1 只需要把风格作为“最小启动参数”稳定透传，真正对生成内容产生差异属于后续业务 Story。
- Story 1.5 是 Epic 3 与 Epic 5 输入壳层的共享前置能力。

### Technical Guardrails

- 风格差异通过 `AgentConfig` 数据预设体现，不允许改全局品牌主色。
- 组件应可在视频输入页和课堂输入页复用，避免复制两套选择逻辑。
- 默认风格必须存在，用户不应被迫先选择风格才能继续。
- 视觉反馈只能是局部 avatar、tag、indicator 或点缀色，不应改变整页主题。

- 架构文档中的 `AgentConfig` 更偏跨端目标结构，当前前端可先在 `features/agent` 落位，再通过服务层透传到后端。
- 当前已有 `framer-motion`、`zustand`、`react-hook-form` 等依赖，但本 Story 不应过度设计；保持轻量下拉足够。

### Testing Requirements

- 覆盖默认风格渲染。
- 覆盖切换不同风格后的局部 indicator 变化。
- 覆盖请求 payload 或 mock task 中的透传字段。
- 覆盖不会修改全局主题变量的约束。
