# Story 1.5: 用户配置系统（个人简介与学习偏好）

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 准备发起学习会话的用户，
I want 在首次使用时设置我的个人简介和学习偏好，
so that 系统可以根据我的特点自动分配或生成合适的 AI 老师和同学。

## Acceptance Criteria

1. 用户首次使用或进入个人配置页面时，可以完成配置表单，系统保存用户的昵称、个人简介和语言选择，配置可在后续任务创建时自动透传。
2. 用户输入个人简介时，系统保存配置，个人简介作为用户背景信息供后端生成 Agent persona 时参考，个人简介长度限制在 10-500 字符。
3. 用户选择语言时，系统保存配置，默认为 'zh-CN'，可选 'en-US'。
4. 用户创建视频或课堂任务时，用户配置作为 `userProfile` 或 `userRequirements` 字段稳定透传，后端根据配置动态生成或匹配合适的 AI agents。
5. 用户在 mock 模式下开发时，配置可保存到 localStorage，真实模式下配置通过 API 保存到用户 profile。

## OpenMAIC 参考模型

```typescript
// 用户输入自己的特点（OpenMAIC 实际实现）
interface UserRequirements {
  requirement: string;        // 主题/需求
  userNickname?: string;      // 昵称
  userBio?: string;           // ⭐ 个人简介
  language: 'zh-CN' | 'en-US';
  webSearch?: boolean;        // 是否启用联网搜索
}

// 系统根据用户配置智能分配 agents
interface AgentConfig {
  id: string;
  name: string;
  role: string;              // teacher / student / assistant
  persona: string;           // 根据用户配置生成
  allowedActions: string[];  // 自动分配权限
}
```

## Dev Notes

### Business Context

- 用户配置系统是后端动态生成 AI agents 的核心输入，不是简单的风格选择器。
- 系统根据用户的个人简介（userBio），由后端智能分配或生成合适的 AI 老师和同学 persona。
- Story 1.5 是 Epic 3（视频）与 Epic 5（课堂）任务创建的前置共享能力。

### 与原描述的区别

| 原描述（错误） | 新描述（正确） |
|-------------|-------------|
| 简单下拉选择"4种老师风格" | 用户输入个人简介和学习偏好 |
| 用户被动选择预设 | 用户主动描述自己 |
| 固定 4 种风格 | 系统动态生成/匹配 |
| 风格选择器组件 | 完整用户配置系统 |

### Technical Guardrails

- Agent 的 persona 由后端根据用户配置（userBio）生成，前端只负责收集和透传用户数据。
- 个人简介是自由文本输入，不是预设选项。
- 语言选择默认为 'zh-CN'，可选 'en-US'。
- 配置必须在任务创建时稳定透传，不得丢失或遗漏。
- Mock 模式与真实模式使用相同的数据模型，确保切换顺畅。
