# Story 1.5: 用户配置系统（个人简介与学习偏好）

Status: ready-for-dev

## Story

As a 准备发起学习会话的用户，
I want 在首次使用时通过渐进式引导设置我的个人简介、性格偏好和AI导师偏好，
so that 系统可以根据我的特点自动分配或生成合适的AI老师和同学。

## 用户流程（基于新设计稿）

```
登录/注册成功
    ↓
个人信息简介页（01-login.html → view-profile）
    ├─ 上传头像（可选）
    ├─ 填写个人简介（可选）
    └─ 跳过按钮 → 直接进入导览页
    ↓
[条件分支]
    ├─ 已填写简介 → 直接进入导览页
    └─ 未填写简介 → 进入信息收集页（02-信息收集页.html）
        ├─ Step 1: 性格类型选择（5选1）
        └─ Step 2: AI导师偏好选择（12标签多选）
    ↓
导览页（03-导览页.html）- 3步产品功能介绍
    ↓
进入主页
```

## Acceptance Criteria

1. **个人信息简介页**
   - 用户登录/注册成功后进入个人信息简介页
   - 支持上传头像（可选）
   - 支持填写个人简介（自由文本，200字限制）
   - 提供"跳过"按钮，可直接跳过到导览页
   - 点击"下一步"时，已填写简介则进入导览页，未填写则进入信息收集页

2. **信息收集页（两步渐进式）**
   - **Step 1: 性格类型选择**（单选）
     - 目标明确，专注结果的行动派
     - 对世界充满好奇的探索者
     - 踏实严谨，喜欢按部就班
     - 乐于交流，思维发散的社交派
     - 天马行空，不拘一格的创意家
   - **Step 2: AI导师偏好选择**（多选）
     - 幽默风趣、严密逻辑、脑洞大开
     - 严格督学、循循善诱、朋友般陪伴
     - 直击核心、旁征博引、温和鼓励
     - 互动狂魔、冷静客观、充满激情
   - 每步都提供"跳过"按钮
   - 支持返回上一步

3. **导览页**
   - 3步产品功能介绍（学习计划生成、沉浸式课堂输入、一键视频生成）
   - 左侧文字说明，右侧GIF演示区域
   - 提供跳过导览按钮
   - 支持前后翻页

4. **数据持久化**
   - 用户配置保存到数据库（通过RuoYi API）
   - Mock模式可降级到localStorage
   - 配置在后续任务创建时自动透传

5. **配置透传**
   - 任务创建时，用户配置作为`userProfile`字段透传
   - 后端根据配置动态生成/匹配合适的AI agents

## 数据模型设计

### 数据库表设计（基于RuoYi规范）

```sql
-- 用户配置表
CREATE TABLE IF NOT EXISTS xm_user_profile (
    id               bigint(20)      NOT NULL AUTO_INCREMENT COMMENT '主键',
    user_id          bigint(20)      NOT NULL                   COMMENT '用户ID（关联sys_user.user_id）',
    avatar_url       varchar(500)     DEFAULT NULL               COMMENT '头像URL',
    bio              varchar(500)     DEFAULT NULL               COMMENT '个人简介',
    personality_type varchar(50)     DEFAULT NULL               COMMENT '性格类型',
    teacher_tags     varchar(500)     DEFAULT NULL               COMMENT 'AI导师偏好（JSON数组字符串）',
    language         varchar(10)     DEFAULT 'zh-CN'            COMMENT '语言偏好',
    is_completed     tinyint(1)      DEFAULT 0                  COMMENT '是否完成配置（0-否 1-是）',
    create_dept      bigint(20)      DEFAULT NULL,
    create_by        bigint(20)      DEFAULT NULL,
    create_time      datetime        DEFAULT CURRENT_TIMESTAMP,
    update_by        bigint(20)      DEFAULT NULL,
    update_time      datetime        DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uk_xm_user_profile_user_id (user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户配置表';
```

### 用户配置数据结构

```typescript
interface UserProfile {
  id: number;                    // 主键
  userId: number;                 // 用户ID
  avatarUrl?: string;             // 头像URL
  bio?: string;                   // 个人简介
  personalityType?: PersonalityType;  // 性格类型
  teacherTags?: string[];         // AI导师偏好
  language: 'zh-CN' | 'en-US';    // 语言偏好
  isCompleted: boolean;           // 是否完成配置
  createTime: string;             // 创建时间（ISO 8601）
  updateTime: string;             // 更新时间（ISO 8601）
}

// 提交请求DTO
interface CreateProfileRequest {
  avatarUrl?: string;
  bio?: string;
  personalityType?: PersonalityType;
  teacherTags?: string[];
}

// 更新请求DTO
interface UpdateProfileRequest {
  id: number;                     // 主键（更新时必填）
  avatarUrl?: string;
  bio?: string;
  personalityType?: PersonalityType;
  teacherTags?: string[];
}
```

## Dev Notes

### Business Context

- 用户配置系统是后端动态生成AI agents的核心输入
- 采用渐进式引导设计，降低用户填写压力
- 支持多路径跳过，确保不会卡住用户

### 实现路径变更（使用 RuoYi 代码生成器）

**原实现路径**（手写代码）：
- 手写 Entity、Bo、Vo、Mapper、Service、Controller
- 需要严格遵守 RuoYi 规范
- 容易出错，效率低

**新实现路径**（代码生成器）：
1. 大模型负责：创建数据库表、提供代码生成器配置指南
2. 开发者负责：在后台页面填写配置、一键生成代码、下载并集成
3. 手写部分：仅限非标准功能（如 getCurrent()、isCompleted()）

### 与原描述的区别

| 原描述 | 新描述 |
|-------------|-------------|
| 单页面浮窗 | 多页面渐进式引导 |
| 场景选择 | 性格类型 + AI导师偏好 |
| 简单表单 | 两步信息收集 |
| 无导览 | 3步产品功能导览 |

### Technical Guardrails

- 所有用户配置必须透传到后端，不得丢失
- 头像上传为可选功能，不强制
- 信息收集页两步都支持跳过
- 导览页可完全跳过
- Mock模式与真实模式使用相同数据模型

## RuoYi 代码生成器使用说明

### 代码生成器工作流程

1. **导入表** - 从数据库读取表结构，导入到代码生成器
2. **编辑配置** - 配置每个字段的显示类型、查询方式、字典类型等
3. **生成代码** - 一键生成前后端完整 CRUD 代码
4. **下载代码** - 下载压缩包，包含所有生成文件

### 生成的代码内容

| 类型 | 文件 | 说明 |
|------|------|------|
| 后端 | Domain.java | 实体类，继承 BaseEntity |
| 后端 | Bo.java | 业务对象，用于接收表单数据 |
| 后端 | Vo.java | 视图对象，用于返回数据 |
| 后端 | Mapper.java | MyBatis Plus Mapper 接口 |
| 后端 | Mapper.xml | MyBatis XML 映射文件 |
| 后端 | Service.java | 服务接口 |
| 后端 | ServiceImpl.java | 服务实现 |
| 后端 | Controller.java | 控制器 |
| 前端 | xxx.js | API 请求封装 |
| 前端 | index.vue | Vue 页面组件 |
| SQL | xxxMenu.sql | 菜单和权限 SQL |

### 代码生成器配置项说明

**显示类型选项**：
- `文本框` - 普通输入框
- `文本域` - 多行文本输入
- `下拉框` - 单选下拉（需要配置字典类型）
- `单选框` - 单选按钮组（需要配置字典类型）
- `复选框` - 多选按钮组（需要配置字典类型）
- `图片上传` - 文件上传组件
- `日期` - 日期选择器
- `日期时间` - 日期时间选择器
- `隐藏` - 不在前端显示

**查询方式选项**：
- `=` - 等于
- `!=` - 不等于
- `>` - 大于
- `>=` - 大于等于
- `<` - 小于
- `<=` - 小于等于
- `LIKE` - 模糊查询
- `BETWEEN` - 范围查询

### 实现路径修正记录（2026-04-04）

**原始方案**（手写代码）：
- 大模型生成完整的 RuoYi 规范代码
- 开发者复制粘贴并修改
- 容易出错，效率低

**新方案**（使用代码生成器）：
- 大模型只负责：创建数据库表 + 提供配置指南
- 开发者在 RuoYi 后台页面填写配置，一键生成代码
- 手写部分仅限非标准功能（如 getCurrent()、isCompleted()）

**修正原因**：
- RuoYi 有成熟的代码生成器，无需手写 CRUD 代码
- 代码生成器保证生成的代码完全符合 RuoYi 规范
- 提高开发效率，减少出错可能
