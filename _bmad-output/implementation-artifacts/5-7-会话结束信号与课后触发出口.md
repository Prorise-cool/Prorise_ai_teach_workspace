# Story 5.7: 会话结束信号与课后触发出口

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 刚完成课堂学习的用户，
I want 课堂结束时收到清晰的课后入口而不是被强制插入正式答题，
so that 我可以保持主叙事完整并在合适时机进入后续巩固。

## Acceptance Criteria

1. 当课堂主叙事进入完成阶段时，系统必须输出可被 Learning Coach 消费的结构化 completion signal，不会在课堂主内容中强制弹出正式 quiz 流程。
2. 当用户希望继续巩固时，结果页提供明确的课后入口，点击后跳转到后续 checkpoint / quiz / path 流程，且当前课堂结果页仍然可以被重新打开与回看。
3. 当 Learning Coach 尚未真实接通时，课后入口依然可以基于冻结的跳转参数与 mock 数据工作，课堂页不需要等待 Learning Coach 全部完成后才可验收。

## Dev Notes

### Business Context

- 课堂主叙事的结束方式直接影响用户对产品节奏的感受；本 Story 的目标是把“会话结束”变成平滑 handoff，而不是突兀切题。
- completion signal 既要服务结果页 CTA，又要服务后续 Learning Coach，因此字段必须兼顾页面消费与后续引擎消费。
- 该 Story 可以先以 mock handoff 完成前端验收，不需要等待 Epic 8 全部实现。

### Technical Guardrails

- completion signal 必须是独立结构，不能把 Learning Coach 私有结构直接塞进课堂主结果。
- 结果页 CTA 只能作为后续动作，不得篡改课堂主内容顺序或把正式答题嵌回主内容流。
- 回跳与再打开课堂结果页的语义必须在入口参数中稳定表达，避免后续路由反复返工。
- mock / real 模式必须使用同一套入口参数形状。

- 覆盖结果页不插入正式 quiz 主流程这一边界。
