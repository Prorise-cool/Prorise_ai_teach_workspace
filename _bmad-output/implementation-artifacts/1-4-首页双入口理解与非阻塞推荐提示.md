# Story 1.4: 首页双入口理解与非阻塞推荐提示

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 首次或回访用户，
I want 首页清楚呈现“主题课堂”和“单题视频”两个入口并在可用时给出推荐，
so that 我可以几乎不犹豫地开始正确的学习流程。

## Acceptance Criteria

1. 首页清晰展示两个主入口、差异说明和明确 CTA，在 `1024px` 宽度下无需滚动即可看到两个入口的主标题、说明与按钮。
2. 推荐提示区只起辅助选择作用，不阻塞双入口主路径；即使推荐逻辑失败，入口仍完全可用。
3. 未登录用户点击任一入口 CTA 时被引导到 `/login` 并保留原始目标意图，登录成功后恢复到相应输入页。

## Tasks / Subtasks

- [ ] 重构首页信息架构（AC: 1, 2）
  - [ ] 用双入口卡片替换当前 scaffold 占位内容。
  - [ ] 为“主题课堂”和“单题视频”写清差异说明与 CTA。
  - [ ] 保证桌面端首屏 1024px 下完整可见。
- [ ] 增加非阻塞推荐提示区（AC: 2）
  - [ ] 推荐区独立于入口卡片，不影响主 CTA 可用。
  - [ ] 推荐数据缺失时展示静态提示或直接隐藏，不渲染错误态阻塞层。
- [ ] 接入入口跳转与意图保存（AC: 3）
  - [ ] 已登录直接去 `/classroom/input` 或 `/video/input`。
  - [ ] 未登录跳到 `/login?returnTo=...` 或等效导航 state。

  - [ ] 覆盖双入口展示、CTA 跳转、未登录重定向、推荐区降级。

## Dev Notes

### Business Context

- 首页的核心职责是“3 秒内让用户理解两个入口差异”，不是展示复杂平台结构。
- 推荐提示是辅助选择机制，不是流程前置门槛。
- Story 1.4 必须消费 Story 1.1 的认证契约和 Story 1.2 的回跳逻辑。

### Technical Guardrails

- 首页路由固定为 `/`，双入口职责固定，不能把课堂 / 视频真正输入逻辑提前塞进首页。
- 未登录点击入口后必须保留目标意图，避免登录成功后回到首页重新选择。
- 当前 MVP 以桌面端为主，至少满足 `1024px` 最小支持宽度。
- 保持小麦统一品牌风格，不因为入口差异引入双主题或过重视觉分叉。

### Testing Requirements

- 覆盖两个入口的标题、说明、CTA 可见性。
- 覆盖推荐区有数据、无数据、失败降级三种情况。
- 覆盖未登录点击入口跳到 `/login` 并附带 `returnTo`。
- 覆盖桌面端布局不需要滚动即可看到双入口主体内容。
