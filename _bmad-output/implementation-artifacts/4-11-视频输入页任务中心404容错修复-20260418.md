# Story 4.11：视频输入页任务中心 404 容错修复

- 日期：2026-04-18
- 范围：`packages/student-web`
- 关联页面：`/video/input`、首页 `/` 顶栏任务中心

## 问题现象

用户在视频输入页创建任务并进入等待页后，再返回输入页或首页时，右上角 bell 任务中心有时显示 `0 项`，同时浏览器网络面板出现多条：

`GET /api/v1/video/tasks/{taskId}/status -> 404 Not Found`

但同一时刻：

- `GET /api/v1/video/tasks?status=pending...` 返回 `200`
- `GET /api/v1/video/tasks?status=processing...` 返回 `200`

## 根因结论

这不是前端请求路径写错，也不是 Vite 代理失效。

真实原因是视频工作区任务中心的聚合 adapter 会先拉取 RuoYi 元数据列表，再对每条活跃任务补拉运行态快照：

1. `/api/v1/video/tasks?status=pending|processing` 返回的是任务元数据列表。
2. `/api/v1/video/tasks/{taskId}/status` 返回的是 FastAPI 运行态快照。
3. 当列表里混入已经过期的旧任务时，状态快照会返回 `404`，语义是“运行态已过期或不存在”。
4. 旧实现对所有快照请求使用 `Promise.all(...)`，只要其中一条 404，整个活跃任务查询就直接 reject。
5. React Query 因此拿不到 `data.items`，输入页 / 首页最终把任务中心渲染成空列表。

## 修复内容

本次修复只改 active-task 聚合层，不改页面结构：

- 在 `video-workspace-task-adapter.ts` 中新增单条快照的 404 判定。
- 当某条任务快照返回 `404` 时，仅跳过这条脏数据。
- 其他错误（如 `401`、`403`、`500`）继续抛出，避免误吞真实故障。
