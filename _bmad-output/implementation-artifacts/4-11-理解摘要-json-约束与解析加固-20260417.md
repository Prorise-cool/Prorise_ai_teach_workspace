# 4-11 理解摘要 JSON 约束与解析加固

- 日期：2026-04-17
- Story：`4-11-视频等待页渐进式产物展示与分段预览`
- 状态：`review`

## 问题现象

等待页“先看这段快速解释”没有展示大模型真实解题摘要，而是回退成题面原文；同时知识点区域可能出现英文分段标题，说明 understanding 子链路失败，preview 退回了 fallback 内容。

典型日志特征：

- `Understanding stage failed, fallback to raw prompt summary`
- `json.decoder.JSONDecodeError: Expecting value: line 1 column 1 (char 0)`

## 根因

### 1. understanding 请求缺少强格式约束

- 文件：`packages/fastapi-backend/app/features/video/pipeline/services.py`
- 原实现只把 `sourcePayload + userProfile` 直接 `json.dumps` 后发给模型，没有明确要求“必须只返回 JSON 对象”。
- 在 `gemini-*` 等模型上，这很容易得到自然语言说明、带前缀解释的文本，或者 fenced JSON，而不是裸 JSON。

### 2. 解析器对 JSON fence 容错不足

- 原 `_parse_jsonish()` 仅尝试：
  - `json.loads(text)`
  - `json.loads(extract_code_from_response(text))`
- `extract_code_from_response()` 偏向代码抽取，对 ` ```json `、前置说明文字、夹带解释的 JSON 文本不稳定。

## 本次修复

### 1. UnderstandingService prompt 加固

- 新增 `_build_understanding_prompt(...)`
- 明确要求模型：
  - 只返回单个 JSON 对象
  - 不输出额外解释、Markdown、标题或前后缀
  - 固定包含 `topicSummary / knowledgePoints / solutionSteps / difficulty / subject`
  - 中文题默认输出简体中文

### 2. `_parse_jsonish()` 解析容错增强

- 先移除 `<think>...</think>`
- 支持提取 ` ```json ... ``` ` / ` ```text ... ``` ` 等 fenced 内容
- 仍兼容旧 `extract_code_from_response(...)`
- 在候选文本中尝试用 `json.JSONDecoder().raw_decode()` 扫描第一个合法 `{` / `[` 起点，支持“前面有解释，后面才跟 JSON”的返回

### 3. 语义级兜底不再只靠 prompt

- 新增语义校验：如果 `topicSummary` 过短、只是题面回声、语言不匹配，或 `knowledgePoints / solutionSteps` 掉成空壳，就不直接把结果透给前端。
- 新增 repair second pass：把上一轮输出和失败原因回灌给模型，要求只返回修补后的 JSON。
- 新增本地 deterministic fallback：
  - `topicSummary` 太弱时，会优先根据 `solutionSteps.explanation` 合成一段可读摘要。
  - `knowledgePoints` 缺失时，会从 `solutionSteps.title` 里提取学生可读短语。
- orchestrator 侧新增分镜回填：如果 preview 里的摘要仍然像题目回声，且 storyboard 已产出有效讲解，则回退到 section lecture lines 重新拼一版等待页摘要。

### 4. backend pytest 阻塞同步修复

- 原阻塞不是视频链路逻辑本身，而是 system Python 里的 `dramatiq 2.1.x` 已不再导出 `dramatiq.middleware.Prometheus`。
- `packages/fastapi-backend/app/infra/redis_client.py` 现已改为可选导入 `Prometheus`，并在 1.x / 2.x 运行时都能安全创建 broker。
- 对应 `tests/unit/core/test_dramatiq_broker.py` 已改为双版本兼容断言，避免测试反过来把新兼容层锁死在旧 API 上。

## 结果预期

修复后，understanding 成功时：

1. 等待页摘要优先显示中文快速讲解，而不是原题题面。
2. 知识点优先显示 understanding 返回的学生可读知识点，而不是 storyboard / outline 的英文标题兜底。
3. 即使模型给了“形式上合法、语义上很弱”的 JSON，后端也会先做 repair 或本地兜底，而不是直接把弱摘要透给前端。
4. backend 回归不再被 `dramatiq` 版本漂移卡死，等待页摘要链路可以在 system Python 下直接跑单测确认。
