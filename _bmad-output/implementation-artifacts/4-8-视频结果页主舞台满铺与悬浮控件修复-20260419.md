# Story 4.8 补充修复：视频结果页主舞台满铺与悬浮控件收口

> 日期：2026-04-19
> 范围：`packages/student-web/src/features/video/pages/video-result-page.tsx`、`packages/student-web/src/features/video/styles/_result.scss`
> 关联 Story：4.8《视频结果页、播放器与结果操作》

---

## 背景

2026-04-19 的结果页视觉对齐中，把 `.xm-video-result__player-frame` 收窄成了 `width: min(100%, 600px)`，导致播放器只停留在主舞台中央一小块区域。右侧 Companion 侧栏虽然仍可正常展开，但视频容器没有吃满除顶栏 / 进度 / 公开横幅 / 侧栏之外的剩余空间，和设计稿“主舞台满铺、控件悬浮覆盖”的结构不一致。

## 本次收口

- 结果页主舞台继续维持 `main + aside` 同级 flex，不改 Companion 侧栏真实宽度与收展动画。
- `xm-video-result__stage` 保持为剩余空间容器，`xm-video-result__player-shell` 不再通过底部 `padding` 给字幕 / Dock 预留真实布局空间。
- `xm-video-result__player-frame` 改为 `position: absolute; inset: 0; width: 100%; height: 100%;`，直接铺满主舞台剩余区域。
- 字幕层 `VideoSubtitle` 与底部 `VideoDock` 继续作为绝对定位浮层存在，只覆盖在播放器之上，不再参与播放器尺寸计算。

## 设计对齐依据

参考设计稿：

- `Ux/03UI:UX 设计素材/001UI 设计稿/04-成品图/01-正式路由页面/05-视频结果页/02-video-result.html`

设计稿里的主舞台核心结构是：

- `main` 为 `flex-1 flex flex-col h-full min-w-0`
- 中央内容区为 `flex-1 relative ... w-full h-full`
- 字幕使用 `absolute bottom-[110px] left-1/2 ...`
- Dock 使用 `absolute bottom-10 left-1/2 ...`

也就是说，字幕和底部控制条天然就是覆盖层，不应该通过额外的内边距把播放器本体压小。

## 验证

- `pnpm --filter @xiaomai/student-web build`
  - 通过，确认 Sass 与结果页打包正常。

## 结论

视频结果页后续如果继续做视觉对齐，必须保持以下边界：

1. Companion 侧栏只通过自身真实宽度压缩 `main`，不能靠覆盖或假留白实现。
2. 播放器容器必须吃满主舞台剩余空间。
3. 字幕、Dock、未来的结果页浮层操作条都应作为覆盖层处理，不能再反向参与主舞台尺寸计算。
