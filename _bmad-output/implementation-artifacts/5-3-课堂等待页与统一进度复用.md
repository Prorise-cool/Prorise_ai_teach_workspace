# Story 5.3: 课堂等待页与统一进度复用

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 等待课堂生成的用户，
I want 课堂等待页复用统一任务体验并支持恢复，
so that 我能在熟悉的交互模式里理解课堂任务进度。

## Acceptance Criteria

1. 当课堂任务已创建且进入执行中时，用户打开 `/classroom/:id/generating` 必须能看到复用统一进度组件的课堂等待页，页面展示阶段状态、阶段描述与恢复动作，且阶段文案聚焦课堂生成、幻灯片、讨论与白板，而不是视频术语。
2. 当页面刷新、断线或事件流暂时不可用时，客户端必须通过 SSE 恢复、快照或 `/status` 查询降级恢复当前阶段与文案，用户不会因为一次刷新就丢失当前等待上下文。
3. 当页面运行在 mock 模式时，前端必须能演示处理中、失败、恢复、降级查询与完成跳转，不依赖真实课堂生成引擎可用性才具备验收条件。

## Tasks / Subtasks

- [ ] 创建课堂等待页路由与阶段映射（AC: 1）
  - [ ] 新建课堂等待页容器，并基于统一任务组件映射 classroom stages。
  - [ ] 定义课堂专属阶段文案，如主题理解、课堂结构生成、讨论编排、白板布局、结果整理。
  - [ ] 保留统一等待体验的视觉骨架和交互节奏。
- [ ] 接入统一 SSE transport 与状态查询降级（AC: 2）
  - [ ] 复用 `services/sse` 的 `Last-Event-ID`、snapshot 与 polling fallback。
  - [ ] 在页面刷新、切后台、短暂断线后恢复任务状态。
  - [ ] 当恢复失败时，明确展示降级提示，而不是让页面无限空白 loading。
- [ ] 打通完成跳转与失败闭环（AC: 1, 2, 3）
  - [ ] 任务完成后跳转到 `/classroom/:id`。
  - [ ] 失败时展示错误码 / 错误提示与重试或返回入口。
  - [ ] 权限失效或任务不存在时返回明确页面态。
- [ ] 建立 mock 场景与测试覆盖（AC: 3）
  - [ ] 增加 classroom 等待页专属 mock stage sequence。
  - [ ] 覆盖处理、完成、失败、snapshot 恢复与 polling fallback。

## Dev Notes

### Business Context

- 等待页是课堂主链路的中间态承接页，既决定用户对长任务的信任感，也决定结果页能否稳定承接完成跳转。
- 该页必须共享 Epic 2 已冻结的统一等待体验，不能再为课堂单独做一套不兼容的任务状态机。
- 本 Story 可以在 mock session 基础上先推进，确保前端不被真实生成链卡住。

### Technical Guardrails

- 课堂等待页必须复用统一任务 SSE 解析与 polling fallback，不允许在页面里再写第二套简化版解析器。
- classroom stage 只允许映射统一任务语义，不允许私自扩展新的公开 SSE 事件类型。
- 等待页必须有明确的失败态、恢复态与权限态，禁止通过“自动刷新重试”掩盖错误。
- 路由跳转应是确定性导航逻辑，不能依赖 `setTimeout` 或白屏 `return null`。
