# Story 4.11: 视频等待页渐进式产物展示与分段预览

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 等待讲解视频生成的用户，
I want 在等待页先看到结构化中间产物并逐步查看已完成片段，
so that 在最终整片生成完成前，我也能确认任务真实推进并提前开始消费部分结果。

## Acceptance Criteria

1. 当视频任务完成 `storyboard` 阶段后，等待页可展示渐进预览面板，至少包含题目摘要、知识点摘要与按顺序排列的分镜卡片（scene title、narration、durationHint）；这些数据来自真实后端产物，而不是前端根据日志文案猜测拼装。
2. 当 `render` 阶段有 section 渲染成功时，系统为该 section 生成稳定可访问的片段地址（如 `sectionClipUrl`），等待页可按顺序追加展示“已完成片段”卡片或播放器；用户刷新页面后仍能恢复这些已完成片段，而不是丢失到仅剩进度条。
3. 等待页渐进预览通过独立预览数据源恢复，推荐新增 `GET /api/v1/video/tasks/:id/preview` 并在现有 `task:progress` / `status.context` 中暴露 `previewAvailable`、`previewUpdatedAt`、`previewVersion` 等轻量信号；前端不需要把完整预览状态硬塞进统一任务快照才能恢复。
4. `/video/:id` 结果页仍保持“最终成片播放器”为唯一成功态入口；在最终 `videoUrl` 尚未生成前，系统不会把 section 片段预览伪装成任务已完成，也不会提前把等待页跳到结果页。
5. `manim_gen` 阶段不承诺“边生成边播放最终视频”；在该阶段系统只展示非视频类中间产物（理解摘要、分镜、ETA、阶段日志），避免对用户做出当前架构无法兑现的流式播放承诺。
6. 当部分 section 渲染失败但整体仍在可恢复或降级范围内时，等待页明确展示各 section 的状态（`pending` / `rendering` / `ready` / `failed` / `omitted`），已完成片段保持可看；若最终合成失败，任务整体仍进入 `failed`，不会因为已有 section 片段就错误标记为 `completed`。

### Business Context

- 最近真实样本显示总耗时约 `594s`，其中 `manim_gen` 单阶段约 `459s`，是当前主瓶颈；单纯延长超时不能解决用户体感问题。
- 当前等待页只有阶段与进度，结果页只认最终 `videoUrl`，导致用户在长时间生成过程中几乎没有可消费内容。
- “渐进式中间产物展示”可以显著改善等待体感，但必须建立在真实中间产物之上，而不是用日志伪装成更细粒度进度。

### Technical Guardrails

- 不改变 `/video/:id` 的成功态语义：结果页仍只消费最终 `VideoResult.videoUrl`，不把 section clip 当作任务完成的替代品。
- 不承诺在 `manim_gen` 阶段播放最终视频；当前架构下该阶段产出的是脚本，不是媒体文件。
- 不直接向前端暴露 raw TTS 音频；前端消费的是 storyboard 预览和已渲染 section clip。
- Preview endpoint 优先读运行态，不依赖 `xm_session_artifact` 长期表做等待页主链路恢复，避免把实时体验绑死在长链回写成功上。
- 已完成片段的发布必须是“按片段成功即发布”，不能等最终 merge 成功后再补发，否则失去渐进价值。
- 若 section clip 发布失败，不得影响最终主任务状态机；该失败应表现为预览能力降级，而不是整体任务直接失败。

### Project Structure Notes

- 渐进预览优先挂在等待页，不与结果页主播放器壳层混用。
- preview 数据源与最终 `result_detail` 分离，避免结果页查询接口在处理中阶段不断膨胀和语义漂移。
- section 片段路径建议复用现有 `/api/v1/video/assets/{asset_key}` 能力，在 development 与真实对象存储场景下保持同构。

### Testing Requirements

- 后端预览测试：`storyboard` 完成后能返回 scene 预览；section 渲染成功后返回 `sectionClipUrl`；运行态过期时返回明确降级结构。
- 前端等待页测试：preview 数据到达后能展示分镜卡片；section 片段到达后能按顺序展示；刷新恢复后数据不丢失。
- 契约测试：`task:progress` 在没有 preview 更新时不强制携带完整 preview payload；有更新时能通过版本信号驱动前端重新拉取。
- 结果页回归测试：存在 preview 数据时，`/video/:id` 在最终 `videoUrl` 缺失前仍不进入 success。
