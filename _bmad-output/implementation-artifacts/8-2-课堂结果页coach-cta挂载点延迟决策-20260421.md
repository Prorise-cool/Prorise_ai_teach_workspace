# 8-2 课堂结果页 Learning Coach CTA 挂载点延迟决策（2026-04-21）

> 归属 Story：`Epic 8 / Story 8.2 会话后入口与 Learning Coach 路由承接`
> 触发场景：`fix/epic-8-9-hardening` 分支的 P1-2 修复条目
> 关联审查报告：`test/epic-8-9-acceptance` 分支审核结论（2026-04-21）

## 背景

Epic 8 / Story 8.2 的验收标准硬性要求：

> Given 用户完成视频或课堂主内容
> When 页面展示后续动作
> Then 用户可通过明确 CTA 进入 checkpoint、quiz 或 path

这意味着 Learning Coach 的入口 CTA 需要在两种结果页上同时存在：

- **视频结果页**（`features/video/pages/video-result-page.tsx`）—— 已通过 `features/video/components/result-header.tsx` 的 `learningCoachTo` prop 实现；当前分支 commit `a168ab6 feat(video): link to learning coach` 与 `0d783b2 fix(video): move learningCoachTo useMemo before early returns` 已落地。
- **课堂结果页** —— **目前不存在**。当前 `packages/student-web/src/features/classroom/` 目录下仅有 `classroom-input-page.tsx`；路由表（`app/routes/index.tsx`）也仅注册了 `classroom/input`。

## 决策

2026-04-21 与产品所有者确认：**当前课堂模块前景未定，可能不会在本轮 MVP 内继续推进**。为避免引入一个"半挂载"的假结果页（以假 CTA 占位但无真实结束态语义），**本轮不新建 `classroom-result-page` 也不在 `classroom-input-page` 上添加 coach CTA**。

该决策的直接影响：
- Story 8.2 "课堂结束后"路径的验收条件目前**不满足**，在 Epic-5 课堂闭环重新开启前持续挂起。
- 视频侧 CTA 不受影响，视频 → Learning Coach 链路完整可用。

## 重启课堂模块时的补齐清单

当课堂模块恢复推进时，至少需要按以下顺序补齐：

1. **结束态结果页**：在 `features/classroom/pages/` 下新增 `classroom-result-page.tsx`，路由 `/classroom/result/:sessionId`。
2. **结束态数据源**：课堂会话结束后把 session payload 通过 `useAuthSessionStore` 或独立 store 提交到结果页；保证 `sessionId`、`sourceType=classroom` 与其他识别字段已准备好。
3. **CTA 挂载**：参考 `features/video/components/result-header.tsx:40-108` 的 `learningCoachTo` 接入方式，在课堂结果页顶部 header 中加入 coach CTA；链接格式参考 `buildLearningCoachSourceSearchParams({ sourceType: 'classroom', sourceSessionId, topicHint })`。
4. **联动 i18n**：复用 `video-content.ts` 中已有的 coach CTA 文案键，或在 `classroom-content.ts` 中镜像同一套文案以保持双入口一致。
5. **E2E 核对 Story 8.2 AC**：课堂结束页的 "先不做 / 进入 checkpoint / 进入 quiz / 查看学习路径" 四组 CTA 动作均可达；与视频侧保持一致的三页独立路由语义（见 P1-1 决策：`/checkpoint/:sessionId`、`/quiz/:sessionId` 不得再聚回单页状态机）。
