# Story 5.9: 课堂输入页联网搜索增强与证据范围配置

Status: backlog

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 需要更完整课堂素材的用户，
I want 在课堂输入页按需开启联网搜索并配置证据范围，
so that 课堂生成可以在必要时参考公开资料或指定来源而不是只依赖裸输入。

## Acceptance Criteria

1. `/classroom/input` 展示显式联网搜索开关与最小证据范围配置，且不挤占主题输入与主 CTA 的主叙事位置。
2. 用户开启联网搜索并提交课堂任务时，相关配置可稳定透传到后端。
3. 联网搜索不可用、结果不足或处于 mock 模式时，页面提供可解释降级提示，用户仍可关闭该能力后继续生成课堂。

## Dev Notes

### Business Context

- 联网搜索已升级为正式业务，不再按“伪业务”处理。
- 该能力属于课堂生成前 / 生成时的证据增强，不新增学生端独立 `/knowledge` 页面。
- 当前版本坚持显式 opt-in，默认不替用户开启公开资料检索。

### Technical Guardrails

- 联网搜索开关必须是显式配置，不能通过脆弱默认值隐式触发。
- 页面即使在联网搜索不可用时也必须保留正常课堂创建路径。
- 证据范围配置只需覆盖最小可用能力，不要借机扩展成独立知识库管理台。
