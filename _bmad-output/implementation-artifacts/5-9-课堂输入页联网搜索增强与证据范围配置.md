# Story 5.9: 课堂输入页联网搜索增强与证据范围配置

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 需要更完整课堂素材的用户，
I want 在课堂输入页按需开启联网搜索并配置证据范围，
so that 课堂生成可以在必要时参考公开资料或指定来源而不是只依赖裸输入。

## Acceptance Criteria

1. 当用户位于 `/classroom/input` 时，页面必须展示显式联网搜索开关与最小证据范围配置，且这些配置不会挤占主题输入与主 CTA 的主叙事位置。
2. 当用户开启联网搜索并提交课堂任务时，联网搜索开关状态与当前证据范围必须稳定透传给后端，后端无需依赖隐式默认规则猜测用户是否允许调用公开资料检索。
3. 当联网搜索不可用、结果不足或当前环境处于 mock 模式时，页面必须展示可解释提示、降级说明或 mock 样例数据，用户仍可关闭该能力后继续生成课堂，而不是被阻断在输入页。

## Tasks / Subtasks

- [ ] 在课堂输入页增加显式联网搜索与最小证据范围配置（AC: 1, 3）
  - [ ] 增加 `webSearchEnabled` 开关与最小 `evidenceScope` 选择区。
  - [ ] 控制其视觉层级，避免破坏主题输入主叙事。
  - [ ] 提供默认关闭与配置说明文本。
- [ ] 扩展课堂创建 payload 与 adapter 透传（AC: 2）
  - [ ] 将 `webSearchEnabled` 与 `evidenceScope` 纳入创建请求。
  - [ ] 保持 mock / real adapter 字段一致，不在页面中做分支组装。
  - [ ] 对非法范围值做前端校验或 adapter 级保护。
- [ ] 打通降级提示与可恢复操作（AC: 3）
  - [ ] 当搜索 provider 不可用时，明确提示并允许关闭能力继续生成。
  - [ ] 当结果不足或当前环境是 mock 时，展示降级说明与样例态。
  - [ ] 禁止把“未检索到公开资料”伪装成已有证据增强。
- [ ] 与 Evidence 域对齐语义而不越界实现（AC: 1, 2, 3）
  - [ ] 对齐 Epic 7 的 evidence scope 语义与联网搜索 provider 命名。
  - [ ] 不在本 Story 中引入来源抽屉、资料上传或独立 `/knowledge` 页面。

## Dev Notes

### Business Context

- 联网搜索是课堂输入页的“按需增强能力”，不是默认强开，也不是独立知识产品入口。
- 本 Story 的目标是把“用户是否允许公开资料增强”显式化、可配置化，而不是把 Evidence 面板提前塞进课堂页。
- 它与 Epic 7 存在强语义对齐关系，但前端页面可先在冻结语义下独立推进。

### Technical Guardrails

- `webSearchEnabled` 必须由用户显式控制，禁止默认开启后再通过文案补救。
- evidence scope 只能表达最小来源范围语义，不能在本 Story 越界实现资料上传、引用展示或术语解释。
- 创建请求字段必须与后端和 mock 资产统一命名，避免 `searchEnabled / useWeb / evidenceScope` 多套命名并存。
- provider 不可用与结果不足必须可解释，不能用伪引用或空成功响应糊弄页面。
