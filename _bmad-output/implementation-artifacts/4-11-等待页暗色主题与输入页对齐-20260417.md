# Story 4.11 等待页暗色主题与输入页对齐

- 日期：2026-04-17
- 关联 Story：4-11-视频等待页渐进式产物展示与分段预览
- 范围：`packages/student-web/src/features/video/components/task-generating-view.scss`
- 状态：review

## 背景

等待页暗色模式与视频输入页的暗色切换不一致，部分卡片、标签、品牌图标、详情区在暗色下仍呈现偏白的半透明底，和输入页的全局 token 体系割裂。

## 根因

1. 等待页 SCSS 仍使用 `.dark ...` 作为暗色覆盖入口。
2. student-web 实际主题切换通过 `document.documentElement.dataset.theme = 'dark'` 驱动，因此 `.dark` 覆盖不会命中。
3. 多个 surface 直接写死为 `rgba(255, 255, 255, ...)`，没有跟随 `theme.css` 的亮暗色 token。

## 实施内容

1. 将等待页局部主题入口改为 `:root[data-theme='dark'] .xm-generating-shell`，与输入页 `_video-input-card.scss` 保持一致。
2. 在等待页局部样式中新增：
   - `--xm-generating-card-surface`
   - `--xm-generating-soft-surface`
   - `--xm-generating-pill-surface`
   - `--xm-generating-screen-chip-surface`
   - `--xm-generating-screen-chip-border`
3. 将以下区域从硬编码白底改为局部 token surface：
   - 顶部品牌图标壳层
   - 阶段导航胶囊外层
   - 进度条轨道
   - 状态标签 / live pill
   - 左侧日志头
   - 右侧摘要 / 分段 / 详情卡
   - 资源阶段统计卡
   - 分段详情 quote box
4. 保持视频屏幕内的 overlay chip 独立视觉，但其暗色值同样改为局部 token，不再直接依赖 `.dark`。

## 验证

- `pnpm --filter @xiaomai/student-web test -- src/features/video/pages/video-generating-page.test.tsx`
- `pnpm --filter @xiaomai/student-web typecheck`

## 后续约束

- 等待页若继续补视觉，不要再新增 `.dark` 选择器分支。
- 暗色切换统一以 `theme.css` + `globals.css` + `:root[data-theme='dark']` 为唯一入口。
- 若要继续对齐输入页气质，优先调 `task-generating-view.scss` 顶部局部 `--xm-generating-*` token，而不是在下方组件块里散落写具体颜色。
