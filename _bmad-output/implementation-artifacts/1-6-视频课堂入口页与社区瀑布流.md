# Story 1.6: 视频/课堂输入入口页 与 首页社区瀑布流

Status: in-progress

## Story

As a 学生用户，
I want 在首页看到清晰的视频创作和课堂创作入口，以及其他成员公开分享的作品瀑布流，
so that 我能快速进入创作，并在社区中发现他人的优质案例获取灵感。

## Acceptance Criteria

1. **视频输入页入口（AC: 入口可达）**
   - 首页顶栏或主操作区存在"创建视频"入口，点击后路由到 `/video/input`（或设计稿指定路径）。
   - `/video/input` 页面视觉还原设计稿 `03-视频输入页/01-input.html`，至少包含：输入框/文本区、提交动作区、必要的占位提示文案。

2. **课堂输入页入口（AC: 入口可达）**
   - 首页顶栏或主操作区存在"创建课堂"入口，点击后路由到 `/classroom/input`（或设计稿指定路径）。
   - `/classroom/input` 页面视觉还原设计稿 `06-课堂输入页/01-input.html`，至少包含：输入框/文本区、提交动作区、必要的占位提示文案。

3. **社区瀑布流（AC: 首页可见）**
   - 首页底部区域展示"社区作品"卡片瀑布流，对齐设计稿 `01-首页与入口/02-home.html` 中的卡片网格布局。
   - 当前阶段使用 mock 数据预加载（至少 8 条，每条含：封面图占位、标题、作品类型标签、作者昵称、浏览量）。
   - 卡片点击可扩展（当前阶段 click 不做跳转或用 `console.log` 占位）。
   - 瀑布流不得渲染管理后台字段（如 `status`、`tenant_id`）。

4. **数据表就绪（AC: 后端就绪）**
   - `xm_user_work` 表 DDL 已添加至 `xm_dev.sql`，字段对齐 RuoYi 标准（含 `tenant_id`、`version`、`del_flag`、`create_dept`、`create_by` 等），并通过 `task_ref_id` 关联 `xm_video_task` / `xm_classroom_session`，通过 `cover_oss_id` 关联 `sys_oss`。
   - 管理员可在 RuoYi 后台通过 `status` 字段（normal/hidden/blocked）控制社区内容可见性。

## Dev Notes

### Business Context

- 本 Story 是用户**进入创作流**的最顶层入口，优先级极高。
- 社区瀑布流当前阶段只用 mock 数据；真实分页接口（`GET /xiaomai/userWork/community`）在后续 Story 联调时接入。
- `xm_user_work.status` 由管理员在 RuoYi 后台操控，前端社区列表接口只返回 `status = 'normal'` 且 `is_public = 1` 的数据。

### Technical Guardrails

- 视频输入页与课堂输入页是两个独立路由页，不共享一个"通用输入"组件——设计稿的两个页面交互差异较大。
- mock 数据文件放在 `features/home/mock/communityWorks.ts`，不引入全局 mock 机制。
- 社区卡片不得自定义 toast / 弹层逻辑，复用 `sonner` 全局通道。
- 页面样式遵守 CLAUDE.md 前端规则：令牌走 `--xm-*`，复杂视觉走 feature SCSS，不硬编码颜色/间距。
- 输入页复用已有的 `shadcn/ui` 原语（如 `Textarea`、`Button`）；不在页面内手写同类交互壳层。

### Reference: OpenMAIC Pattern

OpenMAIC (`references/OpenMAIC/`) 的社区 UI 参考点：
- `app/page.tsx`：首页 `listStages()` 加载卡片 → `ClassroomCard` 组件 → `grid-cols-2 md:grid-cols-3 lg:grid-cols-4`。
- OpenMAIC 用 IndexedDB（Dexie.js）存本地数据，**无多用户共享**；我们的社区瀑布流数据来自 `xm_user_work` 服务端表，是核心差异。
- 卡片数据结构参考：`{ id, title, workType, author, coverUrl, viewCount, likeCount, createdAt }`。

### Reference: xm_user_work 表关键设计决策

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | bigint（Snowflake） | 主键，非 AUTO_INCREMENT，与 RuoYi Snowflake 策略对齐 |
| `tenant_id` | varchar(20) DEFAULT '000000' | **必须有**，多租户隔离 |
| `user_id` | bigint NOT NULL | 关联 `sys_user.user_id`（bigint，不是 varchar） |
| `work_type` | varchar(20) | `video` 或 `classroom` |
| `task_ref_id` | varchar(64) UNIQUE | 对应 `xm_video_task.task_id` 或 `xm_classroom_session.task_id` |
| `cover_oss_id` | bigint | 关联 `sys_oss.oss_id`，不存文件本体 |
| `cover_url` | varchar(500) | 冗余缓存封面直链，减少 JOIN |
| `is_public` | tinyint(1) | 用户控制；0=私有，1=公开到社区 |
| `status` | varchar(20) DEFAULT 'normal' | 管理员后台控制：normal/hidden/blocked |
| `version` | int DEFAULT 0 | 乐观锁，对齐 test_demo 标准 |
| `del_flag` | int DEFAULT 0 | 软删除，对齐 test_demo 标准 |

已知 **`xm_video_task` 和 `xm_classroom_session` 存在字段缺陷**（缺 `tenant_id`、`del_flag`，`user_id` 为 varchar）——这两张表的修正列为技术债，不在本 Story 范围内，但应在下一个 SQL 修正批次中处理。
