# Story 5.10: 课堂结果导出与分享产物

Status: backlog

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 需要沉淀课堂结果的用户，
I want 在课堂结果页导出可分享的课堂产物，
so that 我可以把当前课堂用于备课、外发或后续复用。

## Acceptance Criteria

1. 用户可在 `/classroom/:id` 发起导出并看到明确的导出中、成功、失败或权限不足状态。
2. 当前版本至少返回一种可分享文件产物，优先 `PPTX` 或等效教学分享文件，并明确展示当前导出类型。
3. 导出失败、处理中或后端尚未准备完成时，不影响课堂结果本身的浏览与学习链路。

## Dev Notes

### Business Context

- 课堂结果导出已正式纳入 BMAD，不再视为设计越界。
- 该能力属于前台学习 / 备课消费能力，与后台查询导出、审计导出是不同边界。
- 当前版本要求的是基础导出，不是完整导出平台。

### Technical Guardrails

- 导出入口只能作为结果消费动作，不能抢占课堂主内容区。
- 当前版本只需承认一种基础导出产物，优先 `PPTX`，其余增强能力继续留在后续版本。
- 导出失败不能污染课堂结果本身的完成态与浏览能力。

### Project Structure Notes

- 可先基于 mock 产物链接或假导出状态流实现结果页行为。
- 当前前端真实结构采用 `src/features/*` + `src/app/routes/index.tsx`，课堂结果页需新增到对应 feature 目录，而不是旧的 `src/pages/*`。
- 导出产物如需长期引用，应进入长期业务宿主或对象存储，而不是只停留在运行态。
- 不要把 `Story 10.8` 的后台导出边界错误复用为前台课堂结果导出。
- 在 `5.1` 课堂结果 schema 和结果页壳层稳定前，本 Story 维持 `backlog`。

### Testing Requirements

- 覆盖导出中、成功、失败、权限不足四类状态。
- 覆盖产物链接展示与重试 / 稍后下载提示。
- 覆盖导出失败时课堂结果仍可正常浏览。
