# Story 5.10: 课堂结果导出与分享产物

Status: ready-for-dev

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 需要沉淀课堂结果的用户，
I want 在课堂结果页导出可分享的课堂产物，
so that 我可以把当前课堂用于备课、外发或后续复用。

## Acceptance Criteria

1. 当用户进入已完成的 `/classroom/:id` 并发起导出动作时，页面必须返回明确的导出中、成功、失败或权限不足状态，导出动作不会打断当前课堂主内容浏览。
2. 当当前版本只提供基础导出能力时，系统至少返回一种可分享文件产物，优先 `PPTX` 或等效教学分享文件，页面明确展示当前导出类型，而不是对用户承诺不存在的多格式体系。
3. 当导出失败、处理中或后端仍未准备完成时，页面必须提供重试、稍后下载或降级说明，失败不会污染课堂结果本身的浏览与学习链路。

## Dev Notes

### Business Context

- 导出是课堂结果的消费增强能力，目标是支持备课沉淀、分享或二次使用，而不是扩展成后台导出体系。
- 本 Story 的成功标准是“用户能拿到一个明确、可解释的分享产物或状态”，而不是一次性做全格式导出平台。
- 课堂主内容仍然是主叙事，导出只是在结果页提供消费出口。

### Technical Guardrails

- 导出状态必须独立于课堂结果主浏览状态，导出失败不能把整页打进错误态。
- 当前版本只支持的格式必须在契约和页面文案中写清楚，禁止虚假承诺“多格式导出”。
- 导出产物若走对象存储，应返回稳定引用或下载入口，而不是临时页面 blob 黑盒。
- 不将后台查询 / 审计导出边界混入学生端课堂结果导出 Story。
