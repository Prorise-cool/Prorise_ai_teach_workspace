# Story 1.4: 首页课堂直达入口与顶栏导航分发

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 首次或回访用户，
I want 首页以课堂主入口直接带我进入工作区，并通过顶栏导航进入其他学习入口，
so that 我可以用最短路径开始学习，同时仍能快速找到其他有效入口。

## Acceptance Criteria

1. 首页首屏展示单一主 CTA，并明确进入课堂工作区或 `/classroom/input`；在 `1024px` 宽度下无需滚动即可看到主 CTA 与顶边栏导航。
2. 顶边栏导航可提供其他有效入口或账号动作，不要求用户先在首页做双入口比较；导航本身不依赖推荐逻辑或异步数据才可用。
3. 未登录用户点击首页主 CTA 或任一受保护导航入口时，被引导到 `/login` 并保留原始目标意图，登录成功后恢复到对应目标页面。

## Dev Notes

### Business Context

- 首页的核心职责是“3 秒内把用户送进课堂主工作区，并显式暴露其他可用入口”，不是让用户在首屏做双入口比较。
- 营销落地页由 Story 1.7 单独承接；Story 1.4 只负责默认产品首页 `/`。
- 当前成品图 `02-home.html` 采用单一中心 CTA，不再以双入口卡片作为首页主叙事。
- Story 1.4 必须消费 Story 1.1 的认证契约和 Story 1.2 的回跳逻辑。

### Technical Guardrails

- 首页路由固定为 `/`，主 CTA 直达 `/classroom/input`，不能把课堂 / 视频真正输入表单提前塞进首页。
- 营销落地页固定为 `/landing`，不得与 `/` 混成同一个页面职责。
- 顶栏导航承接其他学习入口，但不应把首页再次做成双入口卡片分流页。
- 未登录点击入口后必须保留目标意图，避免登录成功后回到首页重新选择。
- 当前 MVP 以桌面端为主，至少满足 `1024px` 最小支持宽度。
- 保持小麦统一品牌风格，不因为入口差异引入双主题或过重视觉分叉。

### Project Structure Notes

- 当前 `HomePage` 仍是纯占位内容，本 Story 会把它升级为第一个正式业务页面。
- 现有路由只有首页，适合顺势把认证跳转、课堂输入页和其他入口占位路由一起补入路由树。
- 首页只负责主入口分发与导航，不应承载输入表单和任务创建逻辑，那些属于 Epic 3 / 5。

### Testing Requirements

- 覆盖首页主 CTA、顶栏导航与登录 / 未登录跳转行为。
- 覆盖未登录点击主 CTA 或顶栏入口跳到 `/login` 并附带 `returnTo`。
- 覆盖桌面端布局不需要滚动即可看到主 CTA 与顶栏主体内容。
- 覆盖首页不再出现双入口卡片对比文案。
