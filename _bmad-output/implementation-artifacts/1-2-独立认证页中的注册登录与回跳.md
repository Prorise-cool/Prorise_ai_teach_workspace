# Story 1.2: 独立认证页中的注册、登录与回跳

Status: review

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a 访客，
I want 在独立认证页中完成注册或登录并回到原上下文，
so that 我可以顺畅进入小麦而不会在入口流程里丢失当前位置。

## Acceptance Criteria

1. 访客进入 `/login` 后，可在同一页面切换登录态或注册态并提交有效信息；成功后建立已认证会话并回跳原页面或原意图上下文。
2. 表单输入无效、凭证错误或认证失败时，页面展示明确的字段级或表单级错误提示，用户可在当前页继续修正。
3. 登录页支持键盘操作和可见焦点，不因动画或样式隐藏关键交互焦点状态。

## Dev Notes

### Business Context

- `/login` 是整个产品唯一认证入口，首页与受保护页面只负责把用户带到这里。
- Story 1.2 必须消费 Story 1.1 的认证契约、mock handler 和 `returnTo` 约定。
- 当前版本重点是“可恢复的认证体验”，不是做复杂营销或身份分流页面。

### Technical Guardrails

- 认证页必须是独立页面，不允许首页弹窗完成登录。
- 成功后必须回跳原上下文，未授权重定向不能丢失用户原始意图。
- 关键交互必须支持键盘操作，焦点样式不可被动画或自定义样式吞掉。
- 认证失败文案不能泄露账户是否存在、密码策略等敏感信息。
- 继续沿用现有前端栈：React 19、React Router 7、React Hook Form、Zod、Testing Library。

- 当前 `AppShell` 只提供全局背景和 `Outlet`，认证页可先复用该壳层，不要过早引入第二套 layout。
