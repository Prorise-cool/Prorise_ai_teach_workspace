# Story 4.11 补丁：等待页进度卡长提示词堆叠热修复（2026-04-20）

## 背景

视频等待页 `/video/:taskId/generating` 左侧进度卡片使用 `.xm-generating-shell__title` 渲染“任务标题”。历史实现中该标题直接读取 `sessionStorage` 的 `video-task-draft:${taskId}`，而创建任务时会把用户输入的完整提示词 `values.text` 写入这个 key。  
当提示词较长且包含 Markdown（blockquote/加粗/列表）时，大字号标题会把卡片高度撑爆，出现明显“字堆叠”，并挤压日志区域可视空间。

## 根因

1. `useVideoCreate` 将 `values.text.trim()` 直接写入 sessionStorage 作为“标题”，未做清洗与截断。
2. 等待页标题样式按短标题设计：大字号 + 紧 line-height，不适配长文本。

## 修复

### student-web

1. 新增标题处理工具：`packages/student-web/src/features/video/utils/video-task-title.ts`
   - `normalizeVideoTaskTitle()`：去掉 blockquote/列表前缀与常见 markdown 装饰符，压缩空白。
   - `resolveVideoTaskDraftTitle()`：生成用于 sessionStorage 的短标题（默认 48 chars）。
   - `resolveVideoTaskTitleDisplay()`：等待页展示时输出 `isLong` 供样式分支使用。
2. 创建任务时写入短标题，避免新任务继续把超长 prompt 当标题缓存。
3. 等待页标题展示增加 `is-long` 样式分支：降字号、增行高、3 行 line-clamp，并为标题增加 `title` 属性便于 hover 查看全文。
