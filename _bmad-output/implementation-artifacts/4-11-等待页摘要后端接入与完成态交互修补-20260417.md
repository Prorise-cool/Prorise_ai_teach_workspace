# 4-11 等待页摘要后端接入与完成态交互修补

- 日期：2026-04-17
- Story：`4-11-视频等待页渐进式产物展示与分段预览`
- 状态：`review`

## 本次目标

1. 等待页摘要必须来自后端真实理解结果，而不是题面原文截断。
2. 摘要要尽早出现在 `/preview`，不要等 storyboard 全部完成后才可见。
3. 等待页完成后不再自动跳转结果页，改为用户自己点击顶部按钮进入。
4. 顶部原“返回输入页”主按钮改为“前往结果页”；返回输入页只保留左上品牌图标入口。

## 实现摘要

### 1. FastAPI：preview.summary 改为理解阶段真实产出

- 文件：`packages/fastapi-backend/app/features/video/pipeline/orchestration/orchestrator.py`
- 新增行为：
  - 在 `_run_design_stage()` 的理解阶段先调用 `UnderstandingService`。
  - 用 `topic_summary + solution_steps` 组装等待页 / 结果页共用的摘要 Markdown。
  - 在 storyboard 生成前先写入一版仅含 `summary + knowledgePoints` 的初始 preview，`previewVersion=1`。
  - storyboard 完成后再把 sections 合并进 preview，并递增 `previewVersion`。
- 收口结果：
  - 前端现在会在理解阶段结束后就能通过 `/preview` 拉到可读摘要。
  - 最终结果 `VideoResult.summary` 也改为优先复用 preview 摘要，避免等待页与结果页两套语义。

### 2. student-web：完成态改为手动前往结果页

- 文件：`packages/student-web/src/features/video/pages/video-generating-page.tsx`
- 新增行为：
  - 删除完成态 `setTimeout(... navigate('/video/:id'))` 自动跳转。
  - 顶部 CTA 改为“前往结果页”，任务未完成时默认灰色禁用，完成后启用。
  - 左上品牌图标继续作为返回输入页入口。

### 3. student-web：完成态文案与按钮样式同步调整

- 文件：
  - `packages/student-web/src/app/i18n/resources/video-content.ts`
  - `packages/student-web/src/features/video/components/task-generating-view.scss`

  - 完成态 toast 从“马上进入结果页”改为“可前往结果页查看”。
  - 完成态 ETA 文案从“正在跳转到结果页”改为“结果页已可打开”。
  - 顶部 CTA 改为灰色 token 风格，完成后只增强可点击态，不走高亮主按钮。

## 2026-04-17 晚间补充

### 4. Dramatiq / pytest 环境兼容补齐

- 文件：`packages/fastapi-backend/app/infra/redis_client.py`
- 现象：
  - 工作区 `.venv` 使用 `dramatiq 1.18.x`，`dramatiq.middleware.Prometheus` 可导入。
  - 系统 Python 3.13 环境使用 `dramatiq 2.1.x`，该导出已不存在，导致 `pytest` 在 `tests/conftest.py` 导入链阶段提前失败。
- 修补：
  - 把 `Prometheus` 改为可选导入。
  - 当运行时没有 `Prometheus` 导出时，broker 创建流程优雅降级，不再因为 metrics middleware 缺失而崩溃。
  - 单测同步改为兼容 `dramatiq 1.x / 2.x` 两种运行时。
- 收口结果：
  - 先前被环境阻塞的 backend pytest 现已可在 system Python 下直接执行，不再停在 import 阶段。

## 风险与后续关注

1. 当前等待页摘要来自 `UnderstandingService` 的 `topic_summary + solution_steps`，如果后续要切成“参考答案 + 步骤”双区块，建议在 preview schema 中显式拆字段，而不是继续堆进一个 `summary` 字符串。
2. 当前代码已兼容 `dramatiq 1.x / 2.x` 的 `Prometheus` 差异，但团队本地仍建议优先走 `packages/fastapi-backend/.venv` 统一依赖，避免“同仓两套 dramatiq 行为”带来额外认知负担。
