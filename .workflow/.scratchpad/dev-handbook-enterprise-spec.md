# 企业级开发文档规范基线（v1.0 / 2026-04-25）

> 所有 teammate 改写 docs/01开发人员手册/ 下的文档时必须遵守此基线。

## 1. 文档元信息（每篇文档头部必须有）

```yaml
---
文档编号: PRA-DEV-{section}-{seq}        # 如 PRA-DEV-003-0001
文档名称: 系统架构总览
版本: v1.0.0
状态: Released | Draft | Deprecated
作者: Prorise AI Teach 研发团队
创建日期: 2026-04-25
最近修订: 2026-04-25
适用范围: 全体研发 / 架构师 / 测试 / 运维
密级: 内部公开
---
```

紧接 yaml 必须有「修订记录」表格：

| 版本 | 日期 | 修订内容 | 作者 | 评审 |
|------|------|----------|------|------|
| v1.0.0 | 2026-04-25 | 文档初版 | XX | XX |

## 2. 通用结构（所有文档必备段落）

1. **1. 概述** — 目的 / 适用范围 / 阅读对象 / 术语缩写
2. **2. 引用文件** — 内部文档 + 外部标准（GB/T 8567, IEEE, RFC 等）
3. **3. 正文章节** — 按文档专题展开
4. **N. 附录 A：术语对照** —（可选）
5. **N+1. 附录 B：参考资料** — 链接 / 标准号
6. **修订记录** — 见 §1

## 3. 章节深度

- 标题层级最多 4 级（## → ###  → #### → #####）
- 每节首段 50-200 字概述，避免堆砌空话
- 关键决策用 **ADR 表格**：
  ```
  | 决策ID | 标题 | 状态 | 决策日期 | 背景 | 备选 | 最终决策 | 影响 |
  ```

## 4. 图示规范

- **架构图 / 流程图：必须用 Mermaid**（Markdown 内嵌 ```mermaid ... ```）
- C4 4 层（Context / Container / Component / Code）
- 序列图、状态图、ER 图、部署图：Mermaid `sequenceDiagram` / `stateDiagram-v2` / `erDiagram` / `flowchart`
- 每图必有「图 N-X：描述」标题与简短说明文字

## 5. 代码 / 命令规范

- 代码块标语言（```python / ```bash / ```typescript / ```yaml）
- 命令行示例区分「本地」「Docker」「生产」三态
- 引用代码必须给 file:line 锚点：`apps/fastapi-backend/main.py:42`
- 长代码片段 > 30 行折叠或截断展示要点

## 6. 表格规范

- 必须有表头 + 对齐
- 配置参数必须列：参数名 / 类型 / 默认值 / 说明 / 是否必填
- 接口必须列：方法 / 路径 / 入参 / 出参 / 错误码 / 限流 / 鉴权

## 8. 字数与质量

- **不强制 1 万字**，但每篇必须在以下维度都「展开到位」：
  - 至少 1 个 Mermaid 图
  - 至少 1 个表格
  - 至少 1 处 file:line 代码引用
  - 至少 1 个具体可复现示例 / 命令
  - 至少 1 段「为什么」（不只是「是什么」）
- 禁止：水内容、口水话、AI 自夸、ToDo 占位、空段落

## 9. 实际项目背景（写入文档时引用）

**项目：** Prorise AI Teach（AI 智能体教学平台 + AI 视频生成 SaaS）

**技术栈关键事实：**
- 后端：FastAPI（Python 3.12）+ RuoYi-Plus（SpringBoot 3）+ Dramatiq（任务队列）
- 前端：Vue 3 + RuoYi-Plus-Soybean（管理后台 packages/ruoyi-plus-soybean）+ student-web（学生端）
- 数据：MySQL 8.0 + Redis 7 + MinIO/OSS
- AI：多 Provider 体系（OpenAI / Gemini / Qwen），LLM Router + TTS Provider
- 视频：Code2Video 管道（Manim + voiceover）+ Docker 沙箱渲染
- 部署：Docker Compose（dev）+ K8s（prod 规划）
- BMAD：Epic/Story 驱动开发（_bmad-output/ 为 SoT）

**端口：** FastAPI=8090，RuoYi=8080，前端 dev=5666

**已知技术决策（写入 ADR 时引用）：**
- ADR-001：选 FastAPI 而非 Flask（异步/类型/OpenAPI）
- ADR-002：选 Dramatiq 而非 Celery（轻量/Redis broker）
- ADR-003：选 Code2Video 重写视频管道（替换原有 67 文件 / 11592 行）
- ADR-004：双前端栈（管理后台 + 学生端）
- ADR-005：MemPalace 作为 AI 记忆/规范的唯一入口

## 10. 写作流程（每个 teammate 必须执行）

1. **阅读现有文件** — 该目录下所有 .md 全读一遍
2. **检索代码事实** — 用 Grep/Read 验证技术声明（绝不臆测）
3. **输出新内容** — 用 Write 完整覆盖原文件（不要 Edit 增量）
4. **图表必备** — 至少 1 个 Mermaid
5. **交叉引用** — 跨文档用相对路径链接（`../004-开发规范/0001-编码规范.md`）
6. **完成后** — TaskUpdate 标记 completed，向 team-lead SendMessage 简短汇报

## 11. 禁止事项

- 不得编造代码路径 / 类名 / 端口号（必须用 Grep/Read 验证）
- 不得保留「TODO」「待补充」「TBD」（必须给具体内容）
- 不得复制粘贴大段相同模板凑字数
- 不得删除已存在的真实信息（可重组、可合并、可扩展）
