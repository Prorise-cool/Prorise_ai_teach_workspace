# ManimToVideoClaw

> **Language / 语言:** English | [中文](./README.zh-CN.md)

**ManimToVideoClaw** is an AI-powered pipeline that automatically generates explainer videos from questions, by orchestrating large language models, Manim animation rendering, TTS voice synthesis, and SVG/image generation into a fully automated cloud workflow.

## Architecture Overview

```
User Request
    └─► scenext-forwarding  (API gateway: create task / query status)
            └─► storyboard  (generate storyboard scenes)
                    └─► scenext  (generate Manim code)
                            └─► manimtovideo  (execute Manim → render video → upload to OSS)
                                        └─► Video URL returned to user
```

---

## Modules

| Module | Description |
|---|---|
| `scenext-forwarding` | API gateway — exposes `POST /api/gen_video` and `GET /api/get_status/{id}` |
| `storyboard` | Generates storyboard scenes from the question and sends them to `scenext` |
| `scenext` | Generates Manim animation code from storyboard scenes |
| `manimtovideo` | Executes Manim code, synthesises voice, assembles the final video, and uploads it to OSS |

## Prerequisites

### 1. API Keys

You will need API keys for the following models and services:

| Model / Service | Provider (default) | Config location |
|---|---|---|
| DeepSeek V3 / R1 | Volcengine (火山引擎) | `config.yaml` / `.env` in each module |
| Doubao-Seed-1.6 | Volcengine | same as above |
| Qwen2.5-VL-72B-Instruct | Volcengine | same as above |
| Doubao TTS (voice synthesis) | Volcengine | `manimtovideo` module |
| SVG generation (SVGIO) | SVGIO platform | `manimtovideo/icon/icon_gene.py` |

> The default model provider is **Volcengine**. To switch providers, update the `config.yaml` and `.env` files in each module accordingly.  
> `SF_API_KEY` = SiliconFlow platform key, `DOUBAO_API_KEY` = Volcengine platform key.

### 2. OSS Cloud Storage

An OSS-compatible bucket is required (default: **Alibaba Cloud OSS**).  
Storage-related code is located at `manimtovideo/cloudFunction/cloudFuntion.py`.

### 3. Database (Supabase / PostgreSQL)

Two tables are required: `tasks` and `video_gen_log`.

## Database Setup

### `tasks` table

```sql
create table public.tasks (
  id uuid not null default gen_random_uuid (),
  created_at timestamp with time zone not null default now(),
  user_id uuid not null default auth.uid (),
  title text not null,
  result jsonb null,
  state smallint not null,
  finish_at timestamp with time zone null,
  progress jsonb not null,
  consumption jsonb null,
  scenes jsonb[] null,
  serial_number serial not null,
  config jsonb null,
  constraint tasks_pkey primary key (id),
  constraint tasks_user_id_fkey foreign KEY (user_id) references auth.users (id)
) TABLESPACE pg_default;
```

### `video_gen_log` table

```sql
create table public.video_gen_log (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  user_id uuid not null default auth.uid (),
  processing jsonb null,
  state smallint not null,
  task_id uuid not null,
  manim_res text null,
  manim_code text null,
  title text null,
  constraint video_gen_log_pkey primary key (id),
  constraint video_gen_log_task_id_fkey foreign KEY (task_id) references tasks (id),
  constraint video_gen_log_user_id_fkey foreign KEY (user_id) references auth.users (id)
) TABLESPACE pg_default;
```

After creating the tables in **Supabase**, set the environment variables `SUPABASE_URL` and `SUPABASE_KEY` (secret) in each module.  
To use a different database, search for `supabase` in the codebase and replace the relevant calls.

## Deployment Guide

The system is deployed using **Alibaba Cloud Function Compute**. Each of the four modules is deployed as a separate cloud function and communicates with the others via HTTP APIs.

> **Note:** Only `manimtovideo` has significant compute requirements. The other three modules are lightweight and run on-demand (invoked only when a request arrives), saving cost. Only `scenext-forwarding` must run 24/7.

### Step 1 — Create Database Tables

Create the `tasks` and `video_gen_log` tables (see [Database Setup](#database-setup)) and configure `SUPABASE_URL` / `SUPABASE_KEY`.

### Step 2 — Configure Environment Variables

Review the `.env` file in each module and fill in the required API keys and settings.

### Step 3 — Configure OSS Cloud Storage

Edit `manimtovideo/cloudFunction/cloudFuntion.py` with your OSS credentials and bucket name.

### Step 4 — Deploy `manimtovideo`

Build and deploy the Docker container:

```bash
# from the manimtovideo/ directory
docker build -t manimtovideo .
# deploy the image to your cloud function platform
```

Recommended spec: **4 vCPU / 4 GB RAM**

Note down the URLs for:
- `POST /manimtovideo`
- `POST /deletetask`
- `POST /getlastframeimg`

### Step 5 — Deploy `scenext`

1. Edit `config.yaml` → `api_endpoint` section: fill in `manimtovideo_url`, `deletetask_url`, `getlastframeimg_url`.
2. Build and deploy the Docker container.

Recommended spec: **0.35 vCPU / 512 MB RAM**

Note down the URL for `POST /task`.

### Step 6 — Deploy `storyboard`

1. Edit `config.yaml` → `api_endpoint.scenext_url`.
2. Deploy the code to your server.

Recommended spec: **0.1 vCPU / 128 MB RAM**

Note down the URL for `POST /generate-storyboard`.

### Step 7 — Deploy `scenext-forwarding`

1. Edit `config.yaml` → `api_endpoint.storyboard_url`.
2. Deploy the code to your server.

Recommended spec: **0.1 vCPU / 128 MB RAM**

> This module must remain running **24/7** as it is the entry point for all requests.

## Running Tests

### Generate a Video

```python
import requests

url = "https://<domain>/api/gen_video"
headers = {
    "Authorization": "Bearer xxxxxx",  # placeholder — required
    "Content-Type": "application/json"
}

data = {
    "question": "What is the Pythagorean theorem?",
    "answer": "",
    "questionImages": [],
    "answerImages": [],
    "quality": "m",  # "l" = low, "m" = medium, "h" = high
}

response = requests.post(url, json=data, headers=headers)
print(response.json())
```

```python
task_id = "your-task-id"

headers = {
    "Authorization": "Bearer xxxxxx"  # placeholder — required
}

response = requests.get(url, headers=headers)
print(response.json())
```

## System Configuration

### Configuring the Voice (TTS)

The default TTS engine is **Volcengine Doubao** voice synthesis.

To switch to a different TTS provider:
1. Create a new TTS class file in `manimtovideo/manimtovideo/`.
2. Register the new class by setting the `TTS` field of `code_json` inside the `Task` class in `scenext/handler_fastapi.py`.
3. Configure voice role IDs (teacher voice and student voice) in the `scene['voiceRole']` section inside `storyboard/function/main.py` → `ai_storyboard()`.

### Configuring SVG / Image Generation

The system automatically detects whether a scene requires an SVG illustration and calls an external API to generate it.

Default provider: **SVGIO platform**

To switch providers, edit `manimtovideo/icon/icon_gene.py`.
