# ManimToVideoClaw

> **语言 / Language:** [English](./README.md) | 中文

**ManimToVideoClaw** 是一个 AI 驱动的自动化视频生成流水线，能够根据问题自动生成讲解视频。系统通过编排大语言模型、Manim 动画渲染、TTS 语音合成以及 SVG/图片生成，形成完整的云端自动化工作流。

## 架构概览

```
用户请求
    └─► scenext-forwarding  （API 网关：创建任务 / 查询状态）
            └─► storyboard  （生成视频分镜内容）
                    └─► scenext  （生成 Manim 代码）
                            └─► manimtovideo  （执行 Manim → 渲染视频 → 上传 OSS）
                                        └─► 视频 URL 返回给用户
```

---

## 模块介绍

| 模块 | 说明 |
|---|---|
| `scenext-forwarding` | API 网关，对外暴露 `POST /api/gen_video` 与 `GET /api/get_status/{id}` 接口 |
| `storyboard` | 根据问题生成视频分镜内容，并将分镜发送给 `scenext` 模块 |
| `scenext` | 根据分镜内容生成 Manim 动画代码 |
| `manimtovideo` | 执行 Manim 代码、合成语音、合并视频，并上传至 OSS |

## 前置准备

### 1. API Key

需要以下模型与服务的 API Key：

| 模型 / 服务 | 默认提供商 | 配置位置 |
|---|---|---|
| DeepSeek V3 / R1 | 火山引擎 | 各模块的 `config.yaml` / `.env` |
| Doubao-Seed-1.6 | 火山引擎 | 同上 |
| Qwen2.5-VL-72B-Instruct | 火山引擎 | 同上 |
| 豆包语音合成（TTS） | 火山引擎 | `manimtovideo` 模块 |
| SVG 生成（SVGIO） | SVGIO 平台 | `manimtovideo/icon/icon_gene.py` |

> 默认模型提供商为**火山引擎**。如需切换其他提供商，在各模块的 `config.yaml` 和 `.env` 文件中修改相应配置即可。  
> `SF_API_KEY` 为硅基流动平台 API Key，`DOUBAO_API_KEY` 为火山引擎平台 API Key。

### 2. OSS 云存储

需要一个 OSS 兼容的存储桶（默认：**阿里云 OSS**）。  
云存储相关代码位于 `manimtovideo/cloudFunction/cloudFuntion.py`。

### 3. 数据库（Supabase / PostgreSQL）

需要创建两张数据表：`tasks` 与 `video_gen_log`。

## 数据库配置

### `tasks` 表

```sql
create table public.tasks (
  id uuid not null default gen_random_uuid (),
  created_at timestamp with time zone not null default now(),
  user_id uuid not null default auth.uid (),
  title text not null,
  result jsonb null,
  state smallint not null,
  finish_at timestamp with time zone null,
  progress jsonb not null,
  consumption jsonb null,
  scenes jsonb[] null,
  serial_number serial not null,
  config jsonb null,
  constraint tasks_pkey primary key (id),
  constraint tasks_user_id_fkey foreign KEY (user_id) references auth.users (id)
) TABLESPACE pg_default;
```

### `video_gen_log` 表

```sql
create table public.video_gen_log (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  user_id uuid not null default auth.uid (),
  processing jsonb null,
  state smallint not null,
  task_id uuid not null,
  manim_res text null,
  manim_code text null,
  title text null,
  constraint video_gen_log_pkey primary key (id),
  constraint video_gen_log_task_id_fkey foreign KEY (task_id) references tasks (id),
  constraint video_gen_log_user_id_fkey foreign KEY (user_id) references auth.users (id)
) TABLESPACE pg_default;
```

在 **Supabase** 中创建好数据表后，在各模块中配置环境变量 `SUPABASE_URL` 与 `SUPABASE_KEY`（secret）即可。  
如需使用其他数据库，在代码中搜索 `supabase` 关键字，替换相关数据库调用代码即可。

## 部署流程

本系统使用**阿里云函数计算**进行云部署。4 个模块分别部署为独立的云函数，模块间通过 HTTP API 通信。

> **说明：** 只有 `manimtovideo` 对计算资源有较高要求，其余三个模块配置要求低、按需运行（有请求时才启动），节省成本。只有 `scenext-forwarding` 需要 24 小时持续运行。

### 第一步：创建数据表

按照[数据库配置](#数据库配置)章节创建 `tasks` 与 `video_gen_log` 表，并配置 `SUPABASE_URL` / `SUPABASE_KEY`。

### 第二步：完成环境变量配置

查看各模块的 `.env` 文件，填写所需的 API Key 及相关配置。

### 第三步：配置 OSS 云存储

编辑 `manimtovideo/cloudFunction/cloudFuntion.py`，填入 OSS 账号凭证与 Bucket 名称。

### 第四步：部署 `manimtovideo` 模块

构建并部署 Docker 容器：

```bash
# 在 manimtovideo/ 目录下执行
docker build -t manimtovideo .
# 将镜像部署到云函数平台
```

推荐规格：**4 vCPU / 4 GB 内存**

部署完成后，记录以下接口的访问 URL：
- `POST /manimtovideo`
- `POST /deletetask`
- `POST /getlastframeimg`

### 第五步：部署 `scenext` 模块

1. 编辑 `config.yaml` → `api_endpoint` 部分，填入 `manimtovideo_url`、`deletetask_url`、`getlastframeimg_url`。
2. 构建并部署 Docker 容器。

推荐规格：**0.35 vCPU / 512 MB 内存**

部署完成后，记录 `POST /task` 接口的 URL。

### 第六步：部署 `storyboard` 模块

1. 编辑 `config.yaml` → `api_endpoint.scenext_url`。
2. 将代码部署到服务器。

推荐规格：**0.1 vCPU / 128 MB 内存**

部署完成后，记录 `POST /generate-storyboard` 接口的 URL。

### 第七步：部署 `scenext-forwarding` 模块

1. 编辑 `config.yaml` → `api_endpoint.storyboard_url`。
2. 将代码部署到服务器。

推荐规格：**0.1 vCPU / 128 MB 内存**

> 该模块需要 **24 小时持续运行**，是所有请求的入口。其余三个模块只在有请求时启动，节省费用。

## 运行测试

### 生成视频

```python
import requests

url = "https://<domain>/api/gen_video"
headers = {
    "Authorization": "Bearer xxxxxx",  # 目前只是占位符，但该参数不可少
    "Content-Type": "application/json"
}

data = {
    "question": "什么是勾股定理",
    "answer": "",
    "questionImages": [],
    "answerImages": [],
    "quality": "m",  # "l" 低质量，"m" 中等质量，"h" 高质量
}

response = requests.post(url, json=data, headers=headers)
print(response.json())
```

## 系统配置

### 配置视频配音声音

系统默认使用**火山引擎豆包语音合成**模型。

如需切换其他 TTS 厂商：
1. 在 `manimtovideo/manimtovideo/` 目录下新建 TTS 类文件。
2. 在 `scenext/handler_fastapi.py` 的 `Task` 类 `code_json` 的 `TTS` 字段中，将值设置为新增 TTS 类的类名。
3. 在 `storyboard/function/main.py` 的 `ai_storyboard()` 函数中，找到 `scene['voiceRole']` 字段，分别配置老师和学生的声音 ID。

### 配置视频图片或 SVG

系统会自动识别场景是否需要插入 SVG 图示，若需要则调用外部 API 自动生成并插入视频。

默认平台：**SVGIO**

如需切换其他平台，编辑 `manimtovideo/icon/icon_gene.py` 即可。
