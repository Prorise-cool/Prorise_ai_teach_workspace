# Story 2.4 Redis 运行态边界

## 职责范围

Redis 在任务框架中只承担以下短期运行态职责：

1. 任务快照：保存当前最小状态，用于等待页恢复与 `/status` 查询。
2. SSE 事件缓存：保存短期结构化事件，用于 `Last-Event-ID` 补发与快照恢复。
3. Provider 健康缓存：保存短 TTL 的健康探针结果，用于 Failover 决策。
4. 投递映射：保存消息投递到任务 ID 的短期映射，便于异步链路追踪。

## 明确禁止

以下数据不得仅写入 Redis：

1. 视频结果、课堂摘要、Companion 问答记录等长期业务数据。
2. 审计、回放、后台检索所需的长期历史。
3. 文件产物、对象存储地址及其唯一宿主信息。

长期数据统一进入 RuoYi / MySQL，文件产物统一进入 COS。FastAPI 不承担长期业务数据主存储职责。

## 写入规则

1. 所有 `xm_*` 运行态 key 都必须设置 TTL。
2. 运行态缓存保存结构化 JSON，不保存原始 SSE 文本。
3. 恢复逻辑只读取最近快照与短期事件缓存，不依赖数据库回放全部历史。
4. 若调用方尝试将长期业务数据以 `long_term` 范畴写入运行态存储，应直接拒绝。
