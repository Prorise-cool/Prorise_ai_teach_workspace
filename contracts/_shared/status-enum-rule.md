# 状态枚举规则

## 目标

状态枚举是契约资产，不是实现细节。

## 定义要求

- 对外暴露值使用稳定的小写英文 token。
- 每个值必须说明是否为终态。
- 必须明确状态出现位置：接口响应、SSE 事件、详情快照或 Mock 样例。

## 最小字段

每个状态值至少应说明：

- `value`
- `label`
- `terminal`
- `description`
- `relatedStory`

## 变更约束

- 禁止在未升级契约版本的前提下重命名对外状态值。
- 禁止在示例 payload 中使用未登记状态。
- 前端状态机只能消费已冻结状态，不得猜测未登记值。
