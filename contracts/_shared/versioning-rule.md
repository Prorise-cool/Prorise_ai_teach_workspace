# 契约版本规则

## 版本语义

契约资产使用 `x.y.z` 版本语义：

- `major`：破坏性变更，必须新建 `v{major}/` 目录。
- `minor`：向后兼容的字段扩展、示例补充或说明增强。
- `patch`：文档修正、示例更正、无语义变更的整理。

## 目录映射

- `v1/` 对应所有 `1.x.x` 版本。
- `v2/` 对应所有 `2.x.x` 版本。
- 同一主版本的细节变化只写入当前版本目录下的 `CHANGELOG.md`。

## 变更记录最小字段

每条变更记录必须包含：

- 版本号
- 日期
- 变更类型：breaking / compatible / docs-only
- 影响范围：页面、接口、事件、测试或 Story
- 迁移说明

## 发布约束

- 禁止直接覆盖旧版本文件而不留变更记录。
- 禁止只改 schema 不改示例 payload。
- 禁止只改契约不标注影响 Story / 页面。
