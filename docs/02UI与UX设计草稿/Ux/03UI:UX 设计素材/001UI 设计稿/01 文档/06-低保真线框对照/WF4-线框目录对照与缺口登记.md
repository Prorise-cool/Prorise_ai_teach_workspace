## 2. 审计结论总览

### 2.1 已覆盖

以下对象在线框目录中已存在可对应的 HTML 文件：

- 首页与主学习入口：`/`、`/video/input`、`/video/:id/generating`、`/video/:id`
- 课堂主链路：`/classroom/input`、`/classroom/:id/generating`、`/classroom/:id`
- 知识问答：`/knowledge`
- 共享交互：认证载体、统一等待壳层、Teacher Style 面板、确认弹窗、空态与错误态

以下对象当前缺对应正式线框：

- 独立登录页（建议：`/login`）

### 2.2 已覆盖且职责边界已明确

以下线框文件存在，且本轮已明确其职责边界：

- 测验承载页（候选：`/quiz/:sessionId`）
- 学习路径承载页（候选：`/path`）
- `LearningCenter.tsx` 页面组件（`/learning`，负责学习结果聚合）
- `Profile.tsx` 页面组件（`/profile`，只负责个人资料）
- 历史视图（`/history`，归属学习中心结果管理）
- 收藏视图（`/favorites`，归属学习中心结果管理）
- 设置视图（`/settings`，归属个人域平台设置）

### 2.3 冲突

- 线框目录中存在 `01-auth-dialog.html`，但当前口径已改为“登录采用独立路由页”；因此该文件只能被视为历史共享资产参考，不能覆盖 `architecture.md` §10.3.1 / §10.3.3 规定的 `401 -> 清除 Token，跳转登录`、`403 -> 无权限 / 登录失效提示通道`。
- 目前最大的风险不再是“个人域与学习中心职责不清”，而是页面级规格仍未补完；尤其是 `/learning`、`/profile`、`/settings` 以及学习中心下的结果管理视图仍缺更细的规格定义。
- 历史 / 收藏线框文件当前位于“07-个人中心模块”目录下，但本轮已冻结其业务归属属于 LearningCenter 学习结果聚合域；文件位置与业务归属存在映射冲突。

### 2.4 仅归档

以下文件位于 `03-线框图/99-历史归档/`，应视为历史参考，不再作为当前正式口径：

## 5. 缺口与待确认点

### 5.1 页面级规格缺口

以下对象虽然已有线框，但页面级规格仍未完全冻结，不能因为“有线框”就视为可直接开发：

- 课堂输入页：`/classroom/input`
- 课堂结果页：`/classroom/:id`
- 独立登录页（建议：`/login`）
- `/learning`
- `/profile`
- `/settings`
- `/history`
- `/favorites`

### 5.2 交互实现层待确认

以下对象的职责边界已经明确，但交互实现层级仍可继续优化：

- 测验承载页：`/quiz/:sessionId`
- 学习路径承载页：`/path`
- 历史 / 收藏视图在最终实现中采用一级路由、学习中心二级路由还是学习中心内 Tab
- `/settings` 采用独立页还是 `/profile` 内部二级视图

### 5.3 认证语义待确认

- 登录页最终路径是否固定为 `/login`、注册是否同页承载，仍需继续冻结。
- 但无论登录页路径如何，401 / 403 的处理语义必须继续以 `architecture.md` 为准，不能被线框文件反向定义。

## 7. 结论

从“目录覆盖”角度看，当前线框目录已经能为首页、视频主链路、课堂主链路、知识问答，以及测验 / 学习路径 / 学习中心 / 个人域候选承载提供对应文件。  
从“架构对齐”角度看，仍必须明确区分两件事：

- 线框文件存在：说明有可参考的视觉草稿。
- 路由 / 页面边界已冻结：必须以 `architecture.md` 与其衍生的 IA / UF / PL3 文档为准。

因此，当前 WF4 的最终结论是：线框目录覆盖基本可用，但仍有 3 类收口工作未完成：

- 独立登录页线框仍缺失。
- 历史 / 收藏线框的文件位置与当前业务归属不一致。
- 测验、学习路径与学习中心相关页面仍缺更细的页面级规格与最终交互层级定义。
