## 2. 学习主链路总览（Route Map）

### 2.1 两条平行主链路（不混写）

- 课堂链路（主题学习）：用户输入“我想学 XX”主题，生成结构化课堂内容，形成课堂结果页，并在课堂结束后衔接测验与后续学习建议。
- 视频链路（单题讲解）：用户输入题目（文本或图片），生成可播放的讲解视频，形成视频播放页，并在播放页内继续追问与推荐延展。

### 2.2 全局入口到主链路的跳转

- `/`（首页双入口）：
  - 入口 A：进入 `/classroom/input`
  - 入口 B：进入 `/video/input`
  - 顶部“登录/注册”进入统一登录流程；主动入口可用 Dialog，但受保护路由拦截与 401 处理遵循 `architecture.md` 的固定语义。

### 2.3 主链路的统一“长任务三段式”

两条主链路都遵循同一结构：

- 输入页（创建任务与会话配置）：`/classroom/input`、`/video/input`
- 过程页（共享等待壳层 + SSE 进度）：`/classroom/:id/generating`、`/video/:id/generating`
- 结果页（回看与继续学习）：`/classroom/:id`、`/video/:id`

其中 `:id` 在本文统一表示生成链路的资源标识；generating 页以 task 运行态为主，结果页是否与 task ID 同值不在本文提前冻结，详见“冲突 / 待确认点”。

## 3. 视频链路信息架构（/video）

### 3.1 `/video/input`（问答 Agent 输入页，P0）

**页面职责（IA 视角）**：

- 承载“单题讲解视频”的最小输入闭环：题目输入（文本、拍照 OCR、粘贴）+ 会话级老师风格配置 + 主 CTA 创建任务。
- 不承载长任务进度（提交成功后进入 generating 页）。
- 非路由交互挂载：
  - 老师风格面板：附着在输入区附近展开，不是独立页面。
  - 统一登录入口：当需要登录时复用全局登录流程，401 / 403 的处理遵循 `architecture.md` 与认证主流程文档。

**关键出边（跳转）**：

- 提交创建任务成功：跳转 `/video/:id/generating`
- 返回：回到 `/`

### 3.2 `/video/:id/generating`（视频生成进度页，P0）

**页面职责（IA 视角）**：

- 复用“共享等待壳层”，展示统一进度组件与阶段进度。
- 通过 SSE 消费事件 `connected`、`progress`、`provider_switch`、`completed`、`failed`、`heartbeat`、`snapshot`，完成“等待透明、断线恢复、失败可行动”的体验闭环。
- 不展示最终结果内容（完成后自动跳转到结果页）。

**关键出边（跳转）**：

- `completed`：自动跳转 `/video/:id`
- `failed`：留在本页展示错误与动作（再试一次 / 返回输入 / 取消等），不跳转到独立错误页
- 手动返回：回到 `/video/input` 或 `/`（实现策略见“冲突 / 待确认点”）

### 3.3 `/video/:id`（视频播放页，P0）

**页面职责（IA 视角）**：

- 播放器承载视频播放、倍速、全屏等播放能力。
- 提供结果操作入口：分享、收藏等（收藏与个人域沉淀由 RuoYi 承接，页面仍是视频结果语义）。
- 提供“继续问 AI 老师”的局部交互入口（可作为侧栏对话区或局部模块），不强制跳出到其它路由。
- 提供“相关推荐”的延展入口（可跳转到其它 `/video/:id`，或回到 `/video/input` 开新任务）。

**关键出边（跳转）**：

- 继续学习：
  - 从推荐进入其它 `/video/:id`
  - 进入 `/knowledge` 做知识库问答（若产品决定将“追问”统一到知识问答页，需在“冲突 / 待确认点”明确）

## 4. 课堂链路信息架构（/classroom）

### 4.1 `/classroom/input`（初学场景输入页，P0）

**页面职责（IA 视角）**：

- 承载“主题学习”的最小输入闭环：主题输入 + 会话级老师风格配置 + 主 CTA 创建课堂任务。
- 主题输入的核心心智是“我想系统地学”，与 `/video/input` 的“我有道题不会”保持清晰区隔。
- 非路由交互挂载：
  - 老师风格面板：附着在输入区附近展开，不是独立页面。
  - 统一登录入口：当需要登录时复用全局登录流程，401 / 403 的处理遵循 `architecture.md` 与认证主流程文档。

**关键出边（跳转）**：

- 提交创建任务成功：跳转 `/classroom/:id/generating`
- 返回：回到 `/`

### 4.2 `/classroom/:id/generating`（课堂生成进度页，P0）

**页面职责（IA 视角）**：

- 与 `/video/:id/generating` 共用同一“共享等待壳层”与统一任务状态语义。
- 使用同一 SSE 事件模型做进度、断线恢复与失败处理（事件到 UI 映射策略一致）。

**关键出边（跳转）**：

- `completed`：自动跳转 `/classroom/:id`
- `failed`：留在本页提供明确动作（再试一次 / 返回输入 / 取消等）

### 4.3 `/classroom/:id`（虚拟教室结果页，P0）

**页面职责（IA 视角）**：

- 承载课堂结果的可回看闭环：至少可展示结构化课堂内容（例如幻灯片、讲解片段、讨论片段、课堂摘要等）。
- 课堂结束后触发“课后小测”衔接（以弹窗提示引导进入测验，不把测验误写为课堂结果页的子路由）。
- 提供继续学习的入口（知识问答、学习路径建议、推荐等），但不与视频链路混成同一结果页结构。

**关键出边（跳转）**：

- 触发测验：进入测验承载页（候选独立路径：`/quiz/:sessionId`）
- 进入知识问答：跳转 `/knowledge`
- 进入学习路径：进入学习路径承载页（候选独立路径：`/path`；若允许在 P0 仅露出入口占位，详见第 7 节）

## 5. 知识问答 / 测验 / 学习路径链路（横切能力）

### 5.1 `/knowledge`（知识问答页）

**页面职责（IA 视角）**：

- 知识库问答（RAG）、引用来源展示、相关知识点推荐。
- 作为横切能力，可从首页导航、视频结果页、课堂结果页进入。
- 问答记录属于长期数据，应可在学习中心或历史中再次打开（聚合页不在本文展开）。

**关键出边（跳转）**：

- 相关知识点推荐可作为“继续提问”的输入（不强制跳转到其它页面）。
- 若从知识点推荐触发学习辅助（测验、路径），应进入对应承载页（候选为 `/quiz/:sessionId` 或 `/path`，具体触发点见“冲突 / 待确认点”）。

### 5.2 测验承载页（候选独立路由：`/quiz/:sessionId`）

**页面职责（IA 视角）**：

- 承载一轮测验会话：题目展示、作答、提交、结果与解析、完成总结。
- 可从课堂结束后弹窗引导进入，也允许从知识问答或学习路径节点触发进入（按事实源定义的“学习辅助”范围）。

**关键出边（跳转）**：

- 测验完成后返回课堂结果页（`/classroom/:id`）或进入聚合页（如学习中心，非本文范围）。

### 5.3 学习路径承载页（候选独立路由：`/path`）

**页面职责（IA 视角）**：

- 承载学习路径的规划与查看：选择学习目标与周期，生成路径，展示分周/分节点结构，并提供“开始学习 / 保存路径 / 调整计划”等动作入口。
- 可从课堂结果页、知识问答页或学习中心推荐进入。

**关键出边（跳转）**：

- “开始学习”可能跳转到对应内容入口（例如进入 `/classroom/input` 以主题开启课堂，或进入 `/video/input` 以题目开启视频），具体映射关系需后续冻结。

## 6. 共享非路由交互挂载（Non-Route Interactions）

> 本节只定义“挂载位置与边界”，不把它们写成独立正式页面或独立路由。

### 6.1 共享等待壳层（Shared Waiting Shell）

- 挂载位置：仅在 `/video/:id/generating` 与 `/classroom/:id/generating` 两个过程页内部复用。
- 职责：
  - 展示统一任务状态（`pending`、`processing`、`completed`、`failed`、`cancelled`）与阶段进度。
  - 消费统一 SSE 事件模型（`connected`、`progress`、`provider_switch`、`completed`、`failed`、`heartbeat`、`snapshot`）。
  - 断线恢复：优先重连恢复，必要时降级为状态查询（轮询）。
- 明确边界：共享等待壳层不是独立页面，不存在 `/progress-shell`、`/waiting` 之类主路由。

### 6.2 风格面板（Agent Style Panel）

- 挂载位置：`/video/input` 与 `/classroom/input` 的“核心输入区附近”，以头像或胶囊触发器展开面板。
- 职责：展示当前会话风格与可选风格列表（头像、名称、选中态），支持未来扩展更多风格。
- 明确边界：风格面板不是独立页面，不存在 `/style` 之类主路由。

### 6.3 统一登录入口（Login Entry）

- 挂载位置：独立登录页（建议：`/login`）。
- 触发来源：
  - 首页 `/` 顶部“登录/注册”按钮。
  - 访问受保护路由时的统一登录流程。
  - 401 / 403 的鉴权处理。
- 明确边界：`architecture.md` 已固定 401 为“清除 Token，跳转登录”，403 为“提示无权限 / 登录失效”；当前轮不再把主动登录入口写成 Dialog。

## 8. 冲突 / 待确认点

以下仅记录差异与待确认点，不在本文中改写事实源或替代其它 IA 文档结论：

1. `/classroom/:id`（虚拟教室）在 `ux-design-specification.md` 的页面总览中标注“需补充”，但在路由优先级上属于 P0；需要补齐最小可验收的信息结构边界（至少包含：课堂内容展示、课堂结束态、测验引导入口、继续学习入口）。
2. 独立登录页的最终路径与注册承载方式仍需进一步冻结：当前轮按“建议 `/login`，注册作为同页状态切换”执行；若后续要拆出独立 `/register`，需统一到全站 IA。
3. `/knowledge`、测验承载页、学习路径承载页的 P0 / P1 / P2 分层在事实源之间存在差异：`prd.md` 将知识问答与课后小测标为 P0、学习路径标为 P1，但 `ux-design-specification.md` 将其分别标为 P1、P1、P2；需在评审中明确“能力闭环优先级”与“独立路由页面成品化优先级”是否允许拆分。
4. `:id` 的资源语义：生成过程页以 task 运行态为主，而结果页 `/video/:id`、`/classroom/:id` 是否与 task ID 完全一致，或存在“任务 ID -> 结果资源 ID”的二段映射，需要后端接口契约冻结后确认。
5. `:sessionId` 的来源：测验承载页候选路径 `/quiz/:sessionId` 中的 `sessionId` 是否等同于课堂会话 ID、测验会话 ID，或由后端创建的独立 quizSessionId，需要接口契约冻结后确认。
