  致命差距分析（按影响排序）

  GAP-1：音画不同步 — 根因：缺失 manim-voiceover

  参考项目做法：
  # base.j2 模板 — 每个 scene 的动画被 voiceover context manager 包裹
  class MyScene(VoiceoverScene, FlexibleElementsScene):
      def construct(self):
          for scene in scenes:
              tts = ByteTTS(voice=scene.voiceRole)
              self.set_speech_service(tts)
              with self.voiceover(text=scene.voiceText, max_subcaption_len=1024):
                  # 动画代码在这里 — manim-voiceover 自动：
                  # 1. 生成 TTS 音频
                  # 2. 测量音频时长
                  # 3. 让动画时长匹配语音时长
                  # 4. 自动生成 .srt 字幕
                  scene.scene_code
                  self.wait(0.1)

  当前项目做法：
  # compose.py — 渲染后再拼接
  # 1. 先渲染视频（self.wait(duration/8) 猜时长）
  # 2. 再生成音频（各段 TTS）
  # 3. FFmpeg 硬拼
  # → 动画时长和音频时长完全独立，永远对不上

  差距本质： 参考项目在 Manim 渲染时同步生成音频（帧级精度），当前项目是先画后配（秒级偏差）。

  影响： 这是"音画不同步"的唯一根因。当前架构下无论怎么调参数都不可能解决。

  ---
  GAP-2：公式排版错位 — 根因：缺失 MathTex + LaTeX 管道

  参考项目做法（三层保障）：

  1. Prompt 强制要求：
  数学公式仅使用 MathTex 类，不要使用 Tex 类
  MathTex 内的内容必须使用 LaTeX 语法

  2. AST 自动注入参数（ast_fix_code.py）：
  # 自动给 MathTex 注入 tex_template=TexTemplateLibrary.ctex（中文支持）
  # 自动给 Text 注入 font_size=28, font="Noto Sans CJK SC"
  params = {
      'Text': {'font_size': '20', 'font': '"Noto Sans CJK SC"'},
      'MathTex': {'tex_template': 'TexTemplateLibrary.ctex'},
  }

  3. 代码生成 Prompt 要求使用 self.add_elements()：
  必须使用 self.add_elements() 将元素组添加到画面
  元素组 VGroup/Group 内元素要做好布局控制，避免重叠
  坐标系上面的内容与坐标系必须放到一个 VGroup/Group

  当前项目做法：
  # script_templates.py — 默认模板只有 Text
  title_1 = Text("题目引入", font_size=32)
  body_1 = Text("我们先理解题目...", font_size=26)
  group_1 = VGroup(title_1, body_1).arrange(DOWN, buff=0.5)
  self.add(group_1)
  self.wait(max(duration/8, 0.5))  # 猜时长
  self.clear()

  差距本质：
  - 没有 MathTex → 公式用 Text 渲染 → 无法正确显示分数、根号、上下标
  - 没有 ctex 模板 → 中英混排公式必错
  - 没有自动参数注入 → LLM 生成的代码经常漏 font_size、tex_template
  - 没有 FlexibleElementsScene → 布局靠 LLM 猜坐标，频繁重叠/溢出

  ---
  GAP-3：固定2分钟 — 根因：Storyboard 时长归一化 + wait 猜时长

  参考项目做法：
  - 没有 target_duration 概念
  - 时长完全由 TTS 语音驱动：老师讲多少，视频就多长
  - 分镜数量由教师-学生对话自然终止（学生说"老师再见"）

  当前项目做法：
  # storyboard.py — 强制归一化到 90-180 秒
  target_duration: int = Field(ge=90, le=180)  # 默认 120s

  # normalize_storyboard_duration() — 拉伸/压缩场景时长到目标
  # script_templates.py — wait 时长 = duration_hint / 8（纯猜测系数）
  self.wait(max(scene.duration_hint / 8, 0.5))

  差距本质： 时长应该是 TTS 音频驱动的输出，而不是预设的输入。

  GAP-6：场景生成策略 — 逐场景 vs 一次性

  参考项目：
  for scene in scenes:
      # 传入已有代码作为上下文
      opti = prompt_opti(task, scene['imageDesc'], prev_show_code)
      scene_code = await gene_code_agent.text_to_code(task, opti)

      # 逐场景修复：AST → 静态检查 → 渲染验证
      scene_code = await stat_check_fix(task, scene_code)
      scene_code = ast_fix_code(scene_code)
      scene_code, img_url = await render_fix_main(task, current_code, scene_code, scene['id'])

      # 拼接到已有代码
      current_code = render_scene_code_process(prev=current_code, current=scene_code)

  当前项目：
  # 一次性生成全部场景代码
  prompt = "请根据 storyboard 输出可执行的 Manim Python 脚本。\n" + storyboard_json
  code = await llm.generate(prompt)
  # 只在整体渲染失败后修复，不逐场景验证

  差距： 一次性生成 → 后面的场景无法参考前面的代码 → 布局冲突、元素重叠。逐场景生成 + 增量验证 →
  每个场景都确保正确再继续。
