   Video Pipeline ManimCat Alignment Rewrite                                                                  
                                     
 Context

 Current video pipeline (feature/video-pipeline-manimcat-optimize, commit 812acda) has 4 oversized files
 full of dead code and architectural misalignment with ManimCat. The 5-team review identified 12 specific
 gaps where ManimCat has mechanisms we completely lack. User's directive:
 "еұҺеұұд»Јз ҒеӨҡзҡ„ж–Үд»¶зӣҙжҺҘеҲ дәҶйҮҚеҶҷпјҢе®Ңе…ЁеҜ№йҪҗmanimcat".

 Problem: agent.py (1278L), orchestrator.py (1468L), scope_refine.py (805L), gpt_request.py (563L) вҖ”
 bloated, dead code, wrong architecture.

 Goal: Close all 12 gaps, delete oversized files, rewrite from scratch aligned with ManimCat's two-stage
 generation + error handling + retry patterns.

 ---
 Wave 1: Two-Stage Generation (GAP-1) вҖ” Foundation

 Delete engine/agent.py (1278L), split into 3 focused files.

 1a. New engine/scene_designer.py (~200L)

 Port ManimCat's scene-design-stage.ts:
 - generate_scene_design(concept, output_mode, reference_images, ...) -> str
 - Unique seed: md5(concept + timestamp + random) (GAP-5)
 - Vision auto-switch: detect model capability, fallback without images on error (GAP-4)
 - LLM response diagnostics: check finish_reason for truncation/refusal (GAP-6)
 - Message format normalization: handle string/array message formats (GAP-7)
 - Extract <design> tag from response
 - Temperature 0.8, max_tokens 12000 (match ManimCat)

 1b. New engine/code_generator.py (~250L)

 Port ManimCat's code-from-design-stage.ts:
 - generate_code_from_design(concept, scene_design, output_mode, ...) -> str
 - Input: structured scene design (not raw concept)
 - Extract code from markdown/code blocks
 - Temperature 0.7, max_tokens 12000
 - Same seed/diagnostics/normalization as scene_designer

 1c. New engine/agent.py (~300L) вҖ” Lean coordinator

 Keep TeachingVideoAgent class but gut internals:
 - generate_design() вҶ’ calls scene_designer.generate_scene_design()
 - generate_all_code() вҶ’ calls code_generator.generate_code_from_design()
 - Remove: generate_outline(), generate_storyboard(), generate_section_code() (dead code)
 - Remove: get_mllm_feedback(), optimize_with_feedback() (disabled code)
 - Keep: render_full_video_with_sections(), debug_and_fix_code(), render_section()
 - Keep: RunConfig, required_render_successes (public API)

 Preserved public API: TeachingVideoAgent, RunConfig, required_render_successes

 ---
 Wave 2: Error Handling (GAP-2, GAP-3, GAP-8)

 2a. New engine/sanitizer.py (~150L)

 Port ManimCat's sanitizer.ts:
 - STDERR_LIMIT = 4096, STDOUT_LIMIT = 2048
 - truncate_output(text, limit) вҖ” tail-truncate long output
 - extract_code_snippet(error_text, code) вҖ” extract relevant code lines around error
 - infer_error_type(error_text) -> ErrorType вҖ” classify: Import, Syntax, Runtime, LaTeX, etc.
 - sanitize_render_error(stderr, stdout, code) -> SanitizedError

 2b. New engine/render_failure.py (~100L)

 Port ManimCat's render-failure/service.ts:
 - RenderFailureRecord dataclass: error_type, sanitized_message, code_snippet, memory_usage,
 recovery_status, timestamp
 - RenderFailureStore class: persist/queried via Redis (not just logs)
 - record_failure(task_id, section_id, error, code, memory_mb)
 - get_failure_history(task_id, section_id) -> List[RenderFailureRecord]

 2c. Update engine/code_retry.py вҖ” Doom Loop Prevention (GAP-8)

 Add to existing code_retry.py (181L):
 - detect_doom_loop(failure_history: List[RenderFailureRecord]) -> bool
 - Compare last N error signatures (hash of sanitized error)
 - If same error repeats 3+ times вҶ’ raise DoomLoopError, stop retrying
 - Keep existing SEARCH/REPLACE patch logic

 ---
 Wave 3: LLM Robustness (GAP-4, GAP-5, GAP-6, GAP-7)

 3a. Update engine/gpt_request.py (563L вҶ’ ~300L)

 Clean up LLMBridge:
 - Remove dead functions: request_gemini_video_img_token(), request_gpt41_token(), request_gpt41_img(),
 request_gemini_with_video()
 - Keep: LLMBridge class, text_api(), video_img_api(), configure_bridge(), get_bridge()
 - Add: check_response_quality(response) -> bool вҖ” check finish_reason, content length
 - Add: normalize_messages(messages) -> List[dict] вҖ” handle string/array format (GAP-7)

 Wave 4: Orchestrator Rewrite

 Delete orchestration/orchestrator.py (1468L), rewrite as ~400L.

 4a. New orchestration/orchestrator.py (~400L)

 Keep VideoPipelineService but gut run():
 - Remove 611-line monolith

   - _run_design_stage() вҶ’ calls agent.generate_design()

 - Keep SSE progress reporting
 - Quality gate: section failure вҶ’ sanitize вҶ’ record вҶ’ retry (max 4) вҶ’ doom loop check

 Preserved public API: VideoPipelineService, get_video_pipeline_service
