## 立项背景

### 高职教育核心痛点

中国高职教育长期面临"三高三难"困境：

- **三高**：高投入、高难度、高风险
- **三难**：难实施、难观摩、难再现

传统教学视频制作成本高昂，教师数字素养参差不齐，导致优质教学资源严重匮乏。传统方式制作一条 10 分钟的高质量讲解视频，教师需要脚本撰写 2-3 小时、动画制作 5-8 小时、配音剪辑 2-3 小时，总计超过 10 小时。

与此同时，每位学生的学习风格、认知偏好、知识基础各不相同，"一刀切"的教学视频无法满足个性化需求。

### 技术成熟度

AIGC 相关技术在 2025-2026 年迎来突破性进展，使本项目具备技术可行性：
- 大语言模型（GPT/Claude/文心）能力成熟
- 程序化动画（Manim）技术稳定
- 多模态 TTS 技术突破，支持自然流畅的语音合成

### 政策支持

国家职业教育改革方案、教育数字化转型战略、AI+教育融合发展等政策持续释放利好信号。

## 项目范围

### 双入口架构

小麦平台通过 Nginx 统一入口，提供两种使用模式：

- **入口 1（初学场景）**："我想学 XXX"——完整虚拟教室体验，复刻 OpenMAIC 核心功能（课堂生成流水线、LangGraph 编排、动作执行引擎、幻灯片/测验/PBL、多智能体讨论）
- **入口 2（问答 Agent）**："帮我讲解这道题"——单题讲解视频生成，基于 ManimToVideoClaw 合并（题目理解 Agent、分镜生成、Manim 代码生成、渲染 + TTS + 合成）

### MVP 范围

| 优先级 | 功能模块 | 交付目标 | 技术来源 |
|--------|----------|----------|----------|
| P0 | 课堂服务模块 | OpenMAIC 核心功能移植 | OpenMAIC |
| P0 | 视频服务模块 | 单题讲解视频生成 | ManimToVideoClaw |
| P0 | 用户系统 | 注册/登录/个人中心 | RuoYi API 对接 |
| P1 | 4 种 Agent 风格 | 严肃/幽默/耐心/高效 | Prompt Engineering |
| P1 | 视频播放器 | 播放/倍速/进度控制 | 自研 |
| P2 | 学习记录 | 历史记录/收藏管理 | 自研 |
| P2 | i18n 国际化 | 中英文切换 | react-i18next |
