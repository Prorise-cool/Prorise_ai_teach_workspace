## 项目信息

### 架构决策

`story 1-1` 默认采用以下承载方案：

1. **认证能力来源**：RuoYi

## 快速启动

```bash
cd packages/ruoyi-plus-soybean-master
pnpm install
pnpm dev
```

### 已知开发配置

- 开发环境变量文件：`packages/ruoyi-plus-soybean-master/.env.dev`
- 默认后端地址：`VITE_SERVICE_BASE_URL=http://localhost:8080`
- 默认开启 SSE：`VITE_APP_SSE=Y`

## 推荐启动顺序

1. 先启动 RuoYi 后端。
2. 再启动现有前端参考实现，确认认证接口可达。
3. 最后创建或启动业务前台壳层，接入统一认证入口。

## 常见问题

### `pnpm dev` 无法连接后端

先检查 `.env.dev` 中的 `VITE_SERVICE_BASE_URL` 是否仍指向 `http://localhost:8080`。

### Story 1.1 无法开工

通常不是前端命令问题，而是学生端前台壳层尚未落仓。默认先补 `packages/student-web/`，再实现认证入口。
