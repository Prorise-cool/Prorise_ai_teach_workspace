## 项目信息

### 当前后端版图

| 项目 | 路径 | 角色 |
|------|------|------|
| RuoYi-Vue-Plus | `packages/RuoYi-Vue-Plus-5.X/` | 认证、 RBAC 、长期业务数据、后台管理 |
| FastAPI Backend | `packages/fastapi-backend/` | AI 功能服务层，当前仅占位 |

### 已确认的架构边界

1. 注册、登录、权限、长期业务数据由 RuoYi 承接。
2. FastAPI 负责功能执行、任务协调、外部能力调用与实时状态推送。
3. `story 1-1` 首先打通 RuoYi 认证闭环，不在 FastAPI 重造账户体系。

## RuoYi 参考后端启动

### 关键入口

- 启动模块：`packages/RuoYi-Vue-Plus-5.X/ruoyi-admin/`
- 启动类：`org.dromara.DromaraApplication`
- Profile：`local` / `dev` / `prod`，默认 `dev`
- 配置文件：`ruoyi-admin/src/main/resources/application-dev.yml`

## 快速启动

```bash
cd packages/RuoYi-Vue-Plus-5.X
mvn -pl ruoyi-admin -am spring-boot:run -Dspring-boot.run.profiles=dev
```

如果本地 Maven 插件链路未就绪，则退化为：

```bash
cd packages/RuoYi-Vue-Plus-5.X
mvn -pl ruoyi-admin -am clean package -DskipTests
java -jar ruoyi-admin/target/ruoyi-admin.jar --spring.profiles.active=dev
```

## 项目结构

### RuoYi 参考实现

- `ruoyi-admin/`：可运行主应用
- `ruoyi-common/`：公共能力
- `ruoyi-modules/`：业务模块
- `script/docker/`：Docker Compose 与部署脚本

### FastAPI 业务后端

当前 `packages/fastapi-backend/` 仅包含说明文档。正式落仓时最小结构应包括：

- `app/main.py`
- `app/api/`
- `app/features/`
- `app/shared/`
- `tests/`
- `pyproject.toml` 或等效依赖清单
- `.env.example`

默认端口沿用架构约定 `8090`。

## Story 1.1 与后端的关系

1. Story 1.1 直接依赖 RuoYi 认证接口可用。
2. FastAPI 不是 Story 1.1 的首个阻塞项，真正与 FastAPI 强关联的是 Story 1.3 的鉴权一致性。
3. 若在 Story 1.1 期间提前初始化 FastAPI，只建立最小应用壳层，不实现重复认证逻辑。

## 常见问题

### 后端启动但前端仍未登录成功

优先检查 RuoYi 认证接口、 CORS 与前端环境变量，而不是先怀疑 FastAPI。

### FastAPI 是否必须先写

不是。按照当前 Epic 切分， Story 1.1 可以先以 RuoYi 认证闭环为主， FastAPI 在后续鉴权一致性阶段再接入。
