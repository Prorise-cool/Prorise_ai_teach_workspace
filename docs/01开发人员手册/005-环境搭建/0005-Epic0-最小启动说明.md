## 目标

本说明用于满足 `Story 0.1` 的最小启动要求，确保开发者首次拉取仓库后，能够明确三端的角色、启动顺序与当前可验证边界。

注意：工作区即使已经包含后续 Epic 的代码资产或测试，也不代表这些业务 Epic 已全部验收；本说明只定义“能否启动”和“最小 smoke 面”。

## 三端边界

|------|------|----------|----------|

## 前置要求

- Node.js `>= 20.19.0`
- pnpm `>= 10.5.0`
- Python `>= 3.11`
- JDK `17`
- Maven `>= 3.9`
- MySQL `8.0`
- Redis `7.2`

## 推荐启动顺序

1. 启动 `MySQL`、`Redis` 与一个可用的 `RuoYi` 实例。
2. 在仓库根目录执行 `pnpm install`。
3. 执行 `pnpm setup:fastapi-backend` 创建 FastAPI 本地虚拟环境。
4. 复制 `packages/student-web/.env.example`、`packages/fastapi-backend/.env.example` 与 `packages/fastapi-backend/.env.local.example` 到本地环境文件。
5. 依次启动 `RuoYi`、`FastAPI` 与 `student-web`。

## 启动命令

### 学生端

```bash
pnpm dev:student-web
```

### FastAPI

```bash
pnpm setup:fastapi-backend
cp packages/fastapi-backend/.env.example packages/fastapi-backend/.env.defaults
cp packages/fastapi-backend/.env.local.example packages/fastapi-backend/.env.local
pnpm dev:fastapi-backend
```

说明：`pnpm dev:fastapi-backend` 当前会按 `.env.defaults -> .env.<FASTAPI_ENV> -> .env.<FASTAPI_ENV>.local -> .env.local` 的顺序读取配置，越靠后优先级越高；FastAPI 不再从 env 或 token 文件里读取 RuoYi 业务鉴权。

### RuoYi

```bash
cd packages/RuoYi-Vue-Plus-5.X
mvn -pl ruoyi-admin -am spring-boot:run -Dspring-boot.run.profiles=dev
```

## Smoke Checklist

- `student-web` 可以完成 `typecheck` 与 `test`。
- `FastAPI` 可以访问 `GET /health`，返回 `{"status":"ok"}`。
- `FastAPI` 可以访问 `GET /api/v1/video/bootstrap`、`/classroom/bootstrap`、`/companion/bootstrap`、`/knowledge/bootstrap`、`/learning/bootstrap` 等路由。
- `FastAPI` 的 `tasks` 恢复接口与统一契约输出可在本地测试中通过。
- `RuoYi` 可以在 `8080` 端口启动并暴露认证相关接口。
- `ruoyi-modules/ruoyi-xiaomai` 可以纳入 Maven 聚合编译。
- 根目录与启动命令不要求所有业务功能已交付，只要求入口、依赖、基础服务与最小 smoke 面完整。
- Epic 0 的验收口径仍是“最小启动成立”，不是“后续业务 Epic 自动完成”。
