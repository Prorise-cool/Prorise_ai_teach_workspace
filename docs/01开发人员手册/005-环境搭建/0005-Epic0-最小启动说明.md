## 目标

本说明用于满足 `Story 0.1` 的最小启动要求，确保开发者首次拉取仓库后，能够明确三端的角色、启动顺序与占位边界。

## 三端边界

| 模块 | 路径 | 当前职责 | 当前状态 |
|------|------|----------|----------|
| 学生端前台 | `packages/student-web/` | React 学生端业务前台宿主 | 可启动的工程骨架 |
| FastAPI 功能服务 | `packages/fastapi-backend/` | AI 功能服务宿主 | 架构对齐的框架骨架 |
| RuoYi 后端 | `packages/RuoYi-Vue-Plus-5.X/` | 认证、RBAC、长期业务数据、后台管理 | 可启动的参考后端 |
| 小麦业务模块预留 | `packages/RuoYi-Vue-Plus-5.X/ruoyi-xiaomai/` | 后续小麦业务模块落点 | 仅冻结目录边界 |

## 前置要求

- Node.js `>= 20.19.0`
- pnpm `>= 10.5.0`
- Python `>= 3.11`
- JDK `17`
- Maven `>= 3.9`
- MySQL `8.0`
- Redis `7.2`

## 推荐启动顺序

1. 启动 `MySQL`、`Redis` 与一个可用的 `RuoYi` 实例。
2. 在仓库根目录执行 `pnpm install`。
3. 执行 `pnpm setup:fastapi-backend` 创建 FastAPI 本地虚拟环境。
4. 复制 `packages/student-web/.env.example` 与 `packages/fastapi-backend/.env.example` 到本地环境文件。
5. 依次启动 `RuoYi`、`FastAPI` 与 `student-web`。

## 启动命令

### 学生端

```bash
pnpm dev:student-web
```

### FastAPI

```bash
pnpm setup:fastapi-backend
cp packages/fastapi-backend/.env.example packages/fastapi-backend/.env
pnpm dev:fastapi-backend
```

### RuoYi

```bash
cd packages/RuoYi-Vue-Plus-5.X
mvn -pl ruoyi-admin -am spring-boot:run -Dspring-boot.run.profiles=dev
```

## Smoke Checklist

- `student-web` 可以完成 `typecheck` 与 `test`。
- `FastAPI` 可以访问 `GET /health`，返回 `{"status":"ok"}`。
- `FastAPI` 可以访问 `GET /api/v1/video/bootstrap`、`/classroom/bootstrap`、`/companion/bootstrap` 等骨架路由。
- `RuoYi` 可以在 `8080` 端口启动并暴露认证相关接口。
- 根目录与启动命令不要求业务功能可用，只要求骨架、入口与环境说明完整。

## 明确不包含

- 任何正式业务接口实现。
- 任何业务表设计与长期数据落地。
- 任何视频、课堂、Companion、Evidence、Learning Coach 的业务逻辑。
- 将 `references/` 目录直接作为业务代码目录使用。
