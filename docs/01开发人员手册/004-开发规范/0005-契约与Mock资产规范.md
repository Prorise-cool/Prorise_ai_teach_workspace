# 契约与 Mock 资产规范

> **状态**: 已初始化
> **负责人**: Epic 0 Owner
> **最后更新**: 2026-03-29

---

## 目标

统一 `contracts/` 与 `mocks/` 的目录、命名、版本和变更说明规则，确保后续 Contract Story 有稳定资产落点。

## 唯一落点

- schema、状态枚举、错误码、示例 payload 统一进入 `contracts/`
- mock fixtures、状态流样例统一进入 `mocks/`
- 禁止在页面注释、Issue 文本或聊天记录中维护“唯一版本”的契约细节

## 目录规则

### contracts

```text
contracts/<domain>/
├── README.md
└── v{major}/
    ├── README.md
    ├── CHANGELOG.md
    ├── schemas/
    ├── examples/
    ├── errors/
    └── states/
```

### mocks

```text
mocks/<domain>/
├── README.md
└── v{major}/
    ├── README.md
    └── fixtures/
```

## 命名规则

- 域目录：小写英文语义词，例如 `auth`、`task`、`video`
- schema：`<domain>-<resource>.schema.json`
- example：`<resource>.<scenario>.json`
- 错误码：`<domain>-error-codes.md`
- 状态枚举：`<domain>-status.md`

## 版本规则

- 主版本目录使用 `v1`、`v2` 形式
- 同一主版本内的细节变化写入 `CHANGELOG.md`
- 破坏性变更必须新建新的主版本目录

## 发布门禁

- 契约变更必须同时更新示例 payload
- 错误码变更必须同时说明前端影响范围
- Mock 变更必须指向对应契约版本
- 前端不得解析自由文本 `message` 猜业务语义

## 当前模板域

`task/` 是当前模板域，用于验证目录与命名规则，不代表任务契约已正式冻结。
