## 验收项（请按顺序点击）

### A. 登录与入口

1. - [ ] 浏览器打开 `http://localhost:5173/login`。
2. - [ ] 账号 `admin` + 密码 `admin123`，**无需验证码**（dev captcha 已关）。
3. - [ ] 点击"登录" → 2 秒内自动跳回首页 `/`。
4. - [ ] 点击首页"进入工作区" 或 "Start Learning"，跳转到 `/classroom/input`。
5. - [ ] 页面渲染：顶部 Nav（单题讲解 / 学习中心 / 用户菜单）+ badge `CLASSROOM BUILDER` + 标题 "输入主题，即刻生成完整虚拟课堂" + 输入卡片 + "TRY THESE" 建议 + 底部"探索优质课堂案例" feed。

### B. 课堂生成流水线（真 LLM）

6. - [ ] 输入框输入 `用生活案例讲解牛顿第二定律 F=ma 的核心原理和实际应用`（或任意 10 字以上主题），"生成课堂"按钮变为可点击。
7. - [ ] 点击"生成课堂" → 按钮文字变成 `生成中... 5%`（动态进度条），输入区整体 disabled。
8. - [ ] **耐心等 2-4 分钟**（真 Gemini 调 ~13 次 LLM：1 outline + 1 agents + 10 × 2 scene/actions）。进度条数值会逐步更新（20% / 30% / 50% / 72% / 100%）。
9. - [ ] 生成完成后**自动**跳转到 `/openmaic/classroom/classroom_<hash>`。

### C. 播放页三栏布局

10. - [ ] 左栏"课程大纲"：10 条真实场景（例如 1 生活中的推与拉 / 2 加速度 / 3 力与加速度 / 4 虚拟实验室·互动 / 5 质量与惯性 / 6 核心公式 F=ma / 7 原理检测站·测验 / 8 实际应用 / 9 项目实战·项目 / 10 总结）。
11. - [ ] 顶部栏：OPENMAIC badge + 真实课程名 + 亮暗色切换 + 伴学助手按钮 + 返回首页按钮。
12. - [ ] 中央主画布：默认显示"选择一个场景开始学习"。
13. - [ ] 右栏"伴学助手"：课堂笔记 / 互动答疑 Tab。

### D. 幻灯片场景（真内容 + 动作播放）

14. - [ ] 点击左栏第 1 条"生活中的推与拉：牛顿第二定律的起点"。
15. - [ ] 主画布渲染：
      - 蓝色加粗大标题"生活中的推与拉：牛顿第二定律的起点"
      - 两栏对比 "场景 A：空超市推车"（质量 m 小 / 轻推高速度 / 运动易变） + "场景 B：满载推车"（质量 m 大 / 费力启动 / 难改变）
      - 核心物理变量卡片：力 (F) / 质量 (m) / 加速度 (a)，底部"物理学基础：因果关系的定量描述"
16. - [ ] **底部教师气泡**：林老师（或其他 LLM 生成的名字）头像出现，气泡内出现真实讲稿（例如"想象一下，当你推着一辆空的超市车，它的质量 m 较小..."）。

### E. 测验场景

## 未跑通（P1/P2，明确延后）

这些能力 OpenMAIC 参考项目有、我们**目前没接**。用户可接受合并 master，按优先级后续迭代补齐：

### P1（建议近期迭代补齐）

| 项 | 现状 |
|---|---|
| 白板实时绘制（老师一边讲一边画 SVG） | `wb_draw_*` 动作后端生成了，前端 `use-action-player` 占位未接 `Whiteboard` SVG |
| Edge TTS 高音质朗读 | 目前用浏览器 `speechSynthesis`（音色取决于 OS）；Edge TTS Docker 已启可接 |
| ASR 语音输入（麦克风提问） | 按钮占位，未接 |
| Scene 里的插图生成 | slide `type:image` 元素大多 `src` 空，需要接图像生成 provider |
| "最近课堂"卡片列表 | 后端保存在 Redis / Dexie 本地，但 `/classroom/input` 底部未加入口 |
| Scene 自动过渡 | 手动点击场景才切换；播放结束后未 auto next |
| Discussion action（场内主动触发多 Agent 讨论） | 动作类型已定义，未在 action player 中调 `/chat` |
| 字幕 / Lecture Notes 累积 | 右侧笔记 tab 仅占位文案 |
| 课堂导出（PPTX / HTML / ZIP） | OpenMAIC 用 `pptxgenjs` fork + `mathml2omml`，未移植 |
| 真 token 流（scene_outlines_stream 真 per-chunk SSE） | 目前 `stream_llm` 整块产出 |

### P2（高阶特性，工作量大）

| 项 | 说明 |
|---|---|
| Deep Interactive Mode（3D / 小游戏 / 思维导图 / Monaco code editor） | OpenMAIC v0.2.0 新增，要接 xyflow / three.js / Monaco 全套 |
| Agent 控制 interactive widget（老师主动点击演示） | 需要扩展 tool schema 让 agent 与 iframe 通信 |
| LangChain 真 function calling（替代 JSON-in-text） | Provider 协议级改造 |
| Extended thinking（o1 / Claude thinking budget） | Provider 协议级改造 |

### 明确不移植

- OpenMAIC 自带的视频生成 provider（Seedance / Kling / Veo / Sora）：我们已有 Manim 视频管道，不重复。
- OpenMAIC `ACCESS_CODE` 认证：我们走 RuoYi，冗余。
- OpenClaw skills：外部插件入口，与学生端无关。
- OpenMAIC `eval/` 评测面板：教学平台不用。

## 诊断指引（失败时）

| 症状 | 检查点 |
|------|--------|
| 点"生成课堂"后一直停在 5% | 终端看 Dramatiq worker 日志是否在调 LLM（`LLM request provider=openai-202604070402`），若无则 RuoYi `:8080` 或 token 获取失败 |
| 生成完跳转后主画布空白 | DevTools Network → 查 `GET /api/v1/openmaic/classroom/{id}` 响应的 `classroom.scenes[*].content` 字段；scene type 要匹配 `slide/quiz/interactive/pbl` 之一 |
| 登录后立刻跳回登录页 | RuoYi `:8080` 挂了；`curl http://localhost:8080/` 应返回 200 |
| Chat 报错"Recursion limit of 25 reached" | 后端 classroom.agents 为空；重新生成一次课堂让 agents 真正落库 |
| 教师气泡没出现 | store.agents 没水合；刷新页面应自动从 `classroom.agents` 回灌；否则 console 检查网络 `GET /classroom/{id}` 是否带 agents |
