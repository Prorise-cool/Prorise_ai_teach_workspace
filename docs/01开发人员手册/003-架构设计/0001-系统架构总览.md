## 架构概述

**小麦**是中国首个 AIGC 原生虚拟教室平台，采用**双入口架构**为高职教育提供沉浸式个性化学习体验：

- **入口 1（初学场景）**：完整虚拟课堂体验，复刻 OpenMAIC 全部后端能力（LangGraph 多 Agent 编排、课堂生成流水线、动作执行引擎），前端基于 React 19 重做
- **入口 2（问答 Agent）**：单题讲解视频生成，基于 ManimToVideoClaw 的 Manim 高质量动画管线（分镜生成、代码生成 + 自修复、渲染 + TTS + 合成）

系统采用**双后端松耦合**设计：
- **FastAPI (8090)**：ToC 功能微服务层，承载课堂服务和视频服务两大核心模块
- **RuoYi Spring Boot (8080)**：ToB 管理服务层，提供用户管理和 RBAC 权限控制（**保持不动**，仅通过 API 对接）

两个后端通过**共享 MySQL + Redis** 实现数据互通，JWT Token 两边验证，无需 HTTP 互调。前端通过 **Nginx 统一入口**按路径分发请求。

### 关键设计原则

1. **模块松耦合**：课堂服务与视频服务独立开发、独立部署，零共享数据库表、零直接 import
2. **共享层抽象**：LLM/TTS 通过抽象工厂模式，两个模块复用但互不依赖
3. **前端重做**：功能组件参考复用 OpenMAIC，风格组件必须重写（避免抄袭嫌疑）
4. **管理端不动**：RuoYi 仅通过 API 对接，不修改其代码
5. **早期 i18n**：W2 阶段即集成 react-i18next 国际化，降低后期改造成本

## 模块交互关系

### 请求流转路径

```
用户浏览器
    │
    ▼
Nginx 统一入口（按 URL 路径分发）
    │
    ├─ /api/classroom/*  ──▶  FastAPI (8090) 课堂服务模块
    ├─ /api/video/*      ──▶  FastAPI (8090) 视频服务模块
    ├─ /admin/*          ──▶  RuoYi (8080) 管理服务
    ├─ /                 ──▶  ToC 前端静态资源（React 19）
    └─ /admin-ui/*       ──▶  ToB 前端静态资源（Soybean Admin）
```

### 模块间通信方式

| 通信关系 | 方式 | 说明 |
|----------|------|------|
| **ToC 前端 → FastAPI** | RESTful HTTP + SSE 流式传输 | 课堂生成使用 SSE 实时推送进度；视频生成使用异步任务轮询 |
| **ToC 前端 → RuoYi** | RESTful HTTP | 仅用于用户注册/登录/个人信息管理 |
| **FastAPI → MySQL** | SQLAlchemy ORM | 只读用户表，读写课堂/视频业务数据 |
| **RuoYi → MySQL** | MyBatis-Plus | 读写用户表和权限数据 |
| **FastAPI ↔ Redis** | 直连 | JWT Token 验证、Provider 配置缓存（TTL ≤ 5 分钟）、任务状态管理 |
| **RuoYi ↔ Redis** | 直连 | JWT Token 存储、API 限流（Token Bucket） |
| **FastAPI → LLM Provider** | HTTP API | Gemini/Claude 等，含自动降级切换 |
| **FastAPI → TTS Provider** | HTTP API | 豆包/百度/Spark，Kokoro 为本地兜底 |
| **FastAPI → 腾讯云 COS** | SDK | 视频文件存储和分发 |
| **FastAPI → 腾讯云 OCR** | HTTP API | 题目图片识别，不可用时降级为手动输入 |

### 关键数据流

**入口 1 课堂生成流**：
1. 用户输入学习主题 + 选择 Agent 风格 → ToC 前端
2. ToC 前端 → FastAPI 课堂服务 → LangGraph 编排流水线
3. LangGraph 依次调用：场景大纲生成 → 场景内容生成 → 场景动作生成 → Agent Profile 生成
4. 各步骤通过 SSE 实时推送进度到前端
5. 课堂数据持久化至 MySQL，前端渲染虚拟课堂

**入口 2 视频生成流**：
1. 用户上传题目（文本/图片）+ 选择 Agent 风格 → ToC 前端
2. 图片经腾讯云 OCR 识别为文本
3. ToC 前端 → FastAPI 视频服务 → 题目理解 Agent → 分镜生成 → Manim 代码生成
4. Manim 代码执行渲染（含 AI/AST 自动修复）→ TTS 语音合成 → 音视频合成
5. 最终视频上传至腾讯云 COS，返回可播放 URL

## 技术约束

### 性能约束

| 约束项 | SLO 目标 | SLA 承诺 | 优先级 |
|--------|----------|----------|--------|
| API 响应时间（非生成类） | P95 < 200ms | P99 < 500ms | P0 |
| 视频生成端到端延迟 | P95 < 5 分钟 | P99 < 8 分钟 | P0 |
| 课堂生成端到端延迟 | P95 < 5 分钟 | P99 < 8 分钟 | P0 |
| Manim 代码渲染成功率 | ≥ 80% | ≥ 70% | P0 |
| 首屏加载时间 (FCP) | < 1.5s | < 3s | P1 |
| 系统可用性 | ≥ 99.5% | ≥ 99%（赛事期间 ≥ 99.9%） | P1 |

### 架构约束

| 约束项 | 要求 | 依据 |
|--------|------|------|
| **模块独立性** | 课堂服务与视频服务零共享数据库表、零直接 import | 支持独立部署和独立扩展 |
| **RuoYi 不可修改** | 仅通过 API 对接，不修改 RuoYi 代码 | 保持管理端稳定性 |
| **代码复用率** | > 60% 代码来自 OpenMAIC + ManimToVideoClaw | 降低开发成本，加速交付 |
| **前端项目结构** | Feature-Sliced Design | 业界主流前端架构 |
| **后端项目结构** | Clean Architecture + 模块化 | FastAPI 最佳实践 |
| **API 设计规范** | RESTful + OpenAPI 3.0 | 行业标准 |
| **目录结构** | Monorepo（pnpm workspace） | 复用 Soybean 模式 |

### 安全约束

| 约束项 | 实现方案 |
|--------|----------|
| 数据加密 | HTTPS + AES-256（敏感数据） |
| 认证机制 | JWT + 共享 Redis（复用 RuoYi 认证体系） |
| API 限流 | Redis Token Bucket |
| 输入验证 | 前端 Zod Schema + 后端 Pydantic |
| XSS 防护 | React JSX 自动转义 + DOMPurify（富文本场景） |

### 部署约束

| 约束项 | 规范 |
|--------|------|
| 生产环境 | Linux (Ubuntu 22.04+ / CentOS 8+)，腾讯云标准环境 |
| 开发环境 | 支持 Windows/macOS/Linux，Docker Desktop 必装 |
| 容器化部署 | Docker 镜像统一部署，避免环境差异 |
| 云平台 | 腾讯云 COS（文件存储）/ SCF（无服务器函数）/ TencentDB |

### 外部服务降级约束

| 外部依赖 | 降级策略 |
|----------|----------|
| LLM 服务（Gemini/Claude） | 连续 3 次超时自动切换备用 Provider；全不可用时返回友好提示；每 60s 探测恢复 |
| TTS 服务（豆包/百度/Spark） | 按优先级链切换：豆包 → 百度 → Spark → Kokoro（本地兜底）；每 120s 探测恢复 |
| 腾讯云 OCR | 不可用时提示用户手动输入题目文本；每 60s 探测恢复 |
