# 0018 Story 4.7 视频等待页 SSE 修复前端实机验收清单

日期：2026-04-09

## 验收目标

确认用户从视频输入页进入视频等待页后，页面可以在不刷新的前提下实时显示后端任务状态推进；刷新恢复与完成跳转也保持正常。

## 前置条件

- `student-web` 与对应后端环境已经启动。
- 当前环境可创建视频任务；真实环境或 mock 成功流均可。
- 验收人只需要执行浏览器点击与刷新操作，不需要看控制台日志。

## 点击验收清单

- [ ] 打开 `/video/input`，填写有效主题并提交创建视频任务。
- [ ] 页面成功跳转到 `/video/{taskId}/generating`，等待页没有停留在空白或永远 0%。
- [ ] 进入等待页后 5 秒内能看到“连接中 / 处理中 / 阶段文案 / 进度数值”中的任一实时变化，且无需手动刷新。
- [ ] 在任务完成前，至少观察到一次阶段文案或进度百分比继续推进，而不是长时间静止。
- [ ] 当任务进入并行阶段后，进度百分比只会上升或持平，不会出现例如 `58% → 42%` 的倒退。
- [ ] 若环境能触发修复链路，等待页会进入 `manim_fix` 修复态，同时总进度不会因为修复事件回退。
- [ ] 在任务处理中手动刷新一次页面，等待页能恢复当前状态，或进入降级轮询继续显示最新进度，不出现永久空白。
- [ ] 任务成功完成后，页面会自动跳转到结果页，而不是停留在等待页不动。
- [ ] 如果当前环境能触发修复态或失败态，对应提示卡片与按钮显示正常，不出现页面卡死。

## 备注

- 若验收环境没有修复态 / 失败态样本，只需完成成功链路与刷新恢复链路。
- 研发自测已覆盖 SSE 增量消费、hook 协作、等待页页面测试；本清单只关注用户可见行为。
