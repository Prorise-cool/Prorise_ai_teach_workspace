# Story 4.7 视频等待页 SSE 实时状态故障调查

日期: 2026-04-09

## 结论

- 后端 SSE 路由是活的，`StreamingResponse` 也会持续产出事件。
- 前端 `student-web` 当前没有按“流式”消费 SSE，而是把 `text/event-stream` 用 `response.text()` 全量读完再解析。
- 只要后端不主动关闭连接，前端就拿不到中途任何状态更新。
- 此外，后端发出的 `connected` / `heartbeat` 没有 `id` / `sequence`，会被前端 parser 直接丢弃。

## 对业务现象的解释

这解释了为什么你会看到：

1. 后端日志显示事件持续输出。
2. 浏览器网络面板里连接可能保持打开。
3. 等待页却没有实时推进，甚至不会及时降级到 polling。

因为前端卡在“等整个 SSE 响应结束”这一步，后面的解析、状态写入、降级切换都没有机会发生。

## 修复后建议验收清单

用户只需要做前端点击式验收：

1. 从 `/video/input` 创建任务并跳到 `/video/:id/generating`。
2. 在任务仍在运行时，确认进度条和阶段文案能在连接未关闭前实时变化。
3. 打开浏览器网络面板，确认 SSE 连接保持打开时，页面仍持续更新。
4. 手动断网再恢复，确认页面能恢复进度或切换到 polling，而不是静止不动。
5. 任务成功后确认等待页先看到完成态，再跳转到 `/video/:id`。
