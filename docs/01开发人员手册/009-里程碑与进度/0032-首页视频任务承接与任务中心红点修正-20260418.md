# 首页视频任务承接与任务中心红点修正验收清单

> 事实源以 [`../../../_bmad-output/implementation-artifacts/4-11-首页视频任务承接与任务中心红点修正-20260418.md`](../../../_bmad-output/implementation-artifacts/4-11-首页视频任务承接与任务中心红点修正-20260418.md) 为准。本文只保留前端点击验收口径。

## 本次验收要确认什么

1. 没有活跃任务时，首页或输入页右上角 bell 不再错误显示红点。
2. 已有视频任务时，首页 `/` 能直接看到“当前视频任务”状态卡。
3. 首页的任务卡与 bell 任务中心展示的是同一批活跃任务。

## 前置条件

1. 准备一个已登录账号。
2. 分别准备两种状态：
   - 无任何 `pending/processing` 视频任务
   - 至少有一个 `pending` 或 `processing` 视频任务

## 点击验收清单

### 场景 A：无活跃任务

1. 登录后进入首页 `/`。
2. 观察右上角 bell。
3. 确认 bell 没有红点。
4. 点击 bell，确认仍能打开任务中心，并看到“暂无进行中的任务”。

### 场景 B：存在活跃任务

1. 先启动一个视频任务，确保任务仍停留在 `pending` 或 `processing`。
2. 返回首页 `/`。
3. 确认首页 Hero 右侧出现“当前视频任务”卡片。
4. 确认卡片内至少能看到：
   - 任务标题
   - 当前阶段文案
   - 进度条与百分比
   - “继续查看”按钮
   - “取消任务”按钮
5. 点击“继续查看”，确认跳转到对应等待页 `/video/:id/generating`。
6. 再返回首页，点击 bell，确认 dropdown 里能看到与首页卡片同一个任务。
7. 点击首页任务卡里的“取消任务”，确认任务消失，bell 红点同步消失或队列数量同步减少。

## 预期结果

- 首页不再是视频任务状态盲区。
- bell 红点只表示“确实有活跃任务”，不再误报。
- 首页任务卡、输入页任务卡和 bell 任务中心三者数据一致。
