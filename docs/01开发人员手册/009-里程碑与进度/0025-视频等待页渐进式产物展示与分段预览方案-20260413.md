- 核心改动是**重构 pipeline 编排模型**：从 batch 模式（全部生成 → 全部渲染）改为 per-section streaming 模式（逐个 section 走完 generate → render → 合成 → 发布）。
- TTS 前移到 storyboard 之后（~5s），使每个 section 片段在发布时就带音频。
- 用户在等待页能逐段看到可播放的片段视频，而不是盯 7.5 分钟进度条。

## 三段式用户体验

### 阶段 A：分镜可听（~85s，understanding + storyboard + TTS）

storyboard 完成后立即生成 TTS 旁白（~5s），用户可以：
- 看到题目摘要 + 知识点
- 看到分镜卡片（scene title、旁白文本）
- 试听各 section 的旁白音频

### 阶段 B：片段可看可听（per-section streaming）

每个 section 独立走完 `manim_gen → render → 合成音频 → 发布`：
- section 1 完成 → 用户看到可播放的带声音片段
- section 2 完成 → 用户看到可播放的带声音片段
- ...（用户边看边等，manim_gen 在后台继续生成下一个 section）
- 修复是 per-section 的：section 2 在修复不影响 section 1 的观看

### 阶段 C：整片完成 → 跳转结果页（~5s）

所有 section 拼接为最终完整视频，上传后跳转结果页。

## Pipeline 编排对比

### 当前（batch 模式）

```
understanding → storyboard → manim_gen(ALL 459s) → render(ALL 49s) → TTS(5s) → compose → upload
                              ↑ 用户等 7.5 分钟只看到进度条
```

代码位置：
- `orchestrator.py:350` — `agent.generate_codes()` ThreadPool(2) 并行生成所有 section
- `orchestrator.py:354` — `agent.render_all_sections()` ThreadPool(2) 并行渲染所有 section
- `orchestrator.py:397` — `_run_tts()` 在 render 之后

### 目标（per-section streaming）

```
understanding → storyboard → TTS(5s) → 
  [s1: generate → render → 合成音频 → 发布 ✓] →
  [s2: generate → render → 合成音频 → 发布 ✓] →
  ... →
  final_concat → upload
```

关键改动：
1. TTS 前移到 storyboard 之后
2. `generate_codes(ALL) + render_all_sections(ALL)` → per-section 循环调用 `generate_section_code(i) + render_section(i) + compose_section(i) + publish(i)`
3. `agent.py` 已有 `generate_section_code()` 和 `render_section()` 单 section API，不需要新增，只需 orchestrator 改调用方式

## 新状态机

旧：`understanding → storyboard → manim_gen → render → compose → upload`（阶段维度）

新：`understanding → storyboard → tts → [section_loop × N] → final_concat → upload`（混合维度）

SSE 事件：
- `section_progress`：per-section 状态变化（generating → rendering → fixing → ready/failed）
- `section_ready`：section 片段可播放（携带 clipUrl）
- 保留原有 `progress` 和 `completed` 事件

## 结果页边界

- `/video/:id` 结果页仍只消费最终 `VideoResult.videoUrl`
- Section 片段是等待页的渐进预览，不是任务完成的替代品
- 最终合成使用所有成功 section 的片段拼接，拼接产物才是结果页入口
