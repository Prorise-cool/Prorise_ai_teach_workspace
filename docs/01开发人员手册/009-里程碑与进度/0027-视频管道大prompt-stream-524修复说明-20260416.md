# 视频管道大 Prompt Stream 524 修复说明（2026-04-16）

## 适用范围

- Epic 4 / Story 4.3
- 分支：`feature/video-pipeline-manimcat-optimize`
- 目标模块：
  - `packages/fastapi-backend/app/features/video/pipeline/engine/gpt_request.py`
  - `packages/fastapi-backend/app/features/video/pipeline/engine/openai_stream.py`

## 问题摘要

视频管道 `manim_gen` 的 full-code generation prompt 很大，当前上游代理在建立 stream 连接时可能直接返回 `524 A Timeout Occurred`。旧实现无论 payload 大小都会先发 `stream=True`，导致请求在尚未收到首个 chunk 前就被截断，最终表现为 `Code generation LLM call returned None`。

## 本次修复

1. 在 `gpt_request.py` 新增 message payload 字符数估算。
2. 当输入超过 `FASTAPI_VIDEO_LLM_STREAM_MAX_INPUT_CHARS`（默认 `12000`）时，改为 direct non-stream。
3. 在 `openai_stream.py` 给 `create_chat_completion_text()` 增加 `prefer_stream` 参数，兼容：
   - 小 payload：维持原先 `stream -> partial recovery -> non-stream fallback`
   - 大 payload：直接 `stream=False`

## 配置说明

- 新增环境变量：`FASTAPI_VIDEO_LLM_STREAM_MAX_INPUT_CHARS`
- 默认值：`12000`
- 特殊约定：
  - `> 0`：启用保护，超过阈值跳过 stream
  - `<= 0`：禁用保护，恢复始终 stream
