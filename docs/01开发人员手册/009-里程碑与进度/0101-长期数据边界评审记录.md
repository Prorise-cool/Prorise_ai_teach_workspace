> **状态**: 已评审通过
> **负责人**: Epic 10 Owner / 后端负责人 / 数据承接负责人
> **最后更新**: 2026-03-29

## 评审目标

本记录用于证明 Epic 4 ~ 9 的长期回写需求已经逐项映射到明确宿主与最小字段集，后续 Story 可以在不争论数据归属的前提下继续实现。

## 评审结论摘要

1. 长期业务数据统一落在 RuoYi / MySQL，文件产物进入 COS，Redis 只保留运行态与 TTL 缓存。
2. `xm_video_task`、`xm_classroom_session`、`xm_session_artifact`、`xm_companion_turn`、`xm_whiteboard_action_log`、`xm_knowledge_chat_log`、`xm_quiz_result`、`xm_learning_path`、`xm_learning_record`、`xm_learning_favorite` 已完成最小字段基线冻结。
3. wrongbook / recommendation 已明确通过 `xm_quiz_result.result_type + source_result_id + detail_payload_json` 挂接，不再保留“后续再补表”的口头约定。
4. `/learning`、`/history`、`/favorites` 和后台查询 / 导出 / 审计所需字段均能直接从长期表取得，不依赖 Redis 回放。

## Redis-only 数据核查

以下数据明确保留在 Redis 或运行时缓存，且不承担长期真值职责：

1. `xm_task:{task_id}`：任务运行状态。
2. `xm_task_events:{task_id}`：SSE 事件缓存。
3. `xm_video_runtime:{task_id}`：视频任务临时上下文。
4. `xm_classroom_runtime:{session_id}`：课堂临时上下文。
5. `xm_companion_runtime:{session_id}`：Companion 窗口态。
6. `xm_context_window:{session_id}:{anchor}`：锚点上下文窗口。
7. `xm_provider_health:{provider}`：Provider 健康探针。

确认结果：

- Redis-only 清单中不包含学习记录、收藏、任务元数据、问答历史、quiz 结果、learning path。
- Redis 的任何 TTL 到期都不会导致长期业务记录丢失。
