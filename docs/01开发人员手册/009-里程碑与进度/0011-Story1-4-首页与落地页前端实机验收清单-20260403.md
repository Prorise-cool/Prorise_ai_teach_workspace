# Story 1.4 首页与落地页前端实机验收清单

日期：2026-04-03  
范围：`Story 1.4 - 首页课堂直达入口与顶栏导航分发`  
适用前端：`packages/student-web`  
验收地址：`http://127.0.0.1:5173`  
验收分支：`feature/87-story-1-4-home-entry-nav`

## 验收前准备

- 启动 `student-web` 前端开发服务，并确认地址使用 `127.0.0.1`，不要切换为 `localhost`。
- 确认 Java 后端与 FastAPI 后端已启动，前端开发代理可正常转发接口。
- 准备 1 个可用联调账号，例如：
  - 管理员账号：`admin`
  - 管理员密码：`admin123`
- 如登录页启用验证码，按现有联调流程先取 `uuid`，再从 Redis 获取验证码答案。

## 前端实机验收项

### 1. 首页公开访问与首屏结构

操作：

- 浏览器直接打开 `http://127.0.0.1:5173/`

期望：

- 页面保持公开访问，不会自动跳到 `/login`
- 首屏能直接看到顶栏导航、主标题、主 CTA 与底部波浪工作区入口
- 首页不再出现旧的认证一致性控制台内容

### 2. 首页导航到落地页

操作：

- 在首页点击顶栏 `关于我们`、`产品亮点`、`销售计划` 或 `常见问题`

期望：

- 能进入 `/landing`，并准确落到对应锚点区域
- 该过程不触发登录守卫
- 落地页模块顺序、插画与图片资源和设计稿一致，不出现空图或布局错位

### 3. 首页课堂入口的未登录回跳

操作：

- 保持未登录状态，在首页点击 `Start Learning`
- 再点击波浪区的圆形播放按钮

期望：

- 两个入口都会跳到 `/login`
- 地址栏带有 `returnTo=%2Fclassroom%2Finput`
- 不会先进入空白页，也不会丢失原始目标意图

### 4. 落地页主题与语言切换

操作：

- 打开 `http://127.0.0.1:5173/landing`
- 点击顶部亮暗色切换按钮
- 点击顶部语言切换按钮

期望：

- 亮暗色切换后，Hero 主图会跟随切换，页面令牌、卡片、文字与导航样式同步变化
- 语言切换后，导航、Hero 按钮与模块标题变为英文；再次切回可恢复中文
- 切换过程中不出现闪烁、整页报错或图片丢失

### 5. 落地页受保护入口的未登录回跳

操作：

- 在落地页保持未登录状态，点击顶部 `登录`

期望：

- 最终进入 `/login`
- 地址栏带有 `returnTo=%2Fclassroom%2Finput`
- 说明公开页上的受保护入口仍然保留了课堂工作区意图

### 6. 登录后进入课堂输入壳层

操作：

- 在 `/login` 使用真实账号完成登录

期望：

- 登录成功后进入 `/classroom/input`
- 页面可以看到课堂输入壳层文案、当前会话信息与一致性验证按钮
- 顶栏仍可跳回 `/landing`，页面主题与语言切换保持正常

## 建议记录

- 若验收通过，建议在对应 PR 中附 1 张首页截图、1 张落地页截图、1 张登录后进入 `/classroom/input` 的截图。
- 若验收失败，优先记录失败步骤、最终 URL、浏览器主题/语言状态，以及控制台或 Network 中的异常信息。
