# Story 1.3 认证一致性前端实机验收清单

日期：2026-04-03
范围：`Story 1.3 - 登出、401 处理与受保护访问一致性`
适用前端：`packages/student-web`
验收地址：`http://127.0.0.1:5173`
验收分支：`feature/85-story-1-3-auth-consistency`

## 验收前准备

- 启动 `student-web` 前端开发服务，并确认地址使用 `127.0.0.1`，不要切换为 `localhost`。
- 确认 Java 后端与 FastAPI 后端已启动，前端开发代理可正常转发接口。
- 使用真实联调账号：
  - 管理员账号：`admin`
  - 管理员密码：`admin123`
- 若登录页启用验证码，则在获取登录页 `uuid` 后，从 Redis 读取 `global:captcha_codes:<uuid>`，再对 value 执行 `json.loads(...)` 取得答案。

## 前端实机验收项

### 1. 未登录访问受保护首页

操作：
- 浏览器直接打开 `http://127.0.0.1:5173/`

期望：
- 自动跳转到 `/login`
- 页面不白屏，不出现无限重定向
- 页面有明确登录态引导，而不是停留在错误中间态

### 2. 真实登录后进入首页

操作：
- 在 `/login` 输入 `admin / admin123 / 验证码`
- 点击登录

期望：
- 登录成功后进入首页 `/`
- 首页可以看到当前会话信息，例如用户名、角色、权限
- 不会出现“登录成功但仍停留登录页”或“已登录但又被要求重新登录”的状态不一致

### 3. 受保护访问一致性验证

操作：
- 在首页点击“会话探针”按钮

期望：
- 页面出现成功反馈
- 页面展示探针结果，例如 `userId`、`requestId`、`onlineTtlSeconds`
- 说明前端、FastAPI、RuoYi 对当前登录态判断一致

### 4. 权限不足分支验证

操作：
- 在首页点击“权限探针”按钮

期望：
- 页面跳转到 `/forbidden`
- 页面明确提示当前是“无权限”状态
- 不会被伪装成未登录并错误跳回 `/login`
- 说明 `403` 与 `401` 的语义已经正确区分

### 5. 登出与失效后的守卫回退

操作：
- 在首页或 `/forbidden` 页面点击“退出登录”
- 退出后再次手动访问 `http://127.0.0.1:5173/`

期望：
- 点击退出后立即回到 `/login`
- 再次访问 `/` 时会重新被守卫重定向到 `/login`
- 不会短暂进入首页，不会残留上一次会话数据
