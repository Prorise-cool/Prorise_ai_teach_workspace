# 0033-视频输入页任务中心404容错修复-20260418

## 背景

视频输入页与首页顶栏任务中心使用同一份“活跃视频任务”聚合查询。线上 / 本地联调时出现过如下现象：

- bell 下拉显示 `0 项`
- 页面主体没有当前任务卡
- Network 中 `/api/v1/video/tasks?status=pending|processing` 为 `200`
- 但部分 `/api/v1/video/tasks/{taskId}/status` 为 `404`

## 根因

活跃任务列表来自 RuoYi 元数据表，任务快照来自 FastAPI 运行态存储，两者不是同一个数据源。

当列表中残留了已经过期的旧任务时：

- 元数据列表仍会把它当作 `pending/processing` 返回
- 运行态快照则会返回 `404`

旧版 `video-workspace-task-adapter` 对快照 enrich 使用 `Promise.all(...)`，导致任意一条旧任务 404 都会让整批活跃任务查询失败，最终前端渲染成空队列。

## 代码修复

- 文件：`packages/student-web/src/services/api/adapters/video-workspace-task-adapter.ts`
- 策略：对单条快照 `404` 做软过滤，只跳过失效任务；非 404 错误继续抛出。

## 自测结果

- `pnpm vitest run src/services/api/adapters/video-workspace-task-adapter.test.ts --reporter=dot`
- `pnpm vitest run src/services/api/adapters/video-workspace-task-adapter.test.ts src/features/video/pages/video-input-page.test.tsx src/features/home/pages/home-page.test.tsx --reporter=dot`

结果：`3 files / 25 tests passed`

## 前端点击验收清单

1. 在 `/video/input` 提交一个新视频任务并进入等待页。
2. 点击左上返回，回到 `/video/input`。
3. 确认页面主体仍能看到“当前任务”状态卡。
4. 点击右上 bell，确认任务中心至少显示当前有效任务，而不是 `0 项` 空状态。
5. 若本地存在历史脏任务，Network 可以继续看到个别 `status 404`，但 bell 不应再因为这类 404 整体清空。
6. 回到首页 `/`，确认顶栏 bell 与首页任务卡也能继续承接当前任务。
