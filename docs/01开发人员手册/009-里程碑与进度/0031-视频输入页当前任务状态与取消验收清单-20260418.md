# 视频输入页当前任务状态与取消验收清单

> 事实源以 [`../../../_bmad-output/implementation-artifacts/4-11-输入页当前任务状态卡与返回聚焦补齐-20260418.md`](../../../_bmad-output/implementation-artifacts/4-11-输入页当前任务状态卡与返回聚焦补齐-20260418.md) 为准。本文只给前端点击验收口径。

## 本次验收要确认什么

1. 等待页返回输入页后，输入页主体能显式看到“当前任务”状态卡。
2. 用户可以直接从输入页继续回到等待页。
3. 用户可以直接从输入页取消当前任务。
4. 右上角任务中心和主体状态卡的数据保持一致。

## 前置条件

1. 存在至少一个 `pending` 或 `processing` 的视频任务。
2. 最好直接从 `/video/:id/generating` 进入本轮验收，这样可以顺手验证返回聚焦链路。

## 点击验收清单

1. 打开任意一个仍在生成中的视频等待页 `/video/:id/generating`。
2. 点击右上角“返回工作区”。
3. 确认页面跳回 `/video/input` 后，主体输入卡下方出现“当前任务”卡片，而不是只能从右上角 bell 里找任务。
4. 确认当前任务卡里至少能看到：
   - 任务标题
   - 当前状态 / 阶段文案
   - 进度百分比与进度条
   - “继续查看”按钮
   - “取消任务”按钮
5. 点击当前任务卡里的“继续查看”，确认回到原任务的等待页 `/video/:id/generating`。
6. 再次返回输入页后，点击当前任务卡里的“取消任务”，确认：
   - 有成功反馈
   - 当前任务从主体卡或右上角任务中心里消失
   - 页面不会跳到错误页
7. 打开右上角 bell 任务中心，确认它和主体卡展示的是同一批活跃任务；如果主体卡展示的是某个聚焦任务，dropdown 里也能找到同一个任务。
8. 当活跃任务全部取消或完成后，再点 bell，确认还能打开任务中心，并看到“暂无进行中的任务”空态。

## 预期结果

- 输入页不再只是“创建入口”，而是一个可承接当前任务继续/取消的工作区。
- 等待页返回输入页的体验不断裂，用户能立刻知道“任务还在继续，而且就在眼前”。
- bell 任务中心负责看全局队列，主体卡负责突出当前任务，两者信息一致。
