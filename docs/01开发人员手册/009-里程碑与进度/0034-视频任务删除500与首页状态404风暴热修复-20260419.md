# 0034-视频任务删除500与首页状态404风暴热修复-20260419

## 适用范围

本记录对应 `Story 4.11` 的一次热修补丁，覆盖首页 `/`、视频输入页 `/video/input`、等待页取消动作以及 FastAPI 视频删除接口。

## 问题摘要

- 删除已完成视频任务时，后端会返回 `500 COMMON_INTERNAL_ERROR`。
- 取消或删除任务后，首页 / 输入页 bell 里的任务不会立刻清空。
- 旧任务仍残留在 workspace 列表时，前端会继续轮询该 taskId 的 `/status`，形成重复 `404`。

## 修复摘要

### 后端

- 删除流程改成“先更新 RuoYi 元数据，再删 Redis 运行态”。
- 删除成功后补清 task runtime、task events、preview/cancel_request/fix_log/result_detail 等视频运行态残留，以及 `xm_video_runtime_auth:{task_id}`。
- 删除路由成功响应显式返回 `taskId/status`，避免 dict 误走 `build_success_envelope()` 再次触发 500。

### 前端

- bell 任务中心增加 stale taskId 抑制，单条 `status 404` 不会在下一轮轮询继续追打。
- 取消成功后立即写入 cancelled snapshot，并从 bell 缓存移除任务。
- 删除成功后立即移除 bell 缓存项并清理 `task-status / task-preview / result` detail query。
