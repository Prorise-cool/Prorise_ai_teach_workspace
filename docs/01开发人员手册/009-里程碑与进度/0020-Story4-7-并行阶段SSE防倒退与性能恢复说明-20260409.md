## 关键改动

- 后端恢复 `understanding+storyboard` 合并、TTS scene 并行、Docker 镜像预热。
- 阶段区间改为 `0–12 / 13–25 / 26–42 / 43–58 / 59–65 / 66–78 / 79–94 / 95–100`。
- `VideoPipelineService._emit_stage(...)` 增加全局进度防倒退守卫。
- `manim_fix` 在 render 失败后仍会进入修复态，但不会把全局进度拉回更低百分比。
- 逐场景 Manim 增量生成增加语法校验；默认分镜目标时长恢复为 `120s`。
- `OpenAICompatibleLLMProvider` 与 `DoubaoTTSProvider` 复用实例级 `httpx.AsyncClient`，减少并行场景与多段 TTS 的重复建连开销。

- 当前成功样本仍略高于三分钟线（约 `181.26s`），说明本地串行瓶颈已基本清完，但上游 LLM 延迟与分镜数量波动仍会影响最终耗时。
