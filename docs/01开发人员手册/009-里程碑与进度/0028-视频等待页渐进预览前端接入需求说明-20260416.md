# 视频等待页渐进预览前端接入需求说明

> 本文是开发手册入口，用于告诉后续前端协作者“这次接入要完成什么、在哪些代码位置承接、哪些边界不能动”。事实源仍以 [`../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进预览前端接入需求说明-20260416.md`](../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进预览前端接入需求说明-20260416.md) 为准。

## 这份文档回答什么

- 等待页需要从“总进度条”升级成怎样的三段式体验。
- 前端必须消费哪些接口与事件。
- student-web 哪些文件拥有接入职责。
- 最终结果页与 section 预览之间的边界是什么。

## 这份文档不回答什么

- 不给出组件、hook、store 的具体代码写法。
- 不把 section 片段预览包装成“任务提前完成”。
- 不把开发态本地资产路由写成线上正式契约。

## 接入结论

1. 创建成功后入口不变，仍旧是 `/video/:id/generating`。
2. 等待页必须新增 preview 语义，展示 `summary`、`knowledgePoints`、section 列表、旁白试听和 ready section clip。
3. 等待页必须同时消费 `/status`、`/preview`、SSE 三类信息，而不是只靠 SSE。
4. 结果页 `/video/:id` 仍只消费最终完整视频，不消费 section clip。

## 必须维持的边界

1. section clip 只用于等待页。
2. 最终成功态只由 `/result` 和 `/video/:id` 结果页承接。
3. 刷新恢复与断线恢复不能退化。
4. preview 数据必须来自真实后端结构，不能靠 message 文本猜。

## 前端点击验收清单

1. 打开 `/video/input`，确认“生成质量预设”下拉默认是“均衡讲解”，切换三档预设时，时长/分段/并发/渲染质量/布局摘要会同步变化。
2. 点击“高级参数”，确认 `目标时长 / 分段数量 / 并发生成数 / 渲染质量 / 布局偏好` 都可调，并且关闭弹窗后仍保留刚才设置。
3. 从 `/video/input` 创建一条新视频任务，确认仍进入 `/video/:id/generating`，且顶部状态岛显示总进度与当前阶段。
4. 等到 preview 首次可用后，确认左侧出现摘要、知识点、section 列表，而不是只有单一总进度条。
5. 某个 section 进入 `ready` 后，点击顶部 section 轨道或卡片按钮，确认预览区能播放该段 clip，卡片仍保留旁白试听入口。
6. 在至少一段已 ready 时刷新页面，确认摘要、section 状态与 ready clip 能恢复，不会全部清空回初始态。
7. 观察一个仍在 `rendering` / `fixing` / `failed` 的 section，确认只有该段展示对应状态，整页不会误切到任务失败页。
8. 等待任务最终完成，确认页面自动跳转 `/video/:id`，结果页播放的是最终整片而不是 section clip。
