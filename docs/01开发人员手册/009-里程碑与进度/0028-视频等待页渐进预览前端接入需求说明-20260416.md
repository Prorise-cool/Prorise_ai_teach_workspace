# 视频等待页渐进预览前端接入需求说明

> 本文是开发手册入口，用于告诉后续前端协作者“这次接入要完成什么、在哪些代码位置承接、哪些边界不能动”。事实源仍以 [`../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进预览前端接入需求说明-20260416.md`](../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进预览前端接入需求说明-20260416.md) 为准。

## 这份文档回答什么

- 等待页需要从“总进度条”升级成怎样的三段式体验。
- 前端必须消费哪些接口与事件。
- student-web 哪些文件拥有接入职责。
- 最终结果页与 section 预览之间的边界是什么。

## 这份文档不回答什么

- 不给出组件、hook、store 的具体代码写法。
- 不把 section 片段预览包装成“任务提前完成”。
- 不把开发态本地资产路由写成线上正式契约。

## 接入结论

1. 创建成功后入口不变，仍旧是 `/video/:id/generating`。
2. 等待页必须新增 preview 语义，展示 `summary`、`knowledgePoints`、section 列表、旁白试听和 ready section clip。
3. 等待页必须同时消费 `/status`、`/preview`、SSE 三类信息，而不是只靠 SSE。
4. 结果页 `/video/:id` 仍只消费最终完整视频，不消费 section clip。

## 代码归属点

- 等待页编排：`packages/student-web/src/features/video/pages/video-generating-page.tsx`
- SSE 消费：`packages/student-web/src/features/video/hooks/use-video-task-sse.ts`
- 状态恢复：`packages/student-web/src/features/video/hooks/use-video-task-status.ts`
- 全局阶段映射：`packages/student-web/src/features/video/config/video-stages.ts`
- 等待页状态机：`packages/student-web/src/features/video/stores/video-generating-store.ts`
- 创建任务 adapter：`packages/student-web/src/services/api/adapters/video-task-adapter.ts`
- 后端视频接口：`packages/fastapi-backend/app/features/video/routes.py`
- 统一任务 SSE / status：`packages/fastapi-backend/app/api/routes/tasks.py`

## 必须维持的边界

1. section clip 只用于等待页。
2. 最终成功态只由 `/result` 和 `/video/:id` 结果页承接。
3. 刷新恢复与断线恢复不能退化。
4. preview 数据必须来自真实后端结构，不能靠 message 文本猜。
