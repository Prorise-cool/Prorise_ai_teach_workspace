# Story 4.11 FastAPI 渐进预览后端实现说明

日期：2026-04-13

> 事实源以 [`../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进式产物展示与分段预览.md`](../../../_bmad-output/implementation-artifacts/4-11-视频等待页渐进式产物展示与分段预览.md) 与 [`../../../_bmad-output/implementation-artifacts/4-11-fastapi-渐进预览后端实现说明-20260413.md`](../../../_bmad-output/implementation-artifacts/4-11-fastapi-渐进预览后端实现说明-20260413.md) 为准；本文只做开发手册导航与协作说明。

## 本次范围

- 只修改 `packages/fastapi-backend`。
- 不包含 `student-web` 等待页、结果页或 mock handler 的前端接入。
- 重构策略按“有用则留，无用删除”执行，没有为旧 batch 流程额外保留兼容壳层。

## 后端已完成内容

- 将视频流水线从 batch 编排重构为 per-section streaming：
  - `generate_section_code`
  - `render_section`
  - `compose section clip`
  - `publish section clip`
- 将 TTS 前移到 storyboard 之后，让等待页未来能先听旁白，再逐段看到带声音片段。
- 为运行态新增 preview 数据结构，支持保存：
  - `summary`
  - `knowledgePoints`
  - `sections[].status`
  - `sections[].audioUrl`
  - `sections[].clipUrl`
- 新增 `GET /api/v1/video/tasks/{task_id}/preview`。
- 新增 SSE 事件：
  - `section_progress`
  - `section_ready`

## 协作边界

- 前端 agent 后续需要接入 `/preview` 与 section 级 SSE，并把等待页改成三阶段展示。
- 结果页职责不变，仍只消费最终 `VideoResult.videoUrl`。
- Story 当前仍保持 `in-progress`，等前端接入与验收完成后再整体收口。
