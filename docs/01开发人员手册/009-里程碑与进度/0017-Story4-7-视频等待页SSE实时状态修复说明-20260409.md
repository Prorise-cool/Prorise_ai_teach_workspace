# 0017 Story 4.7 视频等待页 SSE 实时状态修复说明

日期：2026-04-09

## 结论

- 本次修复只落在 `student-web` 前端 SSE transport。
- 核心改动是把等待页的 SSE 消费从“整段 `response.text()` 后再解析”改成“基于 `ReadableStream` 的增量解析”。
- 后端 `connected` / `heartbeat` 仍未补齐 `id` / `sequence`，前端已增加兼容层，但不会把这类临时 id 写入 `Last-Event-ID`。

## 代码改动

### `packages/student-web/src/services/sse/index.ts`

- 新增 `streamTaskEventResponse(...)`，边读边解析 `text/event-stream`。
- 引入 transient identity 兼容，让 `connected` / `heartbeat` 可以参与本地状态流推进。
- `createRealTaskEventStream()` 只会用标准事件 id 更新恢复游标，避免 transient id 污染重连语义。

### `packages/student-web/src/services/sse/task-event-stream.test.ts`

- 增加长连接场景测试，验证在 SSE 响应关闭前就能拿到第一条事件。
- 把 `connected` 缺少标准 `id` 的真实后端行为纳入测试，验证前端兼容层可用。
