# 视频结果页"视频不可用"根因分析

> 日期：2026-04-12
> 现象：所有视频任务完成后，前端结果页必定显示"视频不可用，视频文件已过期或不可访问"
> 状态：已按方案 A 落地热修，并补充超时 / 运行态语义收口；`2026-04-12 21:59 CST` 真实 E2E 已验证

## 根因一句话

**Pipeline 把完整结果（含 videoUrl）存到了 Redis `result_detail` key，但结果查询 API 只从文件系统（`detail_ref`）读取，而 `detail_ref` 从未被设置，降级到 Redis 只读到 `{status: completed}` 没有 result 对象，前端判定视频缺失。**

## 调用链追踪

### Step 1: Pipeline 完成 — 结果只写 Redis，不写文件系统

文件：`packages/fastapi-backend/app/features/video/pipeline/orchestration/orchestrator.py:428-433`

```python
detail = merge_result_detail(None, status="completed", result=video_result.model_dump(...))
runtime.save_model("result_detail", detail)  # 写到 Redis key: xm_video_task:{id}:result_detail
```

- 结果保存到了 Redis 的 `result_detail` key
- **没人把 `detail` 写入 `LocalAssetStore` 文件系统**
- **没人更新任务元数据的 `detail_ref` 字段**

### Step 2: 任务创建时 — `detail_ref` 为 None

文件：`packages/fastapi-backend/app/features/video/service/create_task.py:213-220`

```python
metadata_request = VideoTaskMetadataCreateRequest(
    task_id=task_id,
    user_id=access_context.user_id,
    status=TaskStatus.PENDING,
    summary=summary or "视频任务待处理",
    # detail_ref 未传入 → 始终为 None
)
```

且整个视频模块生命周期中没有任何地方更新 `detail_ref`。

### Step 3: 结果查询 API — 走了降级路径

文件：`packages/fastapi-backend/app/features/video/service/result_service.py:64-87`

```python
# 路径1: 从文件读取（需要 detail_ref 指向文件）
if snapshot.detail_ref and self._asset_store.exists(snapshot.detail_ref):
    detail = self._asset_store.read_result_detail(snapshot.detail_ref)  # ← 永远不走这里
    return detail

# 路径2: 降级从 Redis 读基本状态
if runtime_store is not None:
    state = runtime_store.get_task_state(task_id)  # 只返回 {status, progress, ...}
    if state is not None:
        status = str(state.get("status") or "processing")
        return VideoResultDetail(task_id=task_id, status=status)
        # ⚠️ result=None！没有 videoUrl！
```

因为 `detail_ref` 始终为 `None`，**永远走路径2**，返回 `VideoResultDetail(status="completed", result=None)`。

**关键遗漏**：路径2 从 `runtime_store.get_task_state()` 读取的是 TaskScheduler 写入的基本状态，而非 Pipeline 保存到 `xm_video_task:{id}:result_detail` 的完整结果。两个是不同的 Redis key。

Redis key 对比：
| Key | 写入者 | 内容 |
|-----|--------|------|
| `xm_video_task:{id}:result_detail` | Pipeline `runtime.save_model()` | 完整 VideoResultDetail（含 videoUrl） |
| `xm_task_runtime:{id}:state` | TaskScheduler `_emit_snapshot()` | 仅 {status, progress, message, ...} |

### Step 4: 前端判定

文件：`packages/student-web/src/features/video/hooks/use-video-result.ts:72-80`

```typescript
if (data.result && data.result.videoUrl) {
    viewStatus = 'success';           // ← 不走这里（result=null）
} else if (data.status === 'completed' && (!data.result || !data.result.videoUrl)) {
    viewStatus = 'video-missing';     // ← 走这里 → "视频不可用"
}
```

API 返回 `{status: "completed", result: null}` → 前端匹配到 `video-missing` → 显示错误提示。

## 数据流全景图

```
Pipeline 完成
  │
  ├─ runtime.save_model("result_detail", detail)
  │    → Redis key: xm_video_task:{id}:result_detail  ✅ 写了
  │
  ├─ LocalAssetStore.write_json(???)                     ❌ 没写
  │
  └─ snapshot.detail_ref = ???                           ❌ 没设
       │
       ▼
GET /api/v1/video/tasks/{id}/result
  │
  ├─ snapshot.detail_ref → None                          ❌ 跳过文件读取
  │
  └─ runtime_store.get_task_state()                      ⚠️ 只读到 {status}
       → 返回 VideoResultDetail(status="completed", result=None)
       │
       ▼
前端 use-video-result.ts
  └─ data.status==="completed" && !data.result
       → viewStatus = "video-missing"
       → 显示"视频不可用"                              ❌ 用户看到这个
```

## 修复方案

### 方案A（最小改动，推荐）：在降级路径读取 Redis result_detail

在 `result_service.py` 的 Redis 降级分支中，尝试读取 Pipeline 已保存的 `result_detail`：

```python
# result_service.py, 降级分支
if runtime_store is not None:
    state = runtime_store.get_task_state(task_id)
    if state is not None:
        status = str(state.get("status") or "processing")
        # 尝试从 Redis 读取完整 result_detail
        from app.features.video.pipeline.orchestration.runtime import (
            VideoRuntimeStateStore,
        )
        rt = VideoRuntimeStateStore(runtime_store, task_id)
        detail = rt.load_model("result_detail", VideoResultDetail)
        if detail is not None:
            return detail
        return VideoResultDetail(task_id=task_id, status="processing" if status == "pending" else status)
```

**优点**：改动最小（1个文件，约10行），利用已有的 Redis 数据。
**风险**：Redis result_detail 有 TTL，过期后仍会丢失（但这与当前行为一致）。
