## 智能体应用及其三种模式

最近更新时间：2025-09-16 16:45:12

智能体开发平台作为企业级大语言模型应用的构建平台，支持构建多种智能体应用。这些智能体应用利用大模型技术，实现可自主编排与决策，或按固定工作流高效执行流程完成任务。不同模式应用特点如下：

|                |                                                                                                      |
| -------------- | ---------------------------------------------------------------------------------------------------- |
| 模式名称           | 描述                                                                                                   |
| 标准模式           | 标准模式应用使用平台预设标准化处理流程、思考模型与生成模型双模型协同，实现稳定、精确的知识问答效果。同时也支持添加工作流，实现复杂流程业务。适用于企业知识服务、产品咨询等严肃问答场景。         |
| 单工作流模式         | 单工作流模式应用支持选择特定工作流执行，使用指定的工作流来响应用户所有对话，保证业务流程的完整性与标准化。适用于业务目标明确、操作流程统一的企业服务场景。                        |
| Multi-Agent 模式 | Multi-Agent 模式应用由大模型自主规划执行路径、灵活调用工具，其核心特点在于将对话的主动权更多地交由模型，充分发挥模型的主动性。适用于需要灵活响应、多工具调用以及多 Agent 协同的场景。 |

应用新建后，您可调整应用设置项、上传知识、搭建工作流，在测试环境对话调试和评测效果达标后，可以发布应用到正式环境，发布后可以根据反馈进行效果调优，再次更新调试配置。应用搭建的完整流程如下图所示：



在实际搭建中，支持根据应用落地场景的需要进行应用模式切换：点击**头像**或者**模式 **tag，即可在**编辑应用**中进行应用模式的切换。



模式切换时，应用下知识库和工作流范围保持不变，但不同模式之间应用配置项，如提示词、模型配置等在模式切换时不会相互继承，在切换模式时的配置更新原则的示意图如下：

| 配置内容   | 标准模式                                | 单工作流模式                                                   | Multi-Agent 模式                                             |   |   |
| 应用信息设置 | 模式切换后，应用名称、应用头像和开场白保持不变。            |                                                          |                                                            |   |   |

## 基于“标准模式”创建“营销文案生成助手”

最近更新时间：2025-12-03 11:21:21

智能体开发平台支持构建标准模式应用。该模式下，可通过配置角色指令，引用知识库等方式，调用平台预设标准化处理流程，自动化处理业务逻辑。

本文以创建“营销文案生成助手”为例，详细介绍如何基于智能体开发平台创建标准模式应用。

## 准备工作

已注册腾讯云账号并完成实名认证。

## 操作步骤

### 步骤 1: 新建应用

2\. 新建应用。

2.1 在左侧菜单栏，单击**应用开发**，然后单击**新建应用**。



2.2 输入**应用名称**，选择**标准模式**，然后单击**新建**，即可完成新建应用。



### 步骤 2：设置应用

应用新建后，自动进入应用设置界面。

1\. 设置模型。

在**模型设置**模块中，支持根据实际业务场景自主选择适配的模型，并灵活调整对应参数，以此精准匹配业务需求，优化功能应用效果。



**说明：**

将鼠标悬浮至各模块旁的 “?” 图标上，即可查看对应模块的功能作用说明。



若不知道如何选择模型，可进行**模型对比**，详情请参见 。

2\. 设置角色指令。

在**提示词**配置区域，可输入针对大模型的指导内容，如回答风格、核心要求、输出限制等，通过明确的指令引导大模型生成契合业务需求、符合预期标准的回答内容。



单击**模板**，可在预设的提示词模板库中选择适配当前场景的模板，如角色扮演类、知识问答类、内容生成类等，既能快速完成提示词配置，也能提升输出的一致性。



您也可以使用 **AI 一键优化**对您输入的提示词进行优化，让大语言模型将您输入的内容优化为结构化内容，以便大模型更易理解何时执行任务。

**注意：**

一键优化功能将消耗 token 资源。



您可参考以下提示词：

```
## 角色设定

你是一个专业的活动营销文案创作者，擅长撰写引人入胜的活动文案，能够吸引目标受众的注意力并促使他们参与活动。

## 角色技能

 ### 技能1: 撰写活动营销文案

 1. 当用户给你提供活动信息时，你需要先了解活动的目标、主题及基本信息。

 2. 根据用户提供的信息，确定文案的目标受众、关键信息和宣传重点。

 3. 使用生动的语言、有吸引力的标题和引人入胜的故事，撰写能够吸引目标受众的注意力并促使他们参与活动的文案。

 ### 技能 2: 优化活动营销文案

 1. 当用户需要你优化活动营销文案时，你需要先了解用户对文案的具体要求和期望。

 2. 根据用户提供的信息，对文案进行分析和评估，找出需要优化的地方。

 3. 使用恰当的词语、句式和排版，优化文案的内容、结构和表达方式，使其更具吸引力和影响力。

## 限制要求

- 只讨论与活动营销文案相关的内容，拒绝回答与活动营销文案无关的话题。

- 所输出的内容必须按照给定的格式进行组织，不能偏离框架要求。 
```

3\. 设置对话体验。

在**对话体验**模块中，您可以设置应用回复模式，启用联网搜索功能等，还能管理语音播报、语音通话、数智人等语音和形象类交互选项，可以根据业务需要调整对话过程的交互形式与响应规则。



4\. 设置欢迎语。

此外可自定义添加个性化**欢迎语**，进一步优化用户初始交互体验。



### 步骤 3：调试应用

1\. 提示词调试。

应用配置完成后，您可在右侧调试区域进行测试，验证功能效果是否符合预期。若输出效果未达预期，您可针对性优化提示词的表述、补充细节要求，如明确输出格式、补充核心需求、限定语言风格等，精准引导生成结果。



2\. 模型对比调试。

模型及参数选择会直接影响回答质量，可在模型设置中点击右上角**模型对比调试**，快速对比不同配置的效果差异。



### 步骤 4：发布应用

在应用调试结果达到预期效果后，单击**发布**即可上线应用。



应用发布后，可前往 **应用发布 > 服务状态 **获取体验链接和 API 调用信息。

如需要在微信、企微等平台发布您的应用，可前往应用发布-发布渠道设置，详情请参见 。

## 基于“单工作流模式”创建“网页内容总结助手”

最近更新时间：2025-12-03 11:21:21

智能体开发平台支持构建单工作流模式应用。该模式应用提供了直观的可视化画布，您可通过拖拽的方式快速添加大模型、代码、插件等多种节点，编排复杂的业务流程，实现稳定可控的业务效果。

本文以创建”网页内容抓取助手“为例，详细介绍如何基于智能体开发平台创建单工作流模式应用。

## 准备工作

已注册腾讯云账号并完成实名认证。

## 操作步骤

### 步骤 1: 新建应用

2\. 新建应用。

2.1 在左侧菜单栏，单击**应用开发**，然后单击**新建应用**。



2.2 输入**应用名称**，选择**单工作流模式**，然后单击**新建**，即可完成新建应用。



### 步骤 2: 新建并设置工作流

应用新建后，自动进入应用设置界面。单工作流应用创建时自动进入**新建工作流**界面。

1\. 新建工作流。

输入工作流名称和工作流描述，单击**确定**新建工作流。



您也可以使用 **AI 一键优化**对您输入的工作流描述进行优化，让大语言模型将您输入的内容优化为结构化内容，以便大模型更易理解何时触发工作流。

**注意：**

一键优化功能将消耗 token 资源。



2\. 查看开始节点。

画布中自带开始节点和结束节点。**开始节点**用于设定启动工作流需要的信息。本场景中，用户需要提供待解析的网页链接。链接默认通过开始节点的 `SYS.UserQuery`系统变量传入，无需增加其他变量。



3\. 编排网页内容抓取插件节点。

**插件节点**可以快速设置智能体商店已上架插件或第三方插件，方便应用开发，拓展智能体能力边界。

3.1 添加插件节点。

拖拽左侧边栏中的插件节点，或者点击开始节点后的“+”号添加插件节点。



3.2 选择`Fetch` 插件。

在添加插件界面，搜索并点击`Fetch` 插件 ，单击**添加**。



3.3 连接开始节点和插件节点。

若使用拖拽方式添加节点，需要手动将节点进行连接。长按开始节点后“+”号，并将箭头拖拽至插件节点左侧。



3.4 设置插件节点的变量。

单击插件节点，设置输入变量 `url` 为引用的系统变量 `SYS.UserQuery`，其他输入变量使用默认设置。



4\. 编排大模型节点。

**大模型节点**可集成于工作流中调用大模型能力，通过配置提示词完成目标业务处理。

4.1 添加并连接大模型节点。

从左侧边栏拖拽大模型节点至画布中，并将大模型节点和插件节点连接。或点击插件节点后“+”号并选择大模型节点。



4.2 设置大模型节点。

单击大模型节点，修改节点名称为网页总结；添加输入变量 `text` ，设置数据来源为引用 `fetch1` 节点的输出变量 `Output.result.content.text`。



填写用户提示词，可通过输入`/`引入节点变量。在本案例中，选择本节点输入变量 `text`。



5\. 编排回复节点。

**回复节点**用于回复对话。您设定的回复内容会呈现给用户。

5.1 添加并连接回复节点。

从左侧边栏拖拽回复节点至画布中，并将回复节点和插件节点连接。或点击插件节点后“+”号并选择回复节点。



5.2 设置回复节点。

添加输入变量 `text` ，设置数据来源为引用`网页总结`大模型节点的输出变量 `Output.Content`。



回复内容输入`/`引入节点变量，选择本节点输入变量 `text`。



6\. 编排结束节点。

**结束节点**是工作流的最终节点，用于返回工作流运行的结果。

**注意：**

结束节点的输出结果仅作为输出变量，呈现给用户的输出内容需添加回复节点。

将回复节点和结束节点连接；新建输出变量`result`，数据来源选择引用`回复 1`节点的输出变量 `Output.Answer`。



7\. 调试工作流。

7.1 单击画布右侧的**调试**。



7.2 输入 `VisitorBizId `参数值，标识当前接入会话用户，并单击**去调试**。



7.3 输入文档链接并回车，检查输出结果。调试过程中可以在画布上观察当前工作流的运行情况，验证工作流是否设置正确。



8\. 启用工作流。

返回至工作流管理页面，启用创建的工作流。



### 步骤 3: 设置应用

1\. 设置模型。

进入应用设置页面，单击**模型设置，**选择模型用于理解问题。



2\. 添加工作流。

在页面**点击添加工作流，**添加网页内容总结助手。

**注意：**

工作流需要调试成功后才能添加。



此外您可以选择为应用添加欢迎语，设置对话体验等增强体验。

### 步骤 4: 调试应用

设置好应用后，您可以在右侧调试区域测试应用是否符合预期。



### 步骤 5: 发布应用

在应用调试结果达到预期效果后，单击**发布**即可上线应用。



应用发布后，可前往**应用发布 > 服务状态**获取体验链接和 API 调用信息。

如需要在微信、企微等平台发布您的应用，可前往应用发布-发布渠道设置，详情参见 。

## 腾讯云智能体开发平台 基于“Multi-Agent 模式”创建“

**URL**: https://cloud.tencent.com/document/product/1759/122573

最近更新时间：2025-12-03 11:21:21

智能体开发平台支持构建 Multi-Agent 模式应用。该模式下，您可以使用多个 Agent 创建智能体应用，实现由多个 Agent 协同完成一个复杂任务。

本文以创建“脱口秀素材创作助手”为例，详细介绍如何基于智能体开发平台创建 Multi-Agent 模式应用。

## 准备工作

已注册腾讯云账号并完成实名认证。

## 操作步骤

### 步骤 1: 新建应用

2\. 新建应用。

2.1 在左侧菜单栏，单击**应用开发**，然后单击**新建应用**。



2.2 输入**应用名称**，选择 **Multi-Agent 模式**，然后单击**新建**，即可完成新建应用。新建后自动进入应用设置界面。



### 步骤 2: 新建并设置 Agent

创建“脱口秀素材创作助手”，需要设置四个 Agent，各自的功能如下：

|           |                                           |
| --------- | ----------------------------------------- |
| Agent 名称  | Agent 功能                                  |
| 脱口秀内容创作助手 | 负责处理用户输入并判断移交时机的主 Agent，可自主决策转交其他子 Agent。 |
| 爬爬        | 负责抓取网页内容的子 Agent，解析输入链接并获取对应网页内容。         |
| 缩缩        | 负责内容总结的子 Agent，总结输入内容。                    |
| 乐乐        | 负责脱口秀段子创作的子 Agent，改编输入内容。                 |

#### 2.1 设置主 Agent

创建 Multi-Agent 模式应用后，系统默认创建一个主 Agent。主 Agent 的名称与应用名称保持一致，可通过点击编辑图标修改和编辑 Agent 名称。

**注意：**

如果仅配置单个 Agent，可使用无明确含义的名称。但在多 Agent 协作场景中，各 Agent 的名字请使用有明确含义的名称，以便主 Agent 正确交接其他子 Agent。

2.1.1. 设置 Agent 协同方式

协同方式选择自由转交模式。



2.1.2. 设置模型

单击**模型设置**，选择思考模型用于理解问题和自主决策转交其他子 Agent。



2.1.3. 设置转交描述

转交描述为 Agent 介绍说明，帮助模型判断该 Agent 的作用和转交时机。

您可参考以下转交描述：

```
脱口秀内容创作助手 Agent 是 Agent 应用的核心调度者，负责多 Agent 调度及模糊问题的兜底处理。
```

2.1.4. 设置提示词

在**提示词**部分，可输入针对大模型的精细化指导内容，如任务目标、任务流程、限制说明等，通过清晰明确的提示词指令，精准引导大模型执行指定任务，生成符合预期标准的回答内容。



单击**模板**，可在预设的提示词模板库中选择适配当前场景的模板，如角色扮演类、知识问答类、内容生成类等，既能快速完成提示词配置，也能提升输出的一致性。



您也可以使用 **AI 一键优化**对您输入的提示词进行优化，让大语言模型将您输入的内容优化为结构化内容，以便大模型更易理解何时执行任务。

**注意：**

一键优化功能将消耗 token 资源。



在本案例中，您可参考以下提示词来设置主Agent：

```
# 任务目标

你是 Agent 应用的核心调度者，唯一职责是负责多 Agent 调度及模糊问题的兜底处理，不参与任何具体业务的执行。

# 限制

若用户输入内容不是链接，立即给出提醒：“输入错误，请输入链接。”
```

#### 2.2 设置子 Agent 爬爬

子 Agent 爬爬负责抓取网页内容，解析输入链接并获取对应网页内容。

#### 2.2.1. 添加子 Agent

单击**添加 Agent**，选择**新建。**



填写** Agent 名称**、**转交描述**，其他模块可选填。创建后各内容也可进行修改。



#### 2.2.2. 为子Agent添加工具

在工具栏添加 `fetch` 插件。

**工具栏**支持为 Agent 配置多样化功能插件，如知识库问答、网页爬取、地图定位等。模型将按照提示词设定的任务目标，匹配对应插件自动执行调用流程，来完成复杂任务。



在工作栏单击**添加**，然后在**添加工具**界面搜索并添加 `fetch` 插件。



#### 2.2.3. 设置子Agent

转交描述可参考以下内容：

```
负责抓取网页内容的 Agent，将输入的链接进行解析，获取对应的网页内容。
```

提示词可参考以下内容：

提示词中输入`@`引入已添加插件。

```
# 任务目标

作为内容抓取助手，你需要根据用户输入的链接，获取链接对应网页的内容，并将网页的内容输出给负责内容总结的 Agent。

# 任务流程

1. 使用 fetch 插件获取链接对应的网页内容。

2. 并将网页的内容给 Agent 缩缩。
```

#### 2.3. 设置子 Agent 缩缩

子 Agent 缩缩负责内容总结，将 Agent 爬爬输入的内容进行总结。

##### 2.3.1. 添加子 Agent

单击**添加 Agent**，选择**新建。**



##### 2.3.2. 设置子 Agent

填入 Agent 名称、模型、转交描述、提示词。

转交描述可参考以下内容：

```
负责内容总结的 Agent，将输入的内容进行总结。
```

提示词可参考以下内容：

```
# 任务目标

作为脱口秀素材总结助手，你需要根据 Agent 爬爬的输出结果，按照风趣诙谐的语气总结出有趣的点，并这些有趣的点输出给 Agent 乐乐。

# 限制

尽量简明清晰，输出的字数不超过 100 字。
```

#### 2.4. 设置子 Agent 乐乐

子 Agent 乐乐负责脱口秀段子创作，将 Agent 乐乐输入的内容进行改编创作。

##### 2.4.1. 添加子 Agent

单击**添加 Agent**，选择**新建。**



##### 2.4.2 设置子 Agent

填入 Agent 名称、模型、转交描述、提示词。

转交描述可参考以下内容：

```
负责脱口秀段子创作的 Agent，将输入的内容进行改编创作。
```

提示词可参考以下内容：

```
# 任务目标

作为脱口秀段子创作助手，你需要根据 Agent 缩缩的输出结果进行扩展写作，加入网感词汇（“家人们”“咱就是说”）、互动梗（“你们有没有过……”）、吐槽句式（“不是…… 而是……”）等，形成完整的脱口秀段子。

# 限制

内容尽量诙谐幽默，完整，输出的字数不超过200字。
```

#### 2.5 设置 Agent 转交关系

在应用中添加两个或以上的 Agent 时，系统支持配置多个 Agent 之间的转交关系。请注意，默认创建多个 Agent 后，这些 Agent 不会立即生效，必须与主 Agent 配置转交关系后，主 Agent 才能根据 Agent 描述准确地将任务分配到特定的子 Agent。
