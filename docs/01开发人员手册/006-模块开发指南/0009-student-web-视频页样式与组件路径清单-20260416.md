- 想改输入页“整体骨架和页面气质”，优先看 `workspace-input-shell.tsx` + `input-page-shared.scss`。

### 等待页样式判断建议

- 等待页的大部分视觉问题，第一优先级都在 `task-generating-view.scss`。

- `video-generating-preview-player.tsx` 决定分段试看区的结构密度；如果你觉得“主预览区不高级 / 轨道不直观”，这里就是主要入口。
- `video-preview-section-badge.tsx` 目前走共享 `Badge` + utility classes，如果觉得状态标签不统一，需要同时看 `badge.tsx`。

## 五、建议优先查看顺序

1. 想统一三页的字体、色彩、亮暗色基调：
   看 `theme.css`、`globals.css`。
2. 想统一输入类页面的气质：
   看 `workspace-input-shell.tsx`、`input-page-shared.scss`、`global-top-nav.scss`。
3. 想重点治理视频输入页的审美：
   看 `video-input-card.tsx`、`_video-input-card.scss`、`dialog.tsx`、`popover.tsx`、`input.tsx`。
4. 想重点治理等待页的信息密度和布局：
   看 `task-generating-view.scss`、`video-generating-stage-content.tsx`、`video-generating-preview-player.tsx`。
5. 想重点治理结果页的“高级感”：
   看 `_result.scss`、`_result-header.scss`、`_companion-sidebar.scss`、`_video-dock.scss`。
6. 想确保 i18n 和文案层同步：
   看 `entry-page-content.ts`、`video-content.ts`。
