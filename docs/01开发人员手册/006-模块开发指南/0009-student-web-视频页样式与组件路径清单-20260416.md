### 输入页样式判断建议

- 想改输入页“整体骨架和页面气质”，优先看 `workspace-input-shell.tsx` + `input-page-shared.scss`。
- 想改“真正丑的那块核心输入区”，优先看 `video-input-card.tsx` + `_video-input-card.scss`。
- 想改“底部浏览区卡片感”，优先看 `video-public-feed.tsx` + `_video-public-feed.scss`。
- 想改“高级参数弹框 / 预设下拉的基础质感”，同时看 `_video-input-card.scss` 和 `components/ui/dialog.tsx` / `popover.tsx` / `input.tsx`。

### 2026-04-19 输入页浏览区补充

- 新版浏览区已经不再直接套 `CommunityFeed`，而是视频域自己的 `xm-video-discovery` + `Tabs` 结构。
- 私有区数据链路不是直接读公开列表，而是：
  - `useVideoWorkspaceTasks()` 先拿已完成任务。
  - `resolveVideoResultAdapter().getResult(taskId)` 再补齐封面、时长、公开状态和结果标题。
- 公开区仍走 `usePublicVideos()`，但卡片改成轻量扫读风格，不再把长 `summary` 正文直接铺在卡片正文里。
- 浏览区现在同一时刻只展示一个 tab panel，不再把私有区和公开区同时展开。

### 等待页样式判断建议

- 等待页的大部分视觉问题，第一优先级都在 `task-generating-view.scss`。
- `video-generating-stage-content.tsx` 决定右侧到底显示哪种信息结构，本身也决定“内容看起来是否有价值”。
- 如果摘要区出现“公式 / markdown 不可读”，第一优先级看 `video-generating-rich-content.tsx`，其次看 `globals.css` 里对 `streamdown` 的 `@source`。
- `video-generating-preview-player.tsx` 决定分段试看区的结构密度；如果你觉得“主预览区不高级 / 轨道不直观”，这里就是主要入口。
- `video-preview-section-badge.tsx` 目前走共享 `Badge` + utility classes，如果觉得状态标签不统一，需要同时看 `badge.tsx`。
- 如果等待页暗色模式和输入页不像同一套设计，第一优先级看 `task-generating-view.scss` 顶部的 `--xm-generating-card-surface / soft-surface / pill-surface` 和 `:root[data-theme='dark']`，不要再新增 `.dark` 选择器分支。

### 2026-04-17 补充

- 等待页顶部主按钮“前往结果页”的交互入口仍在 `video-generating-page.tsx`，视觉样式在 `task-generating-view.scss` 的 `.xm-generating-shell__cta`。
- 左上返回输入页的入口目前依附在等待页品牌按钮上，定位同样在 `video-generating-page.tsx`，品牌视觉仍由 `.xm-generating-shell__brand` / `.xm-generating-shell__brand-icon` 控制。
- 完成态按钮、toast 与顶部状态文案都走 `video-content.ts`，如果你要继续改“完成后停留在等待页”的体验，页面入口、SCSS 和 i18n 这三处要一起看。
- 如果出现“分段试看只显示视频一个角、看不到播放键、双击全屏后才正常”，第一优先级看两处：
  - `video-player.tsx` 的 Video.js 初始化模式是否仍是 `fill: true / fluid: false`
  - `task-generating-view.scss` 里 `.xm-generating-player`、`__screen`、`__tracks`、`__details` 的 `min-height / flex-shrink / overflow` 是否还和设计稿保持一致
- 这类问题通常不是接口或 clipUrl 错，而是“外层 16:9 容器 + 内层 Video.js fluid 盒子 + 下方详情滚动区”三者同时在争抢高度。
- 如果你是专门在调等待页暗色视觉，这次已经统一成和输入页一样的机制：`theme.css` + `globals.css` 提供全局 token，`task-generating-view.scss` 只负责等待页局部 surface token，不再靠 `.dark` 补丁式覆盖。
