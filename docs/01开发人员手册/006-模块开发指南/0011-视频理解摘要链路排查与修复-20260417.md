# 视频理解摘要链路排查与修复

> **状态**: 生效
> **最后更新**: 2026-04-17
> **适用范围**: `packages/fastapi-backend/app/features/video/pipeline/`

## 现象识别

如果等待页摘要区出现以下任一情况，优先怀疑 understanding 链路失败：

- “先看这段快速解释”只显示原题题面，没有任何解题说明
- 知识点区域出现 storyboard / outline 的英文标题
- worker 日志出现：
  - `Understanding stage failed, fallback to raw prompt summary`
  - `JSONDecodeError: Expecting value: line 1 column 1`

## 关键文件

| 文件 | 用途 |
|------|------|
| `packages/fastapi-backend/app/features/video/pipeline/orchestration/orchestrator.py` | understanding 失败后是否 fallback，以及 preview 初始 summary 如何生成 |
| `packages/fastapi-backend/app/features/video/pipeline/services.py` | `UnderstandingService` prompt 与 JSON 解析逻辑 |
| `packages/fastapi-backend/tests/unit/video/test_video_pipeline_services.py` | understanding 解析相关单测 |

## 修复要点

### 1. Prompt 必须强约束 JSON

- 不要只把 `sourcePayload` / `userProfile` 裸传给模型
- 必须明确要求：
  - 只返回 JSON 对象
  - 禁止额外解释、Markdown、标题和前后缀
  - 固定字段：`topicSummary / knowledgePoints / solutionSteps / difficulty / subject`

### 2. 解析器必须兼容 fenced JSON

Gemini / OpenAI Compatible 类模型很常见的返回形式：

- `说明如下：```json ... ````
- 带 `<think>...</think>` 包裹
- 前面一段自然语言，后面才开始 JSON

因此 `_parse_jsonish()` 不能只做一次 `json.loads(text)`。

## 当前结论

- 2026-04-17 已补：
  - JSON-only prompt
  - fenced JSON / 前缀说明文本解析容错
- 如果后续仍出现 fallback，优先先看原始 provider 返回内容，而不是先怀疑前端没接住。

## 2026-04-17 第二轮口吻优化

摘要链路打通后，如果出现下面这种现象，也要继续回看 understanding prompt，而不是以为“有内容就算完成”：

- 摘要像证明提纲
- 摘要像竞赛题标准答案目录
- 摘要虽然正确，但不够像老师讲题

这一轮补的不是“能不能返回”，而是“返回得像不像人在讲题”：

- `topicSummary` 从“2-4 句快速解释”升级成“4-6 句老师式快速讲解”
- prompt 明确要求先讲题意、主线、为什么这样做，再提醒一个易错点
- 允许模型输出少量 Markdown / LaTeX 公式，但必须顺手解释
- preview 侧把 `1. **标题**：解释` 改成 `- 标题：解释`，避免前端最终看到的文风又被拼装回提纲样式

如果后续用户反馈“内容还是太像标准答案”，优先检查两件事：

1. provider 原始返回是否已经退回了学术文风
2. preview summary 拼装是否又把 explanation 二次格式化成了强结构步骤列表

## 2026-04-17 第三轮补充

如果看到的现象不是“完全没摘要”，而是下面这些半坏不坏的情况，也不要只盯着 prompt：

- 摘要有内容，但只是题目标签或标题回声
- 知识点又掉回 storyboard 英文短标题
- provider 返回了合法 JSON，但 `topicSummary` 明显太短、不像老师讲解

这时候要看的是后端有没有走到语义级兜底：

1. `UnderstandingService` 现在会先做语义校验，不再因为“JSON 结构合法”就直接放行。
2. 如果摘要太弱，会触发 second-pass repair，让模型按失败原因重写。
3. 如果 repair 后仍然不够好，会根据 `solutionSteps` 本地合成一段老师式摘要。
4. 如果 understanding 阶段最终还只是题面回声，orchestrator 会在 storyboard 产出后用 section lecture lines 回填 `preview.summary`。

## Dramatiq 环境漂移说明

2026-04-17 当天排查里还有一个容易误导人的假象：

- 工作区 `.venv` 的 `dramatiq` 是 `1.18.x`，有 `Prometheus` middleware 导出
- 系统 Python 3.13 环境的 `dramatiq` 是 `2.1.x`，这个导出已经不存在

所以如果你直接跑系统 `pytest`，旧代码会在 `tests/conftest.py -> app.core.security -> app.infra.redis_client` 的导入链提前炸掉，看起来像“视频摘要链路又挂了”，其实只是测试环境先死了。

当前状态：

- `redis_client.py` 已改成 `Prometheus` 可选导入，兼容 `dramatiq 1.x / 2.x`
- `tests/unit/core/test_dramatiq_broker.py` 已同步改成双版本兼容断言
- `python -m pytest packages/fastapi-backend/tests/unit/video -q` 已恢复可跑
