## 基本配置怎么填

在代码生成器顶部的基础配置区域，按下面填写：

| 配置项 | 填写内容 | 为什么这样填 |
|--------|----------|--------------|
| 生成模板 | 单表 | 这是单一业务表，不是主子表结构 |
| 生成包路径 | `org.dromara.xiaomai.user.profile` | 这是 Java 包根，能和小麦模块 `org.dromara.xiaomai` 保持一致，后续补手写接口也方便归类 |
| 生成模块名 | `xiaomai` | 生成器会把它直接拼进接口路径、权限前缀和前端组件目录；填成 `xiaomai` 才会落到 `xiaomai/...` 目录下 |
| 生成业务名 | `user-profile` | 生成器会继续把它拼成 `/xiaomai/user-profile`、`xiaomai/user-profile/index`；这里不要填带 `/` 的路径 |
| 生成功能名 | `用户配置` | 这是后台菜单名称和代码注释标题，用中文业务名最直观，不要直接填表名 |
| 上级菜单 | `小麦业务` | 本仓库已经冻结了小麦后台根菜单，当前这张表应挂到它下面，而不是挂到“系统工具”下 |

补一句最重要的：

- `ruoyi-xiaomai` 是 Maven 模块名，不是这里的“生成模块名”。
- `user/profile` 这种带 `/` 的写法也不适合填在“生成业务名”，因为生成器会自己再拼路径。
- 你如果还没建一级菜单，就先建：`小麦业务`，路径填 `xiaomai`，菜单类型选“目录”，图标建议 `education`。

## 生成后你还要补什么

| 项目 | 是否生成器自动完成 | 说明 |
