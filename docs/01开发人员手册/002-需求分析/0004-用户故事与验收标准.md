## Epic 列表

### E1: 课堂服务核心

**目标**: 将 OpenMAIC 核心功能移植到 FastAPI，实现虚拟课堂生成
**优先级**: P0
**里程碑**: M1-M2
**关联 FR**: FR-CS-001 ~ FR-CS-007

#### US-E1-01: 输入主题生成课堂

> 作为**高职学生**，我希望**输入一个学习主题就能生成完整的虚拟课堂**，以便**快速获得系统化的知识讲解**。

验收标准：
- [ ] AC1: 用户输入主题文本（2-100 字）后点击生成，5 分钟内返回完整课堂内容
- [ ] AC2: 生成失败时显示具体错误原因并提供重试按钮
- [ ] AC3: 生成过程中 SSE 推送进度百分比，前端实时显示

#### US-E1-02: 选择 Agent 风格

> 作为**高职学生**，我希望**选择不同的 AI 老师教学风格**，以便**找到适合自己的学习方式**。

验收标准：
- [ ] AC1: 用户可从严肃/幽默/耐心/高效 4 种风格中选择 1 种
- [ ] AC2: 选择不同风格后生成的课堂内容在语气、用词、举例风格上有明显差异
- [ ] AC3: 默认风格为"耐心型"

#### US-E1-03: 查看幻灯片

> 作为**高职学生**，我希望**课堂中有结构化的幻灯片辅助讲解**，以便**更好地理解知识结构**。

验收标准：
- [ ] AC1: 课堂生成后包含 >= 3 张结构化幻灯片
- [ ] AC2: 每张幻灯片包含标题、要点、可选图示
- [ ] AC3: 幻灯片可在前端按顺序浏览切换
- [ ] AC4: 每张幻灯片内容验证通过（无无效图片链接、无渲染错误）

#### US-E1-04: 交互式测验

> 作为**高职学生**，我希望**课堂结束后有测验检验学习效果**，以便**知道自己是否掌握了知识点**。

验收标准：
- [ ] AC1: 课堂结束后自动生成 3-5 道选择题
- [ ] AC2: 用户提交答案后即时显示对错和解析
- [ ] AC3: 测验结果记录到学习记录

#### US-E1-05: 多 Agent 讨论

> 作为**高职学生**，我希望**课堂中有多个 AI 角色参与讨论**，以便**通过多视角对话加深理解**。

验收标准：
- [ ] AC1: 课堂中 >= 2 个 Agent 角色参与讨论
- [ ] AC2: Agent 发言按 DirectorGraph 调度顺序执行
- [ ] AC3: 用户可作为"学生"角色参与讨论并获得回应

#### US-E1-06: 课堂进度推送

> 作为**高职学生**，我希望**看到课堂生成的实时进度**，以便**知道还需等待多久**。

验收标准：
- [ ] AC1: 课堂生成过程中通过 SSE 每 5s 至少推送 1 次进度更新
- [ ] AC2: 前端显示进度条和当前步骤描述
- [ ] AC3: SSE 连接断开后自动重连

#### US-E1-07: 白板布局管理

> 作为**高职学生**，我希望**多 Agent 讨论时白板内容清晰不重叠**，以便**方便查看每个角色的发言内容**。

验收标准：
- [ ] AC1: 多 Agent 讨论时白板内容分区显示，无重叠
- [ ] AC2: 白板元素自动布局，支持手动调整
- [ ] AC3: 白板历史版本可回溯

### E2: 视频服务核心

**目标**: 将 ManimToVideoClaw 三个 FastAPI 服务合并，实现单题讲解视频生成
**优先级**: P0
**里程碑**: M1-M2
**关联 FR**: FR-VS-001 ~ FR-VS-009

#### US-E2-01: 题目理解

> 作为**高职学生**，我希望**提交一道题目后系统能准确理解题意**，以便**生成针对性的讲解视频**。

验收标准：
- [ ] AC1: 接收文本或 OCR 识别结果后，提取题目类型、知识点、解题步骤结构化 JSON
- [ ] AC2: 处理时间 < 10s
- [ ] AC3: 无法理解的题目返回明确提示而非错误

#### US-E2-02: 分镜生成

> 作为**高职学生**，我希望**系统将解题过程分成清晰的步骤**，以便**视频讲解循序渐进**。

验收标准：
- [ ] AC1: 基于题目理解结果生成 >= 3 个分镜
- [ ] AC2: 每个分镜包含场景描述、动画类型、旁白文本
- [ ] AC3: 分镜总时长控制在 2-5 分钟

#### US-E2-03: Manim 代码生成与自动修复

> 作为**高职学生**，我希望**系统自动生成高质量的数学动画**，以便**直观理解解题过程**。

验收标准（代码生成）：
- [ ] AC1: 基于分镜生成可执行的 Manim Python 代码
- [ ] AC2: 生成的代码通过 Python 语法检查（无 SyntaxError）
- [ ] AC3: 代码包含完整的 Scene 类定义和 construct 方法

验收标准（自动修复）：
- [ ] AC1: Manim 渲染失败后自动分析错误日志并修复代码
- [ ] AC2: 最多尝试 3 次自动修复
- [ ] AC3: 3 次修复均失败后返回错误详情和最后一次修复尝试的代码

#### US-E2-04: 视频渲染与合成

> 作为**高职学生**，我希望**获得一段完整的讲解视频（动画 + 语音）**，以便**像看视频课一样学习**。

验收标准（Manim 渲染）：
- [ ] AC1: 执行 Manim 代码生成 MP4 视频文件
- [ ] AC2: 输出分辨率 >= 720p
- [ ] AC3: 单次渲染超时上限 120s，超时则终止并报错

验收标准（TTS 合成）：
- [ ] AC1: 支持 >= 2 种 TTS 服务商（豆包、百度）
- [ ] AC2: 主服务商不可用时自动切换备用服务商
- [ ] AC3: 合成音频采样率 >= 16kHz，格式为 MP3/WAV

验收标准（视频合成）：
- [ ] AC1: 将 Manim 动画视频与 TTS 音频合成为最终视频
- [ ] AC2: 音画同步误差 < 500ms
- [ ] AC3: 输出格式为 MP4（H.264 + AAC）

#### US-E2-05: COS 上传

> 作为**高职学生**，我希望**生成的视频可以随时访问**，以便**反复观看学习**。

验收标准：
- [ ] AC1: 合成视频自动上传至腾讯云 COS
- [ ] AC2: 上传完成后返回可访问的 CDN URL
- [ ] AC3: 上传失败自动重试 2 次

#### US-E2-06: 图片识别题目

> 作为**高职学生**，我希望**拍照上传题目就能生成讲解视频**，以便**不用手动输入题目文本**。

验收标准：
- [ ] AC1: 支持 JPG/PNG 格式图片上传，<= 5MB
- [ ] AC2: OCR 识别准确率 >= 90%（标准印刷体）
- [ ] AC3: 识别结果返回结构化文本，支持数学公式（LaTeX 格式）

### E3: 视频播放器

**目标**: 基于 Video.js/ReactPlayer 封装自研视频播放器
**优先级**: P1
**里程碑**: M2
**关联 FR**: FR-VP-001 ~ FR-VP-004

#### US-E3-01: 视频播放与控制

> 作为**高职学生**，我希望**方便地控制视频播放（倍速、进度、全屏）**，以便**按自己的节奏学习**。

验收标准（播放）：
- [ ] AC1: 点击播放后 2s 内开始播放视频
- [ ] AC2: 支持暂停/继续
- [ ] AC3: 播放结束后显示"重新播放"和"返回"按钮

验收标准（倍速）：
- [ ] AC1: 支持 0.5x/0.75x/1.0x/1.25x/1.5x/2.0x 六档倍速
- [ ] AC2: 切换倍速后即时生效，无需重新加载
- [ ] AC3: 倍速选择在播放器控制栏可见

验收标准（进度控制）：
- [ ] AC1: 拖动进度条可跳转到任意时间点
- [ ] AC2: 显示当前播放时间和总时长
- [ ] AC3: 进度条显示已缓冲区域

验收标准（全屏）：
- [ ] AC1: 点击全屏按钮进入全屏模式
- [ ] AC2: 全屏模式下所有控制功能正常可用
- [ ] AC3: 按 ESC 或点击退出按钮可退出全屏

### E5: 用户系统

**目标**: 对接 RuoYi API 实现用户认证和管理
**优先级**: P0
**里程碑**: M1-M2
**关联 FR**: FR-UM-001 ~ FR-UM-004

#### US-E5-01: 用户注册与登录

> 作为**新用户**，我希望**快速注册并登录系统**，以便**开始使用小麦的学习功能**。

验收标准：
- [ ] AC1: 用户输入手机号+验证码，点击注册后 3s 内完成账号创建并自动登录
- [ ] AC2: 已注册用户输入正确凭证后 2s 内完成登录并跳转首页
- [ ] AC3: 错误凭证显示明确错误提示，不泄露账号是否存在

#### US-E5-02: JWT Token 认证

> 作为**已登录用户**，我希望**登录状态在多次访问间保持有效**，以便**不用反复登录**。

验收标准：
- [ ] AC1: FastAPI 接口收到请求后从 Redis 验证 JWT 有效性，有效则放行，无效返回 401
- [ ] AC2: Token 过期后自动刷新，用户无感知
- [ ] AC3: 用户登出后 Token 立即失效

#### US-E5-03: 个人资料管理

> 作为**用户**，我希望**编辑自己的个人资料**，以便**完善个人信息**。

验收标准：
- [ ] AC1: 用户可查看和编辑昵称、头像、学校信息
- [ ] AC2: 修改保存后页面即时刷新显示新信息
- [ ] AC3: 头像上传支持 JPG/PNG，<= 2MB

#### US-E5-04: 权限控制

> 作为**系统管理员**，我希望**不同角色有不同的访问权限**，以便**保障系统安全和功能分级**。

验收标准：
- [ ] AC1: 学生角色仅可访问 ToC 功能，无法访问管理接口
- [ ] AC2: 教师角色可访问批量生成功能
- [ ] AC3: 未登录用户访问受保护页面自动跳转登录页
