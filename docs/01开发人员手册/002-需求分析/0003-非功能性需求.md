- Error Budget 每月 1 日重置

## 安全需求（SE）

| NFR-ID | 需求描述 | 实现方案 |
|--------|----------|----------|
| NFR-SE-001 | 数据加密 | HTTPS + AES-256（敏感数据） |
| NFR-SE-002 | 认证机制 | JWT + 共享 Redis（复用 RuoYi） |
| NFR-SE-003 | API 限流 | Redis Token Bucket（复用 RuoYi） |
| NFR-SE-004 | 输入验证 | Zod Schema 验证（前端）+ Pydantic（后端） |
| NFR-SE-005 | XSS 防护 | React JSX 自动转义 + DOMPurify（`dangerouslySetInnerHTML` 场景）+ 后端存储前服务端消毒 |

> **XSS 防护说明**：React 的 JSX 会自动对插值表达式进行转义，可防御大部分 XSS。但当使用 `dangerouslySetInnerHTML` 渲染富文本（如 AI 生成的 HTML 内容）时，必须显式安装并调用 DOMPurify 进行消毒。

## 外部服务降级策略

| 外部依赖 | 故障检测 | 降级行为 | 恢复策略 |
|----------|----------|----------|----------|
| LLM 服务（Gemini/Claude） | 连续 3 次请求超时（30s）或返回 5xx | 自动切换备用 LLM 服务商（Gemini <-> Claude）；全部不可用时返回"服务暂时繁忙"提示并记录告警 | 每 60s 探测主服务商健康状态，恢复后自动切回 |
| TTS 服务（豆包/百度/Spark） | 单次请求超时 15s 或返回错误 | 按优先级链自动切换：豆包 -> 百度 -> Spark -> Kokoro（本地兜底） | 每 120s 探测失败服务商，恢复后恢复优先级 |
| 腾讯云 OCR | 请求超时 10s 或返回错误 | 提示用户手动输入题目文本；前端显示"图片识别暂不可用，请手动输入题目" | 每 60s 探测，恢复后自动启用 |
| 腾讯云 COS | 上传超时 30s 或返回错误 | 视频暂存本地服务器临时目录（保留 24h），返回临时直链；后台异步重试上传 | 每 5 分钟重试上传，成功后替换临时链接为 CDN URL |
| Redis | 连接超时 3s | JWT 验证降级为直接解码验证（不校验黑名单）；缓存降级为直接查库 | 自动重连，恢复后切回正常模式 |
