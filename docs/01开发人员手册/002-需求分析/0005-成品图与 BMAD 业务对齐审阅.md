# 成品图与 BMAD 业务对齐审阅

更新时间：2026-04-02

## 1. 审阅口径

- 审阅对象：`docs/03UI:UX 设计素材/001UI 设计稿/04-成品图/` 下 HTML 成品图。
- 唯一事实基线：`_bmad-output/planning-artifacts/index.md` 及其链接出的 PRD、UX、Architecture、Epic、Story 文档。
- 本文记录两类结果：
  1. 已被 BMAD 正式回填的设计决策。
  2. 仍需继续收口的剩余“伪业务”点。
- 不计入项：
  1. 纯视觉表达与布局风格。
  2. 设计资产文件（`.png`、`.pen`、`assets/`）。
  3. 0 byte 占位文件。

## 2. 2026-03-31 决策更新

- 以下 4 组设计能力，现已明确判定为“旧 BMAD 缺项”，不再按“伪业务”处理：
  1. `01-正式路由页面/01-首页与入口/01-landingPage.html`：独立营销落地页 `/landing`。
  2. `01-正式路由页面/03-视频输入页/01-input.html`：公开视频广场 / 热门题目讲解视频。
  3. `01-正式路由页面/06-课堂输入页/01-input.html`：课堂输入页联网搜索与证据范围配置。
  4. `01-正式路由页面/08-课堂结果页/01-classroom.html`：课堂结果导出。
- 与上述设计对应的 BMAD 事实已补齐：
  1. `FR-UI-009`、`Epic 1 / Story 1.7`：承接 `/landing`。
  2. `FR-VS-010`、`FR-VP-004`、`Epic 3 / Story 3.6`、`Epic 4 / Story 4.10`：承接公开视频发现、公开发布与复用。
  3. `FR-CS-008`、`FR-KQ-007`、`Epic 5 / Story 5.9`、`Epic 7 / Story 7.7`：承接课堂输入页联网搜索增强。
  4. `FR-CS-009`、`Epic 5 / Story 5.10`：承接课堂结果导出。
- 当前正式口径：
  1. `/`：默认进入的产品 `home` 首页，负责课堂主入口直达与顶栏导航分发，不再承接双入口卡片选择。
  2. `/landing`：只在营销投放、宣传、试点招募等场景出现的独立落地页。
  3. `/video/input`：除自主输入外，还可承接公开视频发现与复用。
  4. `/classroom/input`：除主题输入外，还可承接显式联网搜索配置。
  5. `/classroom/:id`：课堂结果消费页可承接基础导出动作。

## 3. 当前总结结论

- 现阶段最重要的审阅结论不是“设计越界”，而是“原 BMAD 漏写”：
  1. `landingPage` 是正式业务。
  2. 视频输入页的公开视频 feed 是正式业务。
  3. 课堂输入页的联网搜索是正式业务。
  4. 课堂结果页的导出是正式业务。
- 因此，上述 4 类能力不应再作为设计整改项，而应作为产品事实源、Epic 与 Story 的执行项。
- 结合当前 HTML 文件内容检索，本轮未再发现比上述 4 类更明确的新“伪业务”字符串证据。
- 仍无法评审的部分仅限 0 byte 占位文件。

## 7. 说明

- `landingPage` 中出现的教师、院校、试点、咨询模块，当前统一解释为“营销包装与线索转化”，不是 app 内自助开通的 ToB 工作台。

- 首页 `02-home.html` 的事实语义以“单一主入口 + 顶栏导航”执行，不再把首页解释为双入口比较页。
- `01-首页与入口/` 目录下的 `png`、`logo`、`assets`、`.pen` 文件属于设计资产，不纳入正式路由业务评审。
