# 调研后设计阶段图像化作业手册（v1）

## 1. 适用前提
- 调研已完成。
- 本手册不包含调研方法、调研流程、调研产出规范。
- 输入物默认已具备：业务目标、核心用户、关键场景、优先级页面范围。

## 2. 本轮范围
- 仅覆盖 UI/UX 设计阶段的图像化表达。
- 仅讨论如何把设计过程与设计决策可视化。
- 主流程边界：IA/用户流 → 线框 → 组件系统 → 高保真 → 开发标注。

## 3. 三种图像化载体：定义与使用时机

### 3.1 流程图（Flow）
- 定义：表达阶段关系、输入输出、依赖与回路。
- 适用：讲清“先做什么、后做什么、哪里可回退”。
- 典型场景：跨角色对齐、里程碑评审、阶段拆解。

### 3.2 看板图（Board）
- 定义：表达任务状态、优先级、阻塞、责任归属。
- 适用：推进执行，控制在制品，管理迭代节奏。
- 典型场景：周迭代、日跟踪、风险暴露。

### 3.3 画廊图（Gallery）
- 定义：并排展示方案、版本、风格差异与取舍证据。
- 适用：视觉对比、方案评审、组件一致性检查。
- 典型场景：风格探索、关键页比稿、版本演进复盘。

## 5. 迭代机制（强制）
- 每轮只改 1~2 个模块。
- 不做一次性大重写。
- 每轮必须记录：
  - 改动模块
  - 改动原因
  - 影响范围
  - 下轮待办
- 若出现范围膨胀，优先砍范围，不牺牲清晰度。

## 6. 验收口径（本手册唯一口径）
- 验收对象：图像表达质量，不是调研质量。
- 验收维度：
  1) 结构清晰：阶段、依赖、回路可读。
  2) 任务可执行：原子任务可分配、可追踪。
  3) 视觉可比较：版本差异可被快速识别。
  4) 比例可落地：16:9 / 4:3 / 9:16 输出一致。
  5) 迭代可持续：单轮改动低成本、可回滚。

## 7. 明确禁止项
- 禁止把调研流程重新塞回本手册。
- 禁止用“补调研”为理由替代设计图像化拆解。
- 禁止产出大而空的总图，不落到任务与模板。
- 禁止一次性改完全部章节。

- 先对齐总流程与原子任务粒度。
- 再进入看板模板与画廊模板细化。
