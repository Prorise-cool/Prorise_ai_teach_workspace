# 交付物命名规范（一页可执行版）

> 原则：统一、好找、避免覆盖。够用即可，不追求复杂体系。

## 1) 命名格式（固定）

`[项目代号]_[节点编码]_[交付物类型]_[版本号]_[日期]_[负责人]`

- 日期格式：`YYYYMMDD`
- 分隔符统一用下划线 `_`
- 文件名不用空格

---

---

## 4) 版本号规则（轻量）

- 格式：`v主版本.次版本`（如 `v2.3`）
- **主版本 +1**：方向变化、结构重做、影响范围大（如 `v1.4 -> v2.0`）
- **次版本 +1**：常规迭代、补页、修订（如 `v2.0 -> v2.1`）
- 同日多次小修可继续加次版本，不用额外后缀

---

## 5) 目录建议（最少够用）

```text
/project_code/
  /01_structure/      # IN0, IA1, UF2, G1, PL3
  /02_wireframe/      # WF4, UV5
  /03_visual_base/    # MB6, DT7, G2
  /04_hifi_and_proto/ # DS8, HI9, PR10
  /05_delivery/       # DEV11, G3, AR12
```

建议每个目录保留一个 `latest` 文件夹（仅放当前有效版本副本）。

---

## 6) 禁止项（务必避免）

- 禁止使用：`最终版`、`最新版`、`改改改`、`new`、`ok了`
- 禁止缺少版本号或日期
- 禁止中英文+符号混用导致不可检索（如空格、括号、中文标点）
- 禁止同一交付物出现两套命名规则并行
