# 调研后启动问卷（设计阶段）

> 适用时机：调研结论已完成，准备进入 IA→用户流→线框→视觉基线→组件→高保真→交付。
> 目标：15-30分钟对齐“是否能立即开工”，避免后续反复返工。

## A. 输入完整性（开工前先确认）
1. 调研结论摘要是否已固定版本？（是/否，链接）
2. 本期业务目标与用户目标各1句话是什么？
3. 本期范围边界是什么？（明确“不做什么”）
4. 是否已有核心功能清单与优先级？
5. 当前缺失的关键输入有哪些？（最多列3项）

## B. 设计边界（做多深、做到哪）
1. 本期必须覆盖的主流程有哪些？
2. 必须覆盖的异常/边界场景有哪些？
3. 哪些页面本期只做占位或低优先级处理？
4. 本期交付深度：线框 / 高保真 / 可点击原型 / 开发标注（勾选）

## C. 品牌与视觉约束
1. 是否有可执行的品牌规范？（Logo/主色/字体/语气）
2. 本期是否允许视觉探索？允许范围是什么？
3. 是否有必须对齐的历史产品视觉资产？
4. 风格决策人是谁？最终拍板机制是什么？

## D. 技术与实现约束
1. 目标平台与端形态：Web / iOS / Android / 小程序 / 响应式
2. 研发栈或组件框架限制是什么？
3. 是否有性能、可访问性、国际化等硬性要求？
4. 有哪些已知技术风险会影响设计实现？

## E. 时间节奏与里程碑
1. 目标上线时间与不可变节点是什么？
2. Gate1/Gate2/Gate3 预计日期？
3. 每周评审频次与固定时段？
4. 若延期，优先砍哪些范围（提前定义）？

## F. 评审机制与协作方式
1. 每个Gate的参与角色是谁？（产品/设计/研发/业务）
2. 评审通过标准采用哪份清单？（默认 `04_质量检查清单.md`）
3. 评审结论记录在何处？谁负责更新？
4. 返工回路约定：Gate1回到IA，Gate2回到风格板，Gate3回到开发标注（确认）

## G. 启动结论（会后5分钟填完）

- 负责人与截止时间：
- 下一步48小时动作：
