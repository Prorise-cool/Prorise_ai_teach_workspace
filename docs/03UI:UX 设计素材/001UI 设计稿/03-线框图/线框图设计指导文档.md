1. **逐页生成**：按照本文档的页面顺序，每次让AI生成一个页面的HTML线框图

4. **迭代优化**：根据实际效果调整设计，重新生成

## 🎨 设计系统基础

在开始设计前，请确保所有页面遵循以下设计系统：

### 色彩系统

| 颜色角色 | HEX | 用途 |
|---------|-----|------|
| **主色（小麦品牌色）** | #f5c547 | 按钮、链接、高亮 |
| **背景色** | #f5ede1 | 页面背景 |
| **表面色** | #ffffff | 卡片、弹窗 |
| **主文字** | #3b1701 | 标题、正文 |
| **次文字** | #6b4421 | 辅助信息 |


### 字体系统

| 层级 | 大小 | 用途 |
|------|------|------|
| **H1** | 48px | 页面主标题 |
| **H2** | 36px | 区块标题 |
| **H3** | 24px | 卡片标题 |
| **Body** | 16px | 标准正文 |
| **Caption** | 12px | 说明文字 |

### 布局原则

- **悬浮导航**：右上角药丸形工具栏
- **视觉聚焦**：核心交互区居中，最大宽度800px
- **留白呼吸感**：模块外边距使用vh单位
- **毛玻璃效果**：backdrop-blur-md/xl

## 01. 落地页/首页

### 📍 页面路径
`/` 或 `/home`

### 🎯 功能需求（FR-UI-001）

**验收标准：**
1. 首页清晰展示"初学场景"和"问答 Agent"两个入口
2. 每个入口有简短说明文字和引导图标
3. 点击入口后 1s 内跳转到对应功能页

### 🎨 UX设计建议

**核心体验原则：**
- **极简至上**：首页聚焦双入口，不做复杂导航
- **智能推荐**：根据用户历史自动推荐入口（首次使用时显示两个入口）
- **零负担选择**：让入口选择变得直觉化、零焦虑

**视觉设计要点：**

1. **布局结构**
   - 顶部：悬浮导航栏（右上角药丸形）
     - 左侧：小麦Logo + 品牌名
     - 右侧：登录按钮、语言切换、主题切换
   - 中央：核心交互区（居中，最大宽度800px）
     - 欢迎语："你好，想学点什么？"
     - 双入口卡片并排展示
   - 底部：简洁的版权信息

2. **双入口卡片设计**
   - **初学场景卡片**
     - 图标：📚 或 自定义图标
     - 标题："我想学"
     - 副标题："系统化学习，完整课堂体验"
     - 示例："我想学递归算法"、"我想学微积分"
     - 颜色：蓝色调（#4a6fa5）
     - 悬停效果：轻微上浮 + 阴影加深

   - **问答Agent卡片**
     - 图标：❓ 或 自定义图标
     - 标题："我有问题"
     - 副标题："单题讲解，快速理解难点"
     - 示例："帮我讲解这道题"、"解释这个概念"
     - 颜色：橙色调（#ff9500）
     - 悬停效果：同上

3. **智能推荐提示**
   - 首次使用：两个入口平等展示
   - 有历史后：在输入框上方显示推荐标签
     - "检测到你上次用了问答功能，继续？"
     - 点击标签直接跳转

4. **背景氛围**
   - 氛围光晕：左上角主色光晕，右下角风格色光晕
   - 动画：呼吸式脉冲（3-4秒周期）
   - 透明度：10%-20%

### 🧩 关键组件

1. **悬浮导航栏组件**
   - 毛玻璃效果：bg-white/60 backdrop-blur-md
   - 圆角：rounded-full
   - 内容：Logo、登录按钮、语言/主题切换

2. **入口卡片组件**
   - 卡片尺寸：280px × 320px
   - 圆角：16px
   - 阴影：0 4px 12px rgba(0,0,0,0.08)
   - 悬停：translate-y(-2px) + shadow-md

3. **推荐标签组件**
   - 药丸形状：rounded-full
   - 背景：当前风格色 + 透明度
   - 文字：白色或深色（根据对比度）

### 🔄 交互流程

```
用户进入首页
    ↓
看到欢迎语 + 双入口卡片
    ↓
【首次使用】
    ├── 点击"我想学" → 跳转到 /classroom/input
    └── 点击"我有问题" → 跳转到 /qa/input

【有历史用户】
    ├── 看到推荐标签（如"继续问答"）
    ├── 点击标签 → 直接跳转对应功能
    └── 或点击其他入口卡片
```

### 📐 状态设计

**正常状态：**
- 两个卡片并排展示
- 氛围光晕缓慢呼吸

**悬停状态：**
- 卡片轻微上浮
- 阴影加深
- 图标/文字微缩放

**点击状态：**
- 卡片轻微下陷
- scale(0.97)
- 1秒内跳转

**加载状态：**
- 骨架屏展示卡片轮廓
- 氛围光晕已渲染

### ✅ 成功标准

- 用户 3 秒内理解双入口区别
- 点击卡片后 1 秒内跳转
- 首屏加载时间 < 1.5s
- 悬停/点击反馈流畅自然

```

├── home-desktop.html       (桌面端首页)
├── home-mobile.html        (移动端首页 - 预留)
└── assets/                 (图片资源)
```

## 02. 登录注册流程

### 📍 页面路径
`/login` `/register` `/profile/setup`

### 🎯 功能需求（FR-UM-001~003）

**验收标准：**
1. 用户输入手机号+验证码，点击注册后 3s 内完成账号创建并自动登录
2. 已注册用户输入正确凭证后 2s 内完成登录并跳转首页
3. 错误凭证显示明确错误提示，不泄露账号是否存在
4. 用户可查看和编辑昵称、头像、学校信息

### 🎨 UX设计建议

**核心体验原则：**
- **极简操作**：最少输入字段，最快完成流程
- **即时反馈**：每一步都有明确的状态提示
- **零思考**：自动聚焦、自动跳转、智能提示

**视觉设计要点：**

1. **登录/注册页面布局**
   - 居中卡片式设计（最大宽度400px）
   - 小麦Logo + 欢迎语
   - 手机号输入框（带区号选择器）
   - 验证码输入框（带倒计时按钮）
   - 登录/注册按钮（主色填充）
   - 底部：用户协议勾选框

2. **输入框设计**
   - 圆角：8px
   - 边框：2px solid #e6dcc8
   - 聚焦时：边框变为主色 #f5c547
   - 错误时：边框变为红色 #ff4d4f
   - 提示文字：灰色 #6b4421

3. **验证码按钮**
   - 默认状态："获取验证码"
   - 倒计时状态："60s 后重试"（灰色禁用）
   - 重新发送："重新获取"

4. **个人信息完善页**（首次注册后）
   - 头像上传（圆形，支持相机图标）
   - 昵称输入（2-20字）
   - 学校选择（下拉或搜索）
   - 角色选择：学生 / 教师 / 自学者
   - 跳过按钮（右上角）

### 🧩 关键组件

1. **手机号输入组件**
   - 区号选择器：+86（中国）
   - 输入框：11位数字验证
   - 实时格式化：138 0000 0000

2. **验证码输入组件**
   - 6位数字输入框
   - 自动聚焦
   - 输入完成自动提交

3. **头像上传组件**
   - 圆形裁剪预览
   - 支持拖拽上传
   - 文件大小限制提示

### 🔄 交互流程

```
【登录流程】
用户点击"登录"
    ↓
进入登录页
    ↓
输入手机号
    ↓
点击"获取验证码"
    ↓
【验证码发送】
    ├── 成功 → 输入框自动聚焦
    └── 失败 → 显示错误提示

输入验证码（6位）
    ↓
【验证码验证】
    ├── 正确 → 自动登录 → 跳转首页
    └── 错误 → 显示"验证码错误，请重试"

【首次登录】
    ↓
跳转到个人信息完善页
    ↓
上传头像、填写昵称、选择学校
    ↓
点击"完成" → 跳转首页
```

### 📐 状态设计

**输入框状态：**
- 默认：灰色边框
- 聚焦：主色边框 + 阴影
- 错误：红色边框 + 错误提示
- 成功：绿色边框 + ✓ 图标

**按钮状态：**
- 默认：主色填充，可点击
- 禁用：灰色填充，不可点击
- 加载中：加载动画 + 文字"登录中..."
- 成功：短暂绿色 + 跳转

**验证码按钮：**
- 可点击：主色文字
- 倒计时：灰色文字 + 禁用
- 重新发送：主色文字

### ✅ 成功标准

- 登录/注册流程 < 3步完成
- 验证码发送 < 10s
- 错误提示明确友好
- 跳转流畅无卡顿

### 📂 文件命名建议

```
02-登录注册/
├── login.html              (登录页)
├── register.html           (注册页)
├── profile-setup.html      (个人信息完善)
└── assets/
```

## 03. 初学场景流程

### 📍 页面路径
`/classroom/input` `/classroom/style` `/classroom/generating` `/classroom/[id]` `/classroom/[id]/quiz`

### 🎯 功能需求（FR-CS-001~007）

**验收标准：**
1. 用户输入主题文本（2-100 字）后点击生成，5 分钟内返回完整课堂内容
2. 用户可从严肃/幽默/耐心/高效 4 种风格中选择 1 种
3. 课堂生成后包含 ≥ 3 张结构化幻灯片
4. 课堂结束后自动生成 3-5 道选择题
5. 课堂中 ≥ 2 个 Agent 角色参与讨论

### 🎨 UX设计建议

**核心体验原则：**
- **输入即惊喜**：极简输入，零思考
- **风格可见性**：4种风格肉眼可见的差异
- **等待透明性**：真实进度，阶段可见
- **互动参与感**：用户作为"学生"参与课堂

---

#### 3.1 主题输入页 `/classroom/input`

**视觉设计要点：**

1. **布局结构**
   - 顶部：面包屑导航（首页 > 初学场景）
   - 中央：核心输入区（居中，800px）
     - 大标题："想学什么？"
     - 副标题："输入主题，我来帮你生成完整课堂"
     - 输入框：大号输入框（占位符："例如：递归算法"、"微积分基础"）
     - 示例标签：热门主题快捷选择
   - 底部："下一步"按钮（禁用状态，输入后激活）

2. **输入框设计**
   - 高度：80px（大号）
   - 字体：18px（Body Large）
   - 圆角：16px
   - 边框：2px solid #e6dcc8
   - 聚焦时：边框变为主色 + shadow-2xl
   - 背景光晕：聚焦时显示风格色光晕

3. **示例标签**
   - 药丸形状：rounded-full
   - 颜色：浅灰背景 + 深灰文字
   - 点击后：文字自动填入输入框

**关键组件：**
- 大号输入框组件
- 示例标签组件
- 下一步按钮组件

---

#### 3.2 风格选择页 `/classroom/style`

**视觉设计要点：**

1. **布局结构**
   - 顶部：进度指示器（1/3 选择风格）
   - 中央：4个风格卡片（2×2网格或横向滚动）
   - 底部："开始生成"按钮

2. **风格卡片设计**（核心差异化）

   **严肃型老师**
   - 主色：#4a6fa5（蓝色）
   - 头像：专业形象（戴眼镜、正装）
   - 名字："严老师"
   - 一句话："专业严谨，逻辑清晰"
   - 语气示例："递归是一种算法思想，让我们从定义开始..."
   - 动画风格：线条图、流程图
   - 悬停效果：蓝色光晕 + 卡片上浮

   **幽默型老师**
   - 主色：#ff9500（橙色）
   - 头像：活泼形象（微笑、休闲装）
   - 名字："小幽默"
   - 一句话："轻松有趣，寓教于乐"
   - 语气示例："让我们用俄罗斯套娃来理解递归！"
   - 动画风格：卡通图、趣味动画
   - 悬停效果：橙色光晕 + 卡片上浮

   **耐心型老师**
   - 主色：#52c41a（绿色）
   - 头像：温和形象（微笑、温暖）
   - 名字："耐心姐"
   - 一句话："一步一步，不急不躁"
   - 语气示例："别着急，我们先理解这个概念的基础..."
   - 动画风格：慢速演示、分步展示
   - 悬停效果：绿色光晕 + 卡片上浮

   **高效型老师**
   - 主色：#722ed1（紫色）
   - 头像：干练形象（简洁、自信）
   - 名字："快博士"
   - 一句话："直奔主题，快速掌握"
   - 语气示例："核心要点就这三个，记住即可"
   - 动画风格：快速演示、关键帧
   - 悬停效果：紫色光晕 + 卡片上浮

3. **卡片交互**
   - 默认：所有卡片平等展示
   - 悬停：当前卡片放大，其他卡片半透明
   - 选中：边框变为主色 + ✓ 图标
   - 点击"试听"按钮：播放5秒语气示例音频

**关键组件：**
- 风格卡片组件（4个变体）
- 语气示例播放器组件
- 进度指示器组件

#### 3.3 课堂生成等待页 `/classroom/generating`

**视觉设计要点：**

1. **布局结构**
   - 顶部：进度指示器（2/3 生成课堂）
   - 中央：进度展示区
     - 大标题："正在生成课堂..."
     - 进度条：分阶段展示
     - 预计剩余时间："预计还需 2 分 30 秒"
     - 当前步骤描述："正在生成幻灯片..."
   - 底部：取消按钮

2. **分阶段进度条**

   ```
   [✓] 理解主题        (0-20%)
   [✓] 生成大纲        (20-40%)
   [→] 生成幻灯片      (40-70%)
   [ ] 生成测验        (70-90%)
   [ ] 准备完成        (90-100%)
   ```

   - 当前步骤：主色高亮 + 动画
   - 已完成：绿色 ✓ 图标
   - 未完成：灰色

3. **小成就提示**
   - 阶段完成时弹出
   - 文案："大纲已生成！"、"幻灯片完成！"
   - 动画：从右侧滑入，3秒后消失
   - 颜色：当前风格色

4. **等待时的视觉反馈**
   - 背景光晕：当前风格色，缓慢呼吸
   - 进度条动画：平滑过渡
   - 骨架屏：课堂页面骨架预览（可选）

**关键组件：**
- 分阶段进度条组件
- 小成就提示组件
- 预计时间倒计时组件

#### 3.4 课堂主页 `/classroom/[id]`

**视觉设计要点：**

1. **布局结构**（沉浸式课堂布局）

   ```
   ┌─────────────────────────────────────┐
   │ 顶部工具栏（极简）                    │
   │ [返回] [课堂标题] [测验] [分享]        │
   ├─────────────────────────────────────┤
   │ 幻灯片区域（60% 高度）                │
   │ [幻灯片内容]                         │
   │ [< 上一页] [页码] [下一页 >]          │
   ├─────────────────────────────────────┤
   │ Agent 对话区域（30% 高度）            │
   │ [Agent 1 头像] [发言内容]             │
   │ [Agent 2 头像] [发言内容]             │
   │ [用户头像] [输入框] [发送]            │
   └─────────────────────────────────────┘
   ```

2. **幻灯片区域**
   - 宽高比：16:9
   - 背景：白色或浅灰
   - 内容：标题、要点、图示
   - 切换动画：淡入淡出
   - 控制栏：上一页/下一页、页码指示器

3. **Agent 对话区域**
   - 样式：类似聊天界面
   - Agent头像：圆形，带角色颜色边框
   - 发言气泡：圆角矩形，背景色区分角色
   - 当前发言Agent：头像有光晕效果
   - 用户输入框：底部固定

4. **Agent 视觉差异**
   - **严老师**：蓝色边框头像 + 蓝色气泡
   - **小幽默**：橙色边框头像 + 橙色气泡
   - **学生角色**：灰色边框头像 + 灰色气泡
   - **用户**：主色边框头像 + 白色气泡

5. **交互功能**
   - 用户可随时输入问题

### 7.1 Loading 页面

**使用场景：**
- 页面初始加载
- 数据获取中
- 视频缓冲

**设计要点：**
- 小麦 Logo 动画（旋转或脉冲）
- 加载文字："加载中..."
- 背景：品牌色光晕

### 7.2 错误页面

**类型：**
- 404 页面（页面不存在）
- 500 页面（服务器错误）
- 网络错误页面

**设计要点：**
- 友好的错误图标
- 清晰的错误说明
- "返回首页"或"重试"按钮
- 颜色：不要过于刺眼，保持品牌调性

### 7.3 空状态页面

**类型：**
- 历史记录为空
- 收藏夹为空
- 搜索结果为空

**设计要点：**
- 插图或图标
- 说明文字
- 引导操作按钮

### 7.4 确认对话框

**使用场景：**
- 删除操作
- 退出登录
- 重要操作确认

**设计要点：**
- 模态框居中
- 标题 + 说明文字
- 两个按钮：[取消] [确认]
- 确认按钮：主色或红色（危险操作）

### 7.5 Toast 提示

**类型：**
- 成功提示（绿色）
- 错误提示（红色）
- 警告提示（橙色）
- 信息提示（蓝色）

**设计要点：**
- 顶部或底部显示
- 自动消失（3秒）
- 可手动关闭
- 图标 + 文字
