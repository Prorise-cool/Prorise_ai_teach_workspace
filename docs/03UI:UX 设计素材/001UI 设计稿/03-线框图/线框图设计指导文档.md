# 小麦线框图设计指导文档

**文档版本：** v3.3  
**更新日期：** 2026-03-28  
**用途：** 统一新架构下的线框图表达边界、命名规则与验收标准。

---

## 1. 设计骨架

本轮线框图统一基于 5 层产品骨架：

- `Video Engine`：单题讲解视频生成与播放。
- `Classroom Engine`：主题课堂生成与沉浸式讲解。
- `Companion`：会话内嵌伴学层，解释当前这一秒 / 这一步。
- `Retrieval / Evidence Service`（目录沿用 `Knowledge`）：资料证据检索与来源管理承载层。
- `Learning Coach`：会话后学习教练层，承接练习、推荐与路径。

### 1.1 最重要的结论

- `Video` 与 `Classroom` 是两条独立内容链路，不在页面层合并。
- `Companion` 是嵌入式能力，不是独立主路由。
- `Retrieval / Evidence Service` 与 `Learning Coach` 是两个独立域，不再用“资料证据检索 + 学后巩固”模糊打包。
- 学生端不再保留独立 `Knowledge` 路由；资料证据能力只作为结果页与学习中心里的来源抽屉 / 证据面板出现。
- 课堂主叙事保持沉浸，正式 quiz 不得硬插在课堂讲解里。

## 2. 目录与路由

### 2.1 目录归位规则

- `01-正式路由页面/`：所有正式业务页面。
- `01-正式路由页面/` 下只允许按页面建目录，不允许再按“视频模块 / 课堂模块 / Companion 模块”拆 UI 资产。
- 页面里的加载态、展开态、局部内嵌态，都应放回该页面目录内部。
- `02-共享交互与通用状态/`：只放等待壳层、反馈状态、通用对话框、警告类复用交互。
- `04-成品图/01-正式路由页面/` 必须与线框图页面目录一一对齐，不允许重新按模块分桶。

## 3. 三层边界规范

### 3.1 Companion

**定义**  
用户针对“当前上下文锚点”发问的能力层。

**必须表达**

- 当前上下文锚点：视频时间点、课堂步骤、当前 slide、白板段落。
- 追问输入。
- 回答流。
- 白板解释区。
- 不中断主叙事的返回 / 收起逻辑。

**禁止事项**

- 不得单独发明 `/companion` 正式路由。
- 不得把 `Companion` 画成与主内容区完全割裂的第二套产品。

### 3.2 Retrieval / Evidence Service（非路由证据面板）

**定义**  
面向资料、教材、讲义、术语、上传文档的证据检索与来源管理能力层，以来源抽屉 / 证据面板形式嵌入主页面。

**必须表达**

- 来源范围或证据范围选择。
- 来源引用。
- 文档上传 / 解析 / 索引状态。
- 历史记录入口。

**禁止事项**

- 不得承担“当前视频这一步为什么这样”的主解释。
- 不得省略来源依据。
- 不得定义为学生端独立页面入口。

### 3.3 Learning Coach

**定义**  
承接会话后的巩固、练习与路径规划。

**必须表达**

- `checkpoint / quiz / path` 至少一个清晰入口。
- 错题、推荐、下一步动作的承接关系。
- 与学习中心聚合页的回流关系。

**禁止事项**

- 不得在课堂讲解流程中强插正式 quiz。
- 不得把学习教练页画成资料证据面板的另一个皮肤。

---

## 4. 课堂叙事与 quiz 的关系

### 4.1 硬约束

- 正式 quiz 必须是会话后动作，而不是课堂中的必经步骤。
- 课堂里只允许出现轻量 `checkpoint` 提示或“课后试一试”入口。

### 4.2 目标体验

- 课堂主线持续沉浸。
- 练习在用户确认后进入独立 `Learning Coach` 流程。
- 学后闭环回流到 `/learning`，而不是塞回个人中心。

---

## 5. 线框文件编写规则

### 5.1 单路由单主文件

- 一个主路由只对应一个主线框文件。
- 同一路由的加载态、空态、错误态、局部展开态都放在同一页面目录中表达。

### 5.2 共享资产归位

- 等待进度、toast、confirm、error、loading 一律进入共享目录。
- 页面专属对话框、Companion 展开态、局部白板态、来源抽屉态，不再单独拆共享目录，必须跟着所属页面走。

### 5.3 命名规则

- 目录名按页面命名，不按能力模块命名。
- 文件名按页面内状态顺序命名：`01-main`、`02-expanded`、`03-error` 或实际流程名。
- 只有共享状态目录允许继续按交互类型命名。

## 7. Companion 最低表达要求

每个包含嵌入式 Companion 的结果页都至少要有：

1. 上下文条  
标明“当前这一秒 / 当前这一步 / 当前 slide”。

2. 问答流  
包含用户提问、AI 回答、追问建议。

3. 白板解释  
至少能表达结构化解释或分步拆解。

4. 主叙事不中断  
视频继续播放、课堂继续浏览不应被 Companion 劫持。

---

## 8. 提交前自检

- 路由与目录归属是否一致。
- 视频页和课堂页是否都能看见 Companion 的入口与展开态。
- 视频结果页、课堂结果页、学习中心里的来源抽屉是否强调证据来源与解析状态，而不是把证据能力画成独立页面。
- quiz 是否已经从课堂主叙事中剥离。
- `/learning` 是否承担了结果聚合职责。
- 共享反馈状态是否已从业务目录抽离。
