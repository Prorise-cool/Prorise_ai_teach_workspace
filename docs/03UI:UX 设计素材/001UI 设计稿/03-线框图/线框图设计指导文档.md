## 📋 文档使用说明

### 如何使用本文档

1. **逐页生成**：按照本文档的页面顺序，每次让AI生成一个页面的HTML线框图
2. **提供完整上下文**：每次生成时，复制对应页面模块的完整内容给AI
3. **保存到对应文件夹**：生成的HTML文件保存到对应的子文件夹中
4. **迭代优化**：根据实际效果调整设计，重新生成

### 文件夹结构

```
03-线框图/
├── 线框图设计指导文档.md (本文档)
├── 01-首页与入口/
├── 02-课堂模块/
├── 03-视频模块/
├── 04-知识问答模块/
├── 05-课后小测模块/
├── 06-学习路径模块/
├── 07-个人中心模块/
├── 08-非路由交互与通用状态/
└── 99-archive/
```

### 📦 目录重规划规则

- 顶层目录按正式路由模块拆分，不按零散功能点或临时状态拆分。
- 一个主路由对应一个主线框文件，同一路由内部状态统一放回该文件表达。
- 输入页内部展开态，如老师风格面板、OCR 中间态、推荐提示，不再单独拆成步骤文件。
- 非路由交互，如认证弹窗、共享等待壳层、老师风格面板、资料初始化、Toast、错误页，统一放到 `08-非路由交互与通用状态/`。
- 旧的“同一路由拆分页”只放到 `99-archive/`，不再作为正式结构参考。

### ⚠️ 老师风格选择器的线框解释规则

- 老师风格选择器属于输入页中的会话配置入口，不是独立主流程页面。
- 线框中的老师风格稿用于展示输入页内部的展开状态，不代表正式前端新增独立 `style` 路由。
- 正式前端路由边界以 UX 规范、架构文档和 Story 1.4 为准；线框状态稿只负责表达输入页局部交互。

### 🧭 路由优先的线框边界规则

- `/classroom/input`、`/classroom/:id/generating`、`/classroom/:id` 分别对应 3 张主线框。
- `/video/input`、`/video/:id/generating`、`/video/:id` 分别对应 3 张主线框。
- `/quiz/:sessionId` 只保留 1 张主线框，开始态、答题态、结果态、完成态都在同一文件内表达。
- `/path` 只保留 1 张主线框，目标输入、生成中、结果态都在同一文件内表达。
- `/knowledge` 只保留 1 张主线框，引用来源是同路由内部展开态，不单拆新页面。

### 字体系统

| 层级 | 大小 | 字重 | 行高 | 用途 |
|------|------|------|------|------|
| **H1** | 48px | 600 | 1.2 | 页面主标题 |
| **H2** | 36px | 600 | 1.2 | 区块标题 |
| **H3** | 24px | 500 | 1.3 | 卡片标题 |
| **Body Large** | 18px | 400 | 1.5 | 核心输入框正文 |
| **Body** | 16px | 400 | 1.5 | 标准正文 |
| **Body Small** | 14px | 500 | 1.5 | 快捷指令标签、按钮 |
| **Caption** | 12px | 400 | 1.4 | 说明文字 |

### 布局原则

- **悬浮导航**：右上角药丸形工具栏（毛玻璃效果）
- **视觉聚焦**：核心交互区居中，最大宽度800px
- **留白呼吸感**：模块外边距使用vh单位
- **毛玻璃效果**：backdrop-blur-md/xl

### 圆角系统

| Token | 值 | 用途 |
|-------|-----|------|
| **radius-sm** | 4px | 内部小图标、复选框 |
| **radius-md** | 8px | 标准卡片、弹窗内元素 |
| **radius-lg** | 16px | 核心对话框主体 |
| **radius-full** | 9999px | 悬浮导航条、头像 |

## 01. 落地页/首页

### 📍 页面路径

`/` 或 `/home`

### 🎯 功能需求（FR-UI-001）

**验收标准：**

1. 首页清晰展示"初学场景"和"问答 Agent"两个入口
2. 每个入口有简短说明文字和引导图标
3. 点击入口后 1s 内跳转到对应功能页

### 🎨 UX设计建议

**核心体验原则：**
- **极简至上**：首页聚焦双入口，不做复杂导航
- **智能推荐**：根据用户历史自动推荐入口（首次使用时显示两个入口）
- **零负担选择**：让入口选择变得直觉化、零焦虑

**视觉设计要点：**

1. **布局结构**
   - 顶部：悬浮导航栏（右上角药丸形）
     - 左侧：小麦Logo + 品牌名
     - 右侧：登录按钮、语言切换、主题切换
   - 中央：核心交互区（居中，最大宽度800px）
     - 欢迎语："你好，想学点什么？"
     - 双入口卡片并排展示
   - 底部：简洁的版权信息

2. **双入口卡片设计**
   - **初学场景卡片**
     - 图标：📚 或 自定义图标
     - 标题："我想学"
     - 副标题："系统化学习，完整课堂体验"
     - 示例："我想学递归算法"、"我想学微积分"
     - 颜色：蓝色调（#4a6fa5）
     - 悬停效果：轻微上浮 + 阴影加深

   - **问答Agent卡片**
     - 图标：❓ 或 自定义图标
     - 标题："我有问题"
     - 副标题："单题讲解，快速理解难点"
     - 示例："帮我讲解这道题"、"解释这个概念"
     - 颜色：橙色调（#ff9500）
     - 悬停效果：同上

3. **智能推荐提示**
   - 首次使用：两个入口平等展示
   - 有历史后：在输入框上方显示推荐标签
     - "检测到你上次用了问答功能，继续？"
     - 点击标签直接跳转

4. **背景氛围**
   - 氛围光晕：左上角主色光晕，右下角风格色光晕
   - 动画：呼吸式脉冲（3-4秒周期）
   - 透明度：10%-20%

### 🧩 关键组件

1. **悬浮导航栏组件**
   - 毛玻璃效果：bg-white/60 backdrop-blur-md
   - 圆角：rounded-full
   - 内容：Logo、登录按钮、语言/主题切换

2. **入口卡片组件**
   - 卡片尺寸：280px × 320px
   - 圆角：16px
   - 阴影：0 4px 12px rgba(0,0,0,0.08)
   - 悬停：translate-y(-2px) + shadow-md

3. **推荐标签组件**
   - 药丸形状：rounded-full
   - 背景：当前风格色 + 透明度
   - 文字：白色或深色（根据对比度）

### 🔄 交互流程

```
用户进入首页
    ↓
看到欢迎语 + 双入口卡片
    ↓
【首次使用】
    ├── 点击"我想学" → 跳转到 /classroom/input
    └── 点击"我有问题" → 跳转到 /video/input

【有历史用户】
    ├── 看到推荐标签（如"继续问答"）
    ├── 点击标签 → 直接跳转对应功能
    └── 或点击其他入口卡片
```

### 📐 状态设计

**正常状态：**
- 两个卡片并排展示
- 氛围光晕缓慢呼吸

**悬停状态：**
- 卡片轻微上浮
- 阴影加深
- 图标/文字微缩放

**点击状态：**
- 卡片轻微下陷
- scale(0.97)
- 1秒内跳转

**加载状态：**
- 骨架屏展示卡片轮廓
- 氛围光晕已渲染

### ✅ 成功标准

- 用户 3 秒内理解双入口区别
- 点击卡片后 1 秒内跳转
- 首屏加载时间 < 1.5s
- 悬停/点击反馈流畅自然

```
├── 01-home.html            (首页 / 双入口页)
└── assets/                 (图片资源)
```

## 02. 非路由认证与资料初始化

### 📍 页面定位
认证弹窗与资料初始化属于非路由交互资产，不单独代表正式主路由页面。

### 🎯 功能需求（FR-UM-001~003）

**验收标准：**
1. 用户输入手机号+验证码，点击注册后 3s 内完成账号创建并自动登录
2. 已注册用户输入正确凭证后 2s 内完成登录并跳转首页
3. 错误凭证显示明确错误提示，不泄露账号是否存在
4. 用户可查看和编辑昵称、头像、学校信息

### 🎨 UX设计建议

**核心体验原则：**
- **极简操作**：最少输入字段，最快完成流程
- **即时反馈**：每一步都有明确的状态提示
- **零思考**：自动聚焦、自动跳转、智能提示

**视觉设计要点：**

1. **登录/注册弹窗布局**
   - 居中卡片式设计（最大宽度400px）
   - 小麦Logo + 欢迎语
   - 手机号输入框（带区号选择器）
   - 验证码输入框（带倒计时按钮）
   - 登录/注册按钮（主色填充）
   - 底部：用户协议勾选框

2. **输入框设计**
   - 圆角：8px
   - 边框：2px solid #e6dcc8
   - 聚焦时：边框变为主色 #f5c547
   - 错误时：边框变为红色 #ff4d4f
   - 提示文字：灰色 #6b4421

3. **验证码按钮**
   - 默认状态："获取验证码"
   - 倒计时状态："60s 后重试"（灰色禁用）
   - 重新发送："重新获取"

4. **个人信息完善页**（首次注册后）
   - 头像上传（圆形，支持相机图标）
   - 昵称输入（2-20字）
   - 学校选择（下拉或搜索）
   - 角色选择：学生 / 教师 / 自学者
   - 跳过按钮（右上角）

### 🧩 关键组件

1. **手机号输入组件**
   - 区号选择器：+86（中国）
   - 输入框：11位数字验证
   - 实时格式化：138 0000 0000

2. **验证码输入组件**
   - 6位数字输入框
   - 自动聚焦
   - 输入完成自动提交

3. **头像上传组件**
   - 圆形裁剪预览
   - 支持拖拽上传
   - 文件大小限制提示

### 🔄 交互流程

```
【登录流程】
用户点击"登录"
    ↓
打开认证弹窗
    ↓
输入手机号
    ↓
点击"获取验证码"
    ↓
【验证码发送】
    ├── 成功 → 输入框自动聚焦
    └── 失败 → 显示错误提示

输入验证码（6位）
    ↓
【验证码验证】
    ├── 正确 → 自动登录 → 跳转首页
    └── 错误 → 显示"验证码错误，请重试"

【首次登录或首次完成认证】
    ↓
进入资料初始化引导态
    ↓
上传头像、填写昵称、选择学校
    ↓
点击"完成" → 跳转首页
```

### 📐 状态设计

**输入框状态：**
- 默认：灰色边框
- 聚焦：主色边框 + 阴影
- 错误：红色边框 + 错误提示
- 成功：绿色边框 + ✓ 图标

**按钮状态：**
- 默认：主色填充，可点击
- 禁用：灰色填充，不可点击
- 加载中：加载动画 + 文字"登录中..."
- 成功：短暂绿色 + 跳转

**验证码按钮：**
- 可点击：主色文字
- 倒计时：灰色文字 + 禁用
- 重新发送：主色文字

### ✅ 成功标准

- 登录/注册流程 < 3步完成
- 验证码发送 < 10s
- 错误提示明确友好
- 跳转流畅无卡顿

## 03. 课堂模块

### 📍 页面路径
`/classroom/input` `/classroom/:id/generating` `/classroom/:id`

### 🎯 功能需求（FR-CS-001~007）

**验收标准：**
1. 用户输入主题文本（2-100 字）后点击生成，5 分钟内返回完整课堂内容
2. 用户可从严肃/幽默/耐心/高效 4 种风格中选择 1 种
3. 课堂生成后包含 ≥ 3 张结构化幻灯片
4. 课堂结束后自动生成 3-5 道选择题
5. 课堂中 ≥ 2 个 Agent 角色参与讨论

### 🎨 UX设计建议

**核心体验原则：**
- **输入即惊喜**：极简输入，零思考
- **风格可见性**：4种风格肉眼可见的差异
- **等待透明性**：真实进度，阶段可见
- **互动参与感**：用户作为"学生"参与课堂

---

#### 3.1 主题输入页 `/classroom/input`

**视觉设计要点：**

1. **布局结构**
   - 顶部：面包屑导航（首页 > 初学场景）
   - 中央：核心输入区（居中，800px）
     - 大标题："想学什么？"
     - 副标题："输入主题，我来帮你生成完整课堂"
     - 输入框：大号输入框（占位符："例如：递归算法"、"微积分基础"）
     - 老师风格触发器：输入框右上角或近输入框位置的头像 / 胶囊触发器
     - 示例标签：热门主题快捷选择
   - 底部："下一步"按钮（禁用状态，输入后激活）

2. **输入框设计**
   - 高度：80px（大号）
   - 字体：18px（Body Large）
   - 圆角：16px
   - 边框：2px solid #e6dcc8
   - 聚焦时：边框变为主色 + shadow-2xl
   - 右上角为老师风格触发器预留固定位置
   - 背景光晕：聚焦时显示风格色光晕

3. **示例标签**
   - 药丸形状：rounded-full
   - 颜色：浅灰背景 + 深灰文字
   - 点击后：文字自动填入输入框

**关键组件：**
- 大号输入框组件
- 老师风格触发器组件（展开后复用 3.2 的面板状态稿）
- 示例标签组件
- 下一步按钮组件

---

#### 3.2 输入页内部状态要求（包含老师风格面板展开态）

**视觉设计要点：**

1. **布局结构**
   - 保持 `3.1` 的输入页主体可见，不切换到独立流程页
   - 在核心输入框右上角或近输入框位置放置头像 / 胶囊触发器
   - 点击触发器后，在输入框附近展开老师风格面板，而不是整页跳转
   - 继续使用输入页原有主 CTA 发起生成，不新增独立的“开始生成”页面按钮或步骤页

2. **面板中的风格项设计**
   - MVP 默认提供 4 种老师风格：严肃、幽默、耐心、高效
   - 每个风格项至少展示：头像、名称、选中态
   - 可选增强信息：一句话风格说明、简短语气示例、试听动作

   - 发起生成：仍由输入页主 CTA 完成，风格选择只是会话配置项

**线框文件要求：**

## 04. 视频模块

### 🎯 功能需求（FR-VS-001~009）

**验收标准：**
1. 接收文本或 OCR 识别结果后，提取题目类型、知识点
2. 支持 JPG/PNG 格式图片上传，≤ 5MB

## 09. 非路由交互与通用状态

### 9.1 Loading 页面

**使用场景：**
- 页面初始加载
- 数据获取中
- 视频缓冲

**设计要点：**
- 小麦 Logo 动画（旋转或脉冲）
- 加载文字："加载中..."
- 背景：品牌色光晕

### 9.2 错误页面

**类型：**
- 404 页面（页面不存在）
- 500 页面（服务器错误）
- 网络错误页面

**设计要点：**
- 友好的错误图标
- 清晰的错误说明
- "返回首页"或"重试"按钮
- 颜色：不要过于刺眼，保持品牌调性

### 9.3 空状态页面

**类型：**
- 历史记录为空
- 收藏夹为空
- 搜索结果为空

**设计要点：**
- 插图或图标
- 说明文字
- 引导操作按钮

### 9.4 确认对话框

**使用场景：**
- 删除操作
- 退出登录
- 重要操作确认

**设计要点：**
- 模态框居中
- 标题 + 说明文字
- 两个按钮：[取消] [确认]
- 确认按钮：主色或红色（危险操作）

### 9.5 Toast 提示

**类型：**
- 成功提示（绿色）
- 错误提示（红色）
- 警告提示（橙色）
- 信息提示（蓝色）

**设计要点：**
- 顶部或底部显示
- 自动消失（3秒）
- 可手动关闭
- 图标 + 文字
