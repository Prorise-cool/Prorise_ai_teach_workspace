## 1. 范围与目标

### 1.1 本文必须覆盖

- `/classroom/input`：主题输入、老师风格（Agent Style）会话配置、提交创建课堂任务。
- `/classroom/:id/generating`：等待与恢复（SSE 进度、断线恢复快照、失败与重试）。
- `/classroom/:id`：结果浏览（幻灯片切换、白板阅读）、多 Agent 讨论片段挂载位置、进入课后小测与返回课堂。
- 异常与恢复：生成失败、断线恢复、局部数据缺失。
- 明确 P0 与增强槽位（不阻断 P0 的后置增强）。

### 1.2 体验总目标（对齐事实源）

- 长任务过程可见，断线可恢复，完成后可回看。
- 课堂链路跑通：主题输入 → 课堂生成 → 浏览幻灯片/白板 → 完成测验 → 可回到课堂继续浏览，并沉淀到历史/学习中心。
- 不要求用户理解 LLM/Manim/TTS 等技术概念；错误提示面向用户，不暴露调试信息。

## 2. P0 与增强槽位（结论先行）

### 2.1 P0（必须）

- 输入与提交：在 `/classroom/input` 可完成主题输入与提交；输入为空或校验失败有明确提示。
- 等待与恢复：`/classroom/:id/generating` 展示统一任务状态与阶段进度；支持基于 Redis 运行时状态与事件缓存的断线恢复，并提供可行动的失败态（重试、返回输入、稍后在历史打开等）。
- 结果浏览：`/classroom/:id` 支持顺序浏览幻灯片（上一页/下一页/目录跳转至少一种）；支持白板阅读模式（白板可读性不因布局问题破坏主链路）。
- 多 Agent 讨论片段：在结果页有明确挂载区，至少支持“讨论摘要片段”展示；允许以学生身份参与输入并获得回应（最小可演示闭环）。
- 课后小测：从课堂结果页进入测验并返回课堂（完成后可回到 `/classroom/:id`），满足“完成测验”的 E2E 场景。
- 异常与降级：生成失败、断线、局部数据缺失均不应把用户引入死路，必须提供替代动作。

### 2.2 增强槽位（不阻断 P0，可渐进增强）

- 多 Agent 深度讨论形态：多角色并行对话、引用溯源、讨论时间轴、可折叠线程等。
- 白板高级布局：复杂排版优化、自动避免重叠的更强策略、可视化标注工具等。
- 更复杂的测验形态：分层题型、错题本联动、学习路径联动等。
- 学习中心/历史/收藏的深度联动入口与运营模块。

## 3. 课堂主流程总览（Happy Path）

1. 用户从首页或其它入口进入 `/classroom/input`。
2. 用户输入“我想学 XXX”的主题，可选择老师风格（可用默认风格）。
3. 用户点击“生成课堂”（主 CTA）提交。
4. 创建成功后进入 `/classroom/:id/generating`：
   - 展示阶段与进度。
   - SSE 更新进度，断线可恢复。
5. 生成完成自动进入 `/classroom/:id`：
   - 浏览幻灯片（Slide）。
   - 进入白板阅读（Whiteboard）。
   - 浏览多 Agent 讨论片段（Discussion）。
6. 用户进入测验承载页（候选独立路径：`/quiz/:sessionId`）：
   - 作答并提交，查看对错与解析。
   - 点击“返回课堂”回到 `/classroom/:id`。
7. 用户可在历史视图或 LearningCenter 聚合入口再次回看该课堂结果（此处仅声明入口，不在本文展开个人域 IA）。

## 4. `/classroom/input`：主题输入与提交

### 4.1 进入条件与拦截

- 该路由为受保护路由：未登录访问时进入统一登录流程；401 / 403 处理遵循 `architecture.md` 的固定语义，登录成功后可回到 `/classroom/input`。

### 4.2 页面核心区块（流程相关）

- 主题输入区（P0）：
  - 输入方式：文本输入为主（可扩展推荐主题、历史主题等为增强槽位）。
  - 校验：空输入不可提交；超长、非法字符等按后端校验结果呈现用户可理解提示。
- 老师风格（Agent Style）会话配置（P0）：
  - 入口：附着在输入区附近的触发器（头像或胶囊等）。
  - 语义：风格是“老师人格预设”，只做局部点缀，不做页面级主题切换。
- 主 CTA（P0）：`生成课堂`（或同义），点击后创建课堂任务。

### 4.3 提交流程（P0）

1. 用户点击主 CTA。
2. 前端执行最小校验（空、明显非法）。
3. 调用创建任务接口（FastAPI 侧编排，长期记录索引由 RuoYi 承接为后续沉淀）。
4. 创建成功：
   - 获取 `classroomTaskId`（对应 `:id`）。
   - 跳转 `/classroom/:id/generating`。
5. 创建失败（错误态，P0）：
   - 网络错误/超时：提示重试。
   - 服务不可用：提示稍后重试，必要时说明系统将自动切换能力提供方（Provider），但不暴露技术名词与内部状态。

### 4.4 可恢复点（P0）

- 用户离开输入页不会产生“半成品课堂”，只有在任务创建成功后才进入生成态。
- 若用户创建成功后误关闭页面：可通过打开 `/classroom/:id/generating` 恢复等待；若已完成则进入 `/classroom/:id`（见第 5 节与第 6 节）。

## 5. `/classroom/:id/generating`：等待、断线恢复与失败收口

### 5.1 页面职责（P0）

- 复用“统一等待壳层”语义，展示任务五态：`pending`、`processing`、`completed`、`failed`、`cancelled`。
- 通过 SSE 消费事件：`connected`、`progress`、`provider_switch`、`completed`、`failed`、`heartbeat`、`snapshot`。
- 明确可行动：等待中可返回入口或取消（若产品支持取消），失败时可重试或回到输入页。

### 5.2 进入时的状态判定（P0）

- 若任务状态为 `pending/processing`：停留本页等待并展示进度。
- 若任务状态为 `completed`：自动跳转 `/classroom/:id`。
- 若任务状态为 `failed/cancelled`：停留本页展示失败/取消态与动作（见 5.5）。

> 备注：状态来源优先 SSE；若 SSE 不可用或断线，允许降级为状态查询（轮询）以完成恢复闭环。

### 5.3 SSE 断线恢复（P0）

- 断线检测：超过一定时间未收到 `heartbeat` 或 SSE 连接错误。
- 恢复策略：
  - 自动重连。
  - 重连成功后优先依据 Redis 中的运行时状态与事件缓存补发恢复页面；`snapshot` 仅用于对齐当前 `status/progress/stage`。
  - 若多次重连失败：降级为状态查询，并提示用户“网络不稳定，正在尝试恢复”。

### 5.4 等待页可见信息（P0）

- 阶段（stage）与进度（progress），配合用户可理解的阶段描述（不暴露技术细节）。
- Provider 切换提示（增强为主，P0 可简化为“正在切换服务以继续生成”）。

### 5.5 失败与可行动收口（P0）

失败态必须提供明确动作集合，避免死路：

- 主动作（P0）：`再试一次`。
  - 语义：对同一任务触发重试，或创建新任务并跳转到新 `:id`（二者属于实现策略，需后续与后端契约冻结；本文只要求用户侧行为可理解）。
- 次动作（P0）：`返回输入`（回到 `/classroom/input`，可带入原主题作为预填属于增强）。
- 替代动作（P0）：`稍后再看`（提示可去历史 / 学习中心打开已落库记录；课堂历史与会话摘要按 `architecture.md` 由 RuoYi / MySQL 承担长期持久化）。
- 文案要求（P0）：失败原因用“可理解摘要”表达，例如“服务繁忙”“生成超时”“内容生成失败”，不暴露内部错误码、堆栈或 Provider 名称。

### 5.6 生成过程中用户离开与恢复（P0）

- 用户刷新/关闭页面后再次打开 `/classroom/:id/generating`：
  - 若仍在进行：通过 Redis 运行时状态、事件缓存补发与 `snapshot` 对齐恢复到当前进度。
  - 若已完成：自动进入 `/classroom/:id`。
  - 若已失败：展示失败态并可重试。

## 6. `/classroom/:id`：结果浏览、白板阅读、讨论片段与测验入口

### 6.1 页面职责（P0）

- 展示课堂结果的最小可读结构：幻灯片浏览 + 白板阅读 + 讨论片段 + 课后小测入口。
- 允许用户刷新或直达该 URL 后恢复：
  - 若结果就绪：正常展示。
  - 若仍在生成：引导回到 `/classroom/:id/generating`（或展示“继续生成中”的恢复条）。
  - 若查询失败或数据缺失：降级展示可用部分，并提供替代动作。

### 6.2 结果信息结构（P0 最小集）

- 幻灯片区域（P0）：
  - 支持切换：`上一页` / `下一页`（至少一种）。
  - 支持快速定位（增强槽位）：目录、缩略图、进度条、搜索。
- 白板阅读区域（P0）：
  - 白板与幻灯片的关系允许两种承载方式：
    - 方式 A：每一页幻灯片关联一个白板视图（推荐用于 P0 的直觉映射）。
    - 方式 B：白板是课堂的独立 Tab（增强也可）。
  - 白板可读性要求：关键元素不应严重重叠，复杂布局问题不阻断主链路（允许降级）。
- 多 Agent 讨论片段挂载区（P0）：
  - 位置建议：与幻灯片/白板并列的侧栏或下方分区。
  - P0 展示形态：讨论摘要片段列表（按章节或关键问题分段），点击可展开阅读。
  - P0 参与形态：提供“学生提问输入框”最小形态，提交后得到回应（不要求多线程并行与复杂引用）。
- 课后小测入口（P0）：
  - 位置建议：课堂完成态的“下一步”区域，或固定在页面顶部/侧栏的行动区。

### 6.3 幻灯片切换与阅读（P0）

- 默认进入：展示第 1 页或“上次阅读页”（后者为增强槽位，需要持久化）。
- 切换行为：
  - 翻页按钮或键盘左右键（键盘为增强槽位）。
  - 页码提示（例如“3 / 12”）。
- 局部数据缺失降级（P0）：
  - 若某一页缺少正文要点：展示“该页内容暂不可用”，允许继续翻页。
  - 若幻灯片列表缺失但有白板：自动进入白板阅读模式，并提示“幻灯片暂不可用”。

### 6.4 白板阅读（P0）

- 白板加载失败或布局异常（P0）：
  - 降级为“文本版要点”或“仅展示不重叠的关键块”，并提示用户可继续查看幻灯片。
  - 不允许整页崩溃或遮挡主要 CTA（例如测验入口）。

### 6.5 多 Agent 讨论片段（P0 与增强分层）

- P0：讨论片段是课堂结果的一部分，可阅读，可作为学习补充；至少提供：
  - 讨论摘要卡片（谁在讨论、讨论主题、结论摘要）。
  - 展开后可阅读完整片段（可滚动）。
  - 学生参与入口：单轮输入并获得回应（多轮对话为增强槽位）。
- 增强槽位：
  - 多角色并行对话与时间轴。
  - 引用对应幻灯片页或白板区域的“定位锚点”。
  - 讨论片段与知识问答 `/knowledge` 的联动跳转。

### 6.6 进入课后小测与返回课堂（P0）

#### 6.6.1 进入小测（P0）

- 入口触发：用户点击“开始小测/课后测验”。
- 跳转：进入测验承载页（候选独立路径：`/quiz/:sessionId`）。
  - `sessionId` 来源：可由课堂结果中返回，或由点击进入时创建（契约待冻结，见“冲突 / 待确认点”）。

#### 6.6.2 小测完成后的回流（P0）

- 小测页必须提供明确的“返回课堂”动作，回到 `/classroom/:id`。
- 回到课堂后：
  - 可展示“测验已完成”的状态提示（增强槽位：显示得分、错题入口等）。

### 6.7 结果页的异常与恢复（P0）

- 结果未就绪：提示“课堂仍在生成中”，提供“继续等待”按钮跳转 `/classroom/:id/generating`。
- 查询失败或权限问题：
  - 401：清除 Token 并跳转登录；成功后回到当前结果页。
  - 404/不可用：提示“该课堂不可用”，提供返回入口与去历史列表的动作（若历史存在）。
- 局部数据缺失（P0）：按模块降级展示，不阻断“浏览其它模块 + 进入测验/返回入口”。

## 7. 失败、断线与局部缺失：关键分支（必须覆盖）

### 7.1 生成失败（P0）

- 发生位置：`/classroom/:id/generating` 或从结果页发现状态为 `failed`。
- 用户可选动作（至少 2 个，P0）：
  - 再试一次。
  - 返回输入。
  - 稍后再看（若可回看则引导去历史/学习中心）。

### 7.2 断线恢复（P0）

- 发生位置：生成页 SSE 断线，或用户刷新/切换网络。
- 系统行为（P0）：
  - 自动重连，并基于 Redis 运行时状态与事件缓存恢复；`snapshot` 只用于当前状态对齐。
  - 重连失败则降级查询。
  - 用户可见反馈：明确正在恢复，并提供“手动重试”按钮。

### 7.3 局部数据缺失（P0）

- 可能缺失类型（示例，不穷举）：
  - 幻灯片缺失或部分页缺失。
  - 白板缺失或渲染失败。
  - 讨论片段缺失。
  - 测验数据未生成或不可用。
- 降级原则（P0）：
  - 只降级缺失模块，不牵连其它模块。
  - 保留可行动入口：返回输入、继续等待、进入可用模块、进入/返回测验（若测验不可用则给出替代动作）。

## 8. P0 验收对齐（来自事实源的 E2E 约束）

本文档对齐以下 P0 E2E 关键路径（以事实源为准，本文不扩写细节）：

- 初学场景输入主题 → 生成课堂 → 查看幻灯片 → 完成测验。
- 长任务过程中进度展示正常，连接中断后状态可恢复。
- 任务完成后可在历史记录或学习中心重新打开结果。

## 9. 冲突 / 待确认点（只记录，不在本文裁决）

1. 老师风格选择器的控件形态：
   - `epics.md`（FR-CS-002）出现“输入框中以下拉框形式展开”的描述。
   - `ux-design-specification.md` 强调“输入框附近触发器展开轻量面板（头像或胶囊）”。
   - 处理：本文只冻结“附着在输入区附近的会话配置入口”这一语义；具体控件形态待成品图或交互规范定稿。
2. `/quiz/:sessionId` 的 P0/P1 优先级差异：
   - `prd.md` 的 P0 E2E 明确包含“完成测验”。
   - IA 文档中 `/quiz/:sessionId` 存在被标为 P1 的情况（作为独立页面成品化节奏）。
   - 处理：本文按“能力必须可演示（P0）”冻结测验入口与回流流程，但不裁决“独立正式页面成品化优先级”。
3. `:id` 与 `:sessionId` 的资源语义：
   - `/classroom/:id` 的 `:id` 是否与任务 ID 完全一致，或存在任务 ID 到结果资源 ID 的映射。
   - `/quiz/:sessionId` 的 `sessionId` 是否等同于课堂会话 ID、测验会话 ID，或由后端创建独立 quizSessionId。
   - 处理：本文只依赖“可恢复/可回看/可回流”的用户侧语义，具体 ID 契约待接口冻结。
