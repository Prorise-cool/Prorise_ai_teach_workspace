## 1. 关键对象与统一约束（对齐事实源）

### 1.1 统一任务状态（Task Status）

长任务统一 5 态：

| 状态 | 含义 | 页面主要承载 |
|---|---|---|
| `pending` | 已受理未开始 | `/video/:id/generating` |
| `processing` | 执行中 | `/video/:id/generating` |
| `completed` | 成功完成 | 自动跳转 `/video/:id` |
| `failed` | 失败完成 | `/video/:id/generating` 失败态 + 操作 |
| `cancelled` | 已取消/中止 | `/video/:id/generating` 收口态 + 操作 |

### 1.2 SSE 事件类型（Waiting Page Contract）

等待页消费的 SSE 事件类型固定为：
`connected`、`progress`、`provider_switch`、`completed`、`failed`、`heartbeat`、`snapshot`。

### 1.3 老师风格（Agent Style）约束

- 老师风格是“会话配置（AgentConfig）”，通过头像/胶囊触发器在输入框附近展开面板完成选择。
- 老师风格只影响“局部 teacher indicators 与内容行为”，不触发页面级主题切换。
- 风格项结构必须支持未来扩展超过 4 种风格，不写死为页面中心的固定 4 卡片布局。

## 2. 主流程闭环（Happy Path）

### 2.1 入口与前置（登录态）

路由骨架对齐：视频链路为 ` /video/input → /video/:id/generating → /video/:id`，均属于受保护路由；未登录访问与 401 / 403 处理遵循 `architecture.md` 的固定鉴权语义。

### 2.2 页面 A：视频题目输入页（`/video/input`）

#### 2.2.1 页面目标

- 极简输入，用户 3 秒内理解并开始。
- 三种输入方式共用同一个题目输入区域与同一个主 CTA。
- 风格选择器在输入区附近完成，不新增独立 style 页面。

#### 2.2.2 文本输入（Typing）

1. 用户在题目输入框输入或粘贴文本。
2. 前端做最小校验（非空、合理长度等）。
3. 用户点击主按钮“开始生成讲解视频”触发创建任务。

#### 2.2.3 图片上传（Upload）

1. 用户选择“上传/拍照”（实现可为上传控件或调用相册/摄像头）。
2. 系统触发 OCR 识别流程。
3. OCR 成功：将识别文本自动回填到“可编辑”的题目输入框，用户可继续编辑。
4. 用户点击主按钮创建任务。

#### 2.2.4 粘贴图片（Paste Image）

1. 用户在输入区聚焦时执行 `Ctrl/Cmd + V`。
2. 前端自动检测剪贴板内容：
   - 若为文本：直接回填输入框。
   - 若为图片：进入 OCR 识别流程并回填到可编辑输入框。

#### 2.2.5 OCR 失败转手输（Fallback to Manual Typing）

触发条件：OCR 失败或识别质量不足，无法可靠提取题目。

可用路径必须存在：

- 页面明确提示“请手动补充题目文本，或更换图片重试”。
- 输入框保持可编辑状态，用户可以直接打字继续流程，不陷入死路。

#### 2.2.6 老师风格选择器（展开/收起/选中）

位置：输入框右上角或近输入框位置，以“头像/胶囊触发器”呈现当前会话风格。

交互与状态：

| 用户动作 | 面板状态 | UI 反馈 |
|---|---|---|
| 点击触发器 | 展开 | 展示风格列表（头像、名称、选中态） |
| 点击空白处或再次点击触发器 | 收起 | 面板关闭，不打断输入 |
| 点击某个风格项 | 选中并保持展开或自动收起（实现二选一，需一致） | 触发器头像/文案更新；选中项高亮/勾选 |

规则：

- 未选择时存在默认风格，且默认风格应“可见且已预选”。
- 选中风格需被写入当前会话配置，并在创建任务时透传到下游生成链路。

#### 2.2.7 创建任务（Create Video Task）

触发：用户点击主 CTA“开始生成讲解视频”。

最小行为（对齐史诗与架构约束）：

1. 前端调用统一任务资源创建视频任务：`POST /api/v1/video/tasks`。
2. 成功返回后拿到 `taskId`（与路由中的 `:id` 对齐，格式示例 `video_{timestamp}_{short_uuid}`）。
3. 立即路由跳转到等待页：`/video/:id/generating`。

## 3. 等待页（事件驱动）与结果页衔接

### 3.1 页面 B：视频生成进度页（`/video/:id/generating`）

#### 3.1.1 页面目标

- 等待透明、不焦虑：阶段列表 + 进度 + 文案 + 预计剩余时间。
- 过程可恢复：断线重连后通过快照恢复阶段与进度。
- 失败可行动：失败时给出用户可理解摘要，并提供“再试一次/返回输入/返回首页”等动作。

#### 3.1.2 SSE 首次连接（SSE Connected）

- SSE 连接地址：`GET /api/v1/video/tasks/{taskId}/events`（EventSource）。
- 收到 `connected`：提示“已连接到服务器”（可为轻提示，不要求强提示）。

#### 3.1.3 等待页阶段与文案（示例）

阶段划分（用于 UI 呈现与用户心智）：

- 理解题目（10-20s）
- 生成分镜（30-60s）
- 生成 Manim（60-120s）
- 渲染动画（60-120s）
- TTS 配音（20-40s）
- 合成视频（20-40s）

#### 3.1.4 SSE 事件驱动的 UI 行为与“结果页衔接”

等待页必须以事件为核心驱动，并明确“何时跳转结果页”：

| SSE 事件 | 等待页行为 | 与结果页衔接（必须明确） |
|---|---|---|
| `progress` | 更新阶段状态、阶段进度条、阶段描述文案与整体百分比 | 不跳转，持续等待 |
| `provider_switch` | 展示 toast：“已切换到备用服务” | 不跳转，继续执行；可在页面保留“已降级/已切换”提示点 |
| `heartbeat` | 无 UI 变化（后台保活） | 不跳转 |
| `snapshot` | 断线重连后用快照恢复当前 `status/progress/stage` | 若快照显示 `completed`，立即执行与 `completed` 同等的跳转逻辑 |
| `completed` | 将任务标记为完成态 | 自动跳转到 `/video/:id`，并在结果页执行“首次加载”流程 |
| `failed` | 展示失败原因与可理解提示 | 不跳转，停留本页进入失败态（见 4.2） |

#### 3.1.5 断线恢复与降级（Degrade）

等待页需要覆盖以下恢复策略：

1. 断线重连：携带 `Last-Event-ID` 重连，优先恢复 SSE 实时流。
2. 超过重试上限：降级到轮询任务状态接口（事实源中称为 `/status`），持续刷新页面状态。
3. 降级期间：UI 需要明确“当前为降级模式（网络不稳定）”，但流程仍可完成。

### 3.2 页面 C：视频播放结果页（`/video/:id`）

#### 3.2.1 结果页进入方式（必须覆盖）

- 从等待页 `completed` 自动跳转进入（主路径）。
- 从历史记录/学习中心/收藏等回看入口直接进入（回访路径）。

#### 3.2.2 结果页加载与“仍在生成”处理（衔接约束）

由于用户可能从外部入口或刷新直接访问 `/video/:id`，结果页需要处理任务未完成场景：

- 若结果查询显示 `status ∈ {pending, processing}`：结果页应提示“仍在生成”，并引导或自动跳转至 `/video/:id/generating` 继续等待。
- 若 `status = failed/cancelled`：结果页可提示“结果不可用”，并引导至 `/video/:id/generating` 查看失败原因与重试动作，或回到 `/video/input` 重新发起。

#### 3.2.3 播放器（播放/倍速/拖拽/全屏）

播放器必须支持以下交互（对标用户习惯）：

| 功能 | 用户动作 | 反馈 |
|---|---|---|
| 播放/暂停 | 点击或空格 | 图标切换 + 状态变化 |
| 进度拖拽 | 拖拽进度条 | 实时预览时间；跳转到已缓冲位置 |
| 倍速 | 点击倍速按钮 | 弹出选项：0.5x / 1x / 1.5x / 2x；即时生效且可见 |
| 全屏 | 点击全屏按钮 | 进入/退出全屏 |

#### 3.2.4 继续提问（Continue Ask）

结果页提供“继续问 AI 老师”的侧边交互区或入口壳层：

- 用户可在结果页内继续输入追问，不必跳出到其他工具。
- 本 UF 文档只定义“入口壳层与交互衔接”，完整追问后端能力若未就绪，允许先占位并提示“功能建设中/稍后可用”（不暴露技术细节）。

#### 3.2.5 收藏（Favorite）

结果页提供“收藏/取消收藏”入口：

- 收藏状态在结果页可见，并可回写到收藏视图与回看域。
- 收藏为可逆操作，默认不需要二次确认。

#### 3.2.6 分享（Share）

结果页突出分享按钮，支持（至少在交互规格层表达）：

- 复制链接：复制视频播放链接。
- 生成海报：下载包含视频封面、标题、二维码的海报。
- 分享到微信：以二维码等形式承载（具体实现依赖渠道能力，见“冲突 / 待确认点”）。

#### 3.2.7 重新生成（Regenerate）

结果页提供“重新生成”入口（对齐 PRD 基础操作要求）：

- 语义：以当前题目与风格为默认预填，创建一条“新任务”（新 `taskId`），并跳转到新任务的等待页。
- 不覆盖旧结果，旧结果仍可在历史中回看（最终以 RuoYi 长期数据为准）。

#### 3.2.8 返回（Back）

结果页提供返回入口：

- 返回到 `/video/input`（继续输入新题）或 `/`（回到双入口）。
- 若从历史/收藏进入，返回策略可为“返回列表页”，该行为需要在路由层统一（见“冲突 / 待确认点”）。

## 4. 异常流程（Error Paths）

### 4.1 输入阶段异常

#### 4.1.1 图片不可用

- 不支持格式或文件过大：提示“图片格式/大小不支持”，允许重新选择图片或改为文本输入。

#### 4.1.2 OCR 失败或识别质量不足

- 必须提供手动输入替代路径（输入框可编辑）。
- 提示用户可更换图片重试，但不强迫走图片路线。

#### 4.1.3 创建任务失败（HTTP / 校验 / 服务不可用）

- 保留用户已输入内容与已选风格，不清空输入。
- 给出用户可理解错误提示与“重试/稍后再试/返回”动作。
- 若判断为外部服务不可用：可提示“已尝试降级/稍后再试”（不暴露底层 Provider 名称）。

### 4.2 等待阶段异常（含失败与降级）

#### 4.2.1 SSE 连接失败

- 首次连接失败：提示“网络不稳定，正在重试连接”，并进入重试逻辑。
- 超过重试上限：降级到轮询任务状态（/status），页面明确“已进入降级模式”。

#### 4.2.2 任务失败（`failed`）

等待页失败态必须包含：

- 用户可理解的失败摘要（避免只展示技术错误码）。
- 可行动按钮：
  - “再试一次”：基于当前输入参数创建新任务，并跳转新等待页。
  - “返回输入”：回到 `/video/input` 并尽量预填原题目与风格（若实现支持）。
  - “返回首页”：回到 `/`。

错误码与提示（示例，按 UX 规格）：

| error_code | 推荐提示语义（示例） | 备注 |
|---|---|---|
| `LLM_UNAVAILABLE` | “AI 服务暂时繁忙，请稍后重试” | 可提供“再试一次” |
| `MANIM_RENDER_FAILED` | “动画渲染遇到问题，正在自动修复...” | 可能进入自动修复阶段并继续等待 |
| `TTS_ALL_PROVIDERS_FAILED` | “语音合成失败，正在重试...” | 可能触发 Provider 切换 |
| `TIMEOUT` | “生成超时，请检查网络后重试” | 可建议检查网络 |

#### 4.2.3 Provider 切换与服务降级（`provider_switch`）

- 收到 `provider_switch`：toast 提示“已切换到备用服务”。
- 若最终返回“降级结果”（低质量但可用）：仍应走 `completed → 结果页`，并在结果页或结果摘要处标注“本次结果为降级版本”（避免用户误判质量）。

### 4.3 结果页异常

#### 4.3.1 播放失败或资源不可访问（例如 COS 链接失效）

- 提示“视频加载失败”，提供“重试加载/重新生成/返回”入口。
- 若可检测为权限或登录态问题：触发统一登录恢复语义（见“冲突 / 待确认点”）。

#### 4.3.2 结果缺失或被删除

- 提示“该结果不存在或已被删除”，提供“返回输入/返回首页/查看历史”入口。

## 5. 恢复流程（Recovery Paths）

### 5.1 等待页刷新/重进（同一任务）

触发：用户在 `/video/:id/generating` 刷新页面或短暂断网后恢复。

恢复优先级：

1. 重建 SSE 连接并请求断点恢复（Last-Event-ID）。
2. 收到 `snapshot`：恢复阶段、进度、状态。
3. 若恢复失败：降级轮询状态接口，直到 `completed/failed/cancelled`。

关键要求：恢复后必须保证“确定性收口”。

- 一旦状态变为 `completed`：必须跳转结果页 `/video/:id`（即使通过 snapshot 或轮询得到）。
- 一旦状态变为 `failed/cancelled`：必须进入失败/取消收口态并提供下一步动作。

### 5.2 结果页刷新（已完成任务）

触发：用户在 `/video/:id` 刷新页面。

行为：

- 结果页通过结果查询拿到可播放资源与必要摘要后直接渲染播放器。
- 若发现任务并未完成（状态回退或查询到 processing）：按 3.2.2 处理，跳转到等待页继续。

### 5.3 回看入口恢复（历史/收藏/学习中心）

用户从回看入口进入 `/video/:id`：

- 若结果可用：直接播放。
- 若结果不可用（failed/cancelled 或资源缺失）：提示并提供“重新生成/返回输入/返回列表”。

## 6. 页面间跳转（Route Transitions）

| From | To | 触发条件 |
|---|---|---|
| `/` | `/video/input` | 用户选择“我有道题不会”入口 |
| `/video/input` | `/video/:id/generating` | 创建任务成功返回 `taskId` |
| `/video/:id/generating` | `/video/:id` | SSE `completed`，或恢复快照/轮询判定 `completed` |
| `/video/:id` | `/video/:newId/generating` | 用户点击“重新生成”，创建新任务成功 |
| `/video/:id` | `/video/input` 或 `/` | 用户点击“返回” |
| 回看入口（LearningCenter / Profile 内的历史或收藏视图） | `/video/:id` | 用户点击打开某条视频结果（回看） |

## 7. 状态与事件映射（State/Event Mapping）

### 7.1 Task Status → 页面与 UI

| status | 输入页 `/video/input` | 等待页 `/video/:id/generating` | 结果页 `/video/:id` |
|---|---|---|---|
| `pending` | 不出现 | 展示“已受理/排队中”，显示阶段列表 | 若被直达访问，提示并跳转等待页 |
| `processing` | 不出现 | 展示阶段进度与预计剩余时间 | 若被直达访问，提示并跳转等待页 |
| `completed` | 可作为回访入口预填（可选） | 立即跳转结果页 | 展示播放器与结果操作 |
| `failed` | 保留输入，允许重试创建 | 失败态收口，提供再试一次/返回输入等 | 提示结果不可用，引导去等待页查看失败或重新生成 |
| `cancelled` | 保留输入，允许重试创建 | 取消态收口，提供重新发起 | 提示结果不可用，提供重新生成 |

### 7.2 SSE 事件 → 等待页 UI 与跳转（汇总）

| event | UI 最小动作 | 跳转 |
|---|---|---|
| `connected` | 可选轻提示 | 否 |
| `progress` | 更新阶段条与文案 | 否 |
| `provider_switch` | toast 提示已切换 | 否 |
| `heartbeat` | 无 UI | 否 |
| `snapshot` | 恢复状态与进度 | 若为 completed，跳转 `/video/:id` |
| `completed` | 结束等待态 | 是，跳转 `/video/:id` |
| `failed` | 进入失败态 | 否 |

## 8. 冲突 / 待确认点（仅记录，不改写事实源）

1. 主动登录入口的 UI 载体仍有差异：
   - 本文已按 `architecture.md` 固定 401 / 403 语义。
   - 但主动登录入口最终采用 Dialog 还是独立登录壳层，仍需在事实源层继续统一。
2. “返回”落点策略待统一：
   - `/video/:id` 的返回应回到 `/video/input`、`/`，还是“返回上一页（例如历史列表）”，需在全局导航与回跳策略中冻结。
3. “取消任务（cancelled）”的前端暴露程度待确认：
   - UX 规格的统一等待原则包含“用户可以取消任务”，但 UF2 的线框与本次必须项未明确取消按钮位置与权限；需确认是否在等待页提供显式取消入口。
4. 分享渠道实现边界待确认：
   - “生成海报/分享到微信”依赖渠道能力与产物生成服务，需确认 MVP 交付是否仅做到“复制链接”，还是同时提供海报与二维码交互。
