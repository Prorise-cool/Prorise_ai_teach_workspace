## 1. 知识问答流程（`/knowledge`）

### 1.1 进入与初始化

**入口**（示例，不限定 UI 形态）：
- 全局导航进入 `知识问答` → `/knowledge`。
- 从视频/课堂结果页点击“继续问 AI 老师”或“去知识问答深挖” → `/knowledge`。

**页面初始化**：
1. 页面加载后展示：课程选择器、问题输入区、回答区（空态）、历史入口。
2. 若课程列表加载失败：展示可理解的错误提示与“重试”，不阻断返回。

**空态（首次进入）**：
- 回答区展示引导文案（例如“选择课程后提问”），不展示异常页。

### 1.2 提问与流式回答（streaming）

**主流程**：
1. 用户在输入框输入问题（围绕课程知识点、教材内容或术语）并提交。
2. UI 立即插入“用户消息气泡”，并进入“等待回答”状态（loading）。
3. 后端开始流式返回后，UI 以流式方式逐步展示回答内容（逐字/逐段均可，但需体现“正在生成”）。
4. 回答完成后，UI 在同一轮回答下方展示引用来源（若有）与相关知识点推荐（若有）。

**失败与恢复**：
- 若请求失败：展示“重试”与“改写问题”建议。
- 若无可用结果：明确说明“未检索到可靠依据/结果不足”，禁止用编造答案替代；同时给出可执行建议（切换课程、补充上下文、上传文档后再问）。

### 1.3 引用来源展示（Citation）

**展示规则**：
- 有来源时：在回答下方展示来源列表（文档片段、章节或来源标记），不影响主答案阅读。
- 来源不足时：在来源区域展示“来源不足”的明确提示，不隐藏该区域导致用户误以为有来源。

**交互**：
- 点击单条来源：若可用则跳转到原文位置；若不可用则提示“暂不支持定位到原文”，并保留来源摘要信息。

### 1.4 术语解释（Glossary）

**触发方式（至少一种）**：
1. 用户对回答中的术语点击“解释”。
2. 或用户在输入框直接问“术语 X 是什么？”。

**结果呈现**：
- 术语解释以“可理解、简明”的形式展示，并与当前问答上下文联动（例如：在当前回答卡片内展开解释区，或在侧栏展示并标注对应术语）。

**失败**：
- 若术语解释不可用：提示失败原因的用户可理解摘要，并提供重试/改写建议。

### 1.5 上传文档与解析（Document Upload + Parse）

> 备注：文档上传与解析属于后置能力，但本流程必须定义其主路径与失败态，避免后续扩展时破坏现有交互心智。

**主流程**：
1. 用户在 `/knowledge` 发起“上传教材/讲义”等文档上传动作。
2. 前端在上传前校验格式/大小；非法格式或超限时直接拒绝并提示原因。
3. 上传成功后进入“解析中”状态，并提示用户解析完成后可用于检索问答。
4. 解析完成：提示“已加入可检索范围”，并允许用户继续提问（可选：允许用户选择“仅在已上传文档内检索/在课程域内检索”）。

**解析失败**：
- 必须返回明确错误原因（面向用户，不暴露内部敏感信息），并提供“重试上传/更换文件/返回继续问答”。

### 1.6 问答历史入口（History In Knowledge）

**历史入口**：
- `/knowledge` 页面内提供历史入口，允许用户再次查看过往问答。

**空态**：
- 没有历史时展示空状态（例如“暂无问答记录，先提一个问题吧”），不渲染异常页面。

## 2. 小测流程（测验承载页，候选独立路径：`/quiz/:sessionId`）

### 2.1 进入与开始

**入口**：
- 课堂结束后弹窗引导“来做个小测验吧” → 用户点击开始 → 进入测验承载页（若冻结为独立页面，候选路径为 `/quiz/:sessionId`）。
- 从知识问答或学习路径节点入口触发进入测验承载页（当产品在具体触发点与承载形态上定稿后补齐映射）。

**开始页（或开始区块）**：
1. 页面展示：题量进度（如“第 1 题 / 共 N 题”）、题干、单选项、提交按钮。
2. 若题目尚未就绪：展示加载态与重试入口，不把加载写成“生成中等待流”主任务（这里是测验页内部加载态）。

### 2.2 作答与即时反馈

**单题作答**：
1. 用户选择一个选项（单选）。
2. 点击“提交答案”。
3. 系统完成判定后立即反馈正误状态：正确展示 ✅，错误展示 ❌。
4. 同时展示解析内容（可默认展开），让用户理解错误原因。

**下一题**：
- 用户点击“下一题”进入下一题，直到完成全部题目。

### 2.3 完成统计与收口动作

**完成页/完成卡片**：
- 展示正确率、用时、知识点掌握情况摘要。
- 必须提供动作：“再来一次”（重新开始一轮测验）与“返回课堂”（回到对应课堂结果页）。

### 2.4 异常与空态

必须覆盖的状态：
- `sessionId` 无效或找不到会话：提示“测验已失效或不存在”，提供“返回课堂/返回首页”。
- 题目列表为空：提示“暂无可用题目”，提供“重试生成/返回课堂”。
- 判定失败或网络错误：保留当前作答上下文，提供“重试提交”，避免用户被迫从头开始。

## 3. 学习路径流程（学习路径承载页，候选独立路径：`/path`）

### 3.1 目标与生成

**入口**：
- 用户从学习中心聚合入口、结果页辅助入口或未来独立入口进入学习路径承载页（若冻结为独立页面，候选路径为 `/path`）。
- 或系统根据学习记录推荐进入学习路径承载页（入口提示不阻断用户继续当前学习）。

**主流程**：
1. 用户选择学习目标（课程或自定义主题）。
2. 用户选择学习周期（例如 4 周）。
3. 点击“开始规划”。
4. 生成完成后展示路径（按周组织的计划、阶段目标与学习节点）。

### 3.2 保存、再次打开与调整

**保存**：
- 用户点击“保存路径”后，路径应支持再次打开与继续执行。

**调整**：
- 用户进入调整模式后，可修改目标或周期而不需要完全重走流程。
- 调整不会静默覆盖旧版本（需保留上一个版本或提供明确的覆盖确认机制，具体机制见“冲突 / 待确认点”）。

**开始学习**：
- 点击“开始学习”跳转到对应内容入口（例如 `/classroom/input` 或 `/video/input`），具体映射关系需后续冻结。

### 3.3 异常与空态

必须覆盖的状态：
- 服务不可用或规划失败：提示可理解失败原因并提供“重试”；若已有已保存路径，仍可查看。
- 保存失败：提示失败并允许重试保存，不丢失当前生成结果视图。
- 无可生成结果：明确说明原因并建议用户缩小目标或补充信息。

## 4. 历史回看流程（历史视图；候选独立路径：`/history`）

### 4.1 列表浏览与空态

**主流程**：
1. 用户进入历史视图（当前应承载于 `LearningCenter.tsx`；若后续冻结为独立页面，候选路径为 `/history`）。
2. 系统按时间倒序展示历史条目，并清晰区分类型（视频、课堂等；是否包含问答/测验/路径见“冲突 / 待确认点”）。
3. 列表每条记录至少提供动作入口：打开结果、重开（以该记录的输入参数预填创建新任务，进入主链路的创建步骤，而不是进入等待页的内部状态展示）、删除（需要确认）。

**空态**：
- 无历史记录时展示空状态，并提供“去开始学习/去提问”等 CTA（例如返回 `/` 或进入 `/video/input`、`/classroom/input`）。

### 4.2 打开结果与资源不可用

**打开结果**：
- 点击条目“打开结果”跳转到对应结果页（例如 `/video/:id`、`/classroom/:id`）。
- 必须保证“重新打开”不会重复触发原始生成任务。

**资源不可用**：
- 若关联资源不可用或已归档：展示明确不可用说明与下一步建议（例如“重开生成/返回历史列表”），不展示损坏链接为成功打开。

### 4.3 删除确认（Delete with Confirmation）

**主流程**：
1. 用户在历史视图点击“删除”。
2. 弹出确认提示；用户取消时列表保持原样，用户确认删除后列表即时更新。

**删除语义边界**：
- 删除仅删除“历史展示关系/历史记录条目”，不物理删除底层结果资源；也不自动取消已有收藏（若同一结果仍被收藏，可从收藏入口继续访问）。

## 5. 异常状态（跨流程最小集合）

以下状态在 UF2 四条流程中均可能出现，且必须具备一致的用户可理解反馈与可执行动作：
- 来源不足：在 `/knowledge` 的来源区域明确提示“来源不足”，并给出下一步动作（切换课程、补充上下文、上传文档再问）。
- 解析失败：文档解析失败必须给出明确原因与重试入口。
- 空列表：`/knowledge` 内部历史入口、历史视图列表、以及测验题目列表为空时，展示空态与可执行 CTA。
- 删除确认：删除属于破坏性操作，必须二次确认，且删除后列表即时更新。
- 资源不可用：历史记录重新打开失败时，提示不可用并提供重开建议。

## 6. MVP / P1 / P2 分层（交付视角）

> 说明：事实源在“能力优先级”与“路由页面成品化优先级”上存在差异；本文在此只做分层呈现，差异点集中记录在“冲突 / 待确认点”，不自行改写事实源。

### 6.1 MVP（以“可演示闭环”为最低门槛）

- `/knowledge`：支持提问与可阅读回答；失败可重试；无结果不编造；具备问答历史入口与空态。
- 测验承载页（候选：`/quiz/:sessionId`）：支持单选作答、即时正误反馈、解析展示、完成统计、“再来一次/返回课堂”。
- 历史视图（候选独立路径：`/history`）：支持按时间倒序展示、打开结果、资源不可用提示、删除确认与列表更新、空态。
- 学习路径承载页（候选：`/path`）：允许以“入口 + 结构占位”不阻断 MVP；若上线路径规划则必须包含生成失败与保存失败处理。

### 6.2 P1（体验增强与沉淀完善）

- `/knowledge`：引用来源展示、术语解释、课程选择体验增强；问答记录回写并可在聚合页再次打开。
- 文档上传与解析：上传校验、解析进度提示、解析失败原因明确。
- 历史视图：类型筛选/聚合入口增强（如与学习中心互链）。

### 6.3 P2（扩展能力）

- 学习路径承载页（候选：`/path`）：完整学习路径规划（目标/周期/生成周计划/保存/调整/开始学习动作），并支持“既有已保存路径再次打开”。
- 学习结果聚合与收藏管理的深度增强（不在本文展开）。

## 7. 冲突 / 待确认点

1. `/knowledge`、测验承载页（候选：`/quiz/:sessionId`）、学习路径承载页（候选：`/path`）的 P0/P1/P2 分层差异：
   `prd.md`：知识问答 P0、课后小测 P0、学习路径 P1。
   `ux-design-specification.md`：知识问答 P1、课后小测 P1、学习路径 P2。
   待确认：是否允许“能力闭环优先级”与“独立路由页面成品化优先级”拆分。
2. `:sessionId` 的语义与来源：
   待确认 `sessionId` 是课堂会话 ID、测验会话 ID，还是独立 `quizSessionId`。
3. 历史视图在 `LearningCenter.tsx` 内的承载形态与记录类型边界：
   `prd.md` 表述历史包含课堂、视频、问答或学习辅助生成记录。
   IA/路由骨架对“知识问答记录在历史视图中的重开/打开结果形态”未冻结。
   待确认：历史视图是在 `LearningCenter.tsx` 中以 Tab、二级路由还是独立 `/history` 承载；若包含问答记录，其“打开结果”应跳转到 `/knowledge` 并定位到某次对话，还是仅在 `/knowledge` 内部历史查看。
4. 学习路径“调整不覆盖旧版本”的具体交互机制：
   史诗要求“上一个版本不会被静默覆盖消失”，但 UI 形态未冻结。
   待确认：采用版本列表、草稿态，还是覆盖前确认。
5. 文档上传支持的具体格式/大小限制：
   `prd.md` 仅描述“规定格式/非法格式或超限拒绝”，未列出格式清单。
   待确认：支持的扩展名、大小上限与解析时长的 UX 期望。
