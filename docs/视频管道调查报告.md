## 一、项目已有基础设施（完整可用）

### 1. TTS 基础设施（已实现）
- **DoubaoTTSProvider** (`app/providers/tts/doubao_provider.py`)
  - Provider ID: `doubao-tts`，已注册到 ProviderRegistry（priority 20）
  - 对接字节跳动 OpenSpeech TTS API（与参考项目的 ByteTTS 调同一个 API）
  - 配置项: `base_url`, `api_key`, `cluster`, `voice_code`, `encoding`, `speed_ratio` 等
  - 返回 base64 音频数据，经 TTSService 解码为文件
- **TTSService** (`app/features/video/pipeline/tts.py`)
  - 逐场景调用 `failover_service.synthesize()`，支持 provider 链 failover
  - 处理 voice_preference 匹配、audio segments、duration probing（ffprobe）
  - 返回 `TTSResult`（含 audio_segments + total_duration）

### 2. Compose 基础设施（已实现）
- **ComposeService** (`app/features/video/pipeline/compose.py`)
  - FFmpeg 合并 video + audio + subtitles
  - tpad 自动延长视频匹配音频时长
  - ASS/SRT 字幕生成（Source Han Sans CN, 24px）
  - 封面截取
  - 返回 `ComposeResult`（含 video_path, cover_path, duration）

### 3. Provider 基础设施（已实现）
- Protocol → Registry → Factory → FailoverService 完整链路
- `provider_runtime.llm_for(stage)` / `provider_runtime.tts_for(stage)` 按 stage 分配
- 健康检查、指数退避重试、provider 切换全部就绪
- 支持 Settings 和 RuoYi 双来源配置

### 4. 沙箱基础设施
- **DockerSandboxExecutor** (`app/features/video/pipeline/sandbox.py`)
  - `--network none` — 完全无网络（安全隔离）
  - `--read-only`, `--cap-drop ALL`, `--pids-limit 64`
  - 安全扫描禁止 `requests`, `os`, `httpx`, `subprocess` 等导入
  - 本地回退: `LocalSandboxExecutor`（开发/测试用）

## 二、架构差异分析

### 我们的项目 vs 参考项目 (manim-to-video-claw)

| 方面 | 参考项目 | 我们的项目 |
|------|---------|-----------|
| TTS 方式 | VoiceoverScene + ByteTTS **嵌入 Manim 脚本** | **外部 TTS** (DoubaoTTSProvider) + FFmpeg 合并 |
| 音画同步 | manim-voiceover 自动对齐 | ComposeService 用 FFmpeg 合并 |
| 网络需求 | 沙箱需要网络（调用 TTS API） | 沙箱**不需要**网络（纯本地渲染） |
| 安全模型 | 较弱（需要 requests, os, dotenv） | **强隔离**（--network none, 禁止危险导入） |
| TTS API 调用 | 在 Docker 容器内直接调用字节 API | 在 FastAPI 进程内通过 Provider 体系调用 |

### 结论
我们的外部 TTS + FFmpeg 合并架构**更优**（安全隔离、关注点分离、复用 Provider 基础设施）。
不需要引入 VoiceoverScene，不需要给沙箱开放网络。

## 三、当前失败根因

管道在 **render 阶段**失败，错误信息:
```
ModuleNotFoundError: No module named 'manim_voiceover.services.byte_tts'
```

### 根因链路
1. **`base.j2` 模板**（已修复）— 之前引用了 `from manim_voiceover import VoiceoverScene` + `from manim_voiceover.services.byte_tts import ByteTTS`
   - 当前状态: **已修复** — 已移除 VoiceoverScene，使用 `FlexibleElementsScene(MovingCameraScene)`
2. **`script_templates.py`** — `build_default_manim_script()` **仍然**使用 `VoiceoverScene`:
   ```python
   "from manim_voiceover import VoiceoverScene",
   ...
   f"class {DEFAULT_MANIM_SCENE_CLASS}(VoiceoverScene, FlexibleElementsScene):"
   ```
   当 LLM 生成失败回退到默认脚本时，此代码进入沙箱，因无 manim-voiceover 而失败。
3. **Docker 镜像** — manim 版本 0.18.1（参考项目用 0.19.0），未安装 manim-voiceover
4. **`renderer.py`** — `render_full_script()` 仍传递 `TTS` 模板变量给 base.j2（虽无害但不一致）
